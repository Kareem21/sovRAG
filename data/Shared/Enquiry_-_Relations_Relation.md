




# Relations/Relation Enquiry
The Relations/Relation Enquiry function is used to show relations for 
 the selected Master File or user where:

	

- if accessed from a Master File then it shows you the relations 
    	 of the Master File to other Master Files in the Relations screen. 
    	 This can be accessed

	
		

- 
    - in File Maintenance, go to Enquiry &amp; Reporting &gt; 
        		 Enquiry &gt; Relations

		

- 
    - in Viewpoint - Home, select the Master File and click Relations 
        		 from the Ribbon or context menu 

	
	
- if accessed from outside a Master File, then it shows you the 
    	 Master Files related to the selected user in the User Relations screen. 
    	 This can be accessed

	
		

- 
    - in Viewpoint - Home, go to To Do &gt; Dashboard &gt; To 
        		 Do Summary and click Relation Enquiry

		

- 
    - in Viewpoint - Home, go to Configuration &gt; Users &amp; 
        		 Security &gt; Users and click Relation Enquiry

	

<span style="font-weight: bold; font-style: italic;">Note</span>: The 
 User Relations screen can also be accessed from within File Maintenance 
 through General &gt; To Do &gt; To Do Summary.

&nbsp;
## Relations and User Relations Screens
When accessed from a Master File, the screen title will be Relations 
 after the Master File name. If accessed outside of a Master File, then 
 the title will be User Relations after the user name. In Relations, you 
 will be able to choose whether you want to view Child or Parent relations 
 or both for the selected Master File, where:

	

- Child relations - will show Master Files that are linked to 
    	 the currently selected Master File

	

- Parent relations - will show Master Files where the currently 
    	 selected Master File has been added as a relation

For User Relations, what is shown will be Master Files where the selected 
 user is set as the Administrator, Manager or Controller.
