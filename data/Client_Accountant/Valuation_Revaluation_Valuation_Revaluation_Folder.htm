Valuation/Revaluation - Valuation/Revaluation Folder

The details of a valuation/revaluation are displayed in this folder.

  • Account File Valuation

    The valuation of all sub-ledger accounts are shown in both the base and sub-ledger currencies. New valuations are based on the most recent valuation of the Account File's sub-ledgers.

     

    Valuation Date

    This displays the valuation date entered in the Select folder for new valuations; or the valuation date of a selected valuation, if you are looking at an existing valuation.

     

    Reference

    This defaults to 'Account File Val. <Date of valuation>'. Change this if necessary. It is also possible to automate this using User Field (VAL.F.RE). Refer to Automatic Valuation/Revaluation Reference.

     

    Valuation Currency

    The valuation currency is the base currency of the Account File. This cannot be changed.

     

    NAV, O/S Units and <- update

    Click <- update to automatically update the NAV, which is the (re)valued sum of all (selected) assets minus the sum of all (selected) liabilities; and the O/S Units, which is the number of outstanding shares/units held in the Shares/Unit Holder sub-ledger. See also Excl. NAV below.

     

    Note

    Enter any relevant note on the valuation.

     

    Show:-

    Tick or untick the following checkboxes to show/hide the information in the valuation table.

    • Main/sub groups - Breakdown the sub-ledger accounts by main and sub-groups.

    • 0 balance accounts - Sub-ledgers with 0 balance.

    • Valuation details - Valuation amounts attributable to exchange rate differences and to differences in value.

    • NAV summary - The NAV summary is available at Book value and at Valuation values.

     

    Within the valuation table

    Excl. NAV

    These checkboxes are located within the valuation table. Tick the checkbox next to the accounts that you want to omit from the NAV calculation, then click <- Update to refresh  the NAV and O/S Units after omitting values from the calculations.

     

    CCY

    The first occurrence of each currency will be highlighted in grey. If you were to update an exchange rate, ALL same currency exchange rates below this first occurrence will be updated as well. Therefore, if you have different rates for the same currency, we recommend that you make the change from the bottom of the table and move upwards.

     

    Command buttons

    Click on the relevant command button to perform the necessary.

    • Details - Click this to view details for the selected account. This will open the Details screen.

    • Analyses - Click this to open the analyses screen for the selected account.

    • Save - Click this to save the valuation or any changes made to an existing valuation.

    • Cancel/Close - Click to cancel the valuation without saving it. For existing valuations, click Close to exit the folder or Cancel to exit without saving the changes made.

    • Print - Click this to print the valuation details.

    • Post Journal - Click this to post any unrealised valuation accrual journals.

    • Copy Data - Click this to copy the valuation data into Microsoft Excel or Word.

  • Portfolio Valuation

    The Valuation screen shows all sub-ledger accounts that have been linked to the selected portfolio, and their valuation in sub-ledger and portfolio currencies. New valuations will be based on the most recent valuation of the sub-ledger accounts linked to the portfolio.

     

    Valuation Date

    This displays the valuation date entered in the Select folder for new valuations; or the valuation date of a selected valuation, if you are looking at an existing valuation.

     

    Reference

    This defaults to 'Portfolio Val. <Date of valuation>'. Change this if necessary. It is also possible to automate this using User Field (VAL.P.RE). Refer to Automatic Valuation/Revaluation Reference.

     

    Valuation Currency

    The valuation currency is the currency of the selected portfolio, as set up in Account/Master File - Account File - Portfolios. This cannot be changed.

     

    Note

    Enter any relevant note on the valuation.

  • Exchange Revaluation

    The Revaluation screen shows all sub-ledger accounts in currencies that are different from the base currency of the Account File and allows you to correct exchange rate differences in one go. Revaluations utilise the most recent exchange rates available on or before the date of the revaluation.

     

    Revaluation Date

    This displays the revaluation date entered in the Select folder for new revaluations; or the revaluation date of a selected revaluation, if you are looking at an existing revaluation.

     

    Reference

    This defaults to 'Exchange Reval. <Date of revaluation>'. Change this if necessary. It is also possible to automate this using User Field (VAL.X.RE). Refer to Automatic Valuation/Revaluation Reference.

     

    Valuation Currency

    The valuation currency is the base currency of the Account File. This cannot be changed.

     

    Note

    Enter any relevant note on the revaluation.

Exporting Valuation/Revaluation

  1. In the Valuation/Revaluation folder, right-click within the table.

  2. Click the option Export to Excel.

  3. Once complete, it will open in Microsoft Excel.
