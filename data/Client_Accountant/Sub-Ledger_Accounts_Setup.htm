Sub-Ledger Accounts - Setup

The setting up of sub-ledger accounts is done in the Select folder of the sub-ledger screen.

  1. Select the appropriate sub-ledger screen.

  2. In the Select folder, click New.

  3. Enter the information for the new sub-ledger account:

    • Debtor sub-ledger

      • Open Item - Select the Open Items, if relevant. This field will be available if you have any Open Items records in Entity Administration.

      • Master File - If the debtor has been set up as a Master File, then you may select it here. Otherwise, leave this field blank.

      • Sub-ledger Code - Enter a sub-ledger code here. If you had selected a Master File above, the Master File Code defaults to this field. Change this if necessary.

      • Type - Select the sub-ledger type, if relevant. You can add more sub-ledger types in the Sub-ledger Type Values [SLTP] lookup table in System Manager - Business Parameters - Lookup Values. When adding a new sub-ledger type to the lookup table, ensure the code starts with D so that it appears only when you are setting up a Debtor sub-ledger.

      • Debtors (Item) Name - Enter the sub-ledger name. If you had selected a Master File earlier, then the name of the Master File defaults to this field. Change this if necessary.

      • External/Account Code - This field is used to record the external account code of the sub-ledger account, if relevant.

      • Currency Code - Select the account currency.

      • Nominal Account - Link the sub-ledger account to a nominal control account for journal posting and reporting purposes.

      • Portfolio - If the sub-ledger account is to be linked to a portfolio, then select a portfolio from the list.

      • Exchange Revaluation - This refers to the type of revaluation.

      • Last Revalued Exchange - The date when the last exchange revaluation was performed.

      • Valuation - Select the valuation method for the sub-ledger account. The valuation method is fundamental to the Accounting valuation and revaluation functions. If the selected method is Interest/Coupon, the Interest field must also be completed. Additionally, with this valuation method, the account will not be included when the sub-ledger is revalued.

      • Interest % - Enter the percentage interest if your Valuation method is Interest/Coupon.

      • Last Received - The date the interest is last received is used when the Valuation method for the sub-ledger is Interest/Coupon. The field is automatically updated with the journal date when an 'Interest Received/Paid' journal is posted.

      • Revaluation - Select the frequency of the revaluation.

      • Last Revalued Market - The date when last value revaluation was performed.

      • Revaluation are booked to the 'unrealised' accounts - If the checkbox is ticked, the Nominal Account Code shown under Unrealised Value Gains/Losses and Unrealised Exchange Gains/Losses will be used to create revaluation.

      • Capital/Income - This field is normally used for Trust accounting only, and is enabled if the Account File uses a Chart of Accounts template that has been enabled for Capital/Income Segregation. When creating a sub-ledger account you have to specify if journal postings are allowed for capital, income or both types of transactions.

      • Date Opened - This defaults to today's date.

      • Date Closed - This field is used to record the date the account is closed, and is left blank.

      • Note - Enter any notes relating to the sub-ledger account.

      • Active - This is ticked by default. To deactivate the account, untick the checkbox. This will prevent further transactions being posted to the account.
