Sub-Ledger Accounts

There are nine sub-ledger account screens for the different types and they are:

  • Debtors

  • Creditors

  • Bank Accounts

  • Fixed Assets

  • Securities

  • Investments

  • Shares/Unit Holders

  • Products/Services

  • Tax Accounts

These separate screens ensure that your journal entries are as efficient and accurate as possible. For example, if you have to record multiple bank receipts for a client, you can enter journals in the Bank Accounts sub-ledger journal screen. The sub-ledger accounts must be set up before they can be used for journal transactions.

Sub-ledger accounts are created in one of three ways:

  • Manual creation - The setting up of individual sub-ledger accounts is done in the respective sub-ledger journal screens or through Journal Quick Entry screen.

  • Automatic creation via Practice Management posting - The setting up of individual Debtor and Creditor sub-ledger accounts in managed entity (Client) Account Files can be done automatically. This happens when sales invoices are posted to Accounting from Billing. When this happens, a sub-ledger is created with the name <Master File Code>~Currency Code~Nominal Account. For instance, BRIGHT~USD~2010. You must ensure that the accounts for the CoA have been correctly mapped in order to use this feature.

    Where do I map accounts?

    This is done in Viewpoint - Home screen, Configuration > Business Parameters > Practice Management > Account Mapping.

  • Auto creation using tags - The setting up of sub-ledgers linked to a nominal account for new Account File can be done automatically using tags.

In each, you will see the following tabs:

  • Select - refer below.

  • Script - This can be accessed by selecting a journal script and clicking > Script. It shows the individual journal lines for the selected journal script. You may complete or amend the lines as necessary before you proceed to post the journal. This folder is disabled if you do not have any scripts or if you do not select a script for use. The feature 'Accrual Next Day Posting' is available on Journal Script screen. When the checkbox 'Accrual Posting' is ticked, the accrual next day reverse journal will be created.

  • Journal - This can be accessed by clicking > Journal. If you have selected a journal script before you clicked >Journal, it will show you the journal lines for the journal script so you can edit and post a pending journal. If no journal script was selected, you may enter the journal manually.

 

Select tab

In this tab, you will see the following:

 

Options

Click to select the display:

  • Sub-ledger - Shows sub-ledger accounts that you have set up for the type.

  • Open Items - This is available for Debtor and Creditor. When selected, it will show records from File Maintenance > Entity Administration > Assets/Liabilities > Open Items.

  • Statutory - This is available for Bank Account only. When selected, it will show records from File Maintenance > Entity Administration > Bank Accounts.

  • Signatories - This is available for Bank Account only. When selected, it will show signatory records from File Maintenance > Entity Administration > Bank Accounts.

  • Fixed Asset Register - This is available for Fixed Assets only. When selected, it will show records from File Maintenance > Entity Administration > Assets/Liabilities > Assets.

 

Filters

The filters apply to the Sub-Ledger/Record list.

  • Status - You can choose to view by status for both sub-ledgers and records.

  • Type - You can choose to view by the sub-ledger type.

 

Sub-Ledger/Record list

The list shows sub-ledgers that you have set up or available records if you chose other options.

 

Command buttons

Click on the relevant command button to perform the necessary. The buttons apply to the Sub-Ledgers list.

  • New - Click this to set up a sub-ledger account.

  • Edit - Click this to amend the information of the selected sub-ledger account.

  • Delete - Click this to delete the selected sub-ledger account. This is possible only if there is no journal posted to it. Otherwise, consider deactivating it.

  • Settlements - Click this to perform account settlement. This button is available only if the Settlement Method for the selected sub-ledger account is 'Automatically (FIFO Settlement) [8]' or 'Manual Settlement [9]'.

    1. Enter the details:

      • Journal Date - This defaults to today's date.

      • Reference - Enter the reference for the settlement.

      • Description - Enter description for the settlement.

      • Sub-ledger Type, sub-ledger account - Select the sub-ledger type and account where the settlement will be placed. Currently, only posting account's sub-ledgers are allowed.

      • Amount <settlement sub-ledger currency>- Enter the settlement amount. The currency is the currency of the settlement sub-ledger. You can select either DR or CR when entering the settlement amount.

      • Exchange - This is the exchange rate for the settlement based on the journal date. It converts the currency of the settlement sub-ledger to the Account File's base currency.

      • <Base currency> - This is the amount in the Account File's base currency.

      • (Bank) Charges - Enter the bank charges, if any. Note that when there is bank charges, the total amount settled is 'Amount - (Bank) Charges'.

      • Narrative - Enter the narrative for the settlement.

      • Settlement <currency for the selected sub-ledger> - This shows the amount in the currency of the selected sub-ledger but can be changed.

      • Exchange - This is the exchange rate for converting the currency of the selected sub-ledger to the Account File's base currency.

      • <Base currency> - This is the amount in the Account File's base currency.

      • Write off <sub-ledger currency> - Enter any write off amount.

      • Settle this Transaction - Tick this to perform the settlement. Note: When this is ticked, the column filters will be enabled.

    2. To post the journal, click on either

      • Post Journal; or

      • View & Post Journal - this will display the Journal Quick Entry screen and populate the journal details before posting. User can verify the journal details before posting.

  • User Fields - Click this to add user fields to the sub-ledger account.

  • Analyses - Click this to view the analyses done on the sub-ledger account.

  • Statement - Click this to print a statement of the sub-ledger account. Enter the period and select the statement you want printed in the Prepare: <Statement name> screen.

  • Depreciate All - This is available only for Fixed Assets sub-ledgers. Click this to have Viewpoint analyse and suggest depreciation posting for all fixed assets. Refer to Fixed Asset Depreciation.

  • Rebalance All - This is available only for Securities and Investment sub-ledger. Click this to have Viewpoint perform rebalance for all Securities and Investments accounts. Refer to Securities Rebalance.

  • Corp. Actions - Click to view the corporate actions, such as dividends, bonus issues, etc., that have been made with regard to the securities in the Security Master. This function relates to the Securities sub-ledger only. Refer to Security Master - Actions.

 

Additional tabs

Click on the relevant tab to view items for the selected sub-ledger account.

  • Scripts - Displays the journal scripts related to the selected sub-ledger. You can click:

    • > Script - to access the Script tab to create a journal using the journal script for the selected sub-ledger account. This is possible only after you have selected the journal script. In the Script tab, you will see journal lines of the script listed and you need to complete them.

    • > Journal - to access the Journal tab to create a journal manually for the selected sub-ledger account.

  • Transactions - Displays related transactions for the selected sub-ledger account. You can view details for the transactions, print a journal or create a journal based on the selected transaction. If necessary, you can also change non-financial details or the Unit Codes; or if it is a Draft Journal, you can also un-post and edit the journal, or open the AEOI Reporting function. If relevant, you can link a transaction record to documents, binders, file notes and obligations.

  • Journal - Displays related journal lines for the selected sub-ledger type. Similar to Transactions tab, you can view its details, print, create/edit journal details, or open the AEOI Reporting function. In the Journal tab, you can also validate AEOI.

    After the 'Validate AEOI' is performed, it will display the following in the Regime column:

    • Display ‘X’ if there isn't any transaction linked to CRS or FATCA.

    • Display ‘C’ for CRS, or/and ‘F’ for FATCA, or/and 'G' for UK FATCA.

    • Display ‘!’ if the Percentage is not 100%.

  • Settlements - This is seen only if the Settlement Method for selected sub-ledger account is set to 'Automatically (FIFO Settlement) [8]' or 'Manual Settlement [9]'. You can filter by Debit/Credit entries, view open or settled entries. You can choose either:

    • Auto Settle - to settle the transactions using first-in first-out and validates the open amount.

    • Settle - this will open the Account Settlement Linking screen where you can select, edit and enter the amounts to be offset against the selected journal line from the sub-ledger screen. Note that partial payments is possible.
