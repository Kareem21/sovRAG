Statement Import & Processing - Source File Amendment

Introduction - Amendment of the Source File

Statement Import & Processing allows you to amend the source file that are provided by Viewpoint. However, there are some conditions and guideline that you need to follow to amend the source file.

You need to have a basic knowledge on the formatting or syntax and be familiar with the process tags of the source file in order for you to amend the source file.

 

How do I amend the Source File?

It is recommended that you configure and amend the source file with the supervision of Viewpoint support representative. However, you can amend the source file by yourself under several circumstances. For example, changing the format date or currency is permissible without Viewpoint supervision.

 

Steps to Amend the Source File

Follow the steps below to configure the source file:

  1. Ensure that you have the source file in your system.

  2. Right click on the source file and click ‘Edit’ button.

  3. The source file will open using the WordPad / Text Editor or any other application depending on which has been installed in your system.

  4. Please refer to the configurable fields of the source file section in order to understand area of fields in the source file that is configurable or amendable.

  5. After amending the source file, you need to import the newly edited source file into Viewpoint Connect again. Please refer to the ‘Statement Import and Processing’ topics on how to import the source file.

Content of Source File

The contents that are included in the source file are Statement Type, Mapping Fields for Processing File Type, etc. Refer to the configurable fields table.

 

Configurable Fields of the Source File

The following table shows the configurable fields in the source file with the descriptions and examples.

Field Name Description Example

FormatID

This field indicates the statement type specific format code. This code is the unique identifier – particularly used when statement format is imported into client environment where the statement type with the same code will be overwritten.

<FormatID>CitiBankNYTRAN</FormatID>

ValidateBankMF

This field indicates that this is flagged to validate the bank Master File on recognition.

Example of value can be assigned will be ‘Yes/No’.

<ValidateBankMF>False</ValidateBankMF>

RecField

This field indicates the recognised account code based on the field below:

  • SL - Subledger code (AcctItems.SlCode).

  • SE - External/IBAN code (AcctItems.ExtAcctNo).

  • U1 - User Defined Subledger Code - ACODER_2.

  • U2 - User Defined Subledger Code - ACODER_2.

  • BA - Bank Account Code (AcctItems.LinkStatRec).

<RecField>SE</RecField>

SourceDescr

This field indicates the description of the source.

Example: CitiBank NY Business Statement 1.0.

<SourceDescr>CitiBankNYXML</SourceDescr>

SourceNote

This field indicates the source note.

<SourceNote>ABC</SourceNote>

StatType

This field indicated the status type of the transaction. It is represented as below:

  • T – Transactions

  • B – Account Balances

  • M – Transaction and Balances

  • S – Statement Source Configuration

<StatType>T</StatType>

FileType

This field indicates the Source File Format which can be:

  • CSV/F - CSV file with fixed length fields

  • CSV or CSV/V - CSV file with variable length fields

  • XML - XML file

<FileType>XML</FileType>

IsActive

This field indicates the statement type activity status.

  • Yes – Indicates statement type is Active.

  • No – Indicates statement type is Inactive

<IsActive>True</IsActive>

RecDate

This field indicates the Reconciliation Date such as (TransDate, ValueDate, SettleDate). The default is TransDate.

<RecDate>TransDate</RecDate>

 

 

AstatSource.ProcessTags

 

Tags for processing recognised transaction for mapping of the transaction to journal header fields.

The following table is the AstatSource process tags along with the descriptions and examples. You can refer to the table below for each of the process tags.

Process Tags Description Example

[/FORMAT-DATE=/]

This defines the date format used.

[FORMAT-DATE=yyyymmdd]

 

The above means the format date is year, month and day.

[/FORMAT-CULTUREINFO=/]

This tag will follow the statement format date.

[FORMAT-CULTUREINFO=en-UK]

en-UK will recognise date format as used in the United Kingdom which is Day-Month-Year.

[/AMOUNT_NO_REVERSE=1/]

Do not reverse the amount.

[/AMOUNT_NO_REVERSE=1/]

 

The value 1 above means YES which will not reverse the amount. If the value is 0, the amount will be reversed.

[/SUPPRESSZERO=1/]

This tag is used if you do not wish to import transactions with zero amount.

[/SUPPRESSZERO=1/]

 

The above using the value 1, which means YES and it will not import the transaction with a zero amount. If the value is 0, means the user wants to import the transaction with zero amount.

[/DefaultCurrency=<CurrencyCode/]

 

where <CurrencyCode> is the ISO currency code

Default currency. This is used when the transaction line does not contain a currency code.

[/DefaultCurrency=EUR/]

 

The above means the default currency used will be Euro.

[/HEADER.1.VAL=/]

This tag is used to detect the header that contains column names.

[/HEADER.1.VAL=0,Sort Code,Account Number,Currency Code,Amount,Entry Type,Transaction Description,Cheque Number,Narrative/]

[/HEADER.1.VAL-LINE=/]

This tag is used if the header is not in row 1.

[/HEADER.1.VAL-LINE=2/]

 

The above means the header starts in row 2.

[TRLINE-LINE=<X>/]

[TRLINE-LINE=<X>~<Y>/]

 

where <X> is the starting line number and <Y> is the last line number.

This tag is used to define the starting point for the import and applies only to CSV files.

[/TRLINE-LINE=2~100/]

 

The above means the import will apply only to the second line to the 100th line of the file.
[/TRLINE=/]

This tag is used to define the column position or column name.

CSV - [/TRLINE=TR.AccountNumber=P_33:46;TR.StatDate=P_2:12; /]

XML - [/TRLINE=TR.AccountNumber=AccountNumber;/]

This tag is used to define the Debit Credit Handling.

TR.<ColName>=$_DCR_+P_00:00~-P_00:00

This tag is used for Returns Specified Characters

Returns specified numbers of characters from a string. X = Start, Y = Length.

TR.<ColName>=F_0|$_POS_X~Y

This tag is used to define the Negative Value Handling.

TR.<ColName>=P_XX:XX|$_NEG_P_XX:XX=CR

[/TRLINE-GEN_<Field>=<Position>/]

 

where

  • <Field> can TransDate, AccountNumber or TransCur

  • <Position> is defined as L_X|F_Y with X being the line number and Y being the field number

This is used to specify the position of the following fields if these are available in the header but not in the details line:

  • TransDate

  • AccountNumber

  • TransCur

[/TRLINE-GEN_TransDate=L_1|F_4/]

 

The above means the TransDate field is in the first line and at the fourth column.

[/TRLINE=<VpField>=<SourceFileLocation>;...;/]

 

where

  • <VpField> refers to the field name in Viewpoint

  • <SourceFileLocation> refers to the location of the information in the source file from the bank and it can be by:

  • specifying the position as P_X:P_Y with X being the starting position value and Y being the ending position value if the source file is a fixed length CSV file

  • specifying the position of the information as the field number if the source file is a variable length CSV file

  • specifying the column name if the source file is XML

If there is a need to specify a debit and credit fields, then the following is added:

  • $_DCR_-<FieldPosition>~+<FieldPosition> where <FieldPosition> is defined for both credit and debit fields using the field number

  •  $_DCR_P<PositionX>:<PositionY>~+<PositionY> where <PositionX> is the starting position and <PositionY> is the ending position for the credit and debit fields

 

This is used to map Viewpoint field to the information in the source file.

 

[/TRLINE=TR.AccountNumber=P_33:46;TR.StatDate=P_2:12; /]

 

The above maps the following fields to fields in the CSV bank file:

  • AccountNumber to the information in position 33 to 46

  • StatDate to the information in position 2 to 12

 

[/TRLINE=TR.AccountNumber=AccountNumber;/]

 

The above maps the AccountNumber field to the column named AccountNumber in the XML source file.

 

[/TRLINE=TR.TransDate=F_1;TR.AmountTrans=$_DCR_-F_5~+F_4;TR.ALineDescr=F_3;TR.AlineText=F_2;/]

 

The above maps the following fields to fields in the CSV bank file:

  • TranDate to the first field

  • AmountTrans to the fifth field if it is a credit amount and the fourth field if it is a debit amount

  • LineDesr to the third field

  • AlineText to the second field

[/TRLINE-GEN=/] – CSV

This tag is used to get the last string of general tag.

[/TRLINE-GEN_AccountNumber=L_3|F_1|$_POS_LASTSTRING/]

[/DELIMETER=/] – CSV

Delimiter uses for CSV.

[/DELIMETER=;/]

[/IMPORT_ OPTION=/] – CSV

Functions of this tag as below:

  • Use TextFieldParser to split file. This option is used when cell data contain Line Feed.

  • Use to ignore closing statement.

  • Use to ignore opening statement.

[/IMPORT_ OPTION=1/]

[/NUMSEP=/]

This tag is used to define decimal type for the amount.

Available type:

1 = Comma (,)

2 = Dot (.)

3 = Single Quote (‘)

[/NUMSEP=2/]

 

The above means it is using Dot (.) as a decimal point for the amount.

CSV and XML Configuration Differences

CSV and XML file format have different types of configuration within the transaction and balance. Below shows the differences between CSV and XML transaction format.

For example, take note that the format of the process tags for CSV to define the account number is '[TRLINE=TR.AccountNumber=F_2;TR.TransDate=F_3;/]' while the format to define an account number in XML format is ‘[/TRLINE=TR.AccountNumber=AccountNumber;TR.TransDate=TransactionDate]’. Refer to more examples below.

 

CSV – Transaction

<?xml version='1.0' ?>

<ViewPointReconcilationSource>

<FormatID>EQRBCT</FormatID>

<ValidateBankMF>False</ValidateBankMF>

<RecField>SE</RecField>

<SourceDescr>RBSI Transaction-CSV</SourceDescr>

<StatType>T</StatType>

<FileType>CSV/V</FileType>

<RecTags>

</RecTags>

<ProcessTags>

                [/TRLINE=TR.AccountNumber=F_2;TR.TransDate=F_3; /]

                [/FORMAT-DATE=dd/mm/yyyy/]

                [/DELIMETER=,/] <!--this require inside ProcessTags element-->

</ProcessTags>

<IsActive>True</IsActive>

<RecDate>TransDate</RecDate>

</ViewPointReconcilationSource>

 

CSV – Fixed - Transaction

<?xml version='1.0' ?>

<ViewPointReconcilationSource>

<FormatID>EQRBST</FormatID>

<ValidateBankMF>False</ValidateBankMF>

<RecField>SE</RecField>

<SourceDescr>RBSI Transaction</SourceDescr>

<StatType>T</StatType>

<FileType>CSV/F</FileType>

<RecTags>[/FILELIKE=RBSI*TRN.atr/]</RecTags>

<ProcessTags>

                [/TRLINE=TR.AccountNumber=P_20:33;TR.TransDate=P_36:46; /]

                [/FORMAT-DATE=yyyy/mm/dd/]

</ProcessTags>

<IsActive>True</IsActive>

<RecDate>TransDate</RecDate>

</ViewPointReconcilationSource>

 

XML – Transaction

<?xml version='1.0' ?>

<ViewPointReconcilationSource>

<FormatID>RBSIXMLT</FormatID>

<ValidateBankMF>False</ValidateBankMF>

<RecField>SE</RecField>

<SourceDescr>RBSI XML Transaction</SourceDescr>

<StatType>T</StatType>

<FileType>XML</FileType>

<ProcessTags>

   [/TRLINE=TR.AccountNumber=AccountNumber;TR.TransDate=TransactionDate; /]

   [/SUPPRESSZERO=1/]

   [/FORMAT-DATE=yyyy/mm/dd/]

   [/TRLINE-XML=StatementEntry/]

</ProcessTags>

<IsActive>True</IsActive>

<RecDate>TransDate</RecDate>

</ViewPointReconcilationSource>
