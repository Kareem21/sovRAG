Payment Types & Export Batches - Tag Values

The following are tag values that can be added for the export batch configuration.

 

Tag

Explanation and Example

[<Field>.<Setting>=<Value>]

where

  • <Field> can be:

  • PayCharge = Charges

  • PayReference = Payment Reference

  • PayNote = Payment Note

  • <Setting> can be:

  • Visible - to set the field as visible

  • Value - to set a default value

  • Enabled - to set the field as editable

  • <Value> refers to the value for the <Setting>

 

This is to allow the define the field settings.

 

Example:

[PayCharge.Visible=Y]

[PayCharge.Enabled=N]

 

This sets the Charges field as visible but not editable.

 

 

 

 

 

 

[PayReference.Chars=<Value>]

where <Value> can be

  • A special character like hyphen, backslash, etc.

  • A to accept uppercase alphabet.

  • A to accept lowercase alphabet.

  • 0 to accept numbers.

 

This is define acceptable input for the Payment Reference field.

 

Example:

[PayReference.Chars=Aa -0]

 

This will allow input containing uppercase and lowercase alphabet, numbers, hyphen and space so the following are acceptable:

  • Dp - 00012017

  • DEP - Bk1001

 

[PayReference.Chars=Aa-/0]

 

This will allow input containing uppercase and lowercase alphabet, numbers, hyphen and backslash so the following are acceptable:

  • Dp-0001/2017

  • DEP-Bk1/001

 

[PayDate.Validate=<Value>]

where <Value> can be Y for Yes and N for No.

This is to validate the PayDate value on data entry and export.

 

Example:

[PayDate.Validate=Y]

 

If any of the payments' Pay date is before today, the export will be rejected.

 

[AutoRelease=Y]

 

This is to release the payment upon final approval. This means that there is no need for user to manually release the payment.
