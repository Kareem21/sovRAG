Journal Tab

The Journal tab is where you enter journals directly to a nominal posting account or to a sub-ledger account. This screen is also used if you have been using a journal script and now need to add, edit or delete one of the script's journal lines.

In the screen, command buttons are placed at the top of the screen, followed by the journal information in the middle of the screen. The bottom of the screen displays fields for data entry.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • New/Update - Click this to save changes or add a new journal line. The button will change to Update. When you have entered the line details, click this button to save the line.

  • Post - Click this to post your entry. The button will only be enabled if your journal lines balance.

    Why was my journal posted as a draft?

    The Draft Journals option is ticked for the Account File in Account File - General screen. See Draft Journals for more information.

  • Print - Click this to print a voucher from the report template library. The document can either be displayed on the screen in an Excel format or be sent directly to the printer. The template that will be generated will, by default, be the last template used. To change the template, click the Select>> button to view a list of the available templates.

  • Save Pending - Click this to save the journal as a pending journal. This option allows semi-completed transactions, such as those that do not balance, or have incomplete details, to be saved and finished at a later time.

    I saved the journal as a pending journal. Where is it now?

  • Cancel/Cancel Journal - Click this to cancel changes made to an existing journal line OR delete the entire journal. When you edit a journal line, the button shows Cancel, and is used to abort the changes made to that particular line. Once you have updated the journal, the button changes to Cancel Journal, which is used to cancel the transaction and delete all the lines entered.

  • Delete Line - Click this to delete the selected journal line.

 

Show Options...

Tick this to show/hide the following fields:

  • Journal type - This field shows if the journal is a regular journal, which is the default selection, or an accrual journal to be automatically reversed the next day.

  • Default 'Description' - Tick this to automatically populate the Description field of new journal lines with the Description of the first journal line.

  • Default 'Line Date' - Tick this to automatically populate the Line Date field of new journal lines with the Line Date of the first journal line.

  • Default 'Narrative' - Tick this to automatically populate the Narrative field of new journal lines with the Narrative of the first journal line.

  • Display Warnings - Tick this to display warnings. For instance, if you enter a credit amount when a debit amount is expected, a warning will be given, which you may opt to ignore.

 

Journal Date

This field displays the journal date. When you create a new journal, this defaults to today's date. For existing journals, such as those in the Pending screen, this will show the date the journal was created.

 

Reference

Enter the journal reference in this field. If you have the Journal Reference user field (JNLREF) set up, then this will be auto-generated.

What is the Journal Reference (JNLREF) user field used for?

 

C/I

Select if this is a capital or income transaction. Note that this field is available only if the Chart of Accounts of the selected Account File has been enabled for Capital / Income Segregation, as is required for Trust files.

  • Capital/Income Segregation

 

Journal line

This shows the journal lines that you have added. You can:

  • Re-order the journal lines using click-and-drag - click to select the line and drag to desired position.

  • Sort the journal lines - click on the column header to sort by column.

Note: Once you sort by a column, it is no longer possible to re-order the lines.

 

Account type

Select the account type, which may be a nominal account or a sub-ledger account. If you select Nominal, an additional sub-ledger field will appear next to the Account field. The sub-ledger field functions like the Movement Types field (see below) on the control accounts. Selecting the sub-ledger account will allow you to analyse the transactions in the nominal account, which only has one movement type. For instance, if you select the Sales nominal account, you can enter the Debtor sub-ledger to record which sales are credit sales and to whom. You can also select a Bank Account sub-ledger to record cash sales and where you banked in the proceeds.

 

Account

Select the account that the journal line will be posted to.

 

SL

Select the sub-ledger account that the entry will be posted to. This field is only displayed if your account type is Nominal.

 

Movement Type

Select the movement type from the list. The selections available depend on the account type selected.

 

Currency/Amount

Select the currency and amount of the transaction. If your account type selection is Nominal, the currency defaults to the base currency of the Account File, but can be changed if required. If you select a sub-ledger account type, the currency of the sub-ledger account is automatically displayed.

 

Exchange

The exchange rate shown is calculated from the conversion of the currency entered to the base currency, if applicable.

 

Amount in base currency

The system calculates and displays the amount of the journal line in the base currency here.

 

Description

Enter the description of the transaction.

 

Line Date

This defaults to today's date. Change this if necessary.

 

Tax Account

Select a tax account, if applicable. This field is only displayed if you have set up at least one tax account.

Where do I set up tax accounts?

This is done in the Tax Accounts sub-ledger screen.

 

Reconciliation Date

Enter a reconciliation date for the transaction, if applicable. If you are entering a journal based on a third party document, such as a bank statement, then this would be the date that this entry appears in the statement.

Note:

  • Bank Account sub-ledger - This is the field that is completed if you use the auto-reconcile function in the Electronic Bank Statement Import & Processing feature. See Statement Processing for more details.

  • Investment sub-ledger - In a group of companies, this might be the date of documentation, such as a resolution, that states the holding company is investing in the subsidiary, etc.

 

Narrative

Enter a narrative for the transaction, if applicable.

 

Organizational Unit

This field is displayed only if you are using the feature.

How do I enable/disable the Organisational Unit feature?

In Viewpoint - Home screen, click Configuration > Settings > Tools & Utilities > Localise and then select Enable/Disable Organisational Unit under the General group. This ensures that the fields are enabled/disabled throughout Viewpoint.
