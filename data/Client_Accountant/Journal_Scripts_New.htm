Journal Scripts - New

  1. In the Customise Journal Scripts screen, select the sub-ledger account type that you want to create a new journal script for. For example, acquisition of a freehold property would be a Fixed Assets sub-ledger transaction, a sale of listed securities would be a Securities sub-ledger transaction. In the former example, the script will appear in the Select folder of the Fixed Assets journal screen; in the latter, the Securities journal screen.

  2. Click New.

  3. A new line will appear in the journal header section of the screen. By default, the new line will replicate some details from the line above, if applicable. You can also edit these details when necessary.

  4. Complete the fields of the journal header accordingly:

    • Group - Enter the group caption or description. This is used to group similar journal scripts together. If left empty, the program will default to the Group caption of the previous row.

    • Description - Enter the description of the transaction. This will be shown on the Select folder of the journal screen.

    • Type - Select the journal type. For accrual journals, the program will automatically post a reversal journal on (Journal Date + 1 day).

    • Cap/Inc - This field is relevant for Charts of Accounts templates that have been enabled for Capital/Income Segregation. When defining scripts where the C/I segregation is not required, select B to allow both capital and income transactions.

    • Caption Amount - Enter the caption for the Currency/Amount field to precisely describe the amount entered. For instance, you may change this to Total Debit Note or Gross Proceeds Received.

    • Default Reference - Enter any text and a valid field code in the format <text>{Field_code}<text>. See Default Reference in the Examples section below.

    • Active - Tick to ensure that the script can be used. Remove the tick if you no longer want it displayed in the Select folder.

    • Customized - Tick to flag a script as customized to prevent it from being overwritten on (re)import.

    • Code - The system will assign a unique code for the script. This cannot be changed.

    • Help Text - Enter information to assist users in deciding which script to use. Be as concise and precise as possible. If there are certain parameters to the journal, for example: "Only complete line 3 if any Foreign Exchange is involved”, put this information here.

  5. Once the journal header details are complete, enter the individual journal lines to build the complete journal.

  6. In the journal line section, click Add Line.

  7. Enter the details of the journal line:

    • Number - This is an incremental number which is assigned by the system. You may use this in field codes for line referrals.

    • Line Description - Enter the description of the line. This will be displayed on the Script folder of the sub-ledger journal Accounting - Journals, and denotes the purpose of the line. This description is used to provide the user with information about the respective journal line and can become part of the journal.

    • A-Type - Select the nominal or sub-ledger account type to which the journal line is being restricted. This field is always the sub-ledger account type for the first line. For instance, if you are creating a script in the Creditor sub-ledger, the A-Type is C - Creditor for the first journal line. This cannot be changed.

    • Movement - Select the movement code for the journal line. The options available for selection depend on the A-Type selected.

    • DR/CR - Select the movements allowed for the journal line.

    • Account - Specify the accounts that are allowed for selection of the journal line. The code begins with the selection type (like NT.) with the allowable values after the period (.). Multiple values are separated by a comma. See Account in the Examples section below.

    • D-Description - Enter any description, reference or narrative field. See D-Description and D-Narrative in the Examples section below.

    • D-Line Date - The date field can refer to the journal, line or reconciliation date of the header, the system date or dates of preceding rows. See D-Line Date and D-Rec Date in the Examples section below.

    • D-Rec. Date - The date field can refer to the journal, line or reconciliation date of the header, the system date or dates of preceding rows. See D-Line Date and D-Rec Date in the Examples section below.

    • D-Narrative - Enter any description, reference or narrative field. See D-Description and D-Narrative in the Examples section below.

    • Amount - Select the appropriate line-related amount. Refer to Amount in the Examples section below.

    • Tags - Find out more here.

      [@BC]

      This tag forces the posting to nominal account to be in base currency instead of the header currency. This code can be useful, for example, for the gain/loss on sub-ledger account disposal, where you want the gain/loss to be in the base currency instead of in the sub-ledger currency.

       

      [Warning=No]

      This is placed in the first line of the script to skip any warning message that may be displayed when posting the script.

       

      [@JNC=<COS option>]

      This code is specifically designed for ‘sales invoicing’. Place this in journal lines for Products/Services sales to automatically generate the ‘stock issue and cost of sales’ entries for products/services. The cost of sales amount will depend on the options selected:

      • COS.A - Cost of sales/average cost.

      • COS.F - Cost of sales/FIFO.

      • COS.S - Cost of sales/following the sub-ledger valuation setting.

      [@HEXCH]

      Place this code in lines that should use the header exchange rate to calculate the amount, instead of the sub-ledger rate or the rate from the Exchange Rate Master.

      • If you want to change the header rate, you need to first delete the amount on the journal line with this code, and then change the rate in the header. If you change the header rate without first deleting the amount, the line will retain the original header exchange rate.

      [Initiate=FADEPR]

      Place this code in the line for Depreciation Journal Script in order to open the Depreciation screen for the depreciation to be done before the Disposal journal. See also Fixed Asset Depreciation

  8. Once all the journal lines have been added, select the Chart(s) of Account templates that the script can be used with. Tick the checkboxes next to the appropriate templates.

  9. Click Update.

 

Examples

Journal Header

  • Default Reference

    You may want to use the {UFV.FIELDKEY} code for automatic incremental numeric references. This reference will be applied to all lines of the journal.

    This code retrieves the field value from the user defined field with the ‘FIELDKEY’ specified. If the field type is a number, and the value numeric, the program will automatically increment the number.

    For example, replace

    <text #>{Field_code}<text>

    with

    E#{UFV.JNLREF} - UL

    and the result will be:

    E#002 - UL

     

    Refer to Automatic Journal Reference.

 

Journal Line

  • Account

    The following will explain the types of codes that you can use in this line:

     

    #x#ACODE

    This copies the account code from the row specified by x.

     

    SL.X

    Sub-ledger accounts of the specified type.

     

    NT.1,2,3

    Posting accounts of the listed account type(s). In the above example, account types 1, 2 and 3.

     

    NM.1,2,3

    Posting accounts of the listed main group(s). In the above example, main groups 1, 2 and 3.

     

    NS.12,23

    Posting accounts of the listed sub group(s). In the above example, sub-groups 1, 2 and 3.

     

    NA.4567,6789

    The specified posting accounts. In the above example, accounts 4567 and 6789.

     

    AS.ABCDE,BDEF

    Specify the sub-ledger accounts that can be selected. In the above example, sub-ledger accounts ABCDE and BDEF.

     

    DV

    Enter a divider line. The program will create an empty line and ignore any values on this line except for the line description. This is useful for separating line groups in large journals.

     

    #H#CP.{CODE}

    This retrieves the {CODE} account as specified in the properties of the nominal control account linked to the sub-ledger account selected in the journal header. The accepted values are:

    • EXCHGL - Both exchange gain and loss accounts*

    • VALUEGL - Both value gain and loss accounts*

    • INCEXP - Both income and expense accounts*

    • EXCHGAIN - Exchange gain

    • EXCHLOSS - Exchange loss

    • CAPGAIN - Capital gain

    • CAPLOSS - Capital loss

    • INCOME - Income

    • EXPENSES - Expenses

    • UNREALREVAL - Unrealised value revaluation

    • UNREALEXCH - Unrealised exchange revaluation

    • UNREALREVALGAIN - Unrealised value revaluation - Gains

    • UNREALREVALLOSS - Unrealised value revaluation - Losses

    • UNREALEXCHGAIN - Unrealised exchange revaluation - Gains

    • UNREALEXCHLOSS - Unrealised exchange revaluation - Losses

    *The program will automatically switch between the accounts based on the amount value (debit/credit).

    For instance:

    #H#CP.EXCHGAIN = Exchange gain posting account as specified in the properties of the nominal control account linked to the sub-ledger.

     

    SN.X.{CODE}

    This retrieves the sub-ledger accounts of the specified type and posts the line amount to the {CODE} account as specified in the properties of the nominal control account linked to the sub-ledger account selected in this journal line. The accepted values are:

    EXCHGAIN = Exchange gain

    EXCHLOSS = Exchange loss

    CAPGAIN = Capital gain

    CAPLOSS = Capital loss

    INCOME = Income

    EXPENSES = Expenses

    UNREALREVAL = Unrealised value revaluation

    UNREALEXCH = Unrealised exchange revaluation

    456 = The listed (specified) posting account

    Example:

    SN.B.EXCHGAIN = Exchange gain posting account as specified in the properties of the nominal control account linked to the Bank sub-ledger selected in this journal line.

    • EXCHGAIN - Exchange gain

    • EXCHLOSS - Exchange loss

    • CAPGAIN - Capital gain

    • CAPLOSS - Capital loss

    • INCOME - Income

    • EXPENSES - Expenses

    • UNREALREVAL - Unrealized value revaluation

    • UNREALEXCH - Unrealized exchange revaluation

    • 456 - The listed posting account 456.

    For instance:

    SN.B.EXCHGAIN = Exchange gain posting account as specified in the properties of the nominal control account linked to the Bank sub-ledger selected in this journal line.

     

     

    See also:

    • Chart of Accounts - Nominal Accounts

     

  • D-Description and D-Narrative

    This can be a combination of text and field codes in the format <text>{Field_code}<text>. The field codes can refer to another field in the journal header, or a preceding journal line. For example, {#2#LINED} will pull in the Line description from journal line 2.

    Multiple field code values can be used at one time.

    For instance, the narrative for securities purchased might be:

    Purchase of {#H#QNT} {#H#ACCT}

    which will result in

    Purchase of 1000 Google.com.

     

    The following explains the variables that can be used:

     

    {REF}

    This picks up the journal reference.

     

    {#x#LINED}

    This copies the line description from the row specified by x.

     

    {#x#DESCR}

    This picks up the description from the row specified by x.

     

    {#x#NARR}

    This picks up the narrative from the row specified by x.

     

    {#x#QNT}

    This picks up the quantity from the row specified by x.

     

    {#x#CCY}

    This picks up the currency from the row specified by x.

     

    {#x#CAMT}

    This picks up the amount in the currency of the transaction from the row specified by x.

     

    {#x#BAMT}

    This picks up the amount in the base currency from the row specified by x.

     

    {#x#EXCH}

    This picks up the exchange rate from the row specified by x.

     

    {#x#ACCT}

    This picks up the account name from the row specified by x.

     

    For Bank Accounts journal scripts, the following code can be used to pick up value for cheque reference:

    {UFV.CHQREF}

    This picks up the value for the user field 'CHQREF'. Refer to Automatic Cheque Reference.

     

  • D-Line Date and D-Rec.Date

    The following are the variables that can be used in this field:

     

    {NOW}

    This will pick up the system date.

     

    {NOW+0}

    This will pick up the system date plus or minus a number of days. For example: {NOW+10} or {NOW-10}.

     

    {JND+0}

    This will pick up the journal date plus or minus a number of days. For example: {JND+10} or {JND-10}.

     

    {#x#LND+0}

    This will pick up the line date from the row specified by x, plus or minus a number of days. For example: {#H#LND+10} or {#H#LND-10}.

     

    {#x#RCD+0}

    This will pick up the reconciliation date from the row specified by x, plus or minus a number of days. For example: {#H#RCD+10} or {#H#RCD-10}.

     

    HIDE

    Use this to hide (disable) the date. This code can only be specified in the header (1st) line of the script.

     

  • Amount

    The following describes the variables that can be used in this field:

     

    BAL

    This is used to show the offset of the running total of the preceding script lines, including the header.

     

    BLS

    This behaves in the same manner as BAL if the preceding lines and the current line are in different currencies. However, if the preceding lines and the current line are in the same currency, the program will complete for the current amount, the offset of the total of the preceding lines i.e. balance the journal up to the line in the currency amount. The base amount is calculated using the exchange rate of the line.

     

    #x#

    This is used in field codes as {#x#FIELDCODE}. The line referral codes cannot be used for all codes and it cannot refer to journal lines below itself - only to the header or lines above. If this code is omitted, e.g. {FIELDCODE}, the codes will refer to the fields in the same line. The codes that can be used are:

    • #H# = Header (#1# = also refers to header)

    • #P# = Previous line

    • #3# = Line number 3. Note that the first journal line number is 1.

     
    #x#EXS

    This will show the exchange difference between the sub-ledger exchange rate and the exchange rate of the row specified by x.

     

    #x#EXR

    This is the same as #H#EXS with the amount reversed.

     

    #x#EXH

    This will show the exchange difference between sub-ledger rate and the exchange rate at the journal date as retrieved from the daily rate table.

     

    #x#EXG

    This is the same as #x#EXH with the amount reversed.

     

    999

    The script will default to this amount when it is used. Amend the figure to correctly represent the amount you want to use.

     

    #x#9%

    This is the percentage of the amount entered in the row referred by x. In the above example, the rate used is 9%.

     

    #x#AMT

    This will show the amount of the row referred to by x.

     

    #x#AMR

    This is the same as #x#AMT with the amount reversed.

     

    The following codes can only be used on the first (header) line:

    TOT+N

    Total of underlying lines.

     

    BSL+N

    Total book value of the sub-ledger account.

     

    SBS+N

    Book value as per the valuation method set for the Account File.

     

    SBA+N

    Book value average cost.

     

    SBF+N

    Book value FIFO.

     

    TZA+N

    Zero CCY+Base Amount Line

     

    TZV+N

    Zero CCY Amount Line

     

    TZX+N

    Exchange difference (zero CCY amount)

     

    The following code is applicable where the first (header) line refers to a sub-ledger with control quantity only:

     

    SSP+N

    Security Unit Price x Quantity.

     

    Note: +N indicates that the currency value CANNOT be edited by the user. This directive is optional, can be used for all codes and is recommended for the SBx codes.

    For non-base currency (header and journal) lines, the exchange rate and base currency fields are editable, except for lines involving sub-ledgers. For TOT the non-base currency is editable, except where the line refers to sub-ledgers.
