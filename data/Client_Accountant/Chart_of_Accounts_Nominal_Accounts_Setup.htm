Chart of Accounts - Nominal Accounts Setup

  1. Enter the information required to set up an account:

    • Account Code - Enter the account code.

    • Account Type - Specify the account type, whether it is a posting account (denoted by numbers) or a control account (denoted by letters of the alphabet representing the sub-ledger types). System control accounts (denoted by '9') should also be set up for profit/loss transfers and retained earnings (balance sheet). These system control accounts are used by the system exclusively and updated automatically whenever a draft journal or permanent journal are posted.

      Profit/Loss Account Types

      3 - Expense Posting

      2 - Liability Posting

      4 - Income Posting

      B - Bank Account

      9 - System Control

      C - Creditor

      D - Debtor

      E - Security

      F - Fixed Assets

      H - Share/Unit Holders

      I - Investment

      P - Product

      W - Tax Account

      9 - System Control

    • Name - Enter the name of the account.

    • Sub Group - Specify the sub group the account belongs to. The system can only link a balance sheet account to a sub group which has been set up as a balance sheet account.

    • Specify Sub-Ledger - Indicate whether it is mandatory to specify a sub-ledger when posting to this account. This field is only activated when the selected Account Type is a posting account.

    • Ledger Type Allowed - If it is optional or mandatory to specify a sub-ledger type, then select the sub-ledger type that is allowed. This field is only activated when the selected Account Type is a posting account.

    • Settlement Method - By default, this field is not enabled. When enabled, it is available only for control accounts (denoted by letters) and is used to link a debtor/creditor transaction to a bank transaction. The available methods that can be selected are:

      • No settlement required [0] - This is the recommended for account types that are not B, C or D.

      • Through Reconciliation screen [1] - This is the recommended for account type B. The settlement is to be performed through Account File > Accounting > Processing > Reconcile Account screen.

      • Automatically (FIFO settlement) [8] - This is the recommended for account type C and D.

      • Manual Settlement [9] - With this method, the transactions can be settled using the Auto Settle or Settle button in the Sub-Ledger Accounts screen.

    • Capital/Income - Determine if the given account is allocated to income, to capital, or to both income and capital. This is applicable to Trust accounts.

    • Tag Values - You can enter tags to:

      • Limit Master Files that can be linked as a sub-ledger to the nominal account

        [VAL_RELATION=CODE1,CODE2,CODE99]

        This is used to allow only ‘Relations’ with the codes CODE1, CODE2 and CODE99 to be linked.

         
        [VAL_TABLEFIELD=TABLE.CONDITION\message if validation fails]

        where

        • TABLE = The table name for validation; 

        • CONDITION = The WHERE clause. Use (-EN-) for the Account File code and (-MF-) for the related Master File code; 

        • Use a pipe character | to add multiple allowable options to the VAL_TABLEFIELD clause. See example below.

         

        Example

        [VAL_RELATION=DIR,MGR,SEC]

        [VAL_TABLEFIELD=Beneficiaries.EntCode='(-EN-)' AND Beneficiary='(-MF-)'\must be a beneficiary of this entity|BankAccounts.EntCode='(-EN-)' AND BankAddrCode='(-MF-)'\must be related as a banker to statutory bank account]

      • Set Warning and Alert messages

        [Warning=text message]

        This will display the message only if 'Display Warnings' option is ticked.

         

        [Alert=text message]

        This will display the message regardless of whether or not 'Display Warnings' option is ticked.

        How do I use this tag?

        The [CREATE|...] tag is to be defined in the Tag Value field for the nominal account. This can be done when you are creating the nominal account or added to an existing nominal account.

        1. In Viewpoint - Home screen, go to Accounting > Chart of Accounts > Nominal Accounts.

        2. Select the Chart of Account and either click New to create the nominal account of select the existing nominal account the sub-ledger is for.

        3. In Tag Value field, enter the complete tag.

        Note: The creation of the sub-ledgers will apply only to new Account File. Adding the tag to existing nominal account will not create the sub-ledgers in current Account Files.

         

      • Customise depreciation journal entries

        When defining the tags, use the following syntax:

        [<Tag>=<Value>]

        where:

        • <Tag> is the supported tag

        • <Value> can be normal text or field code such as:

          • (-UFV.JNLREF-) - refers to the User Field 'JNLREF'

          • (-SL.<FieldName>\<Format>-) - can be any field from the sub-ledger with or without formatting

          • (-JL1.<FieldName>\<Format>-) - can be any field from journal line 1 (Sub-ledger) with or without formatting

          • (-JL2.<FieldName>\<Format>-) - can be any field from journal line 2 (Nominal Account) with or without formatting

        Note: If using field code, ensure that <FieldName> and <Format> is replaced with the actual field and format option.

         

        Tag

        Explanation

        Example

        REF

        This is to specify the reference for the journal line. The default value for this is from the JNLREF User Field.

        [REF=DEPR~(-UFV.JNLREF-)]

        NomDepr

        This is to specify the Nominal Account for the depreciation entry. The default value for this is the Nominal Account set for Expenses.

        [NomDepr=AP: Expenses]

        Descr_SL

        This is to specify the description for the sub-ledger’s journal entry. The default for this is ‘Depreciation’.

        [Descr_SL=Depreciation]

        Descr_NA

        This is to specify the description for the Nominal Account’s journal entry. The default for this is ‘Depreciation Fixed Assets’.

        [Descr_NA=Depreciation of Fixed Assets]

        Narr_SL

        This is to specify the narrative for the sub-ledger’s journal entry. There is no default narrative set.

         

        Narr_NA

        This is to specify the narrative for the Nominal Account’s journal entry. The default for this contains the sub-ledger name after the text ‘Depreciation of’.

        [Narr_NA=Depreciation of (-SL.NAME-)]

        Residual

        This is to update the Residual Value to the specified value in the Fixed Asset’s Sub-ledger account.

        Creating a new Fixed Asset entry will take the Residual Value from Nominal Account Tag Values with the tag [Residual=X]. For example, with the tag [Residual=9], it will set the Residual Value as '9'.

        If not configured, the system will default the Residual Value as '1'.

        [Residual=9]

         

    • Local Account Code - If you are using the Chart of Accounts set by the holding company but use a local code in preparing local accounts, you can enter the local code here.

    • Local Account Name - If you are using the Chart of Accounts set by the holding company but use a local name in preparing local accounts, you can enter the local account name here.

    • Note - Enter additional notes for the account.

    • Restricted Account - Tick this to set restriction on account. When an account is flagged as 'restricted account', user must have authority level 8 - Power User on the respective journal entry screen to post in this account. This field is only visible if the account type is within the range 1 - 4.

    • Active - This box is automatically ticked on the creation of a new account. You may subsequently remove the tick to inactivate the account if you no longer want to post journals to it.

    • Update the 'Exchange Revaluation' of the linked sub-ledgers - This is only available for control accounts (denoted by letters of the alphabet).

    • Exchange Revaluation - This will be active only when the above option is ticked.

  2. Click Update.
