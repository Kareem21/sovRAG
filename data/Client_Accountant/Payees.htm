Payees

Payees are created in each Account File to facilitate the preparation of bank payment batches and subsequently, of bank payment export files. The creation of a new payee must be approved before any bank payments can be made to them. This feature can also be used together with the Distributions feature in Viewpoint Entity Administration to handle payments to beneficiaries.

This screen is part of the Bank Payments feature, which also encompasses the Payments and Export Batches functions.

 

Payees list

This shows all the payees that have been set up.

 

Command buttons

Click the relevant command button to perform the necessary.

  • New - Click this to create a new payee.

  • Edit - Click this to edit the details of a payee. If an approved payee is edited, the payee must be re-approved before subsequent payments can be made.

  • Delete - Click this to delete the selected payee.

  • Approval - Click this to approve the payee. The approval process depends on how the approval cards were set up.

    This will open the Approval screen allowing you to review the details and approve the payee by clicking Approve.

  • Sub-ledger - Click this to create a new sub-ledger or edit an existing sub-ledger from the Payees screen. When creating a new sub-ledger, the ‘Master File’ and ‘Currency Code’ value will be defaulted from the Payees’ ‘Payee Master File’ and ‘Payments CCY'. When creating a new record from the ‘Sub-Ledger’ button, it will always be defaulted to the Creditor screen (File Maintenance > Accounting > Journals > Creditor). Otherwise, it will find the existing record from both the Debtor and Creditor screens where the field ‘Master File’ and ‘Currency Code’ matched with the Payees field ‘Payee Master File’ and ‘Payments CCY’. Any updates to the sub-ledger record from the Payees screen will be reflected in the Sub-ledger in the Creditor/Debtor screen.

    Note: If you have authority of less than '6-Regular User' to the Creditor screen, you will not be authorised to create/edit the sub-ledger when clicking this button. Similarly, if you have authority of less than '6-Regular User' to the Debtor screen, you will not be authorised to edit the sub-ledger if the sub-ledger is in Debtor screen.
