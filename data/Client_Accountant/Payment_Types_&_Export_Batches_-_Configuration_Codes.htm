Payment Types & Export Batches - Configuration Codes

The following are the codes that can be used to configure the bank export batch .xml file. You may use any or all of the codes shown to set up the file for a particular bank. You must obtain the .xml file requirements from the bank before you begin the configuration process.

  • [/PayStart=....] – This is the 'begin part' of the messages. One instance of the code will be decoded and concatenated to the beginning of the message.

  • [/PayEnd=....] – This is the 'end part' of the messages. One instance of the code will be decoded and concatenated to the end of the message.

  • [/PayInfo=....] – This describes the payments. One code instance will be decoded and included for each payment.

  • [/FieldCode=....] – These are field constants that can be referred to in the [/Pay... codes, and can be customised with client specific codes. Refer to Sample Configuration to see how these are used.

Within the message, the following codes can be used to populate the payment information. You may use (+PREFIX.Field+) or (+PREFIX.Field\Format+).

 

Prefix

Used to retrieve...

PEB

Payment batch fields.

Syntax: (+PEB.Field+)

Refer to Payment batch fields for available fields.

PBK

Payment Bank fields.

Syntax: (+PBK.Field+)

Refer to Payment bank fields for available fields.

PAY

Payment detail fields.

Syntax: (+PAY.Field+)

Refer to Payment details fields for available fields.

PSB

Special fields:

  • (+PSB.ExtAcctNo+) – used to retrieve the ExtAcctNo of the Remittance Sub-ledger.

  • (+PSB.TotalAmount+) – used to retrieve the value of the remittance. Please refer to Export and/or Close Payment Batches for a screenshot of where the information is retrieved.

 

The formatting of the fields follow the formatting options available for Word reports. Additional format available are as follows:

  • F.FixLen=X

    Creates a fixed length field of the specified length by adding spaces to the field to the right or truncates the field if the original is longer than the specified length.

     

  • F.MaxLen=X

    Truncates the field if longer than the specified length.

     

  • F.FillZero=X

    Creates a fixed length field of the specified length by adding 0's to the left of the field. This function does nothing if the original values length is longer than the specified length.

     

  • F.FillZND=X

    Converts number with decimal to zero decimal format, e.g. 11.38 becomes 1138, then specified length by adding 0's to the left of the field. This function does nothing if the original values length is longer than the specified length.

     

  • F.FillSpace=X

    Creates a fixed length field of the specified length by adding spaces to the left of the field. This function does nothing if the original values length is longer than the specified length.

Example:

(+PAY.PayeeName\F.FixLen=20+)

(+PAY.PayAmount\F.FillZND =10+)

Additionally, it is also possible to customise the output using SQL Query Block using the following tags:

  • [/PAYHEADER.SQL=<SQL>/] - This will overwrite the default SQL query for (+PBK.Field+).

  • [/PAYDETAIL.SQL=<SQL>/]  - This will overwrite the default SQL query for (+PAY.Field+).
