Account Settlement

This features allows an accounting transaction for a sub-ledger to be linked to another accounting transaction. When the Practice Management component is not used and invoices and receipts are entered directly into Accounting component, this feature can be used to track whether invoices for a client has been paid or not by linking the invoices and the receipts.

To use this, the Settlement Method must be specified for the account type. This can be done through:

  • Globally in Viewpoint - Home > Accounting > Chart of Accounts > Nominal Accounts.

    1. Select the Chart of Account.

    2. Select the account.

    3. Click Edit.

    4. Select the method in Settlement Method.

     

  • Per Account File in File Maintenance > Accounting > Chart of Accounts.

 

Settlement Methods

Manual Settlement

If the settlement method is Manual Settlement, you need to perform the settlement in the Sub-ledger Accounts screen using either the Auto Settle or Settle command button in the Settlements tab.

 

FIFO Settlement

If the settlement method is Automatically (FIFO Settlement) [8], when a receipt is entered, as soon as it is posted, the system will 'look' for as many transaction(s) to link to it in order to settle it immediately. The settlement will be done with FIFO transactions.

For sales transactions where no receipt transactions have been entered, the system will 'wait' for a receipt transaction which can be used to settle them, also on a FIFO basis.

 

Through Reconciliation Screen

If the settlement method is Through Reconciliation Screen [1]. the Settlements command button in the Sub-Ledger Accounts screen will be disabled, and the user will only be able to do the settlement process through the Account File > Accounting > Processing > Reconcile Account screen.

 

Realised Exchange Gain/Loss

For settlement methods Manual Settlement [9] or Automatically (FIFO Settlement) [8], a Settlements command button will be visible in the Sub-Ledger Account screen. Clicking this command button allows users to key in the settlement amount and the system will create the journal entries for realised exchange gain/loss based on the item(s) settled versus the exchange rate of the selected sub-ledger currency on the screen.

The line being settled can either be partial or full settlement, but 'over settlement' is not allowed.
