Securities Rebalance

This feature automatically recalculates the capital and exchange gains/ losses on disposal of securities. This is especially useful when you do not enter securities transactions consecutively. The Securities Rebalance feature proposes the correcting entries as if the securities transactions were entered following the right date sequence.

Once users have verified the calculations, the corrections can be posted with the click of a button. Journal entries are created automatically to reflect these corrections and update the account balances.

Note: This feature can only correct draft journals – permanent journals can no longer be corrected.

 

Prerequisite

Account Files must be set up with the following options to enable the Securities Rebalance feature to work:

 

Securities Rebalance - Individual Securities

The Securities Rebalance appears as a separate tab in the Analyses of Securities sub-ledgers. Thus, it can be accessed from any of the three screens below:

  • Securities sub-ledger screen in the Account File

  • Accounts Enquiry

  • Trial Balance Enquiry

Securities sub-ledger screen in the Account File

  1. In File Maintenance screen, click Accounting Ribbon button and select Journals > Journal Quick Entry from the navigation listing in Accounting Overview.

  2. Click the Securities tab at the bottom panel and select the relevant Securities sub-ledger.

  3. Once done, click on the Analyses button.

  4. In the screen that appears, click Security Rebalance tab. This lists the proposed correcting entries for your verification.

  5. Once you have verified the calculations, click Post to automatically create the correcting entries.

Accounts Enquiry

  1. In File Maintenance screen, click Accounts Ribbon button in the Opened Master File tab.

  2. Click Account.

  3. Select the relevant Securities sub-ledger and click the Analyses button.

  4. In the screen that appears, click Security Rebalance tab. This lists the proposed correcting entries for your verification.

  5. Once you have verified the calculations, click Post to automatically create the correcting entries.

Trial Balance Enquiry

  1. In File Maintenance screen, click Accounts Ribbon button in the Opened Master File tab.

  2. Click Trial Balance.

  3. Build the Trial Balance.

  4. Select the relevant Securities sub-ledger and click the Analyses - SL button.

  5. In the screen that appears, click Security Rebalance tab. This lists the proposed correcting entries for your verification.

  6. Once you have verified the calculations, click Post to automatically create the correcting entries.

 

Securities Rebalance - All Securities

There is also an option to Rebalance All Securities sub-ledgers in the Account File. This option summarises the proposed correcting entries across all Securities sub-ledgers for your verification.

  1. Select the Account File.

  2. In File Maintenance screen, click Accounting Ribbon button and select Journals > Journal Quick Entry from the navigation listing in Accounting Overview.

  3. Click the Securities tab at the bottom panel and click Rebalance All.

  4. Once you have verified the calculations, click Post to automatically create the correcting entries.

What do I do if I see a warning message?

The following Warning messages will result in no action taken by the system:

  • Journal#[Draft/ Permanent Journal No.] outside posting window > : The journals listed are outside the posting window.

Please open the posting window by modifying the Posting Open From and To dates.

 

  • Journal#[Permanent Journal No.] unable to correct permanent journals > : These are permanent journals which can no longer be corrected. No action should be taken by users.

 

Triggering Securities Rebalance via Journal Scripts

The Securities Rebalance calculations can also be triggered via Journal Scripts. Add 1 of the following Tags to the first line of the Securities Journal Scripts:

  • [Initiate=REBAL] – When you click the Post  button to use this Journal Script, the Security Rebalance window will prompt if there are any unposted corrections. This might be included in a Securities acquisition Journal Script.

  • [After_Post=REBAL] – When you click the Post  button to post the Journal Script entry, the Security Rebalance window will prompt if there are any unposted corrections.

 

Calculations - No Exchange Gain/ Loss

The Securities sub-ledger currency is the same as the Account File’s base currency (USD), in this case USD. Hence, there will be no exchange gains/ losses on disposal.

Proposed Corrections

The proposed corrections are based on the following calculations:

 

All transactions in the Securities sub-ledger before the disposal transaction are taken into account in calculating the disposal cost except for the unrealised revaluation transaction.

In addition, subsequent disposals are calculated as if you had posted the proposed corrections on previous disposals. This in calculating the disposal cost of the second disposal, the corrected disposal cost of the first disposal in used; and in calculating the disposal cost of the third disposal, the corrected disposal costs of the first two disposals are used.
