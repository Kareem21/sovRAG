Valuations

This section briefly explains how the gain/loss in value and exchange rate gain/loss on sub-ledgers are determined, and the journal posting made to account for these gains/losses.

 

Gain/Loss in Value

The valuation methods assigned to sub-ledgers are used to determine the valuation/revaluation value and ultimately, to calculate the gain/loss in value within the Account File Valuation and Sub-ledger Revaluation features.

Each valuation method employs a different calculation method to determine the valuation/revaluation value:

Valuation Method

Calculation Method

Description

Book value

None

Takes the sub-ledger balance as the valuation/revaluation value.

Interest/Coupon

  • Actual/360

    Formula = (n/360) x Interest % x Balance CCY x Exchange Rate

    where n = number of days between Last Interest Received Date and Valuation Date

     

    Example:

    Balance CCY = EUR1000

    Interest % = 10%

    Exchange Rate = 1

    Last Interest Received Date = 01 JUNE 2014

    Valuation Date = 03 JULY 2015

    n = 397 days

     

    Unrealised Total

    = (397/360) x 0.1 x 1000 x 1

    = 110.28

  • Actual/365

    Formula = (n/365) x Interest % x Balance CCY x Exchange Rate

    where n = number of days between Last Interest Received Date and Valuation Date

     

    Example:

    Balance CCY = EUR1000

    Interest % = 10%

    Exchange Rate = 1

    Last Interest Received Date = 01 JUNE 2014

    Valuation Date = 03 JULY 2015

    n = 397 days

     

    Unrealised Total

    = (397/365) x 0.1 x 1000 x 1

    = 108.77

  • Actual/Actual

    Formula = (n/y) x Interest % x Balance CCY x Exchange Rate

    where

    n = number of days between Last Interest Received Date and Valuation Date

    y = number of days in a year with 366 for leap years and 365 for non-leap years

     

    Example 1 (Non-Leap Year):

    Balance CCY = EUR1000

    Interest % = 10%

    Exchange Rate = 1

    Last Interest Received Date = 01 JUNE 2014

    Valuation Date = 03 JULY 2015

    n = 397 days

     

    Unrealised Total

    = (397/365) x 0.1 x 1000 x 1

    = 108.77

     

    Example 2 (Leap Year):

    Balance CCY = EUR1000

    Interest % = 10%

    Exchange Rate = 1

    Last Interest Received Date = 01 JUNE 2014

    Valuation Date = 03 JULY 2016

    n = 763 days

     

    Unrealised Total

    = [(32/366) x 0.1 x 1000 x 1] + [(365/365) x 0.1 x 1000 x 1]

    = 208.74

  • 360/360

    Interest is based on twelve 30-day months, calculated once each month on the same day of the month (i.e. interest is paid in arrears). The 31st of the month is ignored.

     

    Formula = (n/360) x Interest % x Balance CCY * Exchange Rate

    where

    n = number of days between Last Interest Received Date (Date 1) and Valuation Date (Date 2) and is calculated as [(Y2 - Y1) x 360] + [(M2 - M1) x 30] + (D2 - D1) with:  

    • Y2 and Y1 being the year of Valuation Date and Last Interest Received Date respectively;

    • M2 and M1 being the month of Valuation Date and Last Interest Received Date respectively; and

    • D2 and D1 being the day of Valuation Date and Last Interest Received Date respectively.

    If the Valuation Date should fall on a 31st, subtract 1 day so it is [(Y2 - Y1) x 360] + [(M2 - M1) x 30] + (D2 - D1) - 1

     

    Example 1 (Last Interest Received Date falls on a 31st):

    Last Interest Received Date = 01 JANUARY 2014

    Valuation Date = 01 JULY 2014

    Balance CCY = EUR1000

    Exchange Rate = 1.30673

    Interest % = 10%

     

    n is calculated as [(2014 - 2014) x 360] + [(7 - 1) x 30] + (1 - 31) = 150 days

     

    Unrealised Total

    = (150/360) x 0.1 x 1000 x 1.30673

    = 54.45

     

    Example 2 (both dates fall on 31st):

    Journal From = 31 JANUARY 2014

    Valuation Date = 31 JULY 2014

    Balance CCY = EUR1000

    Exchange Rate = 1.5

    Interest % = 10%

     

    n is calculated as [(2014 - 2014) x 360] + [(7 - 1) x 30] + (31 - 31) - 1 = 179 days

     

    Unrealised Total

    = (179/360) x 0.1 x 1000 x 1.5

    = 74.58

     

    Example 3 (Valuation Date falls on a 31st):

    Journal From = 01 JANUARY 2014

    Valuation Date = 31 JULY 2014

    Balance CCY = EUR1000

    Exchange Rate = 1.2

    Interest % = 10%

     

    n is calculated as [(2014 - 2014) x 360] + [(7 - 1) x 30] + (31 - 1) - 1 = 209 days

     

    Unrealised Total

    = (209/360) x 0.1 x 1000 x 1.2

    = 69.67

  2. Adds the calculated interest to the sub-ledger balance to determine the valuation/revaluation value.

Market value - User-entered

  • Higher of Book/Market value

  • Lower of Book/Market value

Compares the user-entered market value to the sub-ledger balance (book value) before selecting the valuation/revaluation value.

  • Market value

Takes the user-entered market value as the valuation/revaluation value.

Market value - Securities Master

  • Higher of Book/Market value

  • Lower of Book/Market value

  1. Calculates the market value as the unit price in the Securities Master multiplied by the quantity held in the sub-ledger.

  2. Compares this calculated market value to the sub-ledger balance (book value) before selecting the valuation/revaluation value.

  • Market value

Calculates the market value as the unit price in the Securities Master multiplied by the quantity in the sub-ledger. Takes this market value as the valuation/revaluation value.

Once the valuation/ revaluation value has been determined, the Account File Valuation and Sub-ledger Revaluation features subtract the sub-ledger balance (book value) from the valuation/revaluation value to calculate the gain/loss in value.

 

Exchange Rate Gain/Loss

The exchange rate gain/loss on foreign currency sub-ledgers is calculated using the Account File Valuation and Exchange Revaluation functions.

The exchange rates on or before the valuation/revaluation date are extracted from the Exchange Rates Master and applied to the valuation/revaluation value determined above. This translates the valuation/revaluation value into the base currency of the Account File.

Each foreign currency sub-ledger has two balances, one in the sub-ledger currency and the other in the base currency of the Account File. The sub-ledger balance in the base currency is subtracted from the valuation/revaluation value (also in the base currency), to determine the exchange rate gain/loss.

 

Journal Entries for Gain/Loss

Accounting has three different valuation functions for sub-ledgers:

  • Account File Valuation - This calculates the unrealised gain/loss in value based on the valuation method assigned to the sub-ledger; and the unrealised exchange rate gain/loss. This functions provides an optional accrual journal for all unrealised gains/losses, which automatically reverses out of the books of account the next day (Journal Date + 1 day). This ensures that the gains/losses are not realised into the accounts permanently.

  • Exchange Revaluation - This calculates the realised exchange rate gain/loss. The optional posting of a regular journal to realise the gain/loss in the accounts does not affect the sub-ledger currency balance, only the sub-ledger balance in the base currency of the Account File will be affected.

  • Sub-ledger Revaluation - This calculates the realised gain/loss in value based on the valuation method assigned to the sub-ledger. This function includes an option to post a regular journal to realise the gain/loss in the accounts.

Each valuation function can automatically create journals for the gain/loss in value and exchange rate gain/loss on sub-ledgers. The posting accounts in the journal entries are determined by the properties of the control account linked to the sub-ledgers:
