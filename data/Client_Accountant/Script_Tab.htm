Script Tab

The individual journal lines of the script that you chose in the Select tab is listed in the lower half of this screen. Entering information in the top half of the screen will automatically complete some or all of the individual journal lines of the script, depending on how the script was set up:

  • You may have to complete certain fields in the individual journal lines such as selecting the appropriate bank account or tax account if the script was set up that way.

  • When a sub-ledger or nominal account is selected in a journal line, a '+' sign appears in front of the journal line. You can copy the line by clicking the '+' sign so that another line appears below the first with the same details.

If you do not want to use the script, or if you need to add more journal lines for the transaction, then click > Journal to move to the Journal tab.

 

Journal Date

The journal date defaults to today's date. Change this if necessary.

 

Reference

This field may be auto-populated by the information coded in the Default Reference field of the selected script and may be overwritten by the user. As an example, an auto-populated entry could be the sub-ledger account type followed by the journal number. For instance, N#10 would indicate a nominal account entry (N), journal number 10.

 

C/I

This defines the journal as a capital or income transaction. This field is only active if the Chart of Accounts linked to the selected Account File has been enabled for Capital / Income Segregation, as is required for Trust files.

 

See also:

  • Capital/Income Segregation

 

Description

This field may be auto-populated with the information coded in the D-Description field of the selected script and can be edited by the user.

 

Narrative

This field may be auto-populated with the information coded in the D-Narrative field of the selected script and can be edited by the user.

 

Quantity

Enter the quantity of the transaction. This field is enabled if the sub-ledger selected has a control on quantity.

 

Currency/Amount

If you have selected a nominal posting account in the first line of the journal script, the currency of the entry defaults to the base currency of the selected Account File. Change this if necessary.

If you have selected a sub-ledger account in the first line of the journal script, the currency of the sub-ledger account is automatically displayed.

Note that the caption of this field is determined by the entry in the Caption Amount field of the journal script.

 

D (Debit) or C (Credit)

This denotes whether the first line of the journal script is a debit or a credit entry. You may be able to change the selection, depending on how the script was set up.

 

Exchange

The exchange rate is shown for the translation from the currency of the journal entry to the base currency of the selected Account File. The exchange rate may be changed if necessary.

 

Amount in base currency

The amount entered in the Currency/Amount field is calculated by Viewpoint and shown here in the Account File's base currency.

 

Line Date

The date here may be auto-populated by the information coded in the D-Line Date field of the selected script. This can be edited by the user.

 

Reconciliation Date

The date here may be auto-populated by the information coded in the D-Rec. Date field of the selected script. This can be edited by the user.

 

Multi CCY/exchange

Tick this on/off flag to switch between single currency or multi-currency journals. Switching off the multi-currency flag will hide some fields to reflect a single currency journal in the base currency of the Account File.

 

Descr/Dates/Narrative

Tick this on/off flag to show the Description, Line Date, Recon. Date and Narrative columns in the journal lines.

 

Quantity

Tick this to show the quantity involved in the journal lines, where applicable.

 

Tax

Tick this to show the Tax columns in the journal lines.

 

Accrual Posting

Tick this to create the accrual next day reverse journal.

 

Show

If you enabled unit codes for journals. you can choose which unit code to display in the journal line table below.

 

Journal line

Individual journal lines for the selected journal script are listed. Click within the cell to add/update the details for the line.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • Post - Click this to post the journal. Note the journal must balance before it can be posted - the last line of the journal will show whether the journal is balanced or not.

    Why was my journal posted as a draft?

    The Draft Journals option is ticked for the Account File in Account File - General screen. See Draft Journals for more information.

  • Print - Click this to print a voucher from the report template library. The document can either be displayed on the screen in an Excel format or be sent directly to the printer. The template that will be generated will, by default, be the last template used. To change the template, click the Select>> button to view a list of the available templates.

  • Save Pending - Click this to save the journal as a pending journal. This option allows semi-completed transactions, such as those that do not balance, or have incomplete details, to be saved and finished at a later time.

    I saved the journal as a pending journal. Where is it now?

  • < Cancel/Back - Click this to go back to the Select tab. All changes made in the Script tab will be lost. If you do not want to lose the data you have entered so far, click Save Pending instead.

  • Journal > - Click this to move to the Journal tab where you can add, delete or amend the journal lines that were set up by the script.
