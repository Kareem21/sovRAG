Statement Import & Processing

This feature is an additional option and is only available if included in your licence.

 

The Statement Import & Processing feature allows you to import an electronic bank statement and:

  • Reconcile or create journals relating to bank account sub-ledgers in Accounting.

  • Process Practice Management receipts.

The first step of that process is to recognise the account numbers in the electronic bank statement:

  • If the account number matches a bank account sub-ledger in Accounting, the program will further determine if there exists a journal line with the same account number, currency, amount, and date as the statement line. If an exact match is found, the journal will be automatically reconciled.

  • If an exact match is not found, but if there exists a journal whose bank account number, currency and amount are the same while the date is within a specified range, the journal will be reconciled. This reconciliation may be either automatic or require user confirmation.

  • If the account number matches that of a bank account sub-ledger in Accounting but no journal line has been created; the amount is a debit entry (i.e. more than zero); and the bank account sub-ledger belongs to a Revenue Company and has been mapped in the Account Mapping of Bank Accounts, the following processes will take place:

    1. Retrieve outstanding invoices (not draft invoices) from Viewpoint Practice Management:

      • The bank statement Narrative is analysed for the invoice number(s) of outstanding invoices and the matching outstanding invoice(s) are retrieved.

      • If none found, the bank statement Narrative is analysed for the Billing File name and the Billing File’s outstanding invoices are retrieved.

      • If none found, all outstanding invoices with the same currency and outstanding amount are retrieved.

      • If none found, users can browse for the Billing File and Matter to retrieve all outstanding invoices with the same currency as the bank receipt and matching the selected criteria.

    2. Users can select an outstanding invoice to settle in full or partially. There is also an option to settle multiple outstanding invoices to a Matter. For multiple settlements, the Matter currency must match the bank receipt currency.

    3. If no outstanding invoices are selected, the entire bank receipt can be posted as a Matter Receipt. This will automatically update the receipts in Practice Management and post a permanent journal in Accounting.

  • If the account number matches that of a bank account sub-ledger in Accounting but no journal line has been created, various rules should be defined to create either a pending journal or draft journal based on different conditions relating to the statement line. How the fields in the new pending or draft journal are to be completed can also be defined in the rules. Additionally, the rule can define whether the pending or draft journal should be automatically posted, or whether the user needs to confirm the posting.

  • Lastly, if the account number is not recognised, the statement line relating to it will be ignored, and will not be processed further.

To access this feature, you can either go to:

  • Viewpoint - Home screen:

    • Click Accounting Ribbon button and select Functions > Statement Import & Processing from the navigation listing in Accounting Overview.

    • In Section Menu, click Home > Accounting > Functions > Statement Import & Processing.

  • File Maintenance screen:

    • Click Accounting Ribbon button and select Processing > Statement Processing (Import on Global screen) from the navigation listing in Accounting Overview.

    • In Section menu, click File > Accounting > Processing > Statement Processing (Import on Global screen).

Doing so will open the Electronic Statement Import & Reconciliation screen where you will see the following folders:

  • Select/Import

  • Process & Reconcile

Before you begin using the feature, ensure that you have:

  • Defined a Source folder, where various bank source files are saved.

  • Defined a Statement folder, where electronic statements will be saved prior to import.

  • Defined an Archive folder, where electronic statements are moved to after import.

  • Set up your journal scripts.

  • Imported your source files.

I cannot see the screen, what do I need to do?

If you cannot view the screen:

  1. Check that it has been enabled in System Manager - Settings - Windows Menu.

  2. Ensure that you have the appropriate authority to access the functions in Configuration > Users & Security > Security Setup.

Where do I obtain the source files?

If Viewpoint does not already support the format you require, then obtain an electronic statement from the bank and send it to your Viewpoint Support representative. A source file will be created for you.

If the bank changes the format of their electronic statement, you will need to send an updated copy to Viewpoint so a new source file can be created.

The source files are in .xml format and apart from being designed to 'read' the bank's statement file, also support the following tags:

[/FORMAT-CULTURE=<FormatCode>/]

This is to define how dates should be recognised and <FormatCode> can be:

  • en-UK - recognise dates as Day-Month-Year

  • en-US - recognise dates as Month-Day-Year

For example, if the original source file recognises the date as Day-Month-Year but the bank's statement file has the date in Month-Day-Year format, you can use this tag to make it compatible with the statement file.

 

[/AMOUNT_NO_REVERSE=1/]

This tag is to ensure that the amount in the bank's statement file will not be reversed i.e. if the amount is a positive value, it will be a positive value in Viewpoint after the import.
