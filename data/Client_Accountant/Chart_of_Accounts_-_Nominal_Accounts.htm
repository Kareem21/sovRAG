Nominal Accounts

The nominal accounts that are linked to a particular Chart of Accounts (CoA) template are created here. Each CoA template contains a number of nominal accounts, which must be linked to a Sub group.

There are three types of nominal accounts: posting accounts, control accounts and system control accounts. The posting and control accounts can be further divided into balance sheet and profit/loss accounts. Note that the system prevents the linking of a balance sheet account to a profit/loss account.

Each control account contains certain properties, which must be specified when creating the template. There are mandatory (such as exchange gains and loses) and optional properties. The advantage of specifying these properties is that the system will automate certain accounting functions such as the creation of valuation/ revaluation journals.

 

Chart of Account

Select the Chart of Account template you want to work on.

 

Account list

This shows all the accounts for the selected CoA template.

 

Command buttons

Click on the relevant command button to perform the necessary.

 

  • New - Click this to add new account.

  • Edit - Click this to edit an existing account.

  • Properties - Click this to edit the properties of the selected account. This button is enabled for control accounts only. Exchange gains/losses, capital gains/loses, income/expenses and unrealised value /exchange revaluations should be linked to posting accounts.

    Specify the accounts and options for the listed items:

    • Realised Exchange Gains/Losses

      • Account - Gains - Select the applicable posting account.

      • Account - Losses - Select the applicable posting account.

      • Exchange Revaluation - Select the applicable method.

        • Realise all Gains/Losses [0] - Selecting this method will realise all exchange gains and losses. By default, this option is displayed.

        • Do not Revalue [1] - Selecting this method will not post any valuation.

        • Realise only losses [2] - Selecting this method will create a valuation journal if there is loss in the exchange revaluation.

        • Realise only Gains [3] - Selecting this method will create valuation journal if there is gain in the exchange revaluation.

        Note: The selected method will be displayed in the Exchange Revaluation screen.

    • Realised Value Gains/Losses

      • Account - Gains - Select the applicable posting account.

      • Account - Losses - Select the applicable posting account.

    • Sub-Ledger related Income or Expenses

      • Income - Select the applicable posting account.

      • Expenses - Select the applicable posting account.

    • Unrealised Value Gains/Losses

      • Account - Gains - Select the applicable posting account.

      • Account - Losses - Select the applicable posting account.

      • Valuation Journal - Select the method for the valuation journal.

        • Detailed on Sub-ledger [0] - Selecting this method will create a valuation journal based on the sub-ledger code and currency.

        • No Valuation Journal [1] - Selecting this method will not create any valuation journal.

        • Summary Journal using Off-set Account [2] - Selecting this method will create a summary of journal using an off-set account. The amount shown in the off-set account is based on the total for the specified date. The selected method will be displayed in the Account File Valuation screen and Sub-ledger Valuation screen.
          Example:
          If there are two different sub-ledgers using the same nominal account, only 1 journal line will be created using the off-set account. The amount is the sum of both sub-ledgers.

        • Detailed Journal using Off-set Account [3] - Select this method will create a detailed journal using an off-set account. Each sub-ledger will be created in the same journal but in different lines. The nominal account will be the off-set account while the currency is based on the Account File base currency. The selected method will be displayed in the Account File Valuation screen and Sub-ledger Valuation screen.

         

      • Off-set Account - This is mandatory if the selected method is method 2 or 3.

    • Unrealised Exchange Gains/Losses

      • Account - Gains - Select the applicable posting account.

      • Account - Losses - Select the applicable posting account.

      • Valuation Journal - Select the method for the valuation journal. The available methods are the same as Unrealised Value Gains/Losses. The selected method will be displayed in the Account File Valuation screen and Exchange Revaluation screen.  

      • Off-set Account - This is mandatory if the selected method is method 2 or 3.

  • Delete - Click this to delete the selected account.

  • Select All - Click this to select all the accounts

  • Deselect All - Click this to deselect all the accounts

  • Apply Globally - Click this to apply change or a new account to all Account Files that are linked to the selected CoA template.

    1. Tick the desired options:

      • Add and/or activate account - Tick this to add or activate the account.

      • Update Account Name - Tick this to update the account name.

      • Update Restricted Account flag (Posting Account only) - Tick this to allow global update on restricted accounts.

      • Update Settlement Method - Tick this to update the Settlement Method. This is available only if the account is a control account (account type is denoted by letters of the alphabet).

        • Update Outstanding Amount/Delete Settlement Linking - Tick this to also perform the following if:

          • Settlement Method is changed from 'No Settlement Required [0]' to 'Automatically (FIFO Settlement) [8]' or 'Manual Settlement [9]' - The outstanding amount will be set to be the same as 'Transaction amount'.

          • Settlement Method is changed from either 'Automatically (FIFO Settlement) [8]' or 'Manual Settlement [9]' to 'No Settlement Required [0]' or 'Through Reconciliation Screen [1]' - The existing link will be deleted.

      • Update Exchange Revaluation to linked sub-ledgers - This option only applies to control accounts. When this option is ticked, the program will update all sub-ledgers’ exchange revaluation using the selected Chart of Account with the exchange revaluation of the selected nominal accounts.

      • Deactivate account - Tick this to deactivate the account.

        • Deactivate Subaccount if any (Account Files) - This is only available if the selected nominal account type is 1, 2 or 4.

      • Also apply to INACTIVE Account Files - Tick this if you want the change(s) to apply to Account File that are not active.

    2. Click Apply.

    Note: If the option 'Update Outstanding Amount/Delete Settlement Linking' is not ticked when you choose to update the Settlement Method, a warning will be shown.
