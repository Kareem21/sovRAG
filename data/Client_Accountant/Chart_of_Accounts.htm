Chart of Accounts

This displays all accounts in the file's Chart of Accounts. The Chart of Accounts template used by the Account File is determined when you set up the file; however, you may still make adjustments to the accounts that are used or not within that template. Major changes such as switching to a new Chart of Accounts must be done using the Archive/Change CoA or BC function.

You can click on the following to minimise or expand the sections in this screen:

  • - To minimise this section and hide the command buttons. In the grid, only the selected row will be seen.

  • - To expand the section and show the command buttons.

Filter

You can filter the accounts by their status.

 

Accounts list

This displays the accounts that are in use.

 

Command buttons

Click the relevant command button to perform the necessary.

  • New Subaccount - Click this to create new sub-accounts. It is available only for the account types Asset, Liability, Expense and Income. Sub-accounts on one Account File will not appear in another Account File. Sub-account ranks equally with the account types (Asset, Liability, Expense and Income) where you can post journals directly into them and into the root nominal. However for reporting, it is advised to not post into the root nominal due to the reporting. By default, this button is not active. You need to activate this in Configuration > Settings > Customise > Field Settings. For Practice Management to Accounting mapping, if the Nominal Account is inactive, the system will post the entry to the first active sub-account based on Account Code.

    • Excel Statement code [.na.] will sum and report the Sub-accounts under the root nominal – this is for comparability with other Account Files where the Sub-accounts don’t exist.

    • Excel Statement code [.nb.] will disclose each Sub-account separately – this means each Account File will have different Sub-accounts showing, depending on what was created in that Account File.

    An example – 6110 is the root nominal and 611001 and 611002 are its Sub-accounts.

    • [.na.] code will show one sum of 12,250,000.00 on 6110.

    • [.nb.] code will show the amounts against 611001 and 611002 as in the screenshot above.

    1. Complete details of the sub-account:

      • Nominal Account - This defaults to the selected nominal account.

      • Account Code - Enter the code for the subaccount.

      • Account Name - This defaults to the selected nominal account but you can modify this.

      • Settlement Method - By default, this field is not enabled. When enabled, it is available only for control accounts (denoted by letters) and is used to link a debtor/creditor transaction to a bank transaction. The default method follows the method set in Home > Accounting > Chart of Accounts > Nominal Accounts but you can change this for the Account File.

      • Restricted Account - Tick this to set restriction on account. When account is flagged as 'restricted account', user must have authority 8 (Power-user) on the respective journal entry screen to post in this account. This field is only visible if the account type is within the range 1 - 4.

      • Active - This will be ticked by default.

      • Update the 'Exchange Revaluation' of the linked sub-ledgers - This is only available for control accounts (denoted by letters of the alphabet).

      • Exchange Revaluation - This will be active only when the above option is ticked.

    2. Click Update.

  • Edit - Click this to edit the name of the selected account in the Chart of Accounts or deactivate the account. If it is a control account (accounts denoted by letters), you can also change the settlement method and update the 'Exchange Revaluation' of the linked sub-ledgers.

    Update the ‘Exchange Revaluation’ of the linked sub-ledgers - this option is unticked by default. When this option is ticked, it will perform batch update on the ‘Exchange Revaluation’ field to all the linked sub-ledgers of the same Nominal Account.

    Example:

    In Nominal Account '2320', the option "Update the ‘Exchange Revaluation’ of the linked sub-ledgers" has been ticked with the 'Exchange Revaluation' field set to 'Realise only Gains [3]'.

    The 'Exchange Revaluation' field will then be updated to all linked sub-ledgers with the Nominal Account '2320' in File > Accounting > Journals

     

    As seen here, the Exchange Revaluation has now set to 'Realise only Gains [3]'.

  • Delete - Click this to delete the selected account. Deleted accounts are moved to the Accounts Not Used list below the Chart of Accounts. This is applicable only in Chart of Accounts screen.

  • Remove Unused - Click this to remove the unused accounts listed at the top panel to the bottom panel of the screen (Accounts Not Used panel). This prevents users from accidentally reactivating an account that is not meant to be used. This is applicable only in Chart of Accounts screen.

 

Accounts Not Used list

This displays accounts that are not used for the Account File.

 

Add

Click to add an unused account back into the Chart of Accounts. This is applicable only in Chart of Accounts screen.
