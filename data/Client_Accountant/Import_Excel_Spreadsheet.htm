Import Excel Spreadsheet

Apart from entering the information into Viewpoint, it is also possible to import the information in using Import Sheet. Viewpoint has prepared a number of Import Sheet templates that can be downloaded from Viewpoint's support portal website.

You may download these import sheets and use as is; or modify them to better suit your needs, e.g. removing fields that are not relevant to you.

For each Excel Import Sheet:

  • The import code {$IMPORT:<SHEET>} must be in cell A1. For example: {$IMPORT:JOURNAL}. The Import Sheet for Identity Register, Classification and Indicia and Reportable Accounts can also contain the following:

    • Import sheet for Identity Register/Classification and Indicia:

      • {$IMPORT:IDREG_Country} - When this is inserted in cell A1, it will compare records based on these fields - FileCode, IdType and Country. If there is an existing record in Viewpoint with the same information for these three fields as in the Import Sheet, then the existing record will be updated with the record from the Import Sheet. Otherwise, a new record is created when you perform import.

      • {$IMPORT:IDREG_UdCode} - When this is inserted in cell A1, it will compare records based on these fields - FileCode, IdType and UdCode. If there is an existing record in Viewpoint with the same information for these three fields as in the Import Sheet, then the existing record will be updated with the record from the Import Sheet. Otherwise, a new record is created when you perform import.

      • {$IMPORT:IDREG_IdCode} - When this is inserted in cell A1, it will compare records based on these fields - FileCode, IdType and IdCode. If there is an existing record in Viewpoint with the same information for these three fields as in the Import Sheet, then the existing record will be updated with the record from the Import Sheet. Otherwise, a new record is created when you perform import.

    • Import sheet for Reportable Accounts:

      • {$IMPORT:VPRA_ACCTACTION} - When this is inserted in cell A1, the Import Sheet will allow information for Reportable Account record and Reportable Account Actions to be entered into the Import Sheet and imported into Viewpoint.

      • {$IMPORT:VPRA_ACCOUNTS} - When this is inserted in cell A1, the Import Sheet is only for importing Reportable Accounts. It will check on existing records based on these fields - FileCode, RepAuthority, AcctCode. If there is an existing record in Viewpoint with the same information for these three fields as in the Import Sheet, then the existing record will be updated with the record from the Import Sheet. Otherwise, a new record is created when you perform import.

      • {$IMPORT:VPRA_ACTIONS} - When this is inserted in cell A1, the Import Sheet is only for importing Reportable Account Actions.

  • Columns in an import sheet cannot be hidden. If you want to hide the information in a column, change the width of the column to the smallest allowed size (0.5 points) instead.

  • Every row that is to be imported must start with {$IR} which must always be in column A.

 

Using Import Sheet

These import sheet must be completed with the relevant details before they are to be imported through Viewpoint Connect Add-ins in Microsoft Excel.

  1. Open the Import Sheet and enter the required information.

  2. In Microsoft Excel, select the Viewpoint add-in menu.

  3. Click Import Sheet. The Import Sheet screen will appear.

  4. Click Import.

  5. Click Close to exit the Import sheet screen when the import is complete.  

 

If the Import Sheet is for importing journals, you can choose:

  • Import & Post - Import only selected row and post to Accounting.

  • Post ALL - Post all rows in the Excel spreadsheet.

 

Note: The Copy button on the left allows user to copy the grid information to the clipboard. It copies the message and import status for analysis.
