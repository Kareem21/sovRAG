Statement Import & Processing - Process & Reconcile Folder

This screen is where the actual processing and reconciliation takes place, and this is also where the rules used for the reconciliation process are defined. The function buttons at the top of the screen are used globally, i.e. on all lines. For instance, filtering the lines, performing enquiries or generating reports. The main section of the screen is where the statement lines are displayed. The function buttons below the statement pane relate to the reconciliation process.

 

Filter

Use this to filter the statement lines:

  • All - Display all lines.

  • Error - Display only lines that show an error after processing (status type: 1, 2).

  • Waiting Confirmation - Display lines that are waiting for user confirmation after processing (status type: 10, 20, 30, 41, 80).

  • Pending Posting - Display lines that are pending for posting (status type: 21, 22, 23, 31, 32, 33, 81, 82, 83).

Reload

Click this to reload the statement lines.

 

Set Filter

Use this to filter the list based on the highlighted account number. Click this button again to remove the filter.

 

Enquiry

This is enabled if the highlighted transaction line is recognised, and displays all lines related to the account number.

  • Statement - Journal - This is the default selection, and places the Statement pane on the left of the Account Reconciliation pane (see Statement pane and Account reconciliation pane, below).

    • Account Number - <selected account number> - This shows the selected account number.

    • Date - from, Filter and - to - These fields allow you to filter the display according to the Transaction Date, Value Date or Settlement Date range.

    • Reconciled and Pending - Tick either check box or both, to filter the list and show only reconciled, pending or both types of transactions. Leave both check boxes empty to display ALL transactions.

  • Journal - Statement - This selection puts the Account Reconciliation pane, which is known as the Journal pane here, on the left of the Statement pane. Any unmatched journals will be displayed in the Journal pane.

    • Date Filter, Date - from and - to - These fields allow you to filter the display according to Journal Date and Reconciliation Date for the date range.

    • Sort By - Use this to sort the display.

    • Non-Reconciled - Tick this to display only unreconciled items.

Account Reconciliation pane

The Account Reconciliation pane is placed to the right of the Statement pane and shows the results of the rules processing.

 

Select

Use this to customise the current view of the grid. Alternatively, right-mouse click on the grid and select Customise from the Context Menu.

 

Select All

Click to select all the statement lines. Alternatively, use Ctrl+A or press the F5 key.

 

Process

Click to process all selected lines or press the F6 key. If you exit the Statement Import & Processing screen at this point, the processed items will be reloaded the next time you use the feature.

 

Confirm

Click to confirm the process when required, or press the F7 key. This will be noted in the Status column of the Statement pane (see above).

 

Action

Click this or press F8 to open a separate menu with the following options:

  • Undo - Select this option to undo the last process for the selected statement lines. The status of the lines will revert to Not processed.

  • Ignore - Select this to ignore selected statement lines. Ideally, you should do this before you process the lines. You may still ignore lines post-processing: this action can be done on all status types except Reconciled - Automatic. See also Ignore (undo) Option below.

  • Disregard for Account Receivable (AR) Settlement - Select this to ignore the selected AR - Settlement statement lines. This action can be done on AR - Settlement status type 3, 40, 44, 45.

  • Process - Select this to process the selected statement lines. This is the same as clicking the Process button (see Process above).

  • Confirm - Select this to confirm the process for selected statement lines. This is the same as clicking the Confirm button (see Confirm above).

  • Commit Journals - Select this to commit the process for the selected statement lines. This is the same as clicking the Commit Journals button (see Commit Journals below). This action is final, and cannot be undone.

  • Assign Rule - Select a rule from the list to apply to the selected statement lines.

  • Reconcile - Select this to reconcile the selected line against a specific journal number, which is entered in the Journal Number field.

  • Ignore (undo) option - Tick the Add to (Remove from) the 'Ignore account codes' list check box if you want to add or remove the selected account number from the Ignore list in the Define Rules screen. This option must be used with the Ignore option to add an account number to the Ignore list; and used with the Undo option to remove an account number from the Ignore list.

  • Perform - Click this button to perform the selected action.

  • Cancel (Esc) - Click this button or the Esc key to close the menu.

Commit Journals

Click this to commit the application of the rules. This action is final and cannot be undone.
