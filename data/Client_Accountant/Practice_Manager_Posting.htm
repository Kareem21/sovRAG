Practice Management Posting

This feature is used to post entries from Practice Management to Accounting in the following situations:

  • Your environment has been set up for manual posting, or

  • You have recently added the Accounting component and now wish to post all past entries from Practice Management to Accounting.

The posting feature will work slightly differently based on the selection made for the handling of PM Accounting Journals in Configuration Ribbon button > Settings > System Defaults > Accounting:

  • None [0] - No postings are made, and no journals are kept.

  • PM Journals [1] - The journals are generated in Practice Management, and if necessary, manually imported into Accounting.

  • CA Journals RC [2] - The journals are posted directly to Revenue Company account(s) only.

  • CA Journals CE [4] - The journals are posted directly to the Client Entity account(s) only.

  • CA Journals RC + CE [6] - The journals are posted directly to both the Revenue Company and the Client Entity accounts.

Note the following:

  • In order for the straight-through posting feature to work correctly, ensure that you have correctly done the account mapping. Failure to do this will cause errors in the posting process.

  • Straight-through posting from Practice Management will not produce draft journals even though you have enabled draft journals when setting up the Account File.

 

Posting journals

  1. Select the functions you want to run:

    • Show Control Balances - Tick this if you want to show differences in the journal totals between Practice Management and Accounting. This same information can be seen in the Journal Posting Reconciliation reports in Billing. If there is any difference, then you should run the Post New Journals (Invoices/Receipts/Payments) option (see below). This function is disabled in the Global - Functions - Practice Management Posting screen.

    • Review All Receipts/Payments - Tick this to check all payments and receipts for any errors such as invalid Billing File, Matter, etc. This function is only available in the Global - Functions - Practice Management Posting screen.

    • Post New Journals (Invoices/Receipts/Payments) - Select this option to post outstanding journals from Billing to Accounting. Enter the From and to dates to post journals in a period, or leave the fields empty to post all outstanding journals.

    • Set Invoices/Receipts/Payments before the 'From' date and not yet posted, to 'Post' done - This option is enabled if you enter a From date for the above option. Tick this if you want to mark all unposted invoices, receipts and payments before the From date as posted. There will be no journals created for these transactions. This function is disabled in the Global - Functions - Practice Management Posting screen.

  2. Click Start to begin the posting process.

  3. At the end, click Open Log to view the log file and see if there are any errors. The file (Recalculate.txt) is located in the location specified in Configuration > Settings > System Defaults > Folders - Temporary Files. The information in the log is overwritten at each posting.

Can I post journals from the Practice Management module?

Yes, you can.

  • To post the journals of a specific file: In File Maintenance screen, click Sales Ledger > Billing File > Recalculate/Journal and tick Post New Journals.

  • To post the journals of all Account Files (this include in-house files): In Viewpoint - Home screen, click Practice manager > Recalculate/Journal and tick Post New Journals.
