Automatic Valuation/Revaluation Reference

It is possible to automate the Reference field as the default reference when creating a valuation or revaluation. This is done by creating user fields in Configuration.

  1. In Viewpoint - Home, go to Configuration Ribbon button > Business Parameters > User Fields > Account File and click New.

  2. Tick General Field.

  3. Enter the following information:

    • Key - Type the relevant code:

      • VAL.F.RE - For Account File Valuation.

      • VAL.P.RE - For Portfolio Valuation.

      • VAL.X.RE - For Exchange Revaluation.

      • VAL.V.RE - For Sub-ledger Revaluation.

    • Field Type - Select Text [11].

    • Caption - Enter a suitable caption.

    • Default Value - This is the starting number of the field. If the default value is 100, the first number assigned will be 101, followed by 102, 103, etc.

    • Default Note - This is used to add text and/or other information to the reference. You may use a mixture of text and the following codes in this field to automatically pick up and display specific information:

      • (-EC-) will be replaced by the Account File code.

      • (-NR-) will be replaced by the sequence number generated. If this code is not included in the Default Note field, the sequence number will be suffixed.

      • (-NOW\format-) will be replaced by today's date following the specified format string. For example:

        • (-NOW\yyyy-) returns the year, i.e. 2011.

        • (-NOW\yyyy.MM-) returns the year and month, i.e. 2011.02.

  4. Click Update.

After you have defined the user field, Viewpoint will automatically add it to the Reference field each time a Valuation or Revaluation is created.

 

Example of how the Default Note field can be used

If you enter the following in this field:

AF:(-EC-)~(-NOW\yyyy.MM-)`(-NR-)

when you create a new journal, the Reference field will return:

AF:INVEST~2011.02`655
