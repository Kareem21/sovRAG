Fixed Asset Depreciation

The Fixed Asset Depreciation function allows the system to systematically reduce recorded cost of fixed assets and create the relevant journal entries for the depreciation. The journal entries created will CREDIT the fixed asset's sub-ledger account and DEBIT the Expenses account for the Nominal Account of the Fixed Asset's sub-ledger. If Unit Codes are used, then the values will follow Unit Code values from last journal entry for the sub-ledger and nominal account. Other fields will follow default values but these can be customised using tags.

At the moment, there is only one depreciation method - Straight Line which needs to be applied to a Fixed Asset sub-ledger account in order to work.

 

Journal Entries for Depreciation

The journal entries that are created will CREDIT the Sub-ledger account and DEBIT the Expenses account for the Nominal Account of the Fixed Asset’s sub-ledger.

 

If Unit Codes are used, the values for all Unit Codes will be based on Unit Code values from last journal entry for the sub-ledger and nominal account.

 

Depreciation Calculation

Straight Line Depreciation Method

This method depreciates the cost of the fixed asset evenly throughout its useful life using the following formula:

Depreciation

 = -

(Previous Balance - Residual Value)

(Number of periods - Current Depreciation Period + 1)
 

Balance

=

Previous Balance + (Fixed Asset Transactions) + Depreciation

 

Transactions with movement codes FA, FD, FE, FO and FV are taken into account for the calculation (but each asset requires an FA journal). Transactions with movement codes FX and FU are excluded.

Once the Balance is less than the Residual Value, then Depreciation = 0

 

As for the start date of the depreciation period, this is based on the Last Year-End date for the Account File:

  • If Last Year-End date is the last day of the month, the depreciation date is the last day of the month.

  • If Last Year-End date is not the last day of the month, the depreciation date will be the same 'day' as Last Year-End date. For example, if the Purchase Date is 15 May 2013 and the Last Year-End date is 15 July 2013, the depreciation period will be on the 15th.

However, if there is a depreciation entry that is manually posted in a period, the depreciation date will now follow the posting date.

Pro-rata Depreciation on Disposal of Fixed Asset

When there is a disposal date, then the calculation will stop one day before the disposal date and the depreciation will then be pro-rated on a daily basis. In this case, it will take into account the following:

  • Number of days between Disposal Date and Last Depreciation Date (X)

  • Number of days between Current Depreciation Date and Last Depreciation Date (Y)

  • Depreciation Value as at Current Depreciation Date (C)

Depreciation Value  = (X / Y) * C

 

For example:

Purchase Value = EUR 8,555.00

Purchase Date = 15/05/2013

Residual Value = EUR 1.00

Frequency = Monthly

Number of Periods = 12

Current Depreciation = -712.83

 

With Disposal Date at 08/05/2014, we will have:

Last Depreciation Date = 15/04/2014

Current Depreciation Date = 15/05/2014

 

So,

X = 23 days (Number of days between 08/05/2014 and 15/04/2014)

Y = 30 days (Number of days between 15/05/2014 and 15/04/2014)

Depreciation Value up to 07/05/2014

= (23 / 30) * (-712.83)

= - 546.50

 

Tags

Tags are available to allow for:

  • Customising the depreciation journal entries

  • Initiate depreciation when using Depreciation Journal Script

 

Customising Depreciation Journals

When posted, the journals used the default values for fields like Reference, Description and Narrative. If you would like to customise, you can do so by using the tags. The tags are to be entered into the Tag Values field of the Nominal Account for Fixed Assets.

  1. In Viewpoint - Home screen, go to Accounting Ribbon button and select Chart of Accounts > Nominal Accounts from the navigation listing in Accounting Overview.

  2. Select the Chart of Account.

  3. Select the Nominal Account and click Edit.

  4. Enter the tags in Tag Values.

    When defining the tags, use the following syntax:

    [<Tag>=<Value>]

    where:

    • <Tag> is the supported tag

    • <Value> can be normal text or field code such as:

      • (-UFV.JNLREF-) - refers to the User Field 'JNLREF'

      • (-SL.<FieldName>\<Format>-) - can be any field from the sub-ledger with or without formatting

      • (-JL1.<FieldName>\<Format>-) - can be any field from journal line 1 (Sub-ledger) with or without formatting

      • (-JL2.<FieldName>\<Format>-) - can be any field from journal line 2 (Nominal Account) with or without formatting

    Note: If using field code, ensure that <FieldName> and <Format> is replaced with the actual field and format option.

     

    Tag

    Explanation

    Example

    REF

    This is to specify the reference for the journal line. The default value for this is from the JNLREF User Field.

    [REF=DEPR~(-UFV.JNLREF-)]

    NomDepr

    This is to specify the Nominal Account for the depreciation entry. The default value for this is the Nominal Account set for Expenses

    [NomDepr=AP: Expenses]

    Descr_SL

    This is to specify the description for the sub-ledger’s journal entry. The default for this is ‘Depreciation’.

    [Descr_SL=Depreciation]

    Descr_NA

    This is to specify the description for the Nominal Account’s journal entry. The default for this is ‘Depreciation Fixed Assets’.

    [Descr_NA=Depreciation of Fixed Assets]

    Narr_SL

    This is to specify the narrative for the sub-ledger’s journal entry. There is no default narrative set.

     

    Narr_NA

    This is to specify the narrative for the Nominal Account’s journal entry. The default for this contains the sub-ledger name after the text ‘Depreciation of’.

    [Narr_NA=Depreciation of (-SL.NAME-)]

    Residual

    This is to update the Residual Value to the specified value if the Residual Value was set as ‘0’ in the Fixed Asset’s Sub-ledger account.

    [Residual=1]

     

  5. Click Update.

  •  Nominal Account Setup.

 

Initiating Fixed Asset Depreciation

This is to open the Depreciation screen when you want to use the Disposal Journal Script. Doing so will allow you to do the depreciation before the disposal journal. The tag is [Initiate=FADEPR] and it is to be entered into Tags field of the Depreciation Journal Script.

Using Fixed Asset Depreciation

Fixed Asset sub-ledger account must be using Straight Line depreciation method with the following information specified:

  • Depreciation Method - Set this to Straight Line

  • Residual Value - Estimated value of the fixed asset at the end of its useful life.

  • Frequency - Interval for the depreciation.

  • Number of Periods - Length of the fixed asset's useful life. This is based on the frequency selected.

 

Depreciation for a single Fixed Asset

This can be done using the Analyses button in Fixed Assets Sub-ledger screen as well as from Account Enquiry screen.

 

Fixed Asset Sub-ledger screen

For one fixed asset, use the Analyses button:

  1. In the Fixed Assets Sub-ledger screen, select the fixed asset sub-ledger account.

  2. Click Analyses.

  3. In the Sub-ledger Account Analyses screen, the Date From and - to defaults to the Account File's posting window.

  4. Click Depreciation tab.

  5. In Depreciation, you will see the proposed depreciation journal entries.

  6. Select the next action:

    • Click Post to post the depreciation journal entries.

    • Click Disposal Date to set a disposal date. The depreciation calculation will be adjusted accordingly.

      1. Once you clicked Disposal Date, the calendar will appear.

      2. Select the disposal date for the fixed asset.

    Note: When user posts the depreciation entry, the record will flag as 'Capital' if the 'Capital/Income' field for the selected sub-ledger is set to 'Both Capital and Income [B]'.

 

Account Enquiry screen

In the Account Enquiry screen, select the fixed asset’s sub-ledger first before you click Analyses.

 

 

Depreciation for all Fixed Assets

To depreciate for all the Account File's fixed assets, click Depreciate All in the Fixed Assets Sub-ledger screen.

 

Depreciation Analyses

At times when you run the Depreciation, instead of just the depreciation journals, you may see warnings.

 

Difference in depreciation

If there had been earlier depreciation postings with values that do not correspond to the system calculation, there will be a warning to let you know:

In this case, the system will create an entry to correct the value when you click Post.

 

Depreciation period is outside Posting Window

If the depreciation posting falls outside the Account File’s posting window, there will also be a warning:

When you click Post, you will be asked if you want to continue. Choosing yes will result in one depreciation journal entry with the Account File's Posting Open From date.
