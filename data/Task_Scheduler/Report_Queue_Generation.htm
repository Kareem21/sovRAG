Report Queue Generation

This job allows for the automatic generation of reports that have been sent to Document Queue. With this job, users do not need to manually choose to generate the reports.

 

Setting up Task Scheduler

  1. Go to Configuration > Business Parameters > Task Scheduler > Agent Job Scheduler.

  2. Click New and ensure that the Job Type is set to 'Report Queue Generation [RPQG]'.

  3. Click Next >.

  4. In Criteria, you can set the following:

    • select the user in the 'User' field. If the job is for all users, leave this blank.

    • specific Word report(s) by selecting the template(s) in the 'Word Reports' field. If the job is for all Word reports, leave this blank.

    • specific Excel statement(s) by selecting the template(s) in the 'Excel Statements' field. If the job is for all Excel statements, leave this blank.

    • specify Document Limit with number for how many records of Document Queue to be processed in a single job execution.

    • select Yes or No for Document Delivery. The Document Delivery is an option to enable or disable  the Document Delivery Manager in this job type. When choosing Yes, it will automatically send out email if the document consists of Delivery Manager tag.

    • specify When the e-mail is sent. Below options can be selected:

      • Store the Document Holder (no e-mail) - The holder will be stored in Document Manager and removed from the Word Reports In-tray.

      • Delete the Document Holder - If you select this option, the Document Holder will be deleted from the Word Reports In-tray when the document is converted to e-mail or printed out.

    Note: When the user selects the templates from only one of the categories (Word Reports or Excel Statements), it will only applies to that selected category and templates.

    Example:

    • When the user selects one/more Word report(s) and did not select any Excel statement, it will only proceed with the selected Word reports.

    • When the user selects one/more Excel statement(s) and did not select any Word report, it will only proceed with the selected Excel statements.

  5. Click Next >.

  6. If an e-mail notification for the job is required, tick Enable E-mail Notification and specify details for the e-mail. If you want different types of e-mail notification, tick Advanced and you will get the following:

    • Success - this is the notification when the job is completed successfully.

    • Fail - this is the notification when the job is not completed.

  7. Click Next >.

  8. In Schedule, enter the job frequency.

  9. Click Done.

When the job has been performed, users will be able to see the generated reports in Document Queue. If e-mail notification was set up, this will be sent out as well.

 

Note: If you ticked Advanced Settings in Setup tab, you can use the following tag as well as MAIL tags:

Tag

Explanation

Example

[/CONDITION-UCode=<UserCode>/]

This is to apply only to reports for the specified user(s).

[/CONDITION-Ucode=SM/]

 

means to only apply to reports sent to Document Queue by the user 'SM'.

[/CONDITION-DocCode=<TemplateCode>/]

This is to apply to the selected Word report(s) only.

[/CONDITION-DocCode=MFUFLDCO/]

 

means to only apply to the Document 'MFUFLDCO' sent to Document Queue by any user.
