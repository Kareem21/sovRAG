Replication Jobs

There are two job types for Replication and they are:

  • Replication - Export [REEX]

  • Replication - Import [REIM]

  • Replication - Download Import File [REDL]

All are used when you have MCE and subscriber environments so that you can export or import settings, Master Files and Address Cards from MCE to the subscriber environments.

 

Setting up Task Scheduler for Export

  1. Go to Configuration > Business Parameters > Task Scheduler > Agent Job Scheduler.

  2. Click New and ensure that the Job Type is set to 'Replication - Export [REEX]'

  3. In Tag Values, you can:

    •  See the following tags:

      • [/TASK_ROUTINE=REEX/]

      • [/Script=/]

        • To define the script, ensure that the following information is specified in this tag:

          • [PATH=<LandingArea>] - This is to specify the location of the Landing Area.

          • [TIMEOUT=<Seconds>] - This is to specify the interval for the next attempt if the export fails.

          • [RETRY=<Number>] - This is to specify the number of attempts for the export.

          • [SUBSCRIBER=<SubscriberGroup>] - This is to specify the subscriber group.

          • [JOB=] - This is to specify what to export. Refer to [JOB=] Tag for more information.

    • Add Mail Job tags to set up e-mail notification for this job.

  4. Click Next >.

  5. In Schedule, enter the job frequency.

  6. Click Done.

Note: The Replication Excel Sheet available on Viewpoint Support Portal can be used to help generate the script.

    • <Parameter> is either CODE=<Value> or CODE_EXCL=<Value> and <Value> is used to indicate the settings to include or exclude

    Parameter Explanation Value Example

    CODE

    This is to specify what to include for the settings to export.

    * - this is to indicate all values for the specified item type.

    [JOB=SS_LUVAL|CODE=*]

    means to export all customised lookup values.

    [JOB=AD_RLTYP|CODE=*]

    means to export all customised relationship types.

    <ValCode> - this is to define specific value(s) using the code. For multiple values, use a comma to separate each value.

    [JOB= SS_LUVAL|CODE=ADMU]

    means to export customised values for the Administrative Mark-up [ADMU] lookup table.

    [JOB=SS_DDVAL|CODE=NATI,ITIT]

    means to export the drop-down tables NATI and ITIT.

    <KeyCode>+<KeyCode> - this applies to certain setting items such as Entity Types where a record has more than one key field and you want to specify these. You can also use a combination of <KeyCode> and * as well.

    [JOB=AD_ENTYP|CODE=CY+CI]

    means to export the entity type ‘CI’ in Cyprus.

    [JOB=AD_ENTYP|CODE=CY+CI,IM+PL]

    means to export only the entity types ‘CI’ in Cyprus and ‘PL’ in Isle of Man.

    [JOB=AD_ENTYP|CODE=CY+*]

    means to export all the Entity types in Cyprus.

    CODE_EXCL

    This is to specify what to exclude for the settings to export.

    Same as for CODE above except for *.

    [JOB= SS_LUVAL|CODE_EXCL=ADMU]

    means to export customised values for all lookup tables except the Administrative Mark-up [ADMU] lookup table.

    [JOB=AD_ENTYP|CODE_EXCLE=CY+CI,IM+PL]

    means to export all Entity types except for the entity types ‘CI’ in Cyprus and ‘PL’ in Isle of Man.

     

  • Master Files

    For exporting Master Files,

    • <Item> is AD_MFALL

    • <Parameter> must include MF=<Condition> and INFO=<Value> to specify the Master File(s) and details of the Master File to export where MF=<Condition> and INFO=<Value> is separated using a colon

    Parameter Explanation Condition/Value Example

    MF

    This is to specify the Master File(s) to export.

    The <Condition> can be:

    • 1=1 to include all Master files.

    • MfNr< 0 to include only Global Master Files.

    • Based on any field in the RefMaster table.

    [JOB=AD_MFALL|MF=1=1|INFO=]

    means to export all Master Files.

     

    [JOB_AD_MFALL|MF=MfNr<0|INFO=]

    means to export all Global Master Files.

     

    [JOB_AD_MFALL|MF=AdminGroup IN ('GROUP1','GROUP2')|INFO=]

    means to export Master Files in Administrator Group 1 and Group 2.

    INFO

    This is to specify the Master File details to export.

    The <Value> can be empty to export default details from the Master File's Setup, General and CDD screens or include additional details with the following:

    • AL to include details from the Alerts screen.

    • ST to include details from Master File Status screen.

    • UF to include details from User Fields - Master File screen.

    For multiple values, separate each value with a semi colon.

    [JOB=AD_MFALL|MF=1=1|INFO=]

    means to export all Master Files and the default details.

     

    [JOB=AD_MFALL|MF=1=1|INFO=AL]

    means to export all Master Files and details from the Master File's Setup, General, CDD and Alerts screens.

     

    [JOB=AD_MFALL|MF=1=1|INFO=AL;ST;UF]

    means to export all Master Files and details from the Master File's Setup, General, CDD and additional details from the Alerts, Master File Status and User Fields - Master File screens.

     

  • Address Cards

    For exporting Address Cards,

    • <Item> is AD_ADDRE

    • <Parameter> is AC=<Condition>

    Parameter Explanation Condition Example

    AC

    This is to specify which Address Card(s) to export.

    The <Condition> can be:

    • 1=1 to include all Address Cards.

    • AddrNr < 0 to include only Global Address Cards.

    • Based on any field in the Addresses table.

    [JOB=AD_ADDRE|AC=1=1]

    means to export all Address Cards.

     

    [JOB=AD_ADDRE|AC=AddrNr<0]

    means to export only Global Address Cards.

     

    [JOB=AD_ADDRE|AC=AddrNr=10001]

    means to export the Address Card 10001.

     

    [JOB=AD_ADDRE|AC=Country='JE']

    means to export all Address Cards where Country is Jersey.

     

    [JOB=AD_ADDRE|AC=Country IN ('JE', 'IM')]

    means to export all Address Cards where Country is Jersey or Isle of Man.

     

 

Sample Tag Configuration

  Explanation

[/TASK_ROUTINE=REEX/]

[/Script=[PATH=C:\Viewpoint\Configuration]

[TIMEOUT=60]

[RETRY=10]

[SUBSCRIBER=GEN]

[JOB=SS_LUVAL|CODE=*]

[JOB=AD_RLTYP|CODE=*]

[JOB=AD_MFALL|MF=MfNr<0|INFO=AL:ST;UF]

[JOB=AD_ADDRE|AC=AddrNr<0]

[SUBSCRIBER=ASA]

[JOB=SS_USRTB|CODE_EXCL=VGCHARGE,IECHARGE]

[SUBSCRIBER=EUR]

[JOB=SS_USRTB|CODE_EXCL=HKCHARGE]/]

The tag configuration indicates an export job where the following will be exported to the Landing Area:

  • For Subscriber Group - GEN:

    • All Lookup Table values.

    • All Relationship types.

    • All Global Master Files with all additional information.

    • All Global Address Cards.

  • For Subscriber Group - ASA:

    • All User Table Values except for those in the User Table 'VGCHARGE' and 'IECHARGE'.

  • For Subscriber Group - EUR:

    • All User Table Values except for those in the User Table 'HKCHARGE'.

Setting up Task Scheduler for Download Import File

You can choose to download the exported files from the landing area to your local storage prior to importing using the task scheduler. This allows you to apply the .xml files in the relevant folders from your local storage to your environment at a later time.

  1. In Viewpoint – Home, go to Configuration > Business Parameters > Task Scheduler > Agent Job Scheduler.

  2. Click New

  3. In the Add New Job screen, specify the following:

    •  Thread Number – Select the thread number for this job.

    • Job Type – Set this to Replication – Download Import File [REDL].

    • Description – This defaults to the description of the job type, but you can change this if necessary.

    • Note – Enter additional notes or remarks if relevant.

  4. Click Next >.

  5. In the Criteria tab, define the following fields:

    • Destination Folder Path - This is to specify the folder path for the downloaded files in your local machine.

    • Viewpoint Username - This is to specify the Viewpoint username to run the application service.

    • Viewpoint Password -  This is to specify the login password for the user.

    • Service Address - This is to specify the application service address.

    • Port - This is to specify the port number for the application service.

    • Windows Authentication - This is to specify if the user’s login mode is set as ‘Windows account login.

  6. Click Next >.

  7. In the Notifications tab, you can set up e-mail notifications for this job. Refer to Add New Job Wizard on how to configure e-mail notifications.

  8. Click Next >.

  9. In the Schedule tab, enter the job frequency.

  10. Click Done.

Note: If an invalid value is specified in the Criteria tab for Replication – Download Import File, an error message will be seen in the Agent Job Log screen when the job is run. This requires you to check and correct the criteria for the job.
