OCR Extraction

The 'OCR Extraction [OCRJ]' job type performs OCR updates on stored documents based on the selection criteria of number of documents, document type, and document extract order. Once the documents are recognised, users can perform document searching based on the text in the document. This job type requires the OCR Registration Code, which can be entered in the Application Server setup in Viewpoint Management. Note: When using OCR Transym, the sequence will be the pdf text first, followed by the text in the image.

 

 

Setting up Task Scheduler

  1. Go to Configuration > Business Parameters > Task Scheduler > Agent Job Scheduler.

  2. Click New and ensure that the Job Type is set to 'OCR Extraction [OCRJ]'.

  3. Click Next >.

  4. In the 'Criteria' tab, enter the values and select the options for the following mandatory fields:

    • Total Document - Enter a numerical value for the number of documents to include in the OCR update process. For example, if the value is 1000, the OCR update will be performed on one thousand stored documents.

    • Document Types - Select the file formats to include in the OCR update process.

    • Document Extract Order - Select 'Oldest to Newest' to perform OCR update starting from the earliest document stored or 'Newest to Oldest' to perform OCR update starting from the latest document stored. Documents already OCR updated will not be included in this selection.

  5. Click Next >.

  6. If an e-mail notification is required, tick Enable E-mail Notification and specify details for the e-mail.

  7. Click Next >.

  8. In Schedule, enter the job frequency.

  9. Click Done.
