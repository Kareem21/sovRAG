Global File Review

User can schedule to periodically run the Global File Review task in Viewpoint Task Scheduler and Notification Agent. This can be done through the new Global File Review [GLFR] job type.

 

Setting up Task Scheduler

  1. Go to Configuration > Business Parameters > Task Scheduler > Agent Job Scheduler.

  2. Click New and ensure the Job Type is set to ‘Global File Review [GLFR]’.

  3. Click Next >.

  4. In Tag Values, add the tags to specify the requirements and review scope for the job.

  5. Click Next >.

  6. In Schedule, enter the job frequency.

  7. Click Done.

When the scheduled Global File Review job is successfully executed, user will be able to see the created review tasks in the File Review screen. Separate records will be created for each defined ‘Group’.

 

Global File Review – Tags

The following tags are used to specify which Master Files to review and review scope for the job.

Tag

Explanation

Example

[Description=<Text>]

This defines the ‘Review Description’ column in the File Review screen.

[Description=AEOI Review]

[<Parameter><Operator><Value>]

 

This defines the Master Files for the review where:

  • <Parameter> can be:

  • MfType - To include/exclude Master Files based on Master File type.

  • MfStatus - To include/exclude Master Files based on Master File status.

  • AdminGroup - To include/exclude Master Files based on the Administrator Group of the Master File.

  • OrgUnit - To include/exclude Master Files based on the Organisational Unit of the Master File.

  • Team - To include/exclude Master Files based on the Team.

  • <Operator> can be:

  • = to include

  • <> to exclude

  • <Value> refers to the code for the parameter e.g. if <Parameter> is MfType, <Value> is the code of the Master File Types [ADTP] lookup values.

To have multiple values, separate with a comma.

[MfType=C,I,T]

means to include Company, Individual and Trust for Master File Type.

 

[MfType<>N,O,P,F]

means to exclude Master Files of the type 'Foundation', 'Other', 'Partner' and 'Fund'.

 

[MfStatus=0]

means to include Master Files that are Open/Active only.

 

[MfStatus<>8]

means to exclude Master Files with the status 'Closed/Inactive'.

 

[AdminGroup=GROUP1,GROUP3]

means to include Master Files in the Administrator Groups 'GROUP1' and 'GROUP3'.

 

[AdminGroup<>GROUP2]

Means to exclude Master Files in the Administrator Group 'GROUP2'.

 

[OrgUnit=ACC,CSC]

means to include Master Files in 'ACC' and 'CSC' Organisational Unit.

 

[OrgUnit<>CSL]

means to exclude Master Files in the 'CSL' Organisational Unit.

 

[Team=T1]

means to include Master Files in the 'T1' Team.

 

[Team<>T2]

means to exclude Master Files in the 'T2' Team.

[ReviewScope=<Value>]

This is to specify the file review scope where <Value> can be:

  • 0 - All review scope

  • 1 - Setup review

  • 4 - Compliance review

  • 8 - Statutory review of managed entities

  • 64 - Meetings review

  • 128 - Filings review

  • 256 - Reviews review

  • 512 - Data Validation review

  • 1024 - AEOI review

  • 2048 - AEOI review of managed entities

  • 4096 - Activities Profiles review

[ReviewScope=0]

means to include all review scopes

 

[ReviewScope=448]

means to include Filings review (128), Meetings review (64) and Reviews review (256)

[128+64+256=448]

[GroupBy=<Field>]

This is to specify the group that will be used to separate the review results where <Field> refers to the grouping and can be:

  • MfType – To group by Master File type.

  • MfStatus – To group by Master File status.

  • AdminGroup – To group by the Administrator Group of the Master File.

  • OrganisationUnit – To group by the Organisational Unit.

  • Team – To group by the Master File team.

[GroupBy=AdminGroup]

means to separate review results according to the Administrator Groups.

If the parameter is defined as [AdminGroup=GROUP1, GROUP2],the review results will be separated into GROUP1 and GROUP2.

If the parameter is defined as [AdminGroup<>GROUP1], the review results will be separated into other available Administrator Groups.

[AssignedTo=<Group>:<User>]

This is to specify the user who will be assigned to the review results where:

  • <Group> can be:

  • Entity - this refers to Master Files that are managed entities and can further be grouped by Administrator Group, Organisational Unit or Team

  • NonEntity - this refers to Master Files that are not managed entities and can also be grouped by Administrator Group, Organisational Unit or Team

  • Default - this will be applied to all Master Files that are not included in the above options

  • <User> can be:

  • AdminCode - user in the Administrator field

  • Admin2Code - user in the Manager field

  • Admin3Code - user in the Controller field

  • User Code

To specify different users for different groups, separate with '|' symbol.

 

[AssignedTo=Entity:AdminCode|NonEntity:sm]

means results for:

  • managed entities will be assigned to the Administrator of the Entity File

  • non-managed entities will be assigned to the user 'sm'

 

[AssignedTo;Entity.GROUP1:MARK|Entity.GROUP2:ANNE|Default:SM]

means results for:

  • managed entities in GROUP1 will be assigned to the user 'MARK'

  • managed entities in GROUP2 will be assigned to the user 'ANNE'

  • all other Master Files will be assigned to the user 'SM'

 

[POSTUPDATE=<Value>]

This is to specify whether the Global File Review results to be posted on the File Review screen.

[POSTUPDATE=1]

means the Global File Review updates can be seen on the File Review screen.
