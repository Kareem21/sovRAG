Name Check Service

To be able to use Viewpoint Name Check Services, the following job types must be set up:

  • Name Check Service – Check Status/Change Status [NCCS]

  • Name Check Service – Submit Name [NCCN]

However, note that the Name Check Service - Check Status/Change Status [NCCS] job type is only required if the service provider is RDC or World-Check.

 

Setting up Task Scheduler: Name Check Service - Submit Name [NCCN]

  1. Go to Configuration > Business Parameters > Task Scheduler > Agent Job Scheduler.

  2. Click New and ensure that the Job Type is set to 'Name Check Service - Submit Name [NCCN]'.

  3. Click Next >.

  4. Enter the details in the Criteria.

    Field

    Description

    Service URL

    Enter the URL to connect to the service provider.

     

    Example:

    http://<ServiceAddress>:<ServicePort>/api/NameServices/SubmitName

    Username

    Enter the username used for Viewpoint Services.

    Password

    Enter the password used for Viewpoint Services.

    Review Scope

    When ‘All’ is selected, it will retrieve Master Files based on the criteria set in the Condition field.

    When ‘Since Last Checked’ is selected, it will match the condition and based on the Change Code entered in the Change Code field.

    Condition

    Enter the condition that will determine the Master Files that will be retrieved for submission. The condition can only be based on fields in the RefMaster table.

     

    Example:

    To retrieve non-managed entities with Master File Status 'Active [0]' and CDD selected

    KY=1 AND Status = 0

     

    To retrieve Master files with RDCC Identity Register Type created

    RefCode IN (SELECT id.RefCode from IdentityRegister id WHERE IdType='RDCC')

    Change Code

    Enter Change Code(s) that are used to retrieve Master Files. This is only applicable when Review Code is ‘Since Last Checked’.

     

    Example:

    ADN for New Master File

    Changes Since

    Enter the start date of the change code. This is only applicable when Review Code is ‘Since Last Checked’.

    Document Code

    Enter the template code. Currently, there is only one template - NCXML.

     

    Example:

    NCXML

    Provider Type

    Select the service provider that you would like to connect to. Currently, the following providers are available:

    • Accuity for Automated Screening Manager (ASM)

    • ComplyAdvantage

    • CDDS for AML Risk

    • RDC

    • World-Check

     

  5. Click Next>.

  6. In the Schedule tab, enter the job frequency.

  7. Click Done.

For Name Check result,

  1. In Viewpoint Home, go to Configuration > Business Parameters > Task Scheduler > Agent Job Log.

  2. Enter ‘Name Check*’ in the Description and click Apply.

  3. Double-click on the record to view full details.

 

Setting up Task Scheduler: Name Check Service – Check Status/Change Status [NCCS]

This is not necessary if the service provider is not RDC or World-Check.

  1. In Viewpoint – Home, go to Configuration > Business Parameters > Task Scheduler > Agent Job Scheduler.

  2. Click New and ensure that the Job Type is set to ‘Name Check Service – Check Status/Change Status [NCCS]’.

  3. Click Next >.

  4. Enter the details in the Criteria.

    Field

    Description

    Service URL

    Enter the URL to connect to the service provider.

     

    Example:

    http://<ServiceAddress>:<ServicePort>/api/NameServices/CheckStatus

    Username

    Enter the username used for Viewpoint Services.

    Password

    Enter the password used for Viewpoint Services.

    Provider Type

    Select the service provider that you used for name submission.

     

  5. Click Next >.

  6. In the Schedule tab, enter the job frequency.

  7. Click Done.
