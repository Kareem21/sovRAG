SSRS Report - Subscriptions and Delivery

This job type will generate an SSRS report and deliver it at a specific time as defined in the subscription. This is an alternative to running the SSRS report on demand. On-demand SSRS reporting requires that you actively select the report for generation each time you want to view the report. With this job type, setting up the subscriptions allow the generation to be scheduled and automate the delivery of the report.

Note: This job type requires at least Microsoft SQL Server 2012 Express LocalDB to be installed in the Application Server environment to run.

 

Setting up Task Scheduler

  1. Go to Configuration > Business Parameters > Task Scheduler > Agent Job Scheduler.

  2. Click New and ensure that the Job Type is set to 'SSRS Report - Subscription and Delivery [SSRS]'.

  3. Click Next >.

  4. In Tag Values, specify the requirements for the job using the tags.

  5. Click Next >.

  6. In Schedule, enter the job frequency.

  7. Click Done.

 

Tag

Explanation

Example

[/TASK_ROUTINE=SSRS/]

This defines the job type as SSRS Report - Subscription and Delivery.

 

[/SSRS_REPORTCODE=<ReportCode>/]

This is to specify the SSRS report to generate.

[/SSRS_REPORTCODE=SCADCOMC/]

 

will generate the SSRS report with the code 'SCADCOMC'.

[/SSRS_PARAMETERLIST=<ParameterCode>/]

This is to define the parameters for the SSRS Report. For multiple parameter, separate each parameter code with a comma.

Optional parameter is not required to define in the parameter list unless it is used in the report.

Note: If a SSRS Report contains a hidden mandatory parameter, it is mandatory to hardcode the values of the parameter into the tag values, otherwise it would result in error when generating the report via the Task Scheduler.

[/SSRS_PARAMETERLIST=RefCode/]

OR

[/SSRS_PARAMETERLIST=RefCode,RevEntity/]

 

[/SSRS_PARAMETER.<ParameterCode>.OPERATOR=<Operator>/]

[/SSRS_PARAMETER.<ParameterCode>.VALUES=<Value>/]

OR

[/SSRS_PARAMETER.<ParameterCode>.OPERATOR=<Operator>/]

[/SSRS_PARAMETER.<ParameterCode>.VALUES=<Value1>|<Value2>/]

 

 

This is to assign an operator and value for a report parameter. For multiple values, separate with a '|'.

Supported operators are:

  • =

  • BETWEEN

  • LIKE

If the optional parameter is not defined in the parameter list, then it will not require to assign the Operator and Value.

Note: If a SSRS Report contains a hidden mandatory parameter, it is mandatory to hardcode the values of the parameter into the tag values, otherwise it would result in error when generating the report via the Task Scheduler.

The following are examples based on data types:

 

[/SSRS_ PARAMETER.RefCode.OPERATOR=LIKE/]

[/SSRS_PARAMETER.RefCode.VALUES=IN%/]

 

Above RefCode mean to retrieve for Code start with 'IN'

 

OR

[/SSRS_ PARAMETER.RefCode.OPERATOR==/]

[/SSRS_PARAMETER.RefCode.VALUES=INVEST|CASEY/]

 

Above RefCode mean to retrieve for 'INVEST' and 'CASEY'

 

For BIT data type. Example, True/False:

[/SSRS_PARAMETER.Approved.VALUES=0/] 

OR 

[/SSRS_PARAMETER.Approved.VALUES=False/] 

 

For currency data type. Example, $750.00:

[/SSRS_PARAMETER.InvoiceTotal.VALUES=750.00/] 

OR 

[/SSRS_PARAMETER.InvoiceTotal.VALUES=750/]

 

For number (integer) data type. e.g. range of Draft Number from 11000 to 12000:

[/SSRS_PARAMETER.ProformaNr.OPERATOR=BETWEEN/] 

[/SSRS_PARAMETER.ProformaNr.VALUES=11000|12000/] 

 

For text (varchar) data type:

[/SSRS_PARAMETER.RevEntity.VALUES=INVEST/]

 

For date data type. e.g. 01-4-2018:

[/SSRS_PARAMETER.DateInvoice.VALUES=01-Apr-2018/]

 

Date values must be in the following formats:

  • d-MMM-yyyy

  • dd-MMM-yyyy

  • d-MMMM-yyyy

  • dd-MMMM-yyyy

[/SSRS_FORMAT=<Format>/]

This is to define the output of the report when sent to the intended recipient. Format can be:

  • PDF (default)

  • Word

  • Excel

[/SSRS_FORMAT=EXCEL/

 

will send the report as an Excel file.]

 

[/MAIL_TO.S=<Recipient>/]

OR

[/MAIL_TO.S=<Recipient1>;<Recipient2>/]

This is to define the e-mail address of the recipient of the report.

For multiple recipients, separate each e-mail address with a semicolon.

Note: SQL cannot be used as retrieve e-mail address.

[/MAIL_TO.S=anne.shannon@manford.com/]

OR

[/MAIL_TO.S=anne.shannon@manford.com;sally@manford.com/]

[/MAIL_SUBJECT.S=<Text>/]

This is to specify the subject of the e-mail with the report.

[/MAIL_SUBJECT.S=SSRS Report Subscriptions and Delivery/]

[/MAIL_BODY.S=<Message>/]

This is to specify the message of the e-mail with the report.

[/MAIL_BODY.S=Attached herewith is the report for your reference./]

 

Note: If necessary to have an e-mail notification when the job fails, the Mail Tags can be used but replace 'S' with 'F'.
