Viewpoint Name Check Services

Viewpoint Name Check Services allows for information in Viewpoint to be extracted and submitted to name check service providers such as World-Check, RDC, CDDS, Accuity and ComplyAdvantage. This means that if you have an account with any of the service provider, you can set up Viewpoint Name Check Services so that the following can take place using Viewpoint Task Scheduler and Notification Agent:

  • Information in Viewpoint is extracted as an XML file

  • The XML file is uploaded to the name check service provider for comparison

  • Results from the comparison is sent back to Viewpoint to be viewed in the Changes screen

 

Requirements

Viewpoint Name Check Services requires the following to be installed and set up:

  • Viewpoint Application Server

  • Viewpoint Task Scheduler and Notification Agent (TSNA)

  • Viewpoint Name Check Services

Additionally, the template for the XML file must also be imported. Both the Viewpoint Name Check Services and the template can be downloaded from the Support Portal. For Viewpoint Application Server and Viewpoint Task Scheduler and Notification Agent, these are to be set up using ViewPointDbMgt.exe.

 

Downloading and Installing the Viewpoint Services .zip File

  1. Download the Viewpoint Name Check Service.zip file from Viewpoint’s Support Portal.

  2. Unzip it to an accessible location folder.

  3. Right-click and run the ViewPointServicesInstaller.exe as administrator.

  4. In the Viewpoint Services screen, enter the required Service details.

    • Service Name – Enter the Viewpoint Services name

    • Program Folder – Enter or browse to the folder where the service is located

    • Service Address – Enter the same Service Address as the Application Server

    • Service Port – Enter a Service Port number

    Note: Both the Service Address and the Service Port will be used for the setting up of the Name Check Service jobs.

  5. Click Install.

  6. When prompted, click OK.

 

Configuring the Viewpoint Services

  1. Once the service has been installed, enter the Settings details which will be required when setting up the Name Check Service jobs.

    • Username – Enter a username for Viewpoint Services.

    • Password – Enter a password for Viewpoint Services.

  2. Select the service provider tab that you would like to integrate with. Enter the necessary details based on the service provider.

  3. Click Save.

  4. Go to Control Panel > System and Security > Administrative Tools > Services to check the status of the Viewpoint Services application. If the Status does not show ‘Running’, right-click on the service and click Start.

Upgrading Viewpoint Services

If you would like to upgrade Viewpoint Services, this is done by first stopping the service through Control Panel > System and Security > Administrative Tools > Services. Once the service has been stopped, proceed with the following:

  1. Uninstall the service by right-clicking and running the ViewPointServicesInstaller.exe as administrator.

  2. Click Uninstall.

  3. When complete, click OK and exit Viewpoint Services.

  4. Download the newer version of Viewpoint Name Check Service.zip file.

  5. Extract the contents to the existing folder for Name Check Services.

  6. Right-click and run the ViewPointServicesInstaller.exe as administrator.

  7. Click Install.

  8. When complete, click OK.

  9. Re-enter the password for the username in Settings.

  10. Select the service provider tab and re-enter the details.

  11. Click Save and exit Viewpoint Services.

 

Importing Name Check XML [NCXML] Template

  1. In Viewpoint - Home, go to Configuration > Document Libraries > Word Reports - Report Folder 'Master FIle [F12]'.

  2. Click Import and select the folder containing the NCXML template.

  3. Click Import - Load List.

  4. Click Import.

  5. Once completed, click Close.

 

Specifying GUID and Data Set ID

If the service provider is Accuity, then you need to specify the DatasetGuid and DatasetId in the Key Information part of the template.

  1. In Viewpoint - Home, go to Configuration > Document Libraries > Word Reports - Report Folder 'Master FIle [F12]'.

  2. Select the XML - Name Check Service template.

  3. Click Edit.

  4. Double-click within Key Information and for the [/MFDetails=/] query, specify the relevant Accuity’s GUID as DatasetGuid and DataSet ID as DatasetID.

  5. Click OK.

  6. Click Update.

  7. Click Refresh.

If you are using multiple dataset from Accuity, then the [/MFDetails=/] query is to be amended to contain all the relevant datasets.  For example:

[/MFDetails=SELECT RM.RefCode, RM.RefType, CASE WHEN RM.RefType='I' THEN 'P' ELSE 'O' END AS InquiryType, CASE WHEN (C.GivenNames IS NULL OR C.GivenNames='') THEN RM.Initials ELSE C.GivenNames END AS GivenNames, '3663b370-ca7d-4ceb-9851-3669fed8d61b' AS 'DatasetGuid', '1092' AS 'DatasetId' FROM RefMaster RM LEFT JOIN Compliance C ON C.AddrCode=RM.RefCode WHERE RefCode IN ((-SC.ListOfRefCode-))

UNION

SELECT RM.RefCode, RM.RefType, CASE WHEN RM.RefType='I' THEN 'P' ELSE 'O' END AS InquiryType, CASE WHEN (C.GivenNames IS NULL OR C.GivenNames='') THEN RM.Initials ELSE C.GivenNames END AS GivenNames, '3b62af7d-6f88-4a09-a487-7d3f388e2dff' AS 'DatasetGuid', '1093' AS 'DatasetId' FROM RefMaster RM LEFT JOIN Compliance C ON C.AddrCode=RM.RefCode WHERE RefCode IN ((-SC.ListOfRefCode-))/]

 

Setting Up Name Check Service Jobs

In Viewpoint Task Scheduler and Notification Agent, there are two new job types introduced for Viewpoint Name Check Services and they are:

  • Name Check Service - Submit Name [NCCN] - This will create the XML report based on the criteria defined for the data for submission via Viewpoint Services for name screening. This job is applicable for submission of data to all supported service providers.

  • Name Check Service – Check Status/Change Status [NCCS] - This checks the status for each batch of name submitted to the service provider. This will retrieve the results from service provider and update the event string for further processing. This job is only applicable for data retrieval from RDC and World-Check. It also allows the update of status for World-Check data screening.

Each job type needs to be created separately for different service providers. Once the job type is being setup, the task will run as scheduled in the job frequency. Refer to Name Check Service for information on setting up these job types.

 

Name Check Status Result in Changes Screen

To view the Name Check Status result, in Viewpoint - Home, click on the Changes button in the Ribbon to open the Changes screen where you will be able to see these Change Codes that relate to Name Check Services:

  • Name Check - Submit [MNCS] - this indicates the Master File submitted to name check service provider

  • Name Check - Negative [MNCN] - this is a negative result returned by the name check service provider i.e. meaning there is no exact match

  • Name Check - Positive [MNCP] - this is a positive result returned by the name check service provider when there is an exact match

  • Name Check - Pending Client Action [MNCE] - this is returned when there is a possible match

If desired, you can link these changes code to allow for further actions such as generating a document or creating a task. Refer to Change Codes for more information.
