CDDS Integration

This job allows for Master File information from Viewpoint to be transferred to an external program - AMLspotter by Customer Due Diligence Solutions (CDDS) using Task Scheduler. Tags are used to specify the following:

  • CDDS service address.

  • Criteria to define the Master Files to include.

  • Mapping between Viewpoint and AMLspotter fields.

 

Setting up Task Scheduler

  1. Go to Configuration > Business Parameters > Task Scheduler > Agent Job Scheduler.

  2. Click New and ensure that the Job Type is set to 'CDDS Integration [CDDS]'.

  3. Tick Advanced Setting.

  4. Click Next >.

  5. In Tag Values, you can see the default tags configuration for the job. Review the tags and update the following:

    • the service address to the service address of your AMLspotter

    • the criteria that will be used to determine which Master File information to be exported

    • the mapping of the fields

  6. Click Next >.

  7. In Schedule, enter the job frequency.

  8. Click Done.

CDDS Integration - Tags

Tag

Definition

Syntax and Example

DOALL

This is to specify the frequency for the data transfer to AMLspotter from the 'Date lastDo All review'. When this is due, the CCODE tag is ignored.

Note: 'Date lastDoAll Review' is the system recorded date when it performs the update to AMLspotter for all changes made in the defined change codes.

[/DOALL=<frequency>]

where <frequency> can be:

  • W - weekly

  • M - monthly

  • Q - quarterly

  • Y - yearly

Example: [/DOALL=w/]

SERVICE_ADDRESS

This is to specify the URL of the AMLspotter program.

[/SERVICE_ADDRESS=<URL>/]

where <URL> must be specified in full.

 

SQL.ALL

This query is to specify information that needs to be transferred. It will be run during the first data transfer and each time the data transfer is due (based on frequency set by DOALL).

[/<ClassPrefix>.SQL.ALL=<Query>/]

where:

  • <ClassPrefix> indicates the class for the information

  • <Query> is SQL query to specify what is to be included in the data transfer. The field MfNr MUST be included in the query and mapped to the respective field.

 

Example:

[/IND.SQL.ALL=SELECT * FROM RefMaster INNER JOIN Compliance ON Compliance.AddrCode=RefMaster.RefCode WHERE CP<>0/]

SQL.DT

This query is to specify updated information that should be transferred based on conditions set using the CCODE tag. This query will run if a data transfer has been performed before and there is EventLog table in the query.

[/<ClassPrefix>.SQL.DT=<Query>/]

 

Example:

[/IND.SQL.DT=SELECT DISTINCT MfNr, Status, Name, KYC_Country1, KYC_SourceFunds, KYC_BackGround, RefMaster.RefCode, Compliance.* FROM RefMaster INNER JOIN Compliance ON  Compliance.AddrCode=RefMaster.RefCode INNER JOIN EventLog ON EventLog.KeyCode=RefMaster.RefCode AND EventCode IN ('(-CCODE-)') WHERE  CP<>0/]    

CCODE

This is to specify Change Codes that will require information to be transferred from Viewpoint to CDDS.

[/<ClassPrefix>.CCODE=<ChangeCode>/]

where <ChangeCode> refers to Viewpoint's Change Code. If there is more than one, separate the codes with commas.

 

Example:

[/IND.CCODE=ADC/]

[/IND.CCODE=ADC,ADN,CPC/]

VALUE

This is to specify the mapping between Viewpoint and AMLspotter i.e. source and destination of the information to be transferred.

[/<ClassPrefix>.VALUE=<Destination>|<Source>/]

where:

  • <Destinationy> refers to the AMLspotter Property (field). Note that this is case-sensitive.

  • <Source> refers to where the information is located in Viewpoint. This can be the database field name in Viewpoint or an SQL query to retrieve the information from Viewpoint. If the information is to be taken from more than one Viewpoint database fields, specify these fields with a plus sign.

Example:

[/IND.VALUE=NoInterne|MfNr/]

[/IDC.VALUE=NoInterne|IdType+IdNumber/]

[/ADDR.VALUE=Addresse|SELECT Adline FROM Addresses WHERE AddrNr=(-AddrNr-)/]

[/IND.VALUE=CycleVie|Status/]    

[/IND.VALUE=DteNaissance|BirthDate/]    

[/IND.VALUE=LieuNaissance|PlaceBirth/]    

[/IND.VALUE=Nationalite|SELECT ElCode FROM LookValues WHERE Name='(-NationalityOrigin-)' AND TabCode='NATI'/]    

[/IND.VALUE=Nationalite2|SELECT ElCode FROM LookValues WHERE Name='(-Nationality-)' AND TabCode='NATI'/]    

[/IND.VALUE=Residence|KYC_Country1/]    

[/IND.VALUE=EtatCivil|MaritalStatus/]    

[/IND.VALUE=ActivitePrincipale|SELECT UserText1 FROM Entity WHERE EntCode='(-RefCode-)'/]    

[/IND.VALUE=OrigineFortune|KCY_SourceFunds/]    

[/IND.VALUE=KycBackground|KYC_BackGround/]    

[/IND.VALUE=NumeroCycleVie|SELECT CASE WHEN status='0' THEN 1 ELSE 0 END AS [isActive] FROM Entity WHERE EntCode='(-RefCode-)'/]

[/ENT.VALUE=DteClotureRelation|SELECT TOP 1 CASE WHEN DateEffective IS NOT NULL THEN DateEffective ELSE DateChange END AS DateEffective FROM RefStatus WHERE RefCode='(-EntCode-)' AND SType='E' AND NewStat='8' ORDER BY SeqNr DESC/]    

[/ENT.VALUE=CycleVie|Status/]

[/ENT.VALUE=Nationalite|IncorpPlace/]

[/ENT.VALUE=TypeSociete|UserCombo2/]    

[/ENT.VALUE=KycBackground|KYC_BackGround/]
