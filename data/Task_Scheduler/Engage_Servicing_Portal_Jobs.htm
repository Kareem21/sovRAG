Engage Servicing Portal Task Scheduler Jobs

In Viewpoint's Engage Servicing Portal (ESP), task scheduler jobs can be set up to:

  • Send e-mails on new user account activation or password reset.

  • Update the ESP Calendar.

  • Generate registers in ESP (registers printing).

  • Receive notification in ESP.

  • Synchronise Debtor Aging from ESP Back-office to Front-office.

  • Automate document sharing to ESP.

  • Process any Notification triggered by the Change from the Back-office (for Followers feature).

  • Retrieve unread notifications from the ESP and send the report to the subscribers who did not log in overnight (more than 24 hours).

  • Retrieve all notifications from the ESP within the week and send the report to the subscribers.

  • Update the pending payment transaction to ‘Expired’ status if it is pending/idle for more than a certain period.

  • Retrieve Back-office records based on the queries stored in System Settings ‘ENTADD’ key and store in the Front-office database. The retrieved record will be displayed in the ESP 'Entities' list as per the tag configured in System Settings ‘ENTLST’ key. See System Settings for details.

Note: The ESP task scheduler jobs are configured in the Front-office Client Portal Administrator (CP-AD). Prior to this, please ensure that the Viewpoint Task Scheduler and Notification Agent has been set up in the Front-office ViewPointDbMgt.

 

Setting up Task Scheduler for ESP User Activation

This job will send out an activation e-mail to the user with a link to activate the user account.

  1. Go to Configuration > Business Parameters > Task Scheduler > Agent Job Scheduler.

  2. Click New and ensure that the Job Type is set to 'Engage Servicing Portal - Activation E-mail [CPLA]'.

  3. Click Next >.

  4. In Criteria, you can change the E-mail Subject and E-mail Template.

  5. Click Next >.

  6. If an e-mail notification for the job is required, tick Enable E-mail Notification and specify details for the e-mail. If you want different types of e-mail notification, tick Advanced and you will get the following:

    • Success - This is the notification when the job is completed successfully.

    • Fail - This is the notification when the job is not completed.

  7. Click Next >.

  8. In Schedule, enter the job frequency.

  9. Click Done.

Setting up Task Scheduler to Automate Document Sharing to ESP

This job automates document sharing from the Document Manager to the ESP Web Portal. Administrator can configure the tag values to allow certain documents to be shared to ESP.

  1. Go to Configuration > Business Parameters > Task Scheduler > Agent Job Scheduler.

  2. Click New and ensure that the Job Type is set to 'Engage Servicing Portal - Document Sharing [CSPD]'.

  3. Click Next >.

  4. In Tag Values, enter the tag accordingly. Click here for details.

    Tag Values

    Description

    DocType

    This refers to the Document Type that the user wishes to store. It can be one or more document types separated in comma (',').

    Example:

    [/DocType=DOCX,PDF/]

    Cabinet

    This specifies the documents to be retrieved from which Filing Cabinet in Document Manager (source) and to be stored into which ESP cabinet. The syntax is:

    [/Cabinet= Document Manager Filing Cabinet key tag:ESP Cabinet code/]. It can be one or more sets of filing cabinet separated in comma (',').

    Example :

    [/Cabinet=REFMAT:AGRE,HREMP:STAT/]

     

    Note: The Document Manager Filing Cabinet key tag can be referred from Home > Configuration > Business Parameters > Document Management > Filing Cabinets.

    The ESP Filing Cabinet code can be referred from Home > Configuration > Business Parameters > General > Lookup Values > Engage Portal Filing Cabinets [CPFC] lookup value.

    FileContain

    This will set the rule in selecting the documents with or without specific keyword in the file name.

    The '*' indicates the keyword position as per example below:

    • Agreement* - the file name shall begin with the word 'Agreement'.

    • *Agreement* - the file name shall contain the word 'Agreement'.

    • *Agreement - the file name shall end with the word 'Agreement'.

    Example :

    [/FileContain=Agreement*,*Minutes*/]

    [/FileContain<>*DRAFT*/]

    RequireApproval

    This will set whether an approval is required when sharing out the document. If 'RequireApproval' is set to 'Y', the document will need to be approved by another user before being shared to ESP. If 'RequireApproval' is set to 'N', the document does not need to be approved and will be shared to ESP automatically.  

    Example :

    [/RequireApproval=N/]

    Note: If [/RequireApproval=/] is not defined or incorrectly defined, by default the document will be approved by the system and gets published to ESP automatically.

    LastRun

    This refers to the task scheduler job last run date. The system will compare the 'LastRun date' with the Document Holder 'Date Last Updated'. Only Document Holder with the 'Date Last Updated' later than the task scheduler job 'LastRun date' will be processed for sharing.

    During the task scheduler job setup, it is recommended to put a past date into the 'LastRun' so that all documents that are after this date will be processed for sharing during the job's first run. If 'LastRun' is not defined during the task scheduler job setup, it will take the current date as its 'LastRun' and will only process document with Document Holder 'Date Last Updated' later than the current date.

    For the subsequent task run, the 'LastRun' field will be updated automatically by the system.

    Example :

    [/LastRun=2018-08-16/]

    Notation:

     =   means equals to

    <>  means not equals to

     

    Example tag values:

    [/DocType=PDF,DOCX/]

    [/Cabinet=REFMAT:AGRE,HREMP:STAT/]

    [/FileContain=Agreement*,*Minutes*/]

    [/FileContain<>*DRAFT*/]

    [/RequireApproval=Y/]

    [/LastRun=2018-08-16/]

     

    Explanation:

    The above configuration will look for document type of PDF and DOCX, from the filing cabinet with key tag 'REFMAT' and will store them into ESP cabinet 'AGRE'; while for the filing cabinet with key tag 'HREMP', the document will be stored into ESP cabinet 'STAT'. Additionally, it will only retrieve documents whereby the file name starts with the word 'Agreement' or contains the word 'Minutes'. It will exclude documents whereby the file name contains the word 'DRAFT'. Likewise, the approval is set to Y; hence, approval is required before the document is published to ESP. The system will process documents where the Document Holder 'Date Last Updated' is after 2018-08-16.

  5. Click Next >.

  6. In Schedule, enter the job frequency.

  7. Click Done.

Successfully shared document is logged in the Audit Trail which can be viewed at:

  • Home > Enquiry > Audit Trails > Document Management; or

  • Audit Trail from the Ribbon's Documents tab.

Setting up Task Scheduler to Retrieve Unread Notifications from the ESP and Send the Report to the Subscribers Who Did Not Login Overnight

This job retrieves unread notifications from the ESP and send the report to the subscribers. It will only send out e-mail to user who did not login overnight (more than 24 hours).

It retrieves notifications on:

  • Documents that the user generated (both successful and failed).

  • User ‘Requests’ status updates.

  • User ‘Requests’ messages received.

  • All items that the user followed which includes:

    • Entity Information

    • Documents

    • Filings

    • Meetings

    • Requests

To set up the task scheduler job:

  1. Go to Configuration > Business Parameters > Task Scheduler > Agent Job Scheduler.

  2. Click New. In the Setup, select Job Type ‘Engage Servicing Portal – Daily Report [CPDR]’.

  3. Click Next >.

  4. In the Criteria, enter the E-mail Subject and E-mail Template.

    The default e-mail template used in ESP Notifications is 'ESP Daily Notification' (Code: CPDR01).

    It is accessible in Configuration > Document Libraries > Other Document Templates, in Document Type 'Email Notification Templates [D03]'

    The e-mail will show all Notifications retrieved from the ESP; however, administrator can choose to exclude any Notification in the e-mail via tag customisation.

     

    Customising Tag to Selectively Exclude (Not Show) Certain Notifications

    The default tag (+Notification+) is used in the e-mail template to show all Notifications retrieved from the ESP.

    To exclude any Notification from appearing in the e-mail, replace the tag with:

    (+Notification|NS:<Subscription Id>+)

    where the ‘Subscription Id’ refers to the Notification to be excluded from the e-mail.

    The ‘Subscription Id’ can be referred from the Notification 'Subscription Id' field in Configuration > Engage Servicing Portal > Global > Notifications.

    For example, to exclude the report on 'Document Generation - Success', replace the default tag with:

    (+Notification|NS:1+)

    The 'Subscription Id' operates in bitwise. If the administrator would like to exclude more than one Notification in the e-mail template, this can be done by adding up the 'Subscription Id' of each Notification.

    For example, to exclude the report on 'Document Generation - Success' and 'Document Generation - Fail', add up both the 'Subscription Id' (1+2=3). Hence, the tag become (+Notification|NS:3+).

    Note: The Subscription Id for ‘Followers’ item is 32. To exclude the ‘Followers’ item updates from appearing in the e-mail report, replace it with the tag (+Notification|NS:32+).

     

     

  5. Click Next >.

  6. If an e-mail notification for the job is required, tick Enable E-mail Notification and specify details for the e-mail. If you want different types of e-mail notification, tick Advanced and you will get the following:

    • Success - This is the notification when the job is completed successfully.

    • Fail - This is the notification when the job is not completed.

  7. Click Next >.

  8. This job runs on a daily basis as per default setting. However, you can set the preferred ‘Start Date’, frequency and occurrence time in the Schedule tab.

  9. Click Done.

Setting up Task Scheduler to Update the Pending Payment Transaction to ‘Expired’ Status When Pending for a Certain Period

This job updates the pending payment transaction in ESP 'Invoices' page to ‘Expired’ status if it is pending/idle for more than 10 minutes.

Note: The ‘Expired’ status timeout is set to default 10 minutes, however this setting can be changed in the Front-office Configuration > Settings > Customise > System Settings, under the key ‘CSPPYT’.

To set up the task scheduler job:

  1. Go to Configuration > Business Parameters > Task Scheduler > Agent Job Scheduler.

  2. Click New. In the Setup, select Job Type ‘Engage Servicing Portal - Payment Record Cleanup [CPPC]’.

  3. Click Next >.

  4. If an e-mail notification for the job is required, tick Enable E-mail Notification and specify details for the e-mail. If you want different types of e-mail notification, tick Advanced and you will get the following:

    • Success - This is the notification when the job is completed successfully.

    • Fail - This is the notification when the job is not completed.

  5. Click Next >.

  6. Set the job frequency to 'Continuous [C]'.

  7. Click Done.

 

Setting up Task Scheduler to Retrieve Back-Office Records Based on the Queries Stored in System Settings ‘ENTADD’ Key and Store the Records in the Front-Office Database

This job retrieves the Back-office records based on the queries in the System Settings ‘ENTADD’ key and stores the retrieved records in the Front-office database. The records will then be displayed in the ESP 'Entities' list as per the tag configured in System Settings ‘ENTLST’ key. This task scheduler job and both the settings in ‘ENTADD’ key and ‘ENTLST’ key is used to customise the Entity's display information in ESP 'Entities' list. It also allows ESP to show the Master File compliance status or any defined status in colour badge. Refer to System Settings for details.

To set up the task scheduler job:

  1. Go to Configuration > Business Parameters > Task Scheduler > Agent Job Scheduler.

  2. Click New and ensure that the Job Type is set to 'Engage Servicing Portal - Entity Update [CPEU]'.

  3. Click Next >.

  4. In Criteria, select the database of the Back-office.

  5. Click Next >.

  6. If an e-mail notification for the job is required, tick Enable E-mail Notification and specify details for the e-mail. If you want different types of e-mail notification, tick Advanced and you will get the following:

    • Success - This is the notification when the job is completed successfully.

    • Fail - This is the notification when the job is not completed.

  7. Click Next >.

  8. In Schedule, enter the job frequency.

  9. Click Done.
