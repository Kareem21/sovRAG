Settlements - Invoice Cancellation

Invoices that have been posted cannot be deleted. As such, if there is an error in the invoice, it will have to be cancelled. To cancel, you can use the Credit Note Wizard which can perform full or partial cancellation of the invoice.

 

Credit/Debit Note Wizard

This function is accessible through the Cancel D/C Note button in Receivables and Settlements - Invoice screen. It allows for outstanding invoices to be cancelled.

 

Invoice Lines

This shows the invoice lines for the selected invoice and available action for the cancellation such as resetting the recurring fee. The actions depend on the invoice line.

 

Pending Actions list

This shows pending actions for the selected invoice.

 

Command buttons

Click on the relevant command button to perform the necessary. The buttons apply to the Pending Actions list.

  • New - Click this to set up a new action.

    1. Enter the required information:

      • File Code - Select the Master File that the action is for.

      • Matter Code - Select the Matter that the action is for, if applicable.

      • Draft Invoice Number - If the action is not for a specific invoice, enter 0. If any other number is entered, system will check if the number is a valid draft number based on the File Code selected. If the number is a valid number, the action is deemed to be for the draft invoice of that number. However, if the system is not able to locate any draft invoice with the number, clicking Update will result in the error message 'Invalid Draft Number'.

      • Receipt/Payment Transaction Number - Similar to Draft Invoice Number, enter 0 if the action is not for a specific transaction. Otherwise, enter the receipt or payment transaction number that the action is for.

      • Category - Select a category for the action. This is for classification. New categories can be set up in the Practice Management Action Categories [PACT] lookup table.

      • Type - Select the type of action. This depends on the category you have selected. New action types can be added in the Practice Management Action Types [PMAC] lookup table.

      • Description - Enter a description for the action.

      • Date Created - This defaults to today's date.

      • Created by - This defaults to the logged in user.

      • Assigned to - Select the user in charge of the action. The action item will be seen in the assignee's Reminders.

      • Date Due - Enter the due date for the action. This is mandatory if the Assigned to field is completed.

      • Date Completed - This is to be completed when the action is done.

      • Completed - This is ticked when the action is done.

      • Currency - Enter the currency code.

      • Currency Amount - Enter the amount.

      • Base Currency Amount - Enter the reporting currency code.

      • Note - Enter additional notes for the actions.

    2. Click Update.

  • Edit - Click this to edit the selected action.

  • Complete - Click this to indicate that the action has been completed.

    1. Enter the Date Completed.

    2. Click Update.

  • Select All - Click this to select all actions.

  • Deselect All - Click this to deselect all actions.

  • Delete - Click this to delete the selected action.

Cancellation options

  • Select All - Tick to select all invoice lines and all action for the invoice lines.

  • Create Credit (Debit) Note - Tick to automatically create the credit/debit note. This is ticked by default and will remain ticked.

  • Prompt Posting screen to post the Credit (Debit) Note - Tick to view the posting screen. This allows for you to change the invoice date and/or journal date if necessary. You can also choose to send credit/debit note to Document Queue to be printed.

  • Settle both the original and newly created Credit (Debit) Note - Tick to automatically settle both the invoice and the credit/debit note.

Settlement Status

This is to assign a status to the resulting credit/debit note. Click to select the relevant settlement status.

 

Create

Click this to cancel the invoice and create the Credit (Debit) Note.

 

Cancel

Click this to exit the screen.

 

 

Using Credit/Debit Note Wizard

  1. Select the invoice.

  2. Click Cancel D/C Note. The Cancel D/C Note button is enabled for both DR Note and CR Note (document type CR_N only) in Sales ledger, however, it is only enabled for document type C_PO in Purchase ledger. The button is also enabled when the invoice has discount of 100% and is not settled.

  3. The Credit/Debit Note wizard is seen. In this screen, you will see the invoices lines. This allows you to specify if you want the invoice to be cancelled in full or not. Tick only the lines that you want to cancel. If you wish to cancel all lines, click Select All.

  4. Tick the applicable actions. The actions depend on the invoice lines and they are:

    • Reset Recurring Fee - This action is available if the invoice line is derived from Recurring Fees and if ticked, will reset the recurring fee's next due date. If the service has been deactivated by the creation of the invoice, this will re-activate it. This action is not available if the invoice line has been partially settled.

    • Reverse Time Entries - This action is available if the invoice line is derived from time entries and if ticked, will reverse the relevant time entries. If the invoice line has been partially settled, this action is not available.

    • Reverse Disbursement Entries - This action is available if the invoice line is derived from disbursement entries and if ticked, will reverse the relevant disbursement entries. If the invoice line has been partially settled, this action is not available.

  5. Select the options for the cancellation.

  6. Click Create to start.

Note: If there is no settlement for the invoice, by default, all invoice lines will be ticked in the Credit Note wizard screen. However, if there has been settlement made to any of the invoice lines, then the line will not be ticked by default.

If the option 'Use Recurring Services for Time Postings' or 'Use Recurring Services for Disbursement Postings' is enabled, reversing time or disbursement entries will also reset the 'Next Invoicing Due' date for the services in Recurring Fees screen.

 

 

Manual Cancellation

There are two methods of cancelling an invoice manually:

  • Settle to Matter, in which both the invoice and the offset credit note are settled to Matter; and

  • Make a receipt/payment to both the invoice and the offset credit/debit note.

Before you can proceed with a manual cancellation:

  • Create the offset credit/debit note, similar to how the system creates an offset credit/debit note when you perform an automatic cancellation. Depending on your internal process, the amount of this offset invoice may be equal to the amount of the invoice or it may be the sum of the portion being cancelled, i.e. the unpaid portion.

  • If the offset invoice cancels the total of the original invoice, you may also create a new invoice/supplier invoice to replace the one that is being cancelled.

Once you have done this, follow the steps below to complete the cancellation.

 

Settle to Matter

Settling to Matter can only be done for Sales invoices. You need to perform the following:

  1. Select the invoice that is being cancelled.

  2. Click Settle to Matter. This will launch the Settle to Matter Account screen.

  3. Tick the Settle Multiple Documents and, if necessary, the Settle by Invoice Line checkboxes. This will list all outstanding invoices for the selected Matter.

  4. Select the invoice that is being cancelled and the offset credit note that you created. If you are cancelling the whole invoice and want to apply the amount received from the client to the replacement invoice, then also select the replacement invoice at this point.

  5. The total Amount Rec'd (DR) should be 0.

  6. You may want to enter relevant text as to why the invoices are being cancelled.

  7. Click Update.

  8. The original invoice has now been settled against the credit note. Where relevant the sum paid to the invoice is now settled to the replacement invoice.

If there is no replacement invoice, the amount received from the client will be held in the Matter account. Refer to  Settlements - Settle to Matter.

 

Make a Receipt/Payment

This method works in a similar manner to a settle to Matter.

  1. Select the invoice that is being cancelled.

  2. Click Receipt/Payment. This will launch the Receipt/Payment screen.

  3. Tick the Settle Multiple Documents and, if necessary, the Settle by Invoice Line checkboxes. This will list all outstanding invoices for the Billing/Supplier File.

  4. Select the invoice that is being cancelled, the offset credit/debit note that you created, and where relevant, the newly-created replacement invoice.

  5. The total Amount Rec'd (DR)/Amount Paid (CR) should be 0. If you are settling a sales invoice, you may need to manually change this to 0.

  6. You may want to enter relevant text as to why the invoices are being cancelled.

  7. Click Update.

  8. The original invoice has now been settled against the credit/debit note. Where relevant the sum paid to the invoice is now settled to the replacement invoice.

 

Will the time and disbursement entries that were linked to this invoice be reversed?

If you have selected the Reverse time/disb on cancel invoice option in Configuration > Settings > System Defaults > Practice Management then the posted entries will be reversed, provided you select Start in Step 5 above.

In this case, identical entries will be created with negative amounts to cancel out the original time and disbursement charges, and then the entries are recreated in Time and Disbursements so that they can be pulled into a new invoice. The dates in the Posting Control Settings will have no bearing on the creation of the replacement entries.
