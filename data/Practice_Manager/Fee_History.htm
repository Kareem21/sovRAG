Fee History

The Fee History screen shows the list of invoiced fees based on the selected Service.

Note: This screen is inaccessible by default. User is required to grant authorisation before accessing this screen.

To grant authorisation, go to Configuration > Users & Security > Security Setup > User Roles, click on Authorities button.

Click File > Sales Ledger > Fee History, change the authority.

 

You can click on the following to minimise or expand the sections in this screen:

  • - To minimise this section and hide the command buttons. In the grid, only the selected row will be seen.

  • - To expand the section and show the command buttons.

 

Service Search Box

The ‘Service’ search box will search all Services found in the ‘Transaction Charging’ and ‘Recurring Fees’ screen. The available Services for selection will be based on the user’s authorisation to both screens.

For example, if the user has authorisation to both ‘Recurring Fees’ and ‘Transaction Charging’ screens, the system will show all the Services belonging to these screens. Whereas, if user has authorisation to only the ‘Recurring Fees’ screen, the Service search box will only list Services in the ‘Recurring Fees’ screen.

 

Upper Grid

The upper grid shows all the invoices that are linked to the selected Service.

 

Bottom Grid

The bottom grid shows the following based on selection of the radio button:

  • Line Source - It will show the source of the invoice line.

    • If the selected upper grid record is Time entry, it will show all the Time entry that belongs to the selected invoice.

    • If the selected upper grid record is Disbursements entry, it will show all the Disbursements entry that belongs to the selected invoice.

    • If the selected record in upper grid is a Recurring Fee, nothing will show.

  • Service Date - System will list all the Time/Disbursements entry/post where the Time/Disbursements entry 'Service Date' is between 'From Date' and 'To Date' of the selected record in the upper grid.

  • Not Invoiced - It will show the Time/Disbursements entries that have not been invoiced. It displays records regardless of whether the top grid is with or without record. The top grid shows all the invoices that are linked to the selected Service whereas the bottom grid shows Time/Disbursements Entries that are yet to be invoiced for the selected Service.

Available button):

  • Recalculate button - This button is only available when the selected Service is of Service Class 'TW’ or 'DW'. System will recalculate Time/Disbursements entries linked to the invoices. Recalculate will be based on FIFO (Date entered and Sequence Number) and is only applicable for Time/Disbursements entries that have not been approved. A validation message will be prompted after the recalculation when there are approved entries which require adjustment. When adjustment is needed, users are required to un-approve and run the Recalculate button again. Note: The shortcut key 'Alt + R' can also be used to trigger the recalculate function.

Note: Alternatively, user can also access this screen by clicking on Fee History button from the Recurring Fees screen or Transaction Charging screen.
