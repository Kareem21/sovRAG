Recurring Fees - Transaction Charge

Organisations that utilise Viewpoint's Practice Management, Accounting and Document Management modules have the option of charging their clients for each journal or journal line that is posted to the client's Accounts in Accounting, as well as charging for documents stored in Document Management. This is done by setting up a Transaction Fee service in the client's Billing File as a recurring fee.

  • The service used is from the Transaction Fee [RT] service class.

  • The number of units to be charged is calculated when you click >>Propose Drafts in the Sales Ledger - Posting of WIP screen.

  • Any errors in the setup will be shown in the Description field of the recurring fee, after the Posting of WIP run.

 

Set up the Recurring Fee

When setting up a recurring fee for the transaction charge feature, the following fields must be set up as described:

  • Service - Select the transaction fee service.

  • Number of Units - Enter '1'. This field will be updated by the system when you click >>Propose Drafts in the Posting of WIP screen.

  • Unit Price - The unit price is picked up based on the rate list used by the Billing File. Change this if necessary.

  • Charge per - Select the frequency of the charge. This should match the Effective and Next Due Dates.

  • Effective - Enter the date the system will start the calculation.

  • Next Invoicing Due - Enter the last date of the date range for the calculation.

See also Recurring Fees - Setup.

 

Calculating the Number of Units to be Charged

To calculate the number of units to be charged, i.e. the number of journals or journal lines, go to Sales Ledger - Posting of WIP screen.

  1. Enter the details required. The Date - from and - to should match the Effective and Next Due Dates in the Recurring Fees screen.

  2. Click >>Propose Drafts.

  3. The Number of Units field in the Recurring Fees folder will be updated with the appropriate number, and you can invoice the fee.

 

See also Posting of WIP.
