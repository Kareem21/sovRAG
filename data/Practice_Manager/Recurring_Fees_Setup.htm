Recurring Fees - Setup

Recurring fees can be set up manually or added during the cloning process when setting up the Billing File. The following shows you how to set up a fee manually.

  1. Click New.

  2. Enter the details of the recurring fee.

    1. Enter details for the new Recurring Fee.

      • Matter - Select the Matter type. The selections available are set up in the Matter Type Codes [MTTR] lookup table.

      • Service - Select the recurring fee from the list of available services.

      • Description - This is default to the service description. Edit if necessary.

      • Currency - Enter the billing currency.

      • Number of Units - Specify the number of units to be billed.

      • Unit Price - This information is filled based on the rate list level defined in the Matters screen.

      • Total Fee - This field shows the total fee that will be charged for the service.

      • Discount % - If you are giving a discount on the service, specify the percentage here.

      • Charge Per - Specify how frequently the service is billed. If you select 'One Off [T]', this service can only be invoiced once. There is no pro-rate calculation and will not be included in Posting Rule for 'Recurring Fees in Arrears Accrual' Posting Type. The Lookup Values in this field is not editable.  

      • Effective - Enter the effective date of the invoice. If the effective date is later than the Next Due Date the system will bill this service on a pro rata basis. For example, if a service is charged annually in arrears, and the service was effective from 01/07/2011 but the next bill due is on 01/01/2011, the system will calculate the amount due for six months.

      • Next Invoicing Due - This is the date the fee is next due.

      • Fee Period From Date - This is a read only field. This will be set when 'Fee Period To Date' is entered and is based in the 'Charge Per' setting. For example, if 'Charge Per' is set to 'Monthly', the date will backdate to one month before the 'Fee Period To Date'.

      • Fee Period To Date - Enter the fee period to date.

      • Last Invoiced - This is the date fee was last invoiced. This field is automatically updated when a service is posted to an invoice.

      • Date Terminated - Enter the date when the recurring fee is being terminated. If the recurring service is used in Time and Disbursement entry, the system will not allow user to create of edit the time/disbursement entry with the date later than this termination date. The system will deactivate the recurring fee via Posting of WIP function when the conditions are met as below:

        • 'One Off' recurring fee - will deactivate when 'Date Terminated' <= 'Today's Date'

        • 'Normal' recurring fee - will deactivate when 'Date Terminated'

      • Reason for Termination - Enter the reason of the fee being terminated.

      • Time Charge Units - This field controls the total number of units that can be entered at WIP stage. The field is inactive by default and can be activated when the option 'Use Recurring Services for Time Entries' is enabled. See Field Settings.

      • Time Charge Amount at - This field controls the charge amount that can be entered in WIP stage. The charge amount will be in Matter Currency if the option 'Multi Rate/CCY List' is enabled, otherwise it will be in Service Currency. This field is inactive by default and can be activated when the option 'Use Recurring Services for Time Entries' is enabled. See Field Settings.

     

  3. Click Update to save the entry.

Note: The system will deactivate the recurring fee via Posting of WIP function when the conditions are met as below:

  •  'One Off' recurring fee - will deactivate when 'Date Terminated' <= 'Today's Date'

  •  'Normal' recurring fee - will deactivate when 'Date Terminated' <= 'Next Invoicing Due'
