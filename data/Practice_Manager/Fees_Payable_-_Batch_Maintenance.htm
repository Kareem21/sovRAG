Fees Payable - Batch Maintenance

Payment batches can be created to group disbursements payable to third parties, in instances when your organisation only wishes to pay them on settlement of client invoices. An example of such a fee is the annual government fee payable for continuation of incorporation, which is a recurring fee chargeable to the Billing File (client), where you only pay the government after settlement of the invoice by the client.

Viewpoint recommends that entities selected for strike off are added to a payment batch designated for strike off.

When you select this function in the Fees Payable screen, the Payment Batch Maintenance screen opens in a separate window.

 

Service

Select the service you want to filter by. By default, only the active batches are displayed. If you want to show the inactive batches then change the active/inactive filter accordingly. This will update the records displayed in the table.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • New - Click this to create a new batch.

    1. Enter the details for the batch for the following:
      • Service - Select the service that the batch is being created for. This ensures that you only add invoice lines of this service to this batch.
      • Currency - The currency of the service, as set up in the Service Master, defaults to this field. Change if necessary.
      • Description - The description of the service, as set up in the Service Master, defaults to this field. Change if necessary.
      • Date Opened - Enter the date the batch is opened. This defaults to today's date, so change this if necessary.
      • Revenue Entity - Select the Revenue Entity that has been holding the fees for payment to the third party.
      • Note - Enter any notes relevant to the batch.
    2. Click Update.

  • Edit - Click this to edit the batch.

    1. Edit the necessary details as only certain fields can be edited as follows:
      • Batch No. - This field displays the batch number and cannot be edited.
      • Service - This field displays the service selected when the batch was created and cannot be edited.
      • Description - This defaults to the description of the service. Change this if necessary.
      • Date Opened - The date the batch was opened is displayed here. Change this if necessary.
      • Revenue Entity - Change the Revenue Entity that has been holding the fees for payment to the third party, if relevant.
      • Note - This field displays any note that was added during the creation of the batch.
      • Currency - The batch currency is displayed here and cannot be changed.
      • Exchange to BC - This field displays the exchange rate to the base currency.
      • Number of Entities - This field displays the number of entities assigned to this batch and cannot be edited.
      • Fees Assigned - This field displays the total fees invoiced and cannot be edited.
      • No. Units Assigned - This field displays the number of invoice lines in this batch and cannot be edited.
      • Fees Paid - Enter the amount of fees paid or use the Export/(Re)Calculate function to calculate this amount.
      • Penalty Paid - Enter the total amount of penalty paid if applicable or use the Export/(Re)Calculate function to calculate this amount.
      • Date Paid - When a batch is completed, closed, and paid, enter the date of the respective fees that are paid to the third party.
      • Total Paid - This field reflects the total amount paid to the third party, which consists of the Fees Paid and Penalty Paid amounts and cannot be edited.
      • Payment Reference - Enter the payment method and/or cheque number as a reference. This field also reflects the reference entered in the Government Fees screen.
      • Status - This displays the status of the batch. Change this if necessary.
      • Batch Open - When a batch is closed, the system no longer allows you to assign/unassign services to/from this batch. Close the batch by removing the tick when payment is being prepared and/or made.
    2. Click Update to save your changes.

  • Delete - Click this to delete the batch.

  • Export/(Re)Calculate - Click this to recalculate the sum of the fees in the batch and export the file for use in third party sites.

    When you use click the Export/(Re)Calculate button, you will access the Government Fees screen, which opens in a separate window. This is where you enter the information required to create the batch payment file:

    • Penalty cut-off - Enter the date when the cut-off takes effect.
    • (Re)Calculate - Click to recalculate the fees payable and any penalty amount. This will update the amounts from 'Payable Fees' and 'Payable Penalty' grid items to 'Fees Paid' and 'Penalty Paid' fields in the Batch Information screen. The fees are calculated as follows:

      • Fees Invoiced - The total of fees invoiced to clients relating to fees payable and penalty.

      • Payable Fees - The total of fees payable included in the batch, excluding penalty lines.

      • Payable Penalty - The total of penalty payable included in the batch.

      • Payable Total - The total of fees and penalty payable included in the batch.

      • Number of Entities - The number of entities included in the batch.

      • Penalty deadline - The deadline for penalties included in the batch.

    • Payment Batch File - This section contains the following fields to create the purchase order :

      • Generate Export File - Tick this to generate an export file. Select the format of the file from the drop-down list next to this checkbox.

      • Export Folder - This field becomes mandatory when the 'Generate Export File' checkbox is ticked. Select the location to save the export file.

      • Update Payment Details - Tick this to update payment details.

      • Reference - This field becomes mandatory when the 'Update Payment Details' checkbox is ticked. Enter a reference for the payment and this will be reflected in the 'Payment Reference' field in the Batch Information screen.

      • Date Paid - Enter the date when the batch was paid. This defaults to today's date. Change if necessary.

      • Close Payment Batch - This check box is also ticked automatically when the 'Update Payment Details' checkbox is ticked. You can untick this if you do not wish to create the purchase order.

      • Print Report - Tick this to print the payment batch report.

      • Create Purchase Order - This checkbox is enabled only when the 'Close Payment Batch' is ticked. Tick this to create the purchase order for the payment batch.

      • Supplier - This field becomes mandatory when the 'Create Purchase Order' checkbox is ticked. Select the Supplier File.

      • Fees - This defaults to the service selected in the Batch Maintenance screen when the 'Create Purchase Order' checkbox is ticked. Change if necessary.

      • Penalty - This defaults to the service selected in the Batch Maintenance screen when the 'Create Purchase Order' checkbox is ticked. Change if necessary.

    • Export/Update - Click this to generate the batch payment file and update the payment details.

  • Report - Click this to generate a report on the batch.

  • Close - Click this to exit the Batch Management window. You will be redirected to the Fees Payable screen.
