Time Entries

You can create Time Entries through multiple ways:

  • Selecting Timesheets from the listing in To Do Summary and clicking on the Time Entries tab.

  • Clicking on Time Ribbon button and selecting the Time Entries tab.

  • In Section Menu, click Home > To Do > Overview > Time > Timesheet > Time Entries tab.

This screen opens as a separate window and is divided into two - on the left is for data entry while the right is the Details window. You may find that some fields are pre-completed if you have switched on the context sensitivity option in the Selection & Options folder or there is a default service code defined in your User record.

In the Details window, you will see information relating to the Billing File, matter, rate, service and work in progress. If there is a limit set for the selected matter, you will be notified via colour highlights in the Details window:

  • Orange colour if 75% of the limit has been reached.

  • Red colour if 100% of the limit or exceeding the limit.

 

Administrator

The administrator defaults to the user who is logged on to the system. If you are authorised to do so, you can select another user.

 

Date

The date field defaults to today's date and should be the date the task was performed.

 

Billing File

Select the Billing File for the time entry. If you want to enter a time entry such as sick leave or a public holiday without assigning it to a Billing File, create a Master File for your own company to which you can assign in-house time entries.

 

Matter

Select the matter for the time entry. A matter can be compared with projects your company is managing for clients. Different invoices will be created for different matters.

 

Activity

Select the activity for the time entry. This activity selection option is being set up in the Activities screen and is linked to the Matter account.

Note: This will only be available if the 'All Time/Disbursement Entries Link to an Activity' option is enabled and will replace the 'Matter' field. The selection options for Activity will only be available for Activities status which is not in New, Cancelled or Completed.

 

Rate List

The rate list defaults to the rate list selected for the Matter in Matters screen. If you are authorised to do so, you can select a different rate list.

Note: When edit time entry, the rate list value will not be displayed.

 

Service

Enter the service for the time entry.

 

Service Date

The 'Service Date' field is seen when the Tag Value on the Service in the Service Master is [SRVDATE.Visible=Y]. For Time Entry with Service Class ‘TW’, this field will validate against the Recurring Fees’ ‘Fee Period From Date’ and ‘Fee Period To Date’ when creating invoice in arrears for Time exceeding the Fixed Fee. When this field is enabled, the Time Entries grid will also show how much chargeable/non-chargeable for the 'Service Date - Fee Details'. Note that for Service Class that is not 'TW', this field will only function to record the date of the service.

 

Fee History

The Fee History button is seen when the Tag Value on the Service in the Service Master is [SRVDATE.Visible=Y] and there is an existing Recurring Fee record for the Service.

Clicking on the Fee History button will launch the Fee History window. The Fee History window shows the list of the invoiced fees for the selected Service.

When user highlights an entry and clicks Select button, it will set 'Fee Period - To Date' as ‘Service Date’ for the Time entry.

 

Currency

This field is completed automatically when you select the Matter or service. If you are using multi-currency rate lists, then the currency follows the Matter currency. If you are not, the currency follows the service currency.

 

No. <unit of measure>

Enter the number of units spent on the service. The unit of measure is predefined in Configuration. You can enter the number of units here or you can use the timer function to keep track of the time you spend on the client. Note that this cannot exceed 24 hours.

Regardless of the unit of measure, you can enter the time using hours and minutes while the program will convert it to the right number of units according to the unit of measure set. The format is: <Hours> * <Minutes>

Exchange

The exchange rate is taken from the Exchange Rate Master. If you are not using multi-currency rate lists, the rate is used to convert the Time Entry total from the service currency to the Matter currency.

 

Charge/<unit of measure>

This value defaults from the rates set up for the service.

I got a 'No rate available' message when I entered time for a particular service. Why is this?

It is possible that no rate was set up for the administrator. In Viewpoint - Home screen, click Configuration > Business Parameters > Practice Management > Setup > Service Rates and check if the rate has been entered for the administrator.

If the rates are there, check the service currency. For services with a different currency from the base currency, you need to enable the Multi Rate/CCY List option in System Defaults.

 

Non-charged

This field displays the amount written off, if any. You can update this field by amending the Total <currency> field to the amount you want to charge. Any difference in the actual value and this amount will be written off.

 

Note: For Time/Disbursements entry with Service Class ‘TW’/ 'DW', the Time/Disbursements entry details window will show the amount 'Charged' and 'Non-charged' based on the following scenarios:

  • Scenario 1: When No Available Fixed Fee AND current Time/Disbursements Entry Fully Chargeable.Example:
    Fixed Fee = 2,000
    Time/Disbursement by Service Date excluding current entry = 3,000
    Time/Disbursement by Service Date invoiced previously = 1,000

    In the current Time/Disbursement entry:
    Current Time/Disbursement entry amount (No. of units * Charge/unit) = 500
    Current Time/Disbursement entry non-charged amount = 0
    Current Time/Disbursement entry charged amount (Time/Disbursement entry ‘Total’ field) = 500

    Hence, in the Time/Disbursement entry details window,
    Fixed Fee ‘Charged’ amount
    = (Time/Disbursement by Service Date excluding current entry) - (Fixed Fee) + (Current Time/Disbursement entry charged amount)
    = 3000 - 2000 + 500
    = 1,500

    Fixed Fee ‘Non-charged’ amount
    = Fixed Fee
    = 2,000

  • Scenario 2: When Current Time/Disbursements Entry Amount < Available Fixed Fee.Example:
    Fixed Fee = 2,000
    Time/Disbursement by Service Date excluding current entry = 1,000
    Time/Disbursement by Service Date invoiced previously = 0

    In the current Time/Disbursement entry:
    Current Time/Disbursement entry amount (No. of units * Charge/unit) = 500
    Current Time/Disbursement entry non-charged amount = 500
    Current Time/Disbursement entry charged amount (Time/Disbursement entry ‘Total’ field) = 0

    Hence, in the Time/Disbursement entry details window,
    Fixed Fee ‘Charged’ amount
    = (Time/Disbursement by Service Date excluding current entry) - (Fixed Fee) + (Current Time/Disbursement entry charged amount)
    = 1,000 – 2,000 + 0
    = -1,000
    This yields a negative value, hence ‘Charged’ amount is zero.

    Fixed Fee ‘Non-charged’ amount
    = (Fixed Fee) - (Time/Disbursement by Service Date excluding current entry) + (current Time/Disbursement entry non-charged amount
    = 2,000 – 1,000 + 500
    = 1,500

  • Scenario 3: When Current Time/Disbursements Entry Amount > Available Fixed Fee AND Current Time/Disbursements Entry Partial Chargeable.Example:
    Fixed Fee = 2,000
    Time/Disbursement by Service Date excluding current entry = 1,500
    Time/Disbursement by Service Date invoiced previously = 1,000

    In the current Time/Disbursement entry:
    Current Time/Disbursement entry amount (No. of units * Charge/unit) = 800
    Current Time/Disbursement entry non-charged amount
    = (Fixed Fee) - (Time/Disbursement by Service Date excluding current entry)
    = 2,000 -1,500
    = 500
    Current Time/Disbursement entry charged amount (Time/Disbursement entry ‘Total’ field)
    = (Current Time/Disbursement entry amount) - (Current Time/Disbursement entry non-charged amount)
    = 800 – 500
    = 300

    Hence, in the Time/Disbursement entry details window,
    Fixed Fee ‘Charged’ amount
    = (Time/Disbursement by Service Date excluding current entry) - (Fixed Fee) + (Current Time/Disbursement entry charged amount)
    = 1,500 - 2,000 + 800
    = 300

    Fixed Fee ‘Non-charged’ amount
    = Fixed Fee
    = 2,000

 

Total <Currency>

This is calculated based on the time spent on the task and the charge per unit of measure.

 

Unit Cost

Unit cost value for Entry for administrator rate will be based on the ‘Unit Cost’ defined for the administrator. Unit cost value for Entry for Non-Administrator rate is sum of ‘Cost – Time’ and ‘Cost – Expenses’ for the service.

Note: When both the 'Multi Rate/CCY List' and 'Multi Rate/CCY List Cost of Services' options are enabled and the Reporting currency differs from the Entry currency, the unit cost is calculated by adding the 'Cost – Time' and 'Cost – Expenses' together and multiplying the latest exchange rate from the entry currency to the Reporting currency.

Why am I not able to see this?

This may have been turned off by ticking the Hide Unit/Total Cost on Time Entry option in System Defaults > Time and Disbursements.

 

Total Cost

The total cost of the administrator’s time is displayed here.

Why am I not able to see this?

This may have been turned off by ticking the Hide Unit/Total Cost on Time Entry option in System Defaults > Time and Disbursements.

 

Analyses Code

If this field is enabled, select the analyses code for the service, or specify the default analysis codes for a service in Service Master.

How do I enable/disable the field?

This is done in Viewpoint - Home screen. Go to Configuration > Settings > System Defaults > Time and Disbursements and tick/untick the Enable Analysis Code on Time Entries option.

 

Description

The service description defaults to this field. If the Default Description with Analysis Code option is turned on, then the description for the analysis code is used instead. This may be overwritten.

Where do I enable the Default Description with Analysis Code?

This is done in Viewpoint - Home screen. Go to Configuration > Settings > System Defaults > Time and Disbursements.
