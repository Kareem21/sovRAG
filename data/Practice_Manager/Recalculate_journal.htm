Recalculate/Journal

The Recalculate function allows you to recalculate invoices/drafts, build statistics and post new journals if you are using the Accounting component. You can access this screen through:

  • Viewpoint - Home screen:

    • Click Sales Ledger Ribbon button and select Sales Ledger > Settings (PM) > Recalculate/Journal from the navigation listing in Sales Ledger Overview.

    • Click Purchase Ledger Ribbon button and select Purchase Ledger > Settings (PM) > Recalculate/Journal from the navigation listing in Purchase Ledger Overview.

    • In Section Menu, click Home > Practice Management > Settings (PM) > Recalculate/Journal.

  • File Maintenance screen:

    • Click Sales Ledger Ribbon button and select Billing File > Recalculate/Journal from the navigation listing in Sales Ledger Overview.

    • Click Purchase Ledger Ribbon button and select Supplier File > Recalculate/Journal from the navigation listing in Purchase Ledger Overview.

    • In Section Menu, click Sales Ledger > Billing File > Recalculate/Journal; or click Purchase Ledger > Supplier File > Recalculate/Journal.

In Viewpoint - Home screen, Recalculate/Journal will recalculate for all Billing and Supplier files. In File Maintenance screen, it will recalculate for the selected Billing or Supplier File.

 

Invoices Recalculate/Posting

Recalculation of invoices and drafts includes the recalculation of sales drafts, invoices, purchase orders and supplier invoices. Data errors in the invoice header, detail lines, transactions and receipts will be corrected in the following way:

Invoice Header

  • Assign default Matter code if empty.

  • Assign "N/A" if client code is empty.

  • Assign Revenue Company if empty, based on the Matter Revenue Company.

  • Assign date voucher if empty.

  • Assign last receipt date.

  • Calculate subtotal amount - Total of invoice line in invoice currency.

  • Calculate invoice total in base currency. - Calculate VAT amount in invoice currency and base currency. - Calculate Rounding difference, i.e., the difference between total invoice line in base currency and invoice base currency.

  • Calculate total paid/received.

  • Calculate total write off.

Invoice Lines

  • Assign service class per Service Master if empty.

  • Assign service type per Service Master if empty.

All Invoice receipts/transactions

  • Recalculate base amount for received amount, write off and charge.

  • Assign invoice number if different (based on draft number).

 

Statistics

The Statistics function will build statistics on invoices, recurring fees and matter balances.

 

Post Journal

If journals are not posted automatically to Accounting, the post journal function will look for unposted invoices, receipts and payments within the specified date range and post them. All unposted invoices, receipts and payments before the specified date range can also be marked as posted - note that these marked transactions will not result in a journal in Accounting
