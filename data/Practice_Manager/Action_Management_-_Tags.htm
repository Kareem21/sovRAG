Action Management - Tags

These tags are to be added to the Billing template in the template properties in the Key Information field. With the tags in place, when the billing document is generated, action(s) specified using the tags will be created.

For example, you can have the action 'Debit Note e-mailed' to be created for each invoice that you generate by adding the tag to the invoice template.

 

Create New Action Tag

This tag will create a new action when you generate the billing document and the syntax is:

[/DMW.Action|<Field>=<Value>|<Field>=<Value>|.../]

 

where:

  • <Field> refers to field of an action and can be:

    • RefCode - This is to specify the File Code.

    • MatterCode - This is to specify the Matter Code.

    • ProformaNr - This is to specify the draft invoice number.

    • ReceiptNr - This is to specify the receipt/payment transaction number.

    • ActionCategory - This is to specify the action category and follows the values in the Practice Management Action Categories [PACT] lookup table.

    • ActionType - This is to specify the action type and is according to the values in the Practice Management Action Types [PMAC] lookup table.

    • AcDescription - This is to specify the action description.

    • AcNote - This is to specify additional notes.

    • AdminAssign - This is to specify the assignee for the action.

    • DateDue - This is to specify the due date. You can also use:

      • (-APP.NOW-) for today's date.

      • (-APP.NOW-)+<Number of days> for future date e.g. (-APP.NOW-)+30 will give you 30 days from today.

    • AcComplete - This is to specify status for the action. Enter 0 to indicate it if not yet complete.

    • AcCur - This is to specify the currency code.

    • AcAmountCur - This is to specify the amount.

    • AcAmountBase - This is to specify the base currency code.

  • <Value> can be:

    • Specified value e.g. ActionCategory=D

    • field code to retrieve details from the invoice e.g. RefCode=(-HDF.RefCode-)

 

Example:

[/DMW.Action|RefCode=(-HDF.RefCode-)|MatterCode=(-HDF.MatterCode-)|ProformaNr=(-HDF.ProformaNr-)|ReceiptNr=0|ActionCategory=D|ActionType=DD|AcDescription=E-mail of debit note (-HDF.ProformaNr-)|AcNote=-|AdminAssign=(-HDF.UCode-)|DateDue=(-APP.NOW-)+2|AcComplete=0|AcCur=(-HDF.CurCode-)|AcAmountCur=(-HDF.InvoiceTotal-)|AcAmountBase=(-HDF.InvoiceBase-)|/]

 

Note: If the action has been created and remains incomplete, generating the same document will not create the same action again. However, if the action has been completed, then a duplicate action can be created for the document.

 

Create Duplicate Action

This is to allow creation of duplicate action even if the existing action is not yet complete. However, this is based on the specified flag and the syntax is:

[/DMW.Action.Duplicate=<Flag>/]

 

where <Flag> can be:

  • D - To allow duplicate action to be created for the same draft number.

  • M - To allow duplicate action to be created for the same Matter code.

  • R - To allow duplicate action to be created for the same receipt number.

 

Example:

[/DMW.Action|RefCode=(-HDF.RefCode-)|MatterCode=(-HDF.MatterCode-)|ProformaNr=(-HDF.ProformaNr-)|ReceiptNr=0|ActionCategory=D|ActionType=DD|AcDescription=E-mail of debit note (-HDF.ProformaNr-)|AcNote=-|AdminAssign=(-HDF.UCode-)|DateDue=(-APP.NOW-)+2|AcComplete=0|AcCur=(-HDF.CurCode-)|AcAmountCur=(-HDF.InvoiceTotal-)|AcAmountBase=(-HDF.InvoiceBase-)|/]

 

[/DMW.Action.Duplicate=D/]
