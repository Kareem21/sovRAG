Disbursement Entries

The Disbursements screen opens as a separate window and is divided into two - on the left is for data entry while the right is the Detail window. You may find that some fields are pre-completed if you have switched on the context sensitivity option in the Selection & Options folder or there is a default service code defined in your User record.

 

Administrator

The administrator defaults to the user who is logged on to the system. If you are authorised to do so, you can select another user.

 

Date

The date field defaults to today's date.

 

Billing File

Select the Billing File for the disbursement entry.

 

Matter

Select the matter for the time entry. A matter can be compared with projects your company is managing for clients. Different invoices will be created for different matters.

 

Activity

Select the activity for the time entry. This activity selection option is being set up in the Activities screen and is linked to the Matter account.

Note:This will only be available if the 'All Time/Disbursement Entries Link to an Activity' option is enabled and will replace the 'Matter' field. The selection options for Activity will only be available for Activities status which is not in New, Cancelled or Completed.

 

Rate List

The rate list defaults to the rate list selected for the Matter in the Matters screen. If you are authorised to do so, you can select a different rate list.

Note: When editing a disbursement entry, the rate list value will not be displayed.

 

Service

Enter the service for the disbursement entry.

 

Service Date

The 'Service Date' field is seen when the Tag Value on the Service in the Service Master is [SRVDATE.Visible=Y]. For Disbursement entry with Service Class ‘DW’, this field will validate against the Recurring Fees’ ‘Fee Period From Date’ and ‘Fee Period To Date’ when creating invoice in arrears for Disbursement exceeding the Fixed Fee. When this field is enabled, the Disbursements grid will also show how much chargeable/non-chargeable for the 'Service Date - Fee Details'. Note that for Service Class that is not 'DW', this field will only function to record the date of the service.

 

Fee History

The Fee History button is seen when the Tag Value on the Service in the Service Master is [SRVDATE.Visible=Y] and there is an existing Recurring Fee record for the Service.

Clicking on the Fee History button will launch the Fee History window. The Fee History window shows the list of the invoiced fees for the selected Service.

When user highlights an entry and clicks Select button, it will set 'Fee Period - To Date' as ‘Service Date’ for the Disbursements entry.

 

Currency

This field is completed automatically when you select the Matter or service. If you are using Multi-currency rate lists, then the currency follows the Matter currency. If you are not, the currency follows the service currency.

 

No of Units/Cost

The field caption changes depending on the type of service selected. Enter the appropriate number or cost.

 

Exchange

The exchange rate is taken from the Exchange Rate Master.

 

Charge/Unit/Markup

The charge/unit or markup percentage defaults from the rates set up for the service.

 

Non-charged

This field displays the amount written off, if any. You can update this field by amending the Total <currency> field to the amount you want to charge. Any difference in the actual value and this amount will be written off.

 

Note: For Time/Disbursements entry with Service Class ‘TW’/ 'DW', the Time/Disbursements entry details window will show the amount 'Charged' and 'Non-charged' based on the following scenarios:

  • Scenario 1: When No Available Fixed Fee AND current Time/Disbursements Entry Fully Chargeable.
    Example:
    Fixed Fee = 2,000
    Time/Disbursement by Service Date excluding current entry = 3,000
    Time/Disbursement by Service Date invoiced previously = 1,000

    In the current Time/Disbursement entry:
    Current Time/Disbursement entry amount (No. of units * Charge/unit) = 500
    Current Time/Disbursement entry non-charged amount = 0
    Current Time/Disbursement entry charged amount (Time/Disbursement entry ‘Total’ field) = 500

    Hence, in the Time/Disbursement entry details window,
    Fixed Fee ‘Charged’ amount
    = (Time/Disbursement by Service Date excluding current entry) - (Fixed Fee) + (Current Time/Disbursement entry charged amount)
    = 3000 - 2000 + 500
    = 1,500

    Fixed Fee ‘Non-charged’ amount
    = Fixed Fee
    = 2,000

  • Scenario 2: When Current Time/Disbursements Entry Amount < Available Fixed Fee.
    Example:
    Fixed Fee = 2,000
    Time/Disbursement by Service Date excluding current entry = 1,000
    Time/Disbursement by Service Date invoiced previously = 0

    In the current Time/Disbursement entry:
    Current Time/Disbursement entry amount (No. of units * Charge/unit) = 500
    Current Time/Disbursement entry non-charged amount = 500
    Current Time/Disbursement entry charged amount (Time/Disbursement entry ‘Total’ field) = 0

    Hence, in the Time/Disbursement entry details window,
    Fixed Fee ‘Charged’ amount
    = (Time/Disbursement by Service Date excluding current entry) - (Fixed Fee) + (Current Time/Disbursement entry charged amount)
    = 1,000 – 2,000 + 0
    = -1,000
    This yields a negative value, hence ‘Charged’ amount is zero.

    Fixed Fee ‘Non-charged’ amount
    = (Fixed Fee) - (Time/Disbursement by Service Date excluding current entry) + (current Time/Disbursement entry non-charged amount
    = 2,000 – 1,000 + 500
    = 1,500

  • Scenario 3: When Current Time/Disbursements Entry Amount > Available Fixed Fee AND Current Time/Disbursements Entry Partial Chargeable.
    Example:
    Fixed Fee = 2,000
    Time/Disbursement by Service Date excluding current entry = 1,500
    Time/Disbursement by Service Date invoiced previously = 1,000

    In the current Time/Disbursement entry:
    Current Time/Disbursement entry amount (No. of units * Charge/unit) = 800
    Current Time/Disbursement entry non-charged amount
    = (Fixed Fee) - (Time/Disbursement by Service Date excluding current entry)
    = 2,000 -1,500
    = 500
    Current Time/Disbursement entry charged amount (Time/Disbursement entry ‘Total’ field)
    = (Current Time/Disbursement entry amount) - (Current Time/Disbursement entry non-charged amount)
    = 800 – 500
    = 300

    Hence, in the Time/Disbursement entry details window,
    Fixed Fee ‘Charged’ amount
    = (Time/Disbursement by Service Date excluding current entry) - (Fixed Fee) + (Current Time/Disbursement entry charged amount)
    = 1,500 - 2,000 + 800
    = 300

    Fixed Fee ‘Non-charged’ amount
    = Fixed Fee
    = 2,000

 

Unit Cost

The field displays the addition of unit cost time and unit cost material of the selected service.

Note: If both 'Mutli Rate/CCY List' and 'Multi Rate/CCY List Cost of Services' options are ticked, the system will retrieve the Unit Cost Time and Unit Cost Expense from the Service Rates (based on entry Rate List and Currency). The sum of Unit Cost Time and Unit Cost Expense will be the value and it will be converted to Reporting currency.

The Unit Cost field will refer to the Cost field if the selected service is from DM service class.

Why am I not able to see this?

This may have been turned off by ticking the Hide Unit/Total Cost on Time Entry option in System Defaults > Time and Disbursements.

 

Total Cost

The total cost of the service is displayed here.

Why am I not able to see this?

This may have been turned off by ticking the Hide Unit/Total Cost on Time Entry option in System Defaults > Time and Disbursements.

 

Total <currency>

This is calculated based on the No. of Units multiplied by the Charge/Unit; or the Cost plus the Markup percentage.

 

Description

The service description defaults to this field.

 

Update

Click this to save the entry.

 

Cancel

Click this to delete your entry.
