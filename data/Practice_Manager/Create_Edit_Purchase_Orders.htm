Create/Edit Purchase Orders

Purchase orders are written authorisations for certain parties to provide goods or services at a specified price. Once a purchase order has been accepted by the supplier and you are billed for it, you can then post the order to an invoice.

A purchase order is a draft invoice (debit or credit) that has not yet been posted to the invoice ledger. As a result, it is possible to edit or delete purchase orders without affecting this ledger.

When the option 'Require Approval - Purchase Order' is enabled in Practice Management system default setting, the approved purchase order will be assigned with an invoice number and cannot be deleted. It will prompt the user to set the invoice line as '0' value if user clicks the Delete button. When this option is not ticked, all the Purchase Orders entries are approved by default, however the draft invoice will not be assigned with an invoice number and can be deleted.

Once the purchase order has been posted as an invoice, it is no longer possible to edit or delete the invoice - instead a debit note must be issued to offset any errors.

The Create/Edit Purchase Orders screen is where you create, edit, delete and post purchase orders. This screen is automatically displayed if you click Copy PO from Purchase Ledger > Supplier Invoices in Billing Enquiry screen.

 

Purchase order table

Lists all purchase orders created for the selected Supplier File.

 

Command buttons

Click the relevant command button to perform the necessary.

  • New - Click this to create a purchase order.

    1. Enter the details for the purchase order.

    2. Click Update.

    3. Enter the lines by clicking New in Details table.

    Note: When manually creating a Debit Note to cancel the existing invoice, you can define the draft number of the invoice to cancel in the field 'Original PO No.'. Any invoice line with an outstanding amount will be copied to the newly created draft invoice.

  • Edit - Click this to edit the selected purchase order.

  • Reopen - Click this to reopen a locked/closed purchase order. Purchase orders are locked when you click Print PO button.

  • Delete PO - Click this to delete the selected purchase order.

    Click Yes to confirm the deletion.

  • Print PO - Click this to print the selected purchase order. It will be sent to Document Queue and from there, you can open the draft for editing or send it for printing. Once a draft has been printed, it will be locked for editing. You need to reopen the draft to make further changes.

  • Invoice - Click this to post the selected purchase orders. You can also post a purchase order in the Purchase Orders and Settlements screens.

    1. In the Invoice Posting window that appears, complete the following information:

      • Invoice Date, Journal Date and Overwrite for (existing) Draft Invoices - When you post an invoice, the invoice and journal dates default to the date when the draft was created. You may overwrite these dates by entering the date you want in the Invoice Date and Journal Date fields, and ticking the corresponding Overwrite for (existing) Draft Invoices checkboxes.

      • Print Document (to document queue) - Tick this checkbox to send a copy of the invoice to the Document Queue for printing or editing.

    2. Click Start to begin the process.

    Note:

    When 'Use Recurring Services for T&D Postings' option is enabled in Home > Configuration > System Defaults > Time and Disbursements, the invoice will be created with the following type:

    • If Entity Administration component is activated - an invoice with document type 'Recurring Fee Invoice Entity [A_CS]' will be created.
    • If Entity Administration component is not activated -  an invoice with document type 'Recurring Fee Invoice Non-Entity [A_GE]' will be created.

    When 'Use Recurring Services for T&D Postings' option is disabled in Home > Configuration > System Defaults > Time and Disbursements, the invoice will be created with the type 'Debit Note [DR_N]'.

     

     

     

     

 

Details table

Lists all individual lines of the selected purchase order.

 

Command buttons

Click the relevant command button to perform the necessary.

  • New - Click this to add a new line for the selected draft. You may repeat this step as many times as necessary.

    1. Enter the details for the detail line:

      • Service - Select the service.

      • Description - This defaults to the description entered for the service in Service Master.

      • Project - Use this field to specify which project the purchase order is related to. If you use Accounting for Revenue Entity accounting and require the project information to flow through with journals posted from Practice Management, you must complete the Unit Code 2 field for the Revenue Entity's Account File. If Unit Code 2 is not set up, selections will be taken from the Project Code [PROJECTS] user table and the information will not be populated with Practice manager journals.

      • Revenue Entity - By default, this field is not enabled. However, if the option 'Enable Intercompany Invoice Line Allocations' in System Defaults > Practice Management is enabled, this field can be used to allow for a different Revenue Entity to be specified for the invoice line.

      • Account Type - This defaults to the account type mapped to the service in Account Mapping. However, if necessary, you can change the account type. If the Revenue Entity field is not enabled, this field is mandatory. Note that this field is seen only if you choose to post journals from Practice Management to Accounting using the setting 'CA Journals RC [2]' or 'CA Journals RC + CE [6]' in System Defaults > Accounting.

      • Account Code - This defaults to the nominal account mapped to the service in Account Mapping. However, you can change this if necessary.

        Journal Movement Code

        Account Code is a sub-ledger.

        The journal line currency is assigned based on the sequence below:

        • Sub-ledger account currency SAME as Base currency - Amount in Base currency.
        • Sub-ledger account currency SAME as Line currency - Amount in Line currency.
        • Sub-ledger account currency SAME as Invoice currency - Amount in invoice currency.

        Movement Code is 'Others' with the code following the sub-ledger e.g. BO when the sub-ledger is Bank Account, DO when the sub-ledger is Debtor.

        Account Code is not a sub-ledger.

        Journal line currency follows the invoice currency.

        Movement Code is NL.
         

      • Currency - This is the currency of the service.

      • Exchange Rate - If the invoicing currency is different from the base currency, the rate is picked up from the Exchange Rate Master. If the rate has not been set up, enter it manually.

      • Number of Units - Enter the number of units you are being billed.

      • Unit Price - Enter the unit price.

      • Discount % - Enter the discount percentage that you have been given, e.g. type 0.15 for a discount of 15%.

      • Total Line - This is calculated by multiplying number of units with unit price, minus any discount.

      • Tax Codes - If you enable 'Use Accounting Tax Sub-ledger as Tax Codes', the tax account mapped in Service Tax Codes screen for Supplier Invoicing will be seen (sub-ledger type 'WPUR' and 'WBSP'). Note that this field is inactive by default. If the option 'Use Accounting Tax Sub-ledger as Tax Codes' is enabled, it becomes a mandatory field.

      • VAT % - Enter the VAT percentage if applicable.

      • Tax Amount - This is calculated as Total Line * VAT % but can be overwritten. However, the edited value must not vary from the calculated value by +/-1.00. To allow for bigger differences, the option 'Disable the Validation of the Tax Amount on Purchase Lines' must be enabled in System Defaults > Practice Management.

      • Date - from - Enter the start date of the billing period.

      • -to - Enter the end date of the billing period.

    2. Click Update.

  • Edit - Click this to edit the selected line for the draft.

  • Delete - Click this to delete the selected line for the draft.

  • Recover - Click this to recover the supplier charge from Billing Files.
