Fees Payable - Strike Off

This function is used to change the status of entities that are to be struck off from their current status to the value that being are defined in the STKOFF and STKSUB system setting key. You can do this for one entity or for all entities at the same time. Viewpoint will check that the selected entities are due to be struck off by checking the Strike Date for the entity.

Viewpoint recommends that you add the entities selected for strike off to a payment batch designated for strike off. If the outstanding debit notes are cancelled by credit notes (see options below), the fee and penalty total for the strike off batch should add up to zero (0) as the credit notes created are also included in the same batch.

When you perform a strike off, the Automatic Strike Off of Entities screen will open in a separate window.

 

Note: Setting key 'STKSUB' is to set the Entity Sub-Status when perform strike off. If Setting key value for STKOFF is a 'Entity Sub Status', please update the value of STKOFF key to the relevant entity status, and STKSUB to the relevant entity sub-status.

 

Due Date Before

Enter the due date. Viewpoint will check for all entities with strike dates before this date and change their status to the value that being defined in the STKOFF and STKSUB system setting key.

 

Practice Management / Sales Ledger

Select handling options for the Sales Ledger of affected entities:

  • Delete Draft debit/credit notes pending - Tick this to delete all pending drafts.

  • Create credit (debit) notes to cancel outstanding debit (credit) notes - Select this if you want the system to create credit (debit) notes.

    • Only include Invoices with the selected fee type included - You can choose this option if you only want to create credit (debit) notes for the selected invoice.

    • Automatically settle all the debit and credit notes - You can also choose this option if you want the notes to be settled.

  • Write-off all the outstanding debit and credit notes - Tick this if you want the outstanding credit and debit notes to be written off.

    • Only include Invoices with the selected fee type included - You can choose this option if you only want to write-off the outstanding for the selected invoice.

  • No credit (debit) note creation and/or write-off - Tick this if you do not want any credit or debit note to be created or written off.

 

Entity - Strike off  

Select form generation option:

  • Automatic Select Forms - Tick this if you want all forms that are triggered for this event to be selected automatically.

  • TEST/REVIEW run - no update and/or credit/debit note creation and settlement - Tick this for a test or review run. The proposed actions will be shown in the log file.

 

Update/Cancel

Click this to start/cancel the process.

 

Open Log

Click this to view the results of the activity. This opens the AutoStrike-log.txt file which can be found in the Viewpoint Archive folder.

 

Close

Click this to exit the screen.

 

I do not want to use the status T when striking off an entity. How do I change this?

Set up the status you want to use in Configuration > Settings > Customise > System Settings. Select VPBILLS - STKOFF and click Edit. Change the value from T to the one you want to use.

 

Note: When you perform Strike Off with the options 'Create credit(debit) notes to cancel outstanding debit(credit) notes' and 'Automatically settle all the debit and credit notes' ticked, for Billing Files or Matter that is inactive, the program will temporarily activate the Billing File and/or Matter to perform the posting and then revert the status to its original value.

If this is only on a test run, then the log file will display a warning with the Billing File status.
