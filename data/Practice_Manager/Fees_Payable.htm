Fees Payable

The Fees Payable functionality supports disbursements that are payable to third parties. In this instance, you only make the payment to the third party on settlement of the invoice by the client. An example of such a disbursement is the annual government fee that is payable for continuation of incorporation. This is a recurring fee chargeable to the Billing File (client), which you only pay to the government after settlement of the invoice by the client. Management of third party fees is done in this screen.

Services that you control using the Fees Payable functionality are recognized by the Billing Event set to '3rd Party AR - Fees Payable [PAR]' and the tags entered for calculation of the fees and due dates in the Service Master.

The Fees Payable screen can be accessed through:

  • Viewpoint - Home screen:

    • Click Sales Ledger Ribbon button and select Sales Ledger > Accounts Receivable > Fees Payable from the navigation listing in Sales Ledger Overview.

    • In Section Menu, click Home > Practice Management > Sales Ledger > Accounts Receivable > Fees Payable.

  • File Maintenance screen:

    • Click Sales Ledger Ribbon button and select Invoicing > Fees Payable from the navigation listing in Sales Ledger Overview.

    • In Section Menu, click File > Sales Ledger > Invoicing > Fees Payable.

 

Filters

You can filter the list by:

  • Service - Select the service.

  • Payable - from / - to - Enter the date range.

  • Select Fees - Select the fee status.

  • Invoice/Draft - Select whether you want o search through invoices or drafts.

  • Batch - If you are looking for a specific batch, select it from the list.

 

Refresh button

Click to refresh the list after you have made your selections to the filters. The invoice lines that match the search criteria will be listed in the grid.

 

Fee list

This shows all the fees that match your selection criteria.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • Batch Maintenance - Click to maintain payment batches.

  • Select/Deselect All - Click to select/deselect all invoice lines listed in the grid.

  • Assign - Click to assign the selected fee(s) to a payment batch. When you do this, the payment batch total will reset to 0, and you will need to recalculate the payment batch. This button will not be enabled if you choose All Fees [Y] in the Select Fees field.

  • Undo Assign - Click to undo the assignment of the selected fee(s) to a payment batch. When you do this, the payment batch total will reset to 0, and you will need to recalculate the payment batch. This button will not be enabled if you choose All Fees [Y] in the Select Fees field.

  • Cancel Fee(s) - Click to cancel the selected fees. This button will not be enabled if you choose All Fees [Y] for Select Fees.

  • Fee (Re)calculation - Click to recalculate the selected fees. This needs to be done to review or recalculate all fee lines for the fee, fee due and penalty dates as well as any changed settings. You must run this function if any Government Fee setting has changed. This button will not be enabled if you choose All Fees [Y] for Select Fees. Any errors in the lines will be shown in the Error and Note fields of the table. Find out more here.

    Error

    • Blank - Click Fee (Re)calc. to calculate the outstanding fees payable.

    • No - There are no errors with the file setup.

    • Yes - Note the error(s) and correct the cause(s). See Notes, below.

     
    Notes

    No Registered Agent (REA) appointed as an Officer/Relation

    This means a Registered Agent needs to be appointed for the Billing File. This is done in the Statutory Officers/Relations screen in Administrator. The Officer/Relation Type should be Registered Agent [REA].

     

    No Registered Agent ID (user defined field with key code 'REA_ID')

    A user field needs to be created in the Master File of the Registered Agent in order to record the agent's ID. The key code for this user field, which is set up in System Manager,  MUST be REA_ID. This user field can also be created in the Entity screens in Administrator, but as only one key code with this ID can be created, your organization should decide where the field will be used, before creating it.

  • Invoice Penalty - Click to invoice penalties caused by late payment of outstanding fees. Penalty invoices are not created where the ‘Penalty date’ is beyond the strike-off date.

  • Strike Off - Click to strike off the selected entity.

 

I set up a recurring fee service for the fees payable functionality but the fee is not shown here. Why?

Services that are set up for the fees payable function must have the Billing Event set to '3rd Party AR - Fees Payable [PAR]'. This is done in the Service Master in Configuration > Business Parameters > Practice Management.
