Ad Valorem Fee

The Ad Valorem Fee [RV] service class is used to calculate fees to be charged based on the asset value. This value can be taken from either:

  • Bank Accounts, Securities, Open items, Fixed Assets in the Entity Administration or

  • Account File Valuation in Accounting

To use this, you need to add it as a service in Service Master and then set it up as a recurring fee in the Recurring Fees screen. Then when you perform Posting of WIP, the fee will be calculated based on the tags used for the service setup. It will also:

  • Update the Recurring Fee description according to what has been defined using [CHARGETEXT] tag.

  • For Account File Valuation in Accounting, the Recurring Fee currency will be set to be the same as Account File's base currency after perform Posting of WIP.

Viewpoint supports different pricing models combining different asset classes on a single Ad Valorem, which is known as Multiple Pricing Model. The following table lists the Asset type and the pricing model that supports it.

Configuration of Service in Service Master

When setting up the service in Service Master, the following needs to be defined using tags in the Tag Values field:

  • Value – Specify where to derive the value from e.g. Bank Accounts, Securities, Open items, Fixed Assets from Entity Administration or Account File Valuation in Accounting.

  • Method – The method to define the percentage for the fee calculation.

  • Minimum/Maximum Fee – If applicable, it is possible to set a minimum or maximum fee.

  • Description – Specify how the invoice line description should be.

For Accounting, user can define which NAV to use for calculation by using tags in the Tag Values field:

  • NAV for calculation – applicable for Account File Valuation where it can be:

    • Highest NAV value in the period specified.

    • Average NAV value in the period specified.

    • Last NAV value in the period specified.

Note: For explanation of the tags, refer to Service Master - Tags.

 

Single Pricing Model

Example 1

This is to calculate fee based on value from Open Items with a scaled percentage.

Tag Configuration in Service Master

  1. In Service Master, enter the code and description for the service.

  2. In Tag Values, enter the following tags:

    [VALUE=EAOI:Open Item 1-USD]
    [METHOD=S]
    [PSCALE=
    1,000,000 ~ 1.5
    5,000,000 ~ 1.0
    999,000,000 ~ 0.5]
    [CHARGETEXT=Asset Value: (-VALUE-); Annual Fee: (-FEEANNUAL-); Fee for the period from (-DATEFROM-) to (-DATETO-) is (-FEECHARGE-)]

  3. Click Update.

Note: For explanation of the tags, refer to Service Master - Tags.

 

In Recurring Fees screen, add the service as a Recurring Fee:

When WIP posting is run, the fee will be calculated from the total values for Open Item transactions within 1-Jan-2016 to 1-Jan-2017.

The Recurring Fee will be updated to as follows:

In the draft invoice, the detail line is as follow:

 

How the calculation works:

First 1,000,000, total fee is 1.5% of the asset value:

1,000,000 * 0.015 = 15,000

The remaining 2,600,000, total fee is 1.0% of the asset value:

(3,600,000 - 1,000,000) * 0.01 = 26,000

Hence, total Fee Charge (in USD) = 15,000 + 26,000 = 41,000.00

 

Note: The Charge Per period should not be longer than a year. For period less than a year e.g. quarterly, calculation is on a daily basis where Fee Charge is calculated as (Number of days in the specified period/Number of days in the year) x Annual Fee charge.

For example: First quarter of 2016 (1-Jan-2016 to 31-Mar-2016)

Annual Fee = 41,000

Number of days in the period = 31 (in January) + 28 (in February) + 31 (in March) = 90

Number of days in 2016 = 366 (2016 is a leap year, the number of days is 366)

Fee Charge = (90 / 366) * 41,000 = 10,081.97

This is to calculate the fee based on NAV value with a scaled percentage. The given Net Asset Value is 7,000,000.00.

How the calculation works:

Total Value for Valuation: 7,000,000.00

Multiple Pricing Model

Example 1

This is to calculate Asset under Management (AuM) where Matter Currency is used.

These tags are defined in the Service Master:

[AuMValue=A]

[CHARGECUR=M]

[M01.VALUE=EAASSET:A.BAType<>ST;F.AssetType<>'';S.SECTYPE<>'';O.OITYPE<>'']

[M01.METHOD=S]

[M01.PSCALE=

450,000~1.0

900,000~0.5

999,000,000~0.25]

 

Note: For explanation of the tags, refer to Service Master - Tags.

 

In the EA Assets, let say the following assets met the criteria of the Asset Type in the above tag:

  • Bank Account with transaction value GBP25,000.

  • Fixed Asset with transaction value EUR15,000,000.

  • Securities with transaction value USD350,000

Posting of WIP is then run from date 1-Jan-2018 to 31-Dec-2018. The calculated total fee is EUR42,804.84.

 

How the calculation works:

When [AuMValue=A] with Matter currency is used, the exchange rate will be based on the logic as explained in [AuMValue=A] in Service Master - Tags. For this example, the system retrieves all the previous AuM transactions until the last day of the month in the Recurring Fee’s ‘Fee Period From Date’.

The Recurring Fee's ‘Fee Period From Date’ = 24-Sep-2018, hence the exchange rate will take the rate on 30-Sep-2018.

On 30-Sep-2018, the exchange rates from the Exchange Rate Master table are:

Base Currency

To Currency

Exchange Rate

GBP

EUR

1.091999147

USD

EUR

0.8418126756

 

Total Asset Value (in EUR)

= (25,000 * 1.091999147) + 15,000,000 + (350,000.00 * 0.8418126756)

= 15,321,934.42

 

Total Fee Calculation (in EUR)

First 450,000, total fee is 0.1% of the asset value:

450,000 * 0.01 = 4,500

Up to 900,000, total fee is 0.5% of the asset value:

(900,000 - 450,000) * 0.005 = 2,250

Up to 999,000,000, total fee is 0.25% of the asset value:

(15,321,934.42 - 900,000) * 0.0025 = 36,054.84

Hence, total fee (in EUR) = 4,500 + 2,250 + 36,054.84 = 42,804.84

 

Example 2

This is to calculate Asset under Management (AuM) where Reporting Currency is used.

These tags are defined in the Service Master:

[AuMValue=A]

[CHARGECUR=R]

[M01.VALUE=EAASSET:A.BAType<>ST;F.AssetType<>'';S.SECTYPE<>'';O.OITYPE<>'']

[M01.METHOD=S]

[M01.PSCALE=

450,000~1.0

900,000~0.5

999,000,000~0.25]

 

Note: For explanation of the tags, refer to Service Master - Tags.

 

In the EA Assets, let say the following assets met the criteria of the Asset Type in the above tag:

  • Bank Account with transaction value GBP25,000.

  • Fixed Asset with transaction value EUR15,000,000. The Transaction Exchange to GBP is 0.879781.

  • Securities with transaction value USD350,000. The Transaction Exchange to GBP is 0.716608.

Posting of WIP is then run from date 1-Jan-2018 to 31-Dec-2018. The calculated total fee is GBP38,181.32.

 

How the calculation works:

When [AuMValue=A] with Reporting currency is used, the exchange rate will be based on the Transaction Date’s rate.

 

Total Asset Value (in GBP)

= 25,000 + (15,000,000.00 * 0.879781) + (350,000.00 * 0.716608)

=13,472,527.80

 

Total Fee Calculation (in GBP)

First 450,000, total fee is 0.1% of the asset value:

450,000 * 0.01 = 4,500

Up to 900,000, total fee is 0.5% of the asset value:

(900,000 - 450,000) * 0.005 = 2,250

Up to 999,000,000, total fee is 0.25% of the asset value:

(13,472,527.80 - 900,000) * 0.0025 = 31,431.3195

Hence, total fee (in GBP) = 4,500 + 2,250 + 31,431.3195 = 38,181.32

 

Example 3

This is to calculate Asset under Management (AuM) where minimum fee is defined based on various currencies.

These tags are defined in the Service Master:

[M01.VALUE=EAASSET:A.BAType<>'';F.AssetType<>'']

[M01.Method=S]

[M01.PSCALE=

1,000,000~0.0

3,000,000~0.171

10,000,000~0.114

20,000,000~0.086

999,000,000~0.05]

[M01.Fee.Min.CHF=1,000]

[M01.Fee.Min.USD=1,500]

[M01.Fee.Min=500]

[CHARGECUR=M]

 

Explanation on the tag: For pricing model ‘M01’, the system retrieves bank accounts of any type and fixed assets of any type to calculate the Ad Valorem fees based on asset value measured by scale.

The calculation is based on the Matter currency. The minimum fee has been defined at CHF1,000 or USD1,500 depending on the currency used. If the Matter currency is not CHF or USD, the minimum fee is fixed at 500 based on the currency used.

 

Example Scenario 1:

Matter Currency = USD

Total Asset Value in Matter Currency = USD60,000

Explanation: The asset value is USD60,000 (which is below the value of 1,000,000), and therefore the Ad Valorem fee is fixed at 0% of the asset value. However, the minimum fee has been defined at USD1,500. Hence, for this scenario, the Ad Valorem Fee is USD1,500.

 

Example Scenario 2:

Matter Currency = GBP

Total Asset Value in Matter Currency = GBP60,000

Explanation: The asset value is GBP60,000 (which is below the value of 1,000,000), and therefore the Ad Valorem fee is fixed at 0% of the asset value. However, the minimum fee has been defined at 500 if using other types of currency than USD or CHF. Hence, for this scenario, the Ad Valorem Fee is GBP500.
