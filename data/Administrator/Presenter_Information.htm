Presenter Information

In certain jurisdictions, the statutory forms require details of the presenter such as the name and address. If there is only for presenter for all managed entities, then the presenter can be specified in the Presenter field in System Default > Entity/BPM. However, if there are different presenters for different jurisdictions, then tags can be added for the jurisdiction in the Country Codes [CNTR] lookup table. The tags will allow for the following:

  • different presenters for different managed entities in the jurisdiction

  • same presenter for all managed entities in the jurisdiction

In Viewpoint's standard statutory form templates, the presenter information that is retrieved will depend on the jurisdiction of the entity where the system will check if there is a tag added for the jurisdiction. If there is no tag added, then the presenter will still be the one defined in System Default > Entity/Forms. However, if tag has been added, then the information will be according to the tag.

 

Note: The jurisdiction can refer to either where the entity is incorporated, or where the entity has a foreign registration. This depends on whether the form template is linked to Country of Incorporation or Country of Registration.

 

How it Works

Two tags are available:

  • PRESENT.RELTYPE

  • PRESENT.MFCODE

When retrieving the presenter information for an entity, the system will check if the jurisdiction contains tag for presenter. If both tags are added, the system will retrieve the presenter information based on PRESENT.RELTYPE tag first. If this is not available i.e. there is no relation for the entity, then it retrieves the presenter based on PRESENT.MFCODE tag. This means that even within one jurisdiction, depending on the entity, it is possible to have the presenter either as an appointed relation, or a defined Master File.

 

Different Presenter for Different Managed Entities

When there is a need to have a different presenter for different managed entities in a jurisdiction, there must be a relationship type for presenter, and presenter must be appointed for the managed entities. Then, ensure that the [PRESENT.RELTYPE=<RelCode>] is added for the jurisdiction in the Country Codes [CNTR] lookup table with the <RelCode> being the code of the relationship type for presenter.

If the entity does not have an active appointed presenter, then presenter will refer to the one defined in System Defaults > Entity/Forms.

 

Same Presenter for all Managed Entities

If the presenter is the same for all managed entities in the jurisdiction, then you only need to add the [PRESENT.MFCODE=<MFCode>] is added for the jurisdiction in the Country Codes [CNTR] lookup table with the <MFCode> being the code of the presenter's Master File.
