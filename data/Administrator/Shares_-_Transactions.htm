Transactions

All share transactions that have been entered are listed here. If there is any error for a share transaction, you can reverse it and re-enter the transaction.

Share transactions follow a certain sequence of events e.g. the shares need to be authorised before they are allotted or transferred so, when reversing a transaction, you need to follow the sequence of events backwards. For instance, to reverse a share authorisation, you must first reverse any share allotments for the shares.

Inconsistencies may develop if the share register if there is a large number of changes made to the transactions. If this occurs, you can rebuild the share register to correct the errors.

There are two sections in this screen and you can click on:

  • - To minimise this section and hide the command buttons. In the grid, only the selected row will be seen.

  • - To expand the section and show the command buttons.

Share Class

This lists share classes for the company.

 

Transactions

This lists details of all transactions that have occurred for a selected share class.

 

Select button

Click to perform search using search criteria.

Enter the search criteria and click Apply.

If you are unsure of how to enter the value, place your mouse within the Search for.. column to view tooltip.

Tooltip will appear to guide data entry for the following field type:

Field Type Tooltip Message
Number Enter 'number and number' for a range (example: 100 and 200)
Date User a number for a date relative to today (example -1 for yesterday) and 'and' for a range (example: -7 and 0)
Search control Start with the '+' sign to search on the code (example: +CODE) or the start of the name
Text Use the * in the search text for wild-card selections

 

 

Command buttons

Click on the relevant command button to perform the necessary.

  • Reverse - Click this to reverse the selected transaction.

    Once you have selected the transaction and clicked Reverse, you need to click Yes to continue or No to cancel.

    You are advised to reverse from last entered transaction (Last In First Out order) until you reach the incorrect transaction that you want to reverse.

     

    Note: For transactions that are related e.g. partial transfer resulting in 'Transfer' and 'New Certificate' transactions, reverse the 'Transfer' transaction and the related 'New Certificate' transaction will be reversed as well.

  • Rebuild - Click this to rebuild the share register. Once you have clicked Rebuild, the system will initiate the process. Click OK when the process is complete This can be used to correct certain data inconsistencies.
    However, for errors that cannot be corrected, you will see these displayed in the Share Transaction Analysis and Rebuild screen. In this screen, are the following buttons:

    • Repair - This is only available if you choose to rebuild a share class that does not utilise certificate number and there was an incomplete reversal (e.g. 'New Certificate' transaction was reversed without reversing the originating transaction). Click this to restore the wrongly reversed transaction.

    • Open Log - This displays the log file for the rebuild.

    • Print Report - This enables you to print the log.

    • OK - Click to close the screen.

  • Edit - Click this to change description of the transaction or add/edit transaction notes.

  • Explore - Click this to open the Share Structure Explorer screen. This button is turned off by default.

  1. Situation where the Rebuild function can be used:

    Orphaned Transactions

    For example, when Issuance of shares is accidentally reversed instead of Transfer of shares. The Rebuild function will prompt error messages in the Share Transaction Analysis and Rebuild screen.

    Shareholders Name Change

    For example, when a shareholder changes its name. The Rebuild function will reflect the new name in all the Shares screens.

    Change of the Number of Decimal Places in Field Formats

    For example, when the number of decimal places in Field Formats is changed and the change corresponds to the values in the Shares screens. The Rebuild function will correct all affected records to the new number of decimal places.
