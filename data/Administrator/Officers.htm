Officers

Records of officers are maintained in Statutory Officers folder for managed entities and Trust Officers for trusts. In both folders, there is a table that lists all officers who have been appointed and resigned. Information from here can be retrieved for registers such as Register of Directors.

 

Filter

You can use the filter to view Appointed, Resigned, Proposed or All officers.

Note: Officers with a Proposed status will not be included in Annual Returns.

 

Officers list

This shows the officers appointed for the Entity File.

 

Command buttons

Click the relevant command button to perform the necessary.

  • Appoint - Click this to appoint an officer. A Master File can only be appointed as an officer if the Relation/Officer option has been ticked in Master File - Setup folder.

    1. Enter the required information.

    2. Click Update.

    3. Tick the form(s) you want to generate for this appointment.

    4. Click OK.

    Note: You may have additional fields depending on the set up of the relationship type.

  • Resign - Click this  to resign an officer.

    1. Enter the resignation details.

    2. Click Update.

  • Re-elect - Click this to re-elect an officer. This may not be available, depending on how the officer type is set up.

    1. Enter the details for the re-election.

    2. Click Update.

  • Confirm - Click this to Confirm the appointment. This will update the status from Proposed to Appointed. The command button is available only if you filter by Proposed status.

  • Edit - Click this to edit the details of an officer.

    1. Edit the necessary details.

    2. Click Update after you have completed the editing.

  • Delete - Click this to delete an officer record.

    Click Yes to confirm the deletion.

  • Move Up - Click this to move the selected record up the list. The order in the table is reflected in forms and minutes.

  • Move Down - Click this to move the selected record down the list. The order in the table is reflected in forms and minutes.

 

Why am I not able to select a Master File to appoint as an officer?

Check the following:

  • Is the Relation/Officer role ticked for the Master File? - If the Relation/Officer is not ticked for the Master File, the Master File will not appear in the list for selection.

    This can be seen in the Master File > Setup folder.

  • Is detailed relationship roles enabled? - Detailed relationship roles only allow a Master File to be appointed to roles indicated in the Relationship roles allowed list in Master File - Setup folder.

    This is set in Configuration > Settings > Tools & Utilities > General /Master File.

Where is the Officer types set up?

This is in Configuration > Business Parameters > Administrator > Relationship Types.

Statutory Officers are in class O, Trust Officers are in class T.

 

Why don't I see some of the fields for officer appointment?

Depending on how the officer type is set up, the following fields may not be active:

  • Date Next Re-election

  • Position

  • Key powers

What happens if the resignation date is in the future?

When you resign an officer and enter a resignation date that is in the future, the officer will still appear in the 'Active/Appointed' list but the status is set to 'R'.

 

Where is the Reason of Resignation list maintained?

This is in Configuration > Business Parameters > General > Lookup Values. Select the Reason Resignation [ORRS] table.
