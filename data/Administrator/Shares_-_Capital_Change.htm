Capital Change

The Capital Change function will be displayed as a separate window containing the following tabs:

  • Options

  • Transactions

  • Summary

To perform capital change, start by selecting the relevant option in Options tab and then review the transactions that are listed in Transactions tab. Finally, click Update in Summary tab to apply the change.

 

Options

In this tab, the following options are available for users to select:

  • Create new share class, cancel existing shares and allot new shares ("Capital Change").

    Use this option if you want to create a new share class (e.g. by specifying a new share class name). In this case, the existing members will be issued new certificate with the new share class name. If the new share class is in a different currency, enter that currency and the nominal value.

    Similarly, if there is a new allotment per share, indicate the amount allotted for each share currently held. The certificate will show the member's new share allotment. When a new share class is created, the certificates relating to the old share class will be cancelled.

    For this option, the following fields can be changed:

    • Description - This defaults to 'Issue [AL]' but can be changed to another value.

    • Transaction Date - This defaults to today's date and refers to the date the change is entered.

    • Effective Date - This defaults to today's date and refers to the date the capital change is confirmed.

    • New Share Type - Select the share class.

    • Extension - This field is disabled as it is system-generated.

    • Share Class Name - This is taken from the description for share type.

    • New Ownership Structure - If there is a change in the ownership structure, click to select the new structure. This is only applicable if the entity supports more than one ownership structure.

    • Certificate Prefix - Enter the certificate prefix, if applicable.

    • Issue Certificates - Tick if you want to issue certificates for this class.

    • Currency - Select the currency for the new share class.

    • Nominal Value - Enter the nominal value of the share.

    • Allot for each share held - Enter the factor to multiply with the existing number of shares to derive the new number of shares, or enter the corresponding number of shares that will be allotted for each share currently held.

    • Note - Enter notes relating to the capital change.

    You can also select these options:

    • Trigger events - This is ticked to trigger the necessary forms for share authorisation.

    • Automatically Select Forms - Tick this to have all relevant forms automatically selected and sent to the Form Queue.

     

  • Allot new shares to existing members proportional to their shareholding at ZERO Issue/Paid Value ("Bonus Issue").

    Use this option when bonus is issued on shares held. This type of issue is often at no cost to the shareholder so the Nominal Value field is disabled. Usually bonus issues represent a multiple of the number of shares currently held by a member and this number is allotted to the current shareholding.

    You will see the following fields and options:

    • Description - This defaults to 'Issue [AL]' but can be changed to another value.

    • Transaction Date - This defaults to today's date and refers to the date the change is entered.

    • Effective Date - This defaults to today's date and refers to the date the capital change is confirmed.

    • Share Class- This defaults to the selected share class.

    • Share Class Name - This is taken from the description for share type.

    • Ownership Structure - This defaults to the structure of the share class.

    • Certificate Prefix - This displays the certificate prefix for this share class.

    • Issue certificates - Tick if you want to issue certificates for the bonus issue.

    • Currency - This displays the currency for the share class.

    • Nominal Value - This displays the nominal value and cannot be edited.

    • Issue for each share held - Enter the number of shares to be issued for each share currently held by a member.

    • Note - Enter notes relating to the capital change.

    • Trigger events - This is ticked to trigger the necessary forms for share authorisation.

    • Automatically Select Forms - Tick this to have all relevant forms automatically selected and sent to the Form Queue.

     

  • Allow new shares to existing members proportional to their shareholding at Proportional Issue Paid Value.

    Use this option when recording a rights issue. This is similar to a Bonus Issue except that the new shares are paid for.

    You will see the following fields and options:

    • Description - This defaults to 'Issue [AL]' but can be changed to another value.

    • Transaction Date - This defaults to today's date and refers to the date the change is entered.

    • Effective Date - This defaults to today's date and refers to the date the capital change is confirmed.

    • Share Class- This defaults to the selected share class.

    • Share Class Name - This is taken from the description for share type.

    • Ownership Structure - This defaults to the structure of the share class.

    • Certificate Prefix - This displays the certificate prefix for this share class.

    • Issue certificates - Tick if you want to issue certificates for the bonus issue.

    • Currency - This displays the currency for the share class.

    • Nominal Value - This displays the nominal value and cannot be edited.

    • Issue for each share held - Enter the number of shares to be issued for each share currently held by a member.

    • Note - Enter notes relating to the capital change.

    • Trigger events - This is ticked to trigger the necessary forms for share authorisation.

    • Automatically Select Forms - Tick this to have all relevant forms automatically selected and sent to the Form Queue.

     

  • Restate existing share class, certificates and transaction history.

    Use this option if you have made a mistake, for example, in entering the currency or nominal value for the selected share class.

    For this option, the following fields can be changed:

    • Description - This defaults to 'Issue [AL]' but can be changed to another value.

    • Transaction Date - This defaults to today's date and refers to the date the change is entered.

    • Effective Date - This defaults to today's date and refers to the date the capital change is confirmed.

    • New Share Type - Select the share class.

    • Extension - This field is disabled as it is system-generated.

    • Share Class Name - This is taken from the description for share type.

    • New Ownership Structure - If there is a change in the ownership structure, click to select the new structure. This is only applicable if the entity supports more than one ownership structure.

    • Certificate Prefix - Enter the certificate prefix, if applicable.

    • New Currency - Select the currency for the new share class.

    • New Nominal Value - Enter the nominal value of the share.

    • Adjustment factor @share - Enter the factor to multiply with the existing number of shares to derive the new number of shares.

    • Note - Enter notes relating to the capital change.

    You can also select these options:

    • Trigger events - This is ticked to trigger the necessary forms for share authorisation.

    • Automatically Select Forms - Tick this to have all relevant forms automatically selected and sent to the Form Queue.

     

Depending on the type of capital change, select the relevant option and enter the required details. Once complete, proceed to the next tab to review.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • Next - Click this to go to the next tab.

  • Cancel - Click this to cancel and exit Capital Change.

 

Transactions

This tab shows the existing transactions for the selected share class and transactions that will be added for the selected capital change option. You can right-click within the grid to copy, export or print the information.

If required, changes can be made to transactions related to capital change can be made before these transactions are updated. Changes that are possible would be to values in white cells. Values in yellow cells CANNOT be changed. Once all relevant changes have been made, go to the next tab.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • Back - Click this to go back to the previous tab.

  • Next - Click this to go to the next tab.

  • Cancel - Click this to cancel and exit Capital Change.

 

Summary

This tab shows an overview of updates for the share class due to the capital change where you have the original transaction followed by After Change section, Change section and Other Share Classes section:

  • After Change - Shows changes that will be applied to the selected share class due to the capital change.

  • Change - Shows the calculated difference between the original and the capital change.

  • Other Share Classes - This is only available if the entity has more than one share class.

If required, you can right-click within the grid to copy, export or print the information.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • Back - Click this to go back to the previous tab.

  • Update - Click this to apply the capital change transactions.

  • Cancel - Click this to cancel and exit Capital Change.
