Combine Share Certificates

Share certificates can be combined if they are of  the same share class and owned by the same shareholder. You can combine certificates when you are allotting shares and when you are transferring shares. In both instances, you need to know the share certificate number you want to add the shares to.

  • Combining during share issuance

    1. Enter the information for the issuance:

      • Description - This defaults to Authorization [AU] but you can change the description.

        More descriptions can be added in Share:Allotment/Issue Types [S_AL] lookup value table.

      • Allottee - Select the Master File of the person or entity the shares are allotted to.

      • Transaction Date - This defaults to today's date and is effectively the date the transaction was entered.

      • Effective Date - Enter the date the shareholders pass a resolution authorising the increase in share capital. If this is left blank, the transaction must be posted at a later date in Posting.

      • Number of Shares - Enter the number of shares to be authorised.

      • Subscriber Shareholder? - Tick this to indicate the shareholder is a subscriber shareholder

      • Consideration/Share - Enter the consideration paid  per share. If this is higher than the nominal value, the system will indicate that the shares are issued at a premium. If it is lower than the nominal value, the system will indicate the shares are issued at a discount.

      • Paid-up/Share - Enter the actual amount paid per share. If the paid value per share is equal to the consideration, the system will indicate that the shares are 'Fully paid'. If the paid-up value is lower, the system will indicate the shares are only 'Partly paid'.

      • Paid - Select the means of payment. If it is not by cash, give details in the Note field. There can only be one payment method for a share transaction.

      • Distinctive Numbers - Enter the distinctive numbers that will be shown on the share certificate. For range of share numbers, it  should be entered as 1-25; 55-100 which indicates a range of 1 to 25 and 55 to 100 for a total of 70 shares. Note the use of dash [-] and semi-colon [;].

      • Certificate No. - Delete the default share certificate number and enter the number of the existing certificate you want to add the shares to.

      • Location - Enter the location of the share certificate, if applicable.

      • Note - Enter additional information for the share transaction.

    2. Click Update.

    3. Tick the form(s) you want to generate.

    4. Click OK.

     

  • Combining during share transfer

    1. Enter the relevant information for the share transfer:

      • Description - This defaults to Transfer [TR] but you can change the description.

        More descriptions can be added in Share:Transfer Types [S_TR] lookup value table.

      • Transferee - Select the individual or entity to whom the shares are being transferred.

      • Transaction Date - This defaults to today's date but can be changed. It is essentially the date the transaction was agreed upon between the shareholders or the date the transaction was entered.

      • Effective Date - Enter the date the transfer was actually registered and (where appropriate) the Stamp Duty was paid.

      • Number of Shares - Enter the number of shares being transferred. This cannot be higher than the total shares. If the number of shares transferred is lower than the total shares, the New Certificate section will be enabled.

        This section is enabled if there are shares left after the transfer. Enter the following information:

        • Description - This defaults to New Certificate [NC] but you can change the description.

          More descriptions can be added in Share:New Certificate Types [S_NC] lookup value table in System Manager.

        • Distinctive Numbers - Enter the distinctive numbers that will be shown on the share certificate. For range of share numbers, it  should be entered as 1-25; 55-100 which indicates a range of 1 to 25 and 55 to 100 for a total of 70 shares. Note the use of dash [-] and semi-colon [;].

        • Certificate No. - This defaults to the first available number but can be changed. The initials following the words 'Certificate no.' indicate the certificate prefix. If you have set the certificate prefix as 'OR' it will appear as 'Certificate No. OR'. Enter zero if you do not require a certificate.

        • Location - Enter the location where the share certificate is kept, if applicable.

         

      • Total Consideration - Enter the total consideration for the shares being transferred. The system will divide that amount by the number of shares and display the consideration per share.

      • Distinctive Numbers - Enter the distinctive numbers that will be shown on the share certificate. For range of share numbers, it  should be entered as 1-25; 55-100 which indicates a range of 1 to 25 and 55 to 100 for a total of 70 shares. Note the use of dash [-] and semi-colon [;].

      • Certificate No. - Delete the default share certificate number and enter the number of the existing certificate you want to add the shares to.

      • Location - Enter the location where the share certificate is kept.

    2. Click Update.

    3. Tick the form(s) you want to generate.

    4. Click OK.

     

Note that when share certificates are combined, the shares held by each certificate will be listed separately but will have the same certificate number.
