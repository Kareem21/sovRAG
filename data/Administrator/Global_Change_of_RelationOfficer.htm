Global Change of Relation/Officer

This is to change appointed Relation/Officer(s) linked to the Master File or User Relation for multiple files. It is available in Viewpoint - Home screen. To access it:

  • Using Ribbon - Click EA Global Ribbon button, select Global Change > Relation/Officer

  • Using Section Menu - Click Home > Entity Administration > Global Change >  Relation/Officer

 

When accessing the function, you will see the following:

  • Settings tab - This is to specify the change and parameters for the change.

  • Selection tab - This will show results based on the parameters set.

 

Performing Global Change of Relation/Officer

  1. Access the function in Viewpoint - Home screen.

  2. Select Relation/Officer(s) for the Global Change information in the Settings tab:

    • Global Change - Select the Global Change action:

      • Resign and Appoint

        Select this to perform a resignation and appointment simultaneously. This applies to relationship types linked to Master Files and depending on the relationship types that you select, may apply to Additional Relations as well.

        For the appointment, it will check to ensure that the replacement Master File is configured appropriately and inform users if:

        • the replacement Master File does not have the relevant Role Allowed option ticked e.g. Statutory/Trust Officer option is not ticked

        • the replacement Master File does not have the Relationship Type selected when Detailed Relationship Roles is enabled in System Defaults > General/Master File

        When the message is seen, click OK and exit Global Change in order to correct the set up of the replacement Master File.

         

      • Resign

        Select this to perform a resignation only. This applies to Master File relations and depending on the relationship types that you select, may apply to Additional Relations as well.

         

      • Resign and Appoint - User Relation

        Select this to perform a resignation and appointment simultaneously for relationship type(s) set up as User Relation.

         

      • Resign - User Relation

        Select this to perform a resignation only for relationship type(s) set up as User Relation.

         

    • From - Select the Master File for the relation/officer that will be resigned/replaced.

    • To - Select the replacement Master File/user. This field is available only if the action is 'Resign and Appoint' or 'Resign and Appoint - User Relation'.

    • Effective Date - This defaults to today's date and refers to the effective date of the change. This date will appear on the appropriate forms.

    • Client/Group - This is optional and it refers to the Client/Group field in the Entity File General screen. If specified, in the Selection tab, you will only see the file of the specified Client/Group.

    • Reason for Resignation - Select the reason for the resignation.

    • Administrator Group - This is optional and it refers to the Administrator Group field in the Master File Setup screen. If specified, in the Selection tab, you will see only file of the specified Administrator Group.

    • Form Selection - Select Automatic if you want all related forms for the change to be sent to Form Queue. Select Prompt if you want to make the selection manually.

    • Relationship Types - Tick the Relationship types to apply the change to.  

    • Master File Types - Tick the Master File types to apply the change to.

    • Entity Status - Tick the Entity statuses to apply the change to.

    • Save Relationship Types, Master File Types and Entity Status selections as Preference - Tick this to save the current selections to be applied for the next Global Change.

  3. Click Next.

  4. In the Selection tab, ensure that the change you want to perform is ticked. You can click:

    • Select All - To tick all records. This is the default.

    • Deselect All - To untick all records.

  5. Click Start.
