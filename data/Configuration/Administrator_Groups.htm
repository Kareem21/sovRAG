Administrator Groups

Administrator Groups are used to separate an organisation into groups, such as departments, teams or even jurisdictions. These groups can be used for added security by ensuring that the user will only be able to access the data of Master Files which his group manages. This will ensure client confidentiality and prevent data corruption by unauthorised users. A user can belong to more than one Administrator Group.

For example:

If a company's secretarial department is divided into three teams that work on 30 different companies, you can set up three Administrator Groups. Set up the user IDs for each individual user in each group and enter the appropriate group code against their name.

When a company is entered into the system, you can specify which group it is administered by, thus allowing all members of that group access to the company's data, and restricting access for everyone else. The access rights that you allocate should be carefully considered in order to protect your data. If in doubt about what access level to give a user group, give them low access rights - you can always give them higher access rights later, if necessary.

If you are using Administrator + User Group Model [2] or higher as your security model, you need to set up your Administrator Groups in the Administrator screen.
