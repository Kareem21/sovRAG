System Defaults - Security

The security for your environment is set up here.

 

Security Model

Show the security model from the system standards. This field is not editable. You are required to select the Security Model from Viewpoint Management.

 

Token Expires In

This field is used to set the number of hours for the password activation or reset token to remain valid. Tokens exceeding the number of hours set will expire and the users will not be able to use the links sent in the e-mails to activate or reset a password. The value for this field must be numerical. The default value is 5.

 

Exception Log

Select the location for the exception log:

  • Log to File [1] - Writes exceptions to file.

  • Log to Database [2] - Stores exceptions to database.

Option [2] will store the errors in 'Exception Log' screen under Configuration > Users & Security > Security Setup.

 

Maintain User/System Access Log

Tick this to keep a record of user activity in Viewpoint. The information will be seen in the User/System Access Log screen. This feature can be useful when trying to determine the source of a user's login problems.

 

Password Expires in (days)

Enter the number of days a password can be used before it expires. Enter '0' to disable the feature.

Note: This option does not apply if the user logs in using Windows Authentication.

 

Minimum Password Length

Enter the minimum number of characters for the password. Enter '0' to disable the feature.

Note: This option does not apply if the user logs in using Windows Authentication.

 

Login Failure Threshold

Enter the maximum number of attempts allowed for user account login. When the login attempt reaches the threshold, the user account will be deactivated. Enter '0' to disable the feature and allow infinite attempts.

Note: This option does not apply if the user logs in using Windows Authentication.

 

Require Complex Passwords

Tick this box to enforce complex password. When this option is enabled, password should consist of a mix of alphanumeric, upper case, lower case and special characters. It will validate the password length as set in the 'Minimum Password Length' field and prompt an 'Invalid Minimum Password Length' message if the password length is less than 6 or more than 20.

Note: This option does not apply if the user logs in using Windows Authentication.

 

Prompt with Last Login

Tick this box if you want the date and time of the user's last login session to be shown at the subsequent login.

 

Disable Multiple Logins

Tick this box to disable multiple logins for the same user account.

 

Apply Master File Security to Reports and Widgets

Tick this box to restrict information retrieved for the reports and widgets to Master Files that the user can access. If this is not ticked, then the reports and widgets will include Master Files that the user cannot access as well. This applies to Custom Reports, Word Reports, Excel Statements, SSRS Reports and widgets. For custom reports, this requires the custom report to have Use Administrator Group Authority = Yes, while the others require MFSEC tag.

 

Enable Two-Step Verification

Tick this box to enable two-step verification for Viewpoint Connect and Local Agent logins. When enabled, after logging in, users must key in a six-digit code in a verification screen to complete the login process.

Note: Please ensure that the workstation and mobile phone are in sync with the current date and time. Otherwise, the six-digit code entered on the verification screen will be invalid.

 

Restrict Master Files to Users

Tick this box to restrict users from seeing Master Files that they cannot access. This applies to Master File Search performed in Viewpoint - Home, File Maintenance, and Billing Enquiry. When accessing Changes screen from Viewpoint - Home, records of changes applied to Master Files that user cannot access will also be hidden. This option applies to the Services Provided Model [5] security model.

Note: Records in File Maintenance and File Enquiry containing Master Files that user cannot access will still be available.

 

Enable Authentication Using E-mail

Tick this to use E-mail ID to login to Viewpoint Connect instead of User Code. This is also applicable to using the Single Sign-On user login mode.

 

User Logout after Idle for (Minutes)

This field defines the idle time in Minutes. User will be logged out after the application is idle over a specific time. By default, the value of this field is Nil. Enter '0' to disable the feature.

 

Login Alert

Enter the displayed warning message to user when there is unauthorised access. The displayed message will have a 'Yes' and 'No' button. Proceeds login on 'Yes' and immediately exits on 'No'.

 

Command buttons:

Click on the relevant command button to perform the necessary.

  • Update - Click this to save changes.

  • Cancel - Click this to cancel the changes.
