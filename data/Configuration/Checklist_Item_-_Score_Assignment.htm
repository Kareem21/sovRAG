Checklist Item - Score Assignment

If necessary, scores can be assigned to the value of checklist item for calculation but how this is done depends on the checklist item's field type. The following are the available methods:

  • Using [WSCORE=] tag

  • Through Risk Ratings screen

  • Using Lookup Formula

 

Using [WSCORE=] Tag

The [WSCORE=] tag is used to assign scores when the checklist item's field type is not 'Lookup Table' or 'User Table'. The tag is to be added to the Values/Default column of the checklist item.

Field Type

Location of Score Tag

Example

Yes/No

Enter the tag in Tag Values field after you click the Browse button in the Values/Default column. Note that the first score is for 'Yes' and the second is for 'No'.

[WSCORE=5|0]

means a score a of 5 is assigned to a Yes value and a score of 0 is assigned to a No value.

 

List without Edit

Enter the tag in Tag Values field after you click Browse button in the Values/Default column. Note that the scores should follow the order of the list values.

List options are specified as Advertising|Agriculture|Aviation|Finance

 

[WSCORE=5|10|5|10]

means a score a of 5 is assigned to Advertising and Aviation and a score of 10 is assigned to Agriculture and Finance.

List with Edit

Enter the tag in Tag Values field after you click Browse button in the Values/Default column. Note that the scores assigned are for the list values. All user input will be assigned a score of zero (0).

Refer to the above.

 

Risk Ratings Screen

This is to be used for assigning scores when the checklist item's field type is:

  • Lookup Value

  • User Table Value

The scores are to be entered in the Value column of the Risk Ratings screen. Refer to Risk Ratings.

 

Note: If scores have been assigned for Lookup Values and User Table Values  using [WSCORE=] and [<ScoreSystem>=] tag earlier, this can be converted to Risk Ratings screen.

 

Lookup Formula

This is a type of formula that can be used to assign a score based on calculation of scores for other checklist items. This requires a checklist item with the field type 'Formula'. Refer to Checklist Item - Field Type: Formula for more information.

 

Score Assignment for Viewpoint Engage Workflow

System has been enhanced to automate the scores and formulas recalculation once a Viewpoint Engage request (Workflow) is submitted to the Back-office. To enable the automatic recalculation, the tag '[ON_SUBMIT=DORATING]' is required to be inserted into the Workflow Step properties. When this tag is configured, the Back-office will run the 'Refresh Rating' function to recalculate the scores and formulas once user clicks the 'Submit' button in the Viewpoint Engage Workflow step.
