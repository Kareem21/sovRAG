Identity/AEOI Register Types

Identity types are to be set up so that they can be selected in the Identity Register screen. When setting up an identity register type, you can define the fields for the type such as:

  • Master File field

  • Text fields

  • Lookup Table fields

  • User Table fields

  • Date fields

  • Checkbox fields

Depending on the type of fields, you may be asked to specify the table code or caption.

 

Note: The checkbox fields are only for recording purposes and are not linked to any functionality.

 

Filters

The filters apply to the identity register type list.

  • Type status  - You can choose to view only active types, inactive ones or all.

  • Customised setting - You can choose to view types that have been customised or not.

 

Select/Apply button

This allows for additional criteria to apply to the identity register type list.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • New  - Click to create a new identity register  type.

    1. Enter details for the new identity type.

    2. Select the applicable fields for the identity register type. If required, specify the caption or table code for the field.

    3. In the Tag Values field, you can add the relevant tag to specify the field properties or have the information updated to a field in another screen.

    4. Click Update.

    Note: To assign score for the types, tick the Risk Rating checkbox.

  • Edit - Click to edit an existing identity register type.

  • Delete - Click this to remove an existing identity register type.

I have created the Identity Register Type but I am not able to see it in the Identity Register screen.

If the Category is set to AEOI, the type will be available in the Classification and Indicia screen.
