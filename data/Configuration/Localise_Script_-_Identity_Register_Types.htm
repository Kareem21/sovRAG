Localise Script Tags - Identity/AEOI Register Types

Tag is available to allow new Identity/AEOI Register Types to be added using Localise Script where the type will have the Customised and Active checkboxes ticked by default.

 

Adding New Identity/AEOI Register Types

When using Localise Script to add a new type, you need to specify the required fields and values for the type. Each field will be separated with the pipe '|'.

 

Syntax

IDTYPE:<IDTypeCode>+<IDTypeCat>=<DBField>=<Value>|...

 

where:

  • <IDTypeCode> refers to the code of the Identity Register type

  • <IDTypeCat> refers to the category of the Identity Register type and can be

    • C for Compliance

    • R for AEOI

  • <DBField> is the database field for the Identity Register type and they are

    • Description – Description field

    • Individual – Apply to Individual Master Files option

    • MFLink1 – Associated Master File field

    • MFLinkCaption1 – Caption for Associated Master File field

    • IdCaption – Caption for Identity Text field, when this is specified, Identity Text checkbox will be ticked

    • LKVCode1 – Table Code for Lookup Value 1 field, when this is specified, Lookup Table Value 1 checkbox will be ticked

    • LKVCode2 – Table Code for Lookup Value 2 field, when this is specified, Lookup Table Value 2 checkbox will be ticked

    • UdCode1Table – Table Code for User Table  1 field, when this is specified, User Table 1 checkbox will be ticked

    • UdCode2Table – Table Code for User Table 2 field, when this is specified, User Table 2 checkbox will be ticked

    • With_Country – Country/Jurisdiction field

    • AdCaption – Caption for Additional Text field, when this is specified, Additional Text checkbox will be ticked

    • With_UserNamePassword – User/Account Name and Password field

    • With_Dates – From Date field

    • With_DateTo – To Date field

    • With_Reminder – Reminder field

    • Ud1Caption – Caption for Check Box 1 field, when this is specified, Check Box 1 checkbox will be ticked

    • Ud2Caption – Caption for Check Box 2 field, when this is specified, Check Box 2  checkbox will be ticked

    • Ud3Caption – Caption for Check Box 3 field, when this is specified, Check Box 3 checkbox will be ticked

    • TagValues – Tag Values field

    • IdNote – Note field

    • WithRating - Risk Rating field

  • <Value> is the value for <DBField>.

    • For <DBField> that are checkboxes, value is 1 to tick the checkbox field and indicate the field is required.

    • For other <DBField>, specifying the caption or Table Code will indicate the field is required.

    • For Tag Values field, the <Value> can include

      • <br> for a line break

      • <pipe> for a | symbol

 

Example

To set up a new code as shown above:

IDTYPE:DRL+C=Description=Driving Licence|Individual=1|IdCaption=Driver number|With_Country=1|With_Dates=1|With_DateTo=1
