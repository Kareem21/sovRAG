User Roles

User Roles are used to control access to screens and functions in Viewpoint through authority levels. These User Roles are then assigned to one or more User Groups. In instances where two or more roles with differing authority levels to the same screen are assigned to the same User Group, the higher authority level will prevail.

 

User Roles list

This shows User Roles that have been set up. Viewpoint comes with a standard set of User Roles but you can create your own User Roles.

  1. Go to Configuration > Business Parameters > General > Lookup Values.

  2. Locate and click to select User Roles [URLS] lookup table.

  3. Click New.

  4. Enter details for the new User Role.

    • Element Code - Enter a code for the User Role you want to create. Each element code must be unique, so if you enter a code that is already in use, you will not be able to create the role.

    • Name - Enter a simple description of the role.

    • Long Name - This field is used to group user roles together in the User Roles screen. For example, if you enter Accounting in this field, this User Role will be grouped together with other roles that have Accounting in this field.

    • Electronic Filing Code - If the user role being created is to be used as a service provided then enter SP in this field, otherwise, leave this blank.

    • Element/Sort Value - This field is not used for User Roles.

    • Tag Values - If the security model is Services Provided Model [5] and the user role is a service provided, then tags for File Review can be entered here.

    • Active - This is ticked by default. Removing the tick will remove the role from the User Roles screen.

  5. Click Update.

 

Authorities button

Click this to assign/amend authority level for the selected User Role.

  1. In the User Role Authority Settings screen, set the appropriate authority level for the screens listed in the Navigation column.

  2. Click OK when complete.

Note: When specifying the authority level for a summary screen, it is not possible to set it to 0 if any of the summary screen's child menus have a higher value.

 

Authority Levels

There are four authority levels that you can set for each screen:

  • 0 - No Access (default value): No access to the screen. Users will not see the screen in their Section Menu.

  • 1 - View Only: View-only access for the screen.

  • 6 - Regular User: Users can view the screen and have access to command buttons with authority required set to level 6 or lower, e.g. New and Edit.

  • 8 - Power User: Users can view the screen and have access to command buttons with authority required set to level 8 or lower, e.g. Delete.
