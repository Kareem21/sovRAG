Import/Export Settings

The Import/Export Settings screen allows you to import or export customised data and settings as well as reports from one Viewpoint database to another. These items are:

  • Customised data refers to values and tables that can be customised including Lookup Values, Drop-down Values, User Tables, To-Do Rules, Windows/Web Menu, Grid Fields, Change Code Links, etc.

  • System settings include Field Settings, Service Master, Service Tax Codes, Account Mapping, Filing Cabinet, Index Fields, User Fields, User Group Security and User Role Authority.

  • Standard Reports and Custom Reports.

  • Word Templates and HTML Templates.

  • Workflow Templates.

  • Activities Profiles.

  • Dashboard Widgets

 

Profile

Select the profile for the settings that you want to export. Switching profiles will change the settings selected. Profiles are created when performing an export, see steps for export below.

 

Delete button

Click to delete the selected profile. You cannot delete the Default profile [DEFT].

 

Settings

This lists all settings supported by this feature organised by categories. To select, you can:

  • Expand the category heading and tick the checkbox for individual items.

  • Tick the checkbox for the category heading to select all settings in the category.

  • Tick the checkbox for the main Import/Export heading to select all settings.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • Import - Click to start the import process. Find out more here.

    When importing settings from another environment, Viewpoint will recognise and list .xml files that contain a <ViewPointCustomisedSetting> tag. The import will only work for the same database type, for instance, importing information from one database to another database. Data in the current database will be deleted if it is dependent data, for instance, data in the VpAGroupUsers table is dependent on the VpUsers table.

    1. Browse to the folder containing the profile you want to import.

    2. Click Import - load list.

    3. Tick the checkbox of items you want to import.

    4. Click Import.

    5. Click Close when completed.

  • Export - Click to start the export process. Find out more here.

    When exporting the settings of one environment to another, you can select the data you want to export, and save this selection under a specific profile name. Once exported, you will see .xml files with the names 'VPSS_<jobcode>.xml' or 'REL_VPSS_<jobcode>.xml'. Each of these .xml files contain all the fields for the database table involved in the selection. For document forms, the actual physical files will be exported to a Form sub-folder within the selected Export folder.

    1. In the Import/Export Settings screen, tick the check box next to the settings that you want to export. Click the + or - nodes to expand or collapse the list.

    2. Click Export.

    3. Enter a name for the profile and click OK.

    4. Browse to the location of the Export folder.

    5. Click Export.

    6. Click Close when the process is completed.

  • Refresh - Click to reload the screen.
