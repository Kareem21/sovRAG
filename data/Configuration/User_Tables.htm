User Tables/Values

User tables can be custom-created for these functions:

  • Prevent data discrepancy by providing a standard list of accepted values for users to select.

  • For referencing tags.

You can create as many user tables as you need, as opposed to lookup tables, which are system-defined.

The upper grid displays the User Tables. When a User Table record is selected, the bottom grid will load the User Values for the selected User Table.

 

User Tables / Values Listing

This shows all the existing user tables in Viewpoint. Click to select the table you require.

 

Command buttons

These apply to the User Table list. Click on the relevant command button to perform the necessary.

  • New - Click this to add a new User Table.

    Enter the relevant information and click Update.

    In the Tag Values field, tags are available to:

    • Control whether new values can be added to the User Table

      [ALLOWNEW=N] prevents users from adding new values to the User Table manually by disabling the New button for the User Value.

      For import actions either through Import/Export Settings or Data Replication, this tag is to be entered in the environment where the User Table will be exported (Source/Master) environment.

      In Data Replication, the Master environment is used to set up the configuration, hence the [ALLOWNEW=N] tag does not limit the Master environment to create new User Value.

      The [ALLOWNEW=N] tag and configurations from Master environment will then replicated to the Subcribers where the tag will take effect and disallow user in creating new User Value.

      In the environment where import is performed (Destination/Subscriber):

      • when the exported .xml file contains value but the values do not exist in the destination/subscriber environment, these will be added.

      • when the exported .xml file does not contain value but the values exist in the destination/subscriber environment, these will be deactivated in the destination/subscriber environment.

      • when the exported .xml file contains the User Table but the User Table does not exist in the destination/subscriber environment, the User Table will be imported.

      • when the exported .xml file does not contain the User Table but the User Table exists in the destination/subscriber environment, the User Table will be deactivated in the destination/subscriber environment.

       If this tag is not entered, the default value is [ALLOWNEW=Y].

    • Specify the subscriber(s) of the User Table

      • [SUBSCRIBER=<Subscriber>] is to specify the subscriber group(s) that the User Table is for. For multiple groups, use a comma to separate e.g. [SUBSCRIBER=GEN,ASA,EUR]

      • [SUBSCRIBER<><Subscriber>] is to specify subscribers that will be excluded.

      See also:

      • Master Configuration Environment (MCE) and Data Replication

       

       

  • Edit - Click this to edit the selected User Table in terms of the:

    • Name field

    • Note field

    • Tag Values field

    • Risk Rating - Tick this to apply risk scores for the values of the User Table.

    • Active option

  • Delete - Click this to delete the selected User Table

Command buttons

These apply to the User Table Value list. Click on the relevant command button to perform the necessary.

  • New - Click this to add a new value to a user table. If the [ALLOWNEW=N] tag is specified for the User Table, this will be inactive.

    1. Select the user table from the User Table list.

    2. Click New.

    3. Enter details for the value.

    4. Click Update.

     

  • Edit - Click this to edit a selected table or value. Further parameter configuration can be done using Tag Values.

  • Delete - Click to delete a user table or value. You will need to confirm the deletion.

Country Matrix [AEOI]

File Review - AEOI Review

Tag for this table is for identification of countries a particular reporting authority has agreed to exchange with.

[<Regime>.RACREATE.<ReportingAuthorityCountryCode>=<ExchangeCountryCode>]

where:

  • <Regime> refers to the reporting regime.

  • <ReportingAuthorityCountryCode> refers to the country of the reporting authority.

  • <ExchangeCountryCode> refers to the country the reporting authority will be exchanging information with.

Example:

[C.RACREATE.BM=AR,BE,BG,CZ,DK,EE,FO,FI,FR,DE,HU,IS,IN,IE,IT,KP,LV,LT,LU,MT,MX,NL ,NO,RO,SM,SC,SK,SI,ZA,ES,SE,GB]

this defines the countries that reporting authority for Bermuda is exchanging information with for the CRS Reporting Regime..

 

[C.RACREATE.BS=NONE]

this specifies that there is no country for exchange for the reporting authority of Bahamas.

 

[C.RACREATE.BM=]

this specifies that the reporting authority of Bermuda will exchange information with all countries.
