Edit Checklist/Workflow Step Editor

Adding of items to a checklist/workflow steps will be through Edit Checklist screen or Workflow Step Editor screen depending on how you are creating the checklist.

Items list

This shows all items for the checklist/workflow step and for every item are the following columns:

  • Checklist item - This is the name or description of the item. The item can be a heading for indented items below it or instructions, or caption for the information required. You can:

    • Enter the text e.g. 'Name:' or 'Please provide the following details'.

    • Use codes to retrieve information from the database e.g. (-IntType\#LT-INTP-). The codes are Database Fields within (-...-) which are retrieved using SQL query or COPY tag; and may include format codes similar to those used in Word templates.

    • Use combination of both text and codes e.g. 'Position held by (-AddrCode\#AD-0-0-) - (-Position-)'.

  • Key - This is an identifier for the item so that it can be referred to in another checklist item. If the information is to be copied using Viewpoint Clipboard, this should be the Database Field. When the workflow consists of a step with Formula field type, the Key syntax for all other steps within the workflow will have restrictions due to Excel being used as the backend calculation tool in Formula field type.

    The Key should adhere to the following syntax:

    Valid characters    

    The first character of the Key can be a letter, an underscore character (_), or a backslash (\). Remaining characters in the Key can be letters, numbers, periods, and underscore characters.

    Note: Do not use uppercase and lowercase characters 'C', 'c', 'R', or 'r' as Key, because they are used as shorthand for selecting a row or column in the Excel sheet. Do not use numbers as first character.

    Cell references disallowed    

    Key cannot be the same as an Excel sheet cell reference, such as Z$100 or R1C1.

    Spaces are invalid     

    Spaces are not allowed as part of a Key. Use the underscore character (_) and period (.) as word separators, such as, Sales_Tax or First.Quarter.

    Name length    

    The Key can contain up to 255 characters.

    Failure to adhere to the above syntax will cause the Key to be omitted when it is use in Formula field type as calculation.

  • Help/Description - This is text that can be used to explain the item further.

  • Comment – This allows the user who design the workflow to add comments related to the checklist items for future maintenance or reference purpose. The information in this column will not be displayed in the Workflow screen.

  • Field Type - This determines the behaviour of the item such as whether or not it can accept input or perform an action. Depending on what is required, ensure that you select the appropriate field type.

  • Style - This is to apply a style to the display of the checklist item and is applicable to all field types. Refer to Checklist Item - Style.

  • Mandatory - This determines if the item is mandatory or not. If it is mandatory, user cannot complete the step if the item is left blank. Caption for mandatory item is displayed in blue text.

  • Required - This is to specify if the item is required or not. If it is required, user can still complete the step if the item is blank and complete it in another workflow step further along the workflow. Caption for required item is displayed in indigo text.

  • Values/Default - This is to specify additional configuration for the item based on the field type of the item. For the majority of the field types, this column also includes a Browse button. For certain field types, the Values/Default column must not be empty.

This list can be exported by performing a right-click and selecting:

  • Copy - This will copy the list to Windows clipboard so that you can paste to preferred destination e.g. to an Excel file or a table in a MS Word document.

  • Export to Excel - This will create an Excel file containing the list.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • Edit Properties - Click this to open the ‘Edit Step Properties’ screen.

  • Update - Click this to save changes made to the workflow step.

  • Cancel - Click this to cancel changes made to the workflow step.

Note: Changes made to the Workflow Step Editor screen will trigger the Step Library Workflow – Edit with Editor [WFPF] change code and the changes will be recorded in the Changes screen.
