Form Library

The templates in this library are usually statutory forms and minutes which are triggered by Changes and can be either:

  • Word document templates in .doc or .docx format.

  • PDF form templates created using XFA (XML Forms Architecture) PDF specification. Refer to Electronic PDF Forms.

The templates here can be generated only when the linked change occurs and users select them in the Change Processing screen.

Note: Certain forms like Annual Return for Jersey (JEC20), allows for the use of Tag Values in Entity Type to determine the sections to include in the template.

 

General/Jurisdiction

This defaults to the jurisdiction specified in System Defaults > Entity/Forms. Tick the General check box to view forms that are not related to any jurisdiction, or else select another jurisdiction to view the forms in that library.

 

Template list

All templates in the selected jurisdiction or General are listed here.

 

Change list

This shows all the changes linked to the selected form template.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • New - Click this to create a new form. For Viewpoint on the Cloud, this can only be performed on the server.

  • Edit - Click this to view or edit the properties of a selected form.

    General

    This check box is ticked for all Master File-related and jurisdiction-independent forms. If this field is completed, the Jurisdiction field (see below) is disabled.

     

    Jurisdiction

    This field shows which jurisdiction a form belongs to.

     

    Code

    This is the unique reference code for the template.

     

    Description

    This is the title of the document, and is displayed when the form is triggered for selection. This is also the description that is displayed in the form queue.

     

    Type

    The form type selections are maintained in the Form Type [FRMT] lookup value table.

     

    Customised (Don't overwrite in upgrade)

    Tick to prevent customised templates from being overwritten when you upgrade Viewpoint or import a template with the same code.

     

    Language

    Select the default language of the form. This does not currently have any impact on the usage of the form. Languages are maintained in the Language Codes [LANG] lookup value table.

     

    Require Filing

    Tick this if the form needs to be filed with the statutory regulatory body of that jurisdiction.

     

    Remind After (No. of days)

    Enter the maximum number of days after the form generation for it to be filed.

     

    Print No. Of Copies

    Specify the number of copies to be printed, i.e. the number to be printed if the document is sent directly to the printer.

     

    Path & File Name

    Enter or browse to the location where the template was saved. All forms should be stored in the Forms directory specified in Configuration > Settings > System Defaults - Folders - Template Forms. Remember that if you link the wrong template, you may end up editing the wrong template. When copying an existing template, the path will default to the jurisdiction sub-folder in the Forms folder. For instance, if you are copying a template in the Isle of Man jurisdiction, the Path & File Name will default to \IM\<formcode>.doc.

     

    Notes

    The notes field is for reference only. For example, you can enter a date on which the form was last reviewed.

     

    Help Text

    Enter any relevant information related to the usage of the form.

     

    Key Information

    This section contains the SQL (Standard Query Language) statements which are used to retrieve the information that is displayed in the forms. Double click on the field with the left mouse button to display the full contents. See also Coding a Document.

     

    Field Details

    This displays the field codes contained in the form. Double-click on the field with the left mouse button to display the full contents. Do not update this field as the contents are automatically updated every time you click on Refresh and should not be edited manually.

     

  • Delete - Click this to delete the selected form template from the Form list. This will also delete any forms in Form Queue that utilises the template. For Viewpoint on the Cloud, this can only be performed on the server.

  • Copy - Click this to make a copy of a selected form. This will copy all the information of the selected form, including the template (Word) document. You will need to give the copy a new code, and this code will be used as the default name of the template document. The copy function is a good way to create a new form if the new form bears some similarities with the original, such as the same change triggers and generated fields, or if the template document has the same layout. See also Edit above.

  • Edit Document - Click this to open the template in your word processor.

  • Refresh - Click this to refresh the template after making any changes. You must do this or your template will not generate correctly. This will also warn you if there are any errors in the template. See Refresh/Option below for more information.

  • Field Wizard - Click this to launch the Field Wizard, which provides help on field codes that are linked to the form's change links. This command button is only activated when at least one change has been linked to the form.

  • Check In - The Check In button is only visible when users are connected via the Cloud Solution. The button is disabled when there are no templates being checked out locally in the client-side. When you open a template file, the Check In button will be enabled. Once completed, close the document and click Check In. The template will be uploaded to the server-side and the local copy in client-side will be deleted.

  • New Change - Click this to add a new change link to the form. This will open up the Change Link screen.

  • Edit Change - Click this to edit the change link details.

  • Delete Change - Click this to delete the selected change link.

  • Import - Click this to import Word templates.

    Word templates can be imported to Viewpoint when you have both the .doc/.docx and .def files.

    1. In the Import screen, browse to the folder containing the template files.

    2. Click Import - Load List.

    3. Tick the templates you want to import.

    4. Click Import.

  • Export - Click this to export the selected Word template.

    To export selected template:

    1. Once you click Export, browse to the folder where the exported files are to be stored.

    2. Click Export.

     

    To export more than one template:

    1. Once you click Export, browse to the folder where the exported files are to be stored.

    2. Click Select Category/Jurisdiction.

    3. Tick the category or jurisdiction.

    4. Click Select Document to see all templates in the selected category or jurisdiction.

    5. Tick the templates you want to export.

    6. Click Export.

  • Refresh/Option - Click this to perform verification on the template. You can select options in the Verifying Documents window.

    The verification can be performed for:

    • Selected Document [Code] - Select this to refresh and verify the selected template only. This is the same as using the Refresh button (see above).

    • Selected Category [Category] - Select this to refresh and verify all templates in the selected jurisdiction or category.

    • Selected Document Type [Library] - Select this to refresh and verify the selected library.

    • All Document Libraries - Select this to refresh and verify all templates in the Document Libraries.

     

    The type of verification is to be selected by ticking the relevant option(s) below:

    Verify Codes and SQL Statements

    Tick this to verify the codes and SQL statements. Errors are shown in red, and need to be resolved before the template will work properly. Warnings are shown in blue and may be ignored if they relate to formatting issues.

     

    Upgrade codes where applicable

    Generally, this will upgrade codes so that they are compatible either in Access or SQL. If you still encounter errors after this, you will need to manually check the SQL queries for the template. The two upgrades are:

    • Upgrade SQL statements - Tick this to upgrade SQL statements. Refer to Convert SQL Function.

    • Upgrade sub-forms to (+BLOCK...+) paragraphs - This is used to create block statements for forms that contain sub-forms. This feature is no longer used starting from ViewPoint version 6.

    Report obsolete sub-form codes as errors

    This will report sub-forms codes that are no longer in use as errors. You may have to check the form to remove any errors. This feature is no longer used starting from ViewPoint version 6.

     

    Add Document Management tags (as set with the forms type lookup value)

    Tick this to add the Document Management index tags to the Key Information field of the selected template(s). The tags are retrieved from the Form Type [FRMT] lookup value table for the document type. Adding these tags help to reduce the time required to store a generated document in Document Manager.

    If the template already contains the index tags, then the existing tags can be maintained or overwritten.

    • Overwrite existing Document Management tags - Tick this to overwrite any existing tags in the Key Information field.

     

    Add Document Management OCR tag in footer where none exists

    Tick this to add the Document Management OCR tag in the first page footer of the template. This tag is used by Document Manager to add a scanned document to an existing document holder.

     

    Generate using OpenXML Technology

    Tick this to use OpenXML technology for generation of Word template documents. The generation does not require Microsoft Office. However, to open the generated document, Microsoft Office 2007 or newer is required.

    When this option is ticked, if the template is in .doc format, this will do the following:

    1. Create and save a .docx copy of the template to the Forms folder.

    2. Change the Path & File Name to the .docx template.

    3. Add the tag [/GENMETHOD=OPENXML/] to Key Information.

    If the template is already in .docx format, then only the tag will be added to the Key Information.

    For templates containing conditional PAGEBREAK command e.g. (+PAGEBREAK=10+) or Microsoft Word field codes, using OpenXML technology will result in loss of functionality of the command and field codes.

    Note: For Viewpoint on the Cloud, this can only be performed on the server.

     

    Command buttons

    Click on the relevant command button to perform the necessary.

    • Start - Click this to start verifying documents.

    • Cancel - Click this to exit the screen.
