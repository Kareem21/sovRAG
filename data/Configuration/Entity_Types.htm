Entity Types

Customising the different entity types that you will use in your organization will result in more accurate and efficient data entry. This customisation includes defining the various entity types in a jurisdiction and the screens that will be used for that entity type.

 

Filters

The filters apply to the entity type list.

  • Type status - You can choose to view only active types, inactive ones or all.

  • Customised setting - You can choose to view types that have been customised or not.

 

Select/Apply button

This allows for additional criteria to apply to the entity type list.

 

Entity Type list

This list shows existing entity types that have been set up. By default, the entity types are grouped by the jurisdiction.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • New - Click this to create a new entity type.

    1. Click New.

    2. Enter the details for the entity type. Note that:

      • Ownership Structure - controls the options seen for the Ownership Structure field when creating a new share class. If the ownership structure starts with 'X', then it is possible to select from two ownership structures:

        • XA - User can select either SA or SB

        • XS - User can select either SA or SS

        • XB - User can select either SB or SN

        • XN - User can select either SS or SN

        • XC - User can select either SA, SB, SN or SS.

      • Tag Values - these are additional codes that can be added and apply only to the entity type. Refer to Entity Type - Tags for more information.

    3. Click Update.

     

     

     

  • Edit - Click this to edit an existing entity type.

  • Delete - Click this to delete an existing entity type.

  • Navigation - Click this to set up the screens for selected entity type.

    1. In the Entity Type Menu screen:

      • Tick the Allow This Menu checkbox to include this screen for the entity type.

      • Un-tick the Allow This Menu checkbox to remove this screen for the entity type.

    2. Click Update.

     

  • Type Mapping - Click this to map or change existing entity types to new ones.

    This is used to change or update entity types throughout Viewpoint. For example, if an existing entity type has been made obsolete or redundant, you can change all entity files of this type to another type by using this function.

    1. In the Entity Types Mapping/Changing screen, select the entity type to map or change.

    2. Select the new jurisdiction in the New Jurisdiction column.

    3. Select the new entity type in the New Entity Type column.

    4. Click Update.

    5. Click Close when complete.

     

  • Duplicate Custom Menu - Click this to copy the Custom Menu set up for the selected entity type and apply it to another entity type.

    If the selected entity type does not have any Custom Menu set up, the message 'Invalid menu configuration!' is displayed.

    For entity types with Custom Menu configured, the Duplicate Custom Menu screen is displayed.

    1. In the Duplicate Custom Menu, tick the entity type you want to apply the custom menu to.

    2. Click Update.

    If necessary, you can also edit the Custom Menu by double-clicking in the Custom Menu field.

 

I have enabled the screen for the entity type, but I cannot see the screen. What should I do?

Check to make sure that you have sufficient access to the screen in your User Group Roles. Find out more in User Roles.
