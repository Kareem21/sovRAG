Checklist/Workflow Step - Tags

These tags are to be entered in the Tag Values field of the checklist/workflow steps.

Purpose

Tag

Explanation

Updates User Fields with checklist item's value or score

 

Note: Updating User Fields from workflows will also perform the tags added in the User Field configuration.

[ON_NEW=UPDATE.UFIELD|KEY=<UFKey>|VALUE=<ItemKey>|FILECODE=<MasterFileItem>]

where:

  • <UFieldKey> refers to the key of the User Field.

  • <ItemKey> refers to the key of the checklist item.

  • <MasterFileItem> refers to the key of the checklist item that will specify the Master File. This is optional.

This will update the specified User Field with the checklist item's value or score once you start the checklist and save your input.

Example:

[ON_NEW=UPDATE.UFIELD|KEY=JERSK1|VALUE=RSK1|FILECODE=MFile]

will take the value from the checklist item 'RSK1' and update it to the User Field of the Master File specified in a checklist item with value ‘MFile’ with the key 'JERSK1'.

 

Note: If the <MasterFileItem> is not defined, the User Field will be updated for the workflow’s Master File by default.

 

[ON_NEW=UPDATE.UFIELD|KEY=JERSK1|VALUE=RSK1.SCORE]

will take the score for the checklist item 'RSK1' and update it to the User Field with the key 'JERSK1'.

 

Note: This does not apply to workflow steps, only File Note checklists.

[ON_EDIT=UPDATE.UFIELD|SOURCE=<StepKey>|KEY=<UFKey>|VALUE=<ItemKey>|VCODE=<ItemKey>|DATEFROM=<ItemKey>|DATETO=<ItemKey>|FILECODE=<MasterFileItem>]

where:

  • <StepKey> refers to the key of the workflow step.

  • <UFKey> refers to the key of the User Field.

  • <ItemKey> refers to the key of the checklist item.

  • <MasterFileItem> refers to the key of the checklist item where the value for the item is a Master File. This is optional.

Additional tag parameters are available to update the Lookup/User Table, Date From, and Date To fields upon step completion:

  • VCODE refers to the Lookup/User Table field available based on the Field Type selection. VCODE is only applicable for Master File/Dates/Lookup [82] and Master File/Dates, User TV [83] field types. Use VALUE for all other field types.

  • DATEFROM refers to the Date From field.

  • DATETO refers to the Date To field.

 

This will update the specified User Field with the checklist item's value or score when the workflow step is completed.

Example:

[ON_EDIT=UPDATE.UFIELD|SOURCE=VPWS002|KEY=JERSK1|VALUE=RSK1|VCODE=ADDRTYPE|DATEFROM=DA1|DATETO=DA2|FILECODE=MFile]

will take the value from the checklist item 'RSK1' in workflow step ‘VPEWS002’ and update it to the User Field 'JERSK1' of the Master File specified in a checklist item with value ‘MFile’. The value for ‘ADDRTYPE’ key updates the Address Type field. The values for Date From and Date To fields are taken from ‘DA1’ and ‘DA2’ values.

 

Update Matter Account User Fields with checklist item's value

 

Note: Updating User Fields from workflows will also perform the tags added in the User Field configuration.

[ON_NEW=UPDATE.UFIELD|KEY=<UFKey>|CATEGORY=<MatterCodeItem>|VALUE=<ItemKey>| FILECODE=<MasterFileItem>]

where:

  • <UFKey> refers to the key of the User Field.

  • <MatterCodeItem> refers to the key of the checklist item that will specify the Matter code. This is optional.

  • <ItemKey> refers to the key of the checklist item.

  • <MasterFileItem> refers to the key of the checklist item that will specify the Master File. This is optional.

 

This will update the specified Matter User Field with the checklist item's value or score once you start the checklist and save your input.

Example:

[ON_NEW=UPDATE.UFIELD|KEY=MUF01|CATEGORY=MC|VALUE=A|FILECODE=MFile]

will take the value from the checklist item 'A' and update it to the User Field of the Master File specified in a checklist item with value ‘MFile’ with the key 'MUF01' for the Matter specified in checklist item 'MC'.

 

Note: If the <MasterFileItem> is not defined, the User Field will be updated for the workflow’s Master by default.

 

[ON_NEW=UPDATE.UFIELD|KEY=MUF01|CATEGORY=MC|VALUE=A.SCORE]

will take the score for the checklist item 'A' and update it to the User Field with the key 'MUF01' for the Matter specified in checklist item 'MC'.

 

Note: This does not apply to workflow steps, only File Note checklists.

[ON_EDIT=UPDATE.UFIELD|SOURCE=<StepKey>|KEY=<UFKey>|CATEGORY=<MatterCodeItem>|VALUE=<ItemKey>|VCODE=<ItemKey>|DATEFROM=<ItemKey>|DATETO=<ItemKey>|FILECODE=<MasterFileItem>]

where:

  • <StepKey> refers to the key of the workflow step.

  • <UFKey> refers to the key of the User Field.

  • <MatterCodeItem> refers to the key of the checklist item that will specify the Matter code. This is optional.

  • <ItemKey> refers to the key of the checklist item.

  • <MasterFileItem> refers to the key of the checklist item where the value for the item is a Master File. This is optional.

Additional tag parameters are available to update the Lookup/User Table, Date From, and Date To fields upon step completion:

  • VCODE refers to the Lookup/User Table field available based on the Field Type selection. VCODE is only applicable for Master File/Dates/Lookup [82] and Master File/Dates, User TV [83] field types. Use VALUE for all other field types.

  • DATEFROM refers to the Date From field.

  • DATETO refers to the Date To field.

This will update the specified Matter User Field with the checklist item's value or score when the workflow step is completed.

Example:

[ON_EDIT=UPDATE.UFIELD|SOURCE=VPWS002|KEY=MUF01|CATEGORY=MC|VALUE=A|VCODE=ADDRTYPE|DATEFROM=DA1|DATETO=DA2|FILECODE=MFile]

will take the value from the checklist item 'A' from the workflow step ‘VPWS002’ and update it to the User Field ‘MUF01’ of the Master File specified in a checklist item with value ‘MFile’ for the Matter specified in checklist item 'MC'. The value for ‘ADDRTYPE’ key updates the Address Type field. The values for Date From and Date To fields are taken from ‘DA1’ and ‘DA2’ values.

 

Specify Rating Group to use

[SCORETAG=<Code>.<Type>]

where:

  • <Code> is the code for the rating group and can be obtained from Risk Ratings screen or the Risk Rating Groups [RRGR] lookup table.

  • <Type> can be:

    • C - Current

    • R - Prepare Revision

    • S - Scenario Evaluation

Note: <Type> is optional. If the <Type> is not specified, the default is Current [C].

This tag can be added in the following Tag Values field in:

  • ‘Edit Step Properties’ screen

  • 'Lookup Value' field type

  • 'User Table Value' field type

  • 'Entity Type' field type

  • 'Service' field type

  • 'User Defined' field type

If the same tag is applied in different locations, the program will take precedence over the field types.  

This specifies the Rating Group to use for the checklist/workflow step's score assignment. It applies to field types 'User Table' and 'Lookup Table'.

Example:

[SCORETAG=IMSS.C] will take the scores for Rating Group 'IMSS' for the type 'Current [C]'.

[SCORETAG=IMSS.R] will take the scores for Rating Group 'IMSS' for the type 'Prepare Revision [R]'.

[SCORETAG=IMSS.S] will take the scores for Rating Group 'IMSS' for the type 'Scenario Evaluation [S]'.

Refer to Checklist Item - Score Assignment and Risk Ratings for more information.

 

Note: For field types ‘Yes/No’, ‘List with Edit’ and ‘List without Edit’, scores can be assigned to the values using [WSCORE=] tag in the Values/Default column for the field type.

[SCORETAG =#<Key>]

where <Key> refers to the current step field key.

 

or

 

[SCORETAG=#WF.<Key>]

where <Key> refers to the previous step field key.

This allows multiple Risk Rating Groups to be configured in a checklist item as selection.

The score assignment can then be referred to the Risk Rating Group selected.

Example:

[SCORETAG=#RRAGR]

Risk Rating Group is selected according to the value of current step field key ‘RRARG’.

or

[SCORETAG=#WF.RRAGR]

Risk Rating Group is selected according to the value of previous step field key ‘RRARG’.  

Allow scores to be displayed

[SHOWSCORE=<Value>]

where <Value> can be:

  • F - To load and show scores assigned only if the checklist/workflow step contains a formula.

  • S - To load and show scores assigned even if the checklist/workflow step does not contain any formula.

  • M - To allow score to be seen only in the Score column.

  • H - To hide all scores assigned and disable the Show Score option.

This is to control if user can view the scores assigned when they are completing the a checklist/workflow step.

If the score is not relevant for the checklist/workflow step; or the score is sensitive and should not be visible to the user, then it can be hidden using this tag.

Allow Score column to be displayed

[SCORECOLUMN=Y]

This is to display the Score column in the workflow. By default, the column is hidden and user can also right-click on the step and choose Show Score in the workflow.

Define the SQL query for the checklist/workflow step

[/<QueryName>=<Query>/]

where:

  • <QueryName> is a name for the SQL and this must be referenced for checklist items that will be displaying the retrieved information.

  • <Query> is the SELECT query that will be retrieving the required information.

The query is a standard SQL SELECT query where you specify the fields to retrieve, the table/view and condition(s), if applicable.

When specifying the condition, you can use the following variables:

  • (-MF-) to refer to the Master File the checklist/workflow is for.

  • (-KEY:<Key>-) to refer to value for the specified key e.g. (-KEY:DR-) means refer to value for the checklist item with the key 'DR'.

  • (-AGROUP-) to refer to the user's Administrator Groups.

  • (-UCODE-) to refer to the user.

  • (-UGROUP-) to refer to the user's User Group.

  • (-FAGROUP-) to refer to the user's default Administrator Group.

  • (-WflowNr-) to refer to the workflow number. This is applicable only to workflow steps.

Note: The user is the user who started/reloaded the checklist or workflow step.

 

Example:

[/AOfficers=SELECT AddrCode, OfficerType FROM Officers WHERE EntCode='(-MF-)' AND Status='A' AND RelClass='O'/]

will retrieve active statutory officers of the Master File the checklist/workflow is for.

 

The checklist items to list the retrieved information must have the CCODE and COPY tags. Refer to Checklist Item - Tags.

Specify Data Source

[/<DataSource>=|0:<Field1>|1:<Field2>|2:<Table>|3:<Condition>|4:<AdditionalField>|5:<ActiveCondition>|/]

where:

  • <DataSource> is the name of the data source.

  • <Field1> refers to the field containing the code and is mandatory.

  • <Field2> refers to the field containing the name/description and is mandatory.

  • <Table> refers to the table or tables containing the fields for the Master File and is mandatory.

  • <Condition> refers to the condition for filtering the selection and is based on fields of the specified <Table>.

    • (-RECLINK.<GridCode>.<Column>-) can be used to retrieve the column value of the grid that is linked to the workflow.

  • <AdditionalField> refers to additional field to be displayed for the selection of the Master File and is optional.

  • <ActiveCondition> refers to the Master File status and is optional. If not specified, the list will only return active Master Files and users will not be able to untick the ‘Active only’ option. If specified, users can choose to tick or untick the ‘Active only’ option.

This is to define the data source and filter it accordingly for selection in the workflow and is for checklist items where field type is 'Master File' or 'User Defined'.

 

Example:

[/MT=|0:MtNr|1:MtDescription|2:q_Meetings|3:MeetCode='(-RECLINK.MTNGENT.MeetCode-)'|/]

The data source ‘MT’ will retrieve the value from the column ‘MeetCode’ of the ‘MTNGENT’ grid that is linked to the workflow.

 

Note: This tag is supported on workflows published to Viewpoint Engage for Master File field type.

Set the calculated score as output for decision step

[OUTPUT=<Key>.SCORE]

This defines the score for the checklist items with the specified key as the source for upcoming decision step. This way, users do not need to manually select the Yes or No branch. Refer to Decision Step.

This tag works for calculated checklist item i.e. where the checklist item's field type is 'Formula'.

 

Note: This tag is supported on workflows published to Viewpoint Engage.

Set the value as output for decision step

[OUTPUT=<Key>.VALUE] or [OUTPUT=<Key>]

where <Key> refers to the checklist item's key.

This defines the value for the checklist items with the specified key as the source for upcoming decision step. This way, users do not need to manually select the Yes or No branch. Refer to Decision Step.

This tag works for calculated checklist item i.e. where the checklist item's field type is 'Formula'.

 

Note: This tag is supported on workflows published to Viewpoint Engage.

Hide the checklist item

[ROW.Hide=<Key>]

where <Key> refers to the checklist item's key.

 

This will not display the checklist item in the workflow step.

 

Note: This tag is supported on workflows published to Viewpoint Engage.

Update workflow step caption with information from step item

[Update.Step_Descr=<NewCaption>]

where <NewCaption> should include the Key of a checklist item along with formatting option like:

  • VALSHOW – For values that contains code and description/name such as a Master File, VALSHOW will display the Master File name; otherwise, only the code is displayed.

  • SCORE – This will show the score of the checklist item.

This applies only to workflows. It will update caption of the step with information from a checklist item in the workflow step. The change is made once the checklist item contains a value that has been saved.

 

Example:

[Update.Step_Descr= Appointment of (+OMF.VALSHOW+)]

will update the caption for the step to include the description/name of the value for the checklist item with the key 'OMF'. If the field type is Master File, adding VALSHOW will display the Master File name.

 

Note: This tag is supported on workflows published to Viewpoint Engage.

Update workflow description with information from step item

[Update.Header_Descr=<NewDescr>]

where <NewDescr> should include the Key of a checklist item along with formatting option like:

  • VALSHOW – For values that contains code and description/name such as a Master File, VALSHOW will display the Master File name; otherwise, only the code is displayed

  • SCORE – This will show the score of the checklist item.

This applies only to workflows. It will update the workflow with information from a checklist item in the workflow step. The change is made once the checklist item contains a value that has been saved.

Example:

[Update.Header_Descr= Appointment of (+OMF.VALSHOW+)]

will update the description of the workflow to include the description/name of the value for the checklist item with the key 'OMF'. If the field type is Master File, adding VALSHOW will display the Master File name.

 

Note: This tag is supported on workflows published to Viewpoint Engage.

Update workflow header administrator and workflow step administrator with information from step item

To update workflow header administrator:

[Update.Header_AdminCode=(+<FLDKEY>+)]

 

To update workflow step administrator:

[Update.Step_AdminCode=(+<FLDKEY>+)]

 

where <FLDKEY> refers to the key of the checklist item.

This applies only to workflows. It will update the administrator of the workflow header or step with information from a checklist item in the workflow step. The change is made upon saving the workflow.

 

Note: This tag is supported on workflows published to Viewpoint Engage.

Show an item if condition(s) is/are met

[ROW.Condition.<Key>=<Condition>]

where:

  • <Key> - refers to the Key of the item that is to be hidden/shown

  • <Condition> - refers to the condition or conditions and must include the Key of the checklist item containing the value to be used for evaluation against the condition(s). For multiple conditions, specify this using ‘AND’ or ‘OR’.

This applies only to workflows. It is to control whether or not an item is seen in the workflow based on specified condition(s).

Note: This tag is supported on workflows published to Viewpoint Engage.

Create invoice through workflow

 

[ON_SUBMIT=CREATEINV]

This tag is mandatory.

This is to be included to ensure an invoice is created when the workflow is completed.

Note: This tag is supported on workflows published to Viewpoint Engage.

Specify invoice details

 

[CREATEINV.SERVCODE=<Key>]

where <Key> refers to the key of the checklist item.

This tag is mandatory if the service line(s) for the invoice is to be specified by checklist items of the field type ‘Lookup Table’, ‘User Table Value’ or ‘Yes/No’.

If only one service line is required and this is specified by a checklist item of the field type ‘Service’, then this tag is not required.

This is to define the service lines based on checklist items configured with the specified keys.

Example:

[CREATEINV.SERVCODE=S1,S2,S3]

will create an invoice where the service lines will be specified by checklist items with the keys S1, S2 and S3.

Note: This tag is supported on workflows published to Viewpoint Engage.

[CREATEINV.MatterType=<Value>]

where <Value> is the matter code from the Matter Type Code [MTTR] lookup table.

This tag is optional.

This is to define the matter type for the invoice.

Example:

[CREATEINV.MatterType=AC]

will filter the matter type according to ‘AC’ (Accounting).

Note: This tag is supported on workflows published to Viewpoint Engage.

[CREATEINV.PostAsFinal=<Value>]

where <Value> can be:

  • Y to post the created invoice.

  • N to retain as draft invoice.

This tag is optional.

This is to specify if the invoice created will be a draft invoice or posted as final invoice.

If not specified, the default is N.

Example:

[CREATEINV.PostAsFinal=Y]

will create and post the invoice.

Note: This tag is supported on workflows published to Viewpoint Engage.

[CREATEINV.Reference=(+<Key>+)]

where <Key> refers to the key of the checklist item.

This tag is optional.

This is to retrieve value from a checklist item for the Reference field of the invoice.

Example:

[CREATEINV.Reference=(+REFE+)]

will insert the value for checklist item with the key ‘REFE’ as the invoice reference.

Note: This tag is supported on workflows published to Viewpoint Engage.

[CREATEINV.HeaderNote=(+<Key>+)]

where <Key> refers to the key of the checklist item.

This tag is optional.

This is to retrieve value from a checklist item for the Header Note field of the invoice.

Example:

[CREATEINV.HeaderNote=(+HEAD+)]

will insert the value for checklist item with the key ‘HEAD’ as the invoice header note.

Note: This tag is supported on workflows published to Viewpoint Engage.

[CREATEINV.FooterNote=(+<Key>+)]

where <Key> refers to the key of the checklist item.

This tag is optional.

This is to retrieve value from a checklist item for the Footer Note field of the invoice.

Example:

[CREATEINV.FooterNote=(+FOOT+)]

will insert the value for checklist item with the key ‘FOOT’ as the invoice footer note.

Note: This tag is supported on workflows published to Viewpoint Engage.

[CREATEINV.Note=(+<Key>+)]

where <Key> refers to the key of the checklist item.

This tag is optional.

This is to retrieve value from a checklist item for the Note field of the invoice.

Example:

[CREATEINV.Note=(+NOTE+)]

will insert the value for checklist item with the key ‘NOTE’ as the invoice note.

Note: This tag is supported on workflows published to Viewpoint Engage.

Prompt the eView page upon creating an invoice via the Workflow in Viewpoint Engage

[CSP_SUBMIT=SHOWEVIEW]

This is to prompt the eView page upon creating an invoice via the Workflow.

[SHOWEVIEW.EVIEW=<Template Code>]

This defines the eView template to be used to display the posted invoice details.

Note: The eView template for Viewpoint Engageis maintained in the Back-office Configuration > Document Libraries > Engage Servicing Portal Templates.

Example:

[SHOWEVIEW.EVIEW=CPSERVC]

will use the eView template ‘CPSERVC’ to display the posted invoice details.

[SHOWEVIEW.LINKCODE=<Link Code>]

This defines the link code (grid code in Back-office Viewpoint Connect) where the invoice details are retrieved. It is the code for the grid containing the record that you wish to link the eView to.

Example:

[SHOWEVIEW.LINKCODE=RCSALESU]

will display the invoice details as retrieved from the link code 'RCSALESU'.

Automate the scores and formulas recalculation for Viewpoint Engage Workflow

[ON_SUBMIT=DORATING]

This will run the 'Refresh Rating' function in the Back-office Workflow to recalculate the scores and formulas once user clicks the 'Submit' button in the Viewpoint Engage Workflow step.

Transfer linked items to new Master file

[ON_SUBMIT=TRANSFERMF]

This is to be included to ensure that relevant linked items are transferred from the parent workflow to the new Master File created by the linked or child workflow when the workflow is completed.

Specify Master File and linked items to be transferred

[TRANSFERMF.MFKEY=<Key>]

where <Key> refers to the key of the checklist item that will specify the Master File.

This is to define the Master File where the linked items will be transferred to.

Example:

[TRANSFERMF.MFKEY=MFile]

will transfer the linked items to a new Master File where the value is specified in a checklist item with key ‘MFile’.

[TRANSFERMF.TYPE=<Code>]

where <Code> can be:

  • BI – Binder

  • DM – Task

  • DH – Document holder

  • TM – Time entry

  • DI – Disbursement entry

  • WM – Main workflow

To include multiple codes, separate each code with a comma symbol. If <Code> is not specified, all linked items will be transferred.

Note: For TM and DI, the linked items will not be transferred if the entry has been posted as draft invoice or if the Sales Ledger module is not active.

 

This is to define the relevant linked items that will be transferred to the new Master File.

Example:

[TRANSFERMF.TYPE=DH,TM,DI]

will transfer linked items including document holder, time entry and disbursement entry to the new Master File.

Update linked Filings, Meetings and Reviews record upon complete and undo step

[ON_SUBMIT=UPDATE.<Macro1>|UPDATE.<Macro2>]

where each macro has to be defined separately:

[<Macro1>.DEF=<Screen>|<Button>]

[<Macro1>.FLD=<Column1>=(+KEY+)|<Column2>=’X’]

[<Macro1>.Condition= <SQL Condition>]

[<Macro1>.UpdateLimit=X]

 

[<Macro2>.DEF=<Screen>|<Button>]

[<Macro2>.FLD=<Column1>=(+KEY+)|<Column2>=’X’]

[<Macro2>.Condition= <SQL Condition>]

[<Macro2>.UpdateLimit=X]

 

Note:

  • The macro code tag [<Macro>.DEF=<Screen>|<Button>] is used to replace the existing tag [<Macro>=<Screen>|<Button>].

  • It is mandatory to define <Macro>.DEF, <Macro>.FLD and <Macro>.Condition for each macro.

  • If <Macro>.UpdateLimit is not defined, it will update only one record whereas if <Macro>.UpdateLimit=0, it will update all records.

This will update the linked records when user completed a step.

Example:

[ON_SUBMIT=UPDATE.FILEACT|UPDATE.FILING|]

will update the linked record in FILEACT macro, followed by the FILING macro.

[ON_UNDO=UPDATE.<Macro1>|UPDATE.<Macro2>]

where each macro has to be defined separately:

[<Macro1>.DEF=<Screen>|<Button>]

[<Macro1>.FLD=<Column1>=(+KEY+)|<Column2>=’X’]

[<Macro1>.Condition= <SQL Condition>]

[<Macro1>.UpdateLimit=X]

 

[<Macro2>.DEF=<Screen>|<Button>]

[<Macro2>.FLD=<Column1>=(+KEY+)|<Column2>=’X’]

[<Macro2>.Condition= <SQL Condition>]

[<Macro2>.UpdateLimit=X]

 

Note:

  • The macro code tag [<Macro>.DEF=<Screen>|<Button>] is used to replace the existing tag [<Macro>=<Screen>|<Button>].

  • It is mandatory to define <Macro>.DEF, <Macro>.FLD and <Macro>.Condition for each macro.

  • If <Macro>.UpdateLimit is not defined, it will update only one record whereas if <Macro>.UpdateLimit=0, it will update all records.

This will update the linked records when user undo a step.

Example:

[ON_UNDO=UPDATE.FILEACT2|UPDATE.FILING2]

will update the linked record in FILEACT2 macro, followed by the FILING2 macro.
To filter new Master File properties according to a desired criteria instead of displaying the properties of the Master File which is used to launch the workflow. This is applicable for checklist items under parent item with ‘Input Field’ field type.

[/InputField.MFPROP=MFPROPERTIES/]

(for all “Input Field”)

 

OR

[/InputField.MFPROP=MFPROPERTIES|<KEY>/]  

(for specific “Input Field”)

 

where <KEY> refers to the key for ‘Input Field’ field type

 

AND

[/MFPROPERTIES=MF.RefType=(+<KEY>+)|EN.IncorpPlace=(+<KEY>+)|EN.EntType=(+<KEY>+)/]

 

where (+<KEY>+) refers to the key for related checklist items in the workflow.

This will filter the selection of properties of new Master File based on Master File Type, Jurisdiction and Entity Type.

 

Note: Ensure to configure the Tag Values in either of the following as well.

  • Configuration > Business Parameters > Entity Administration > Entity Types
  • Configuration > Business Parameters > General > Lookup Values (ADTP)

Example of tag:

[FILTER:SOF.TYPE=DIR]

This will display ‘DIR’ (Director) only in the selection of Officer Type checklist item for the selected Entity Type or Master File Type.

 

Example:

To filter the type of Officer Type to only ‘Director’ and ‘Secretary’ for new Incorporation, configure the tag values for step properties of the checklist item with ‘Input Field’ field type as follows:

[/InputField.MFPROP=MFPROPERTIES|DUPSO/]

[/MFPROPERTIES=

MF.RefType=(+RefType+)|

EN.IncorpPlace=(+IncorpPlace+)|

EN.EntType=(+EntType+)/]

 

Configure [FILTER:SOF.TYPE=DIR,SEC] in the related Entity Type and/or Master File Type.

 

In the workflow, users will only see ‘DIR’ and ‘SEC’ as selection for Officer Type after they select the related Entity Type and Incorporation Place.
Support multiple field keys configuration in one Tag Values field.

(+<Key1>##<Key2>+)

where <Key> refers to the key for related checklist items in the workflow.

The syntax ‘##’ allows multiple keys to be configured in one Tag Values field or in one Macro tag. By using this format, the first key that contains a value will be retrieved as the default value or when users execute updates through Macro.

Example:

AlertNote=(+AlertNoteO##AlertNoteL+)

If ‘AlertNoteO’ contains value, it will be used. If it does not contain value, the value for ‘AlertNoteL’ will be used instead.
Retrieve values as default from a specified workflow.
 

Note:

The values retrieved will appear as read-only.

[ON_ LOAD=COPYWFTOCLIPBOARD]

 

[COPYWFTOCLIPBOARD.WFTK=<TemplateKey>]

where <TemplateKey> refers to the Workflow Template Key.

 

OR

[COPYWFTOCLIPBOARD.WFTN=<FieldKey>] where <FieldKey> refers to the key of the checklist item with Workflow Link Field Type.

 

[COPYWFTOCLIPBOARD.WFSK=<StepKey>] where <Step key> refers to Workflow Step key.

This tag is optional.

Aside for the tags, configure "#" in ‘Clipboard Mapping’. The checklist item will retrieve value from the exact same key in the clipboard field. To refer values from Duplicate Group, use ‘+<Key>’.

 

When the workflow template is open for the first time, fill in and complete the workflow as per normal practice. The next time you use the same workflow template, all the values with ‘Clipboard Mapping’ configured will be auto filled.
To configure an item with Workflow Link Field Type so that it can only link with an existing workflow.

[WF.ExistingHistory=Y]

The linked workflow will not be deleted if the main workflow is deleted as it only serves as reference.
