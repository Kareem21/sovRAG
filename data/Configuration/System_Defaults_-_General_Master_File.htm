System Defaults - General/Master File

The defaults in this screen apply to the environment and Master Files.

 

Coding Method for Master File

Specify the coding method for new Master Files from the system standards. Viewpoint recommends that you carefully consider which coding system you will use before you start using the system because an unsuitable coding system can have serious “knock-on” effects which are time consuming to solve. To avoid problems at a later date, treat your database as you would treat a new filing cabinet: instead of throwing everything in without labelling the sections, choose a coding system which will maximize the efficiency of the system no matter how many files you put in. Note that the code accepts alphanumeric characters (A-Z and 0-9) only except if you enable the option 'Allow Master File Coding in Special Characters'.

  • Alpha 4 Characters Automatic [1] - Automatically assigned four-character code that is derived from the Master File name. For example: CASE.

  • Alpha 6 + Sequence + MF Type [3] - Suggested code made up of six characters derived from the name followed by three numeric characters beginning with 001 and ending with the Master File type. This code can be amended as long as the final code is ten characters long. For example: CASEYL001C.

  • Alpha 6 Characters Automatic [2] - Automatically assigned six-character code that is derived from the Master File name. For example: CASEYL

  • Automatic - Separate Range for EA/BF [14] - Automatically assigned code made up of different prefixes and sequential number for managed, non-managed entities that are billing files and non-managed entities that are not billing files. The prefixes and sequential number are determined by the setup of the Entity File Codes [ADE], Billing File Codes [ADB] and Master File Codes [ADR] in Configuration > Settings > Customise > System Running Numbers.

  • Automatic - Separate Range for Entities [10] - Automatically assigned code made up of different prefixes and sequential number for managed and non-managed entities. The prefixes and sequential number are determined by the setup of the Entity File Codes [ADE] and Master File Codes [ADR] in Configuration > Settings > Customise > System Running Numbers.

  • Manual - 10 Characters [8] - Suggested ten-character code derived from the Master File name. Amend this if required.

  • Manual - 4 Characters [5] - Suggested four-character code derived from the Master File name. Amend this if required.

  • Manual - 6 Characters [6] - Suggested six-character code derived from the Master File name. Amend this if required.

  • Manual - 8 Characters [7] - Suggested eight-character code derived from the Master File name. Amend this if required.

  • Manual - Free Number of Characters [9] - Suggested code derived from the Master File name and may range from two to ten characters. Amend this if required.

  • Numeric Automatic [0] - Automatically assigned code determined by the setup of the Master File Codes [ADR] in the System Running Numbers section.

  • Numeric Manual [11] - Suggested code determined by the setup of the Master File Codes [ADR] in the System Running Numbers section. Amend this if required.

  • User Office + Numeric Automatic[12] - Suggested code is made up of:

    • the code of the User Office assigned to the user (in the user account) creating the Master File; and

    • number value of the Master File Codes [ADR] in the System Running Numbers section

 

Status - New Master File

Select the default status for new Master Files. This is useful if your procedures require that new Master Files be approved, or set to Active, by a Manager. Create your Master File status in the Master File Status [RECS] lookup value table.

 

Omit Country with Address

Use this function if you want to omit the country in an address. For example, if you are in Isle of Man and do not require Isle of Man addresses to display 'Isle of Man' in forms, then enter Isle of Man [IM] in this field.

 

Default Language

Select the default language for your system. This sets the default invoice language to be used for clients and the default language in Document Templates.

 

Environment Name

This field is not editable. User is required to edit Environment Name from ViewPointDbMgt.exe. In System Settings, it is stored under VPCROSS application - ENVIND key.

 

Environment Colour Code

Enter the colour code for the Viewpoint environment. The colour applies to the background of the Section Menu and Breadcrumb. You can click on the Browse button to pick the colour.

To select a colour, you can click on the colour listed in Basic colors section and click OK.

To select a custom colour, add the colour first to Custom colors section:

  1. Click Define Custom Colors >>.

  2. Click to select the colour.

  3. Click Add to Custom Colors.

In System Settings, it is stored under VPCROSS application - ENVINC key.

 

Maximum Master File Search Returns

The value here limits the number of records for Master File search. By default, the value is 'Nil' which means no limit. In System Settings, it is stored under VPCROSS application - MAXSRH key.

 

Display Search Result

When performing a Master File search, if the results exceeds the value set in 'Maximum Master File Search Returns', nothing will be displayed. If you tick this option, then it will display the maximum number of records for the search with a message informing you that the search results exceeds the maximum limit set.

 

Maximum Grid Records

The value here limits the number of records to be loaded to grids throughout Viewpoint. The default is 5000 records but this can be changed according to your needs. Set the value to 0 to disable the limit.

Note: Loading of records will take time and processing power.

 

Add-ins/Local Agent Port Range

The value here limits the TCP Binding Port number range for Add-ins and Local Agent connection to Viewpoint. Enter the port range in the format of x-y, where x is the minimum port number and y is the maximum port number. The port range has to be within 1025 to 65535. Viewpoint will only establish the connection based on the specified port range. The port number for Add-ins and Local Agent will be randomised with each receiving a different number.

The entered value will be stored in the System Settings, under VPCROSS application - PORTRG key.

 

Navigation Menu Access

The options below relate to the navigation menu.

 

Hide Master File Relations for Entities

Tick this to hide the Relations screen in the Master File menu if the selected Master File is an Entity.

 

Hide Compliance Register for Non-Entities

Tick this to hide the Compliance Register in the Documents menu if the selected Master File is not an administered entity (i.e. Entity Administration is not ticked in Master File - Setup).

 

Hide Safe Custody Register for Non-Entities

Tick this to hide the Safe Custody Register in the Documents menu if the selected Master File is not an administered entity (i.e. Entity Administration is not ticked in Master File - Setup).

 

Hide 'Master File - New' Menu in Home Screen

Tick this to hide the New option from the Section Menu in Viewpoint - Home screen.

 

Display v7 Section Menu

Tick this to have the system remember the pin status of the Section Menu in Home and File level.  

 

Generation of Documents

The options in this section relate to documents that are generated from Viewpoint.

 

Surname in UPPER Case

Tick this if you want the last name of a person to appear always in upper case. For example: Benjamin ARCHER.

 

Archive by Master File

This is used when you do not use Document Management but want to archive Viewpoint forms by Master File. When ticked, the generated forms will be stored in individual files in \\server\ViewPoint\Archive\MFILES. Users will be able to view the stored documents by selecting Filing Cabinets in the right-click mouse menu or clicking Documents from the Ribbon in Viewpoint - Home and File Maintenance screens. For more information, refer to Filing Cabinet for Master File.

If you add the Document Management component, untick this option and tick the 'Integrate DM for generated documents' in System Defaults > Document Management. This will give you the Document Management option in your right-click mouse menu and change the location for generated forms to \\server\ViewPoint\Archives. To be able to access previously-generated forms in \\server\ViewPoint\Archive\MFILES, the files need to be moved to \\server\ViewPoint\Archives and renamed. Please contact your Viewpoint support for assistance on this.

Note: When the user has converted the environment to the Cloud Solution, system will disable the 'Archive by Master File' option due to the generated forms and letters in the Cloud Solution will be saved into the DocStore database.

The user can retrieve the previous saved forms and letters from the server \Archive\MFILES folder.

User with Document Manager licence are recommended to use the Document Manager Import Tool to move the generated Forms/Letters from Archive\MFFILES folders to Document Store after the Document Store creation.

 

Auto Check In Document After Generation in Form Queue

When this option is enabled, after generation of a new document in Form Queue, the document will be automatically checked in. This feature is applicable when Viewpoint Connect is connected via Application Server.

 

Other Options

Identity Details Checked for Officer/Share/UBO

Tick this to have Compliance Information option checked when the Relationship or Shareholder role is selected. This means the Compliance folder is activated when you create a new Master File, and you have to complete the information in the screen before you can complete the creation of the file.

 

Allow Individual Master Files as 'Bank'

Tick to allow individuals to act as Banks, and be selected as a Bank in the Bank Accounts screen.

 

Allow 'Entity Administration' for Individual Master Files

Tick to allow individuals to act as Entities. This would be done only if you need to administer to your clients' needs as if they were entities.

 

Warn on Selection of Master File with Status > 0

Tick if you want to be warned if you select Master Files with status other than Active [0].

 

Show 'Obligations' in Tools

Tick this to enable Obligations tab in Tools. This will also include Obligations in the Dashboard list.

 

Synchronise Administrator/Manager/Controller for Entity/Billing

Tick this if you want the Administrator, Manager and Controller fields for the Billing File to be the same as the Entity File. With this option ticked, when you change the selection for the Entity File, the change will apply to the Billing File and vice versa. The synchronisation will also apply to Annual Fees > File Setup.

Note: If tags are used to update user relation to these fields, then the tags take priority over this setting.

 

Enable 'Help Text'

Tick this to turn on the Help Text feature which will display explanatory text for Viewpoint standard fields. The explanatory text is to be entered through Settings > Customise > Field Settings.

 

Enable Notifications

Tick this to enable the Notifications feature. Once this is selected, restart Viewpoint Connect to see the Notifications button in Viewpoint ribbon.

 

Enable Detailed 'Relationship Roles'

Tick this if you want to enable detailed Relationship Roles. When ticked, a Master File can only be appointed to roles that he has been allowed to hold.

 

Automatic Update of City/Region Tables

Tick this if you want to automatically update the City/Region tables when you create new addresses.

 

Include Master File 'User Text' in Search Name

Tick this to include the value entered in the UserTextMR field in the Master File - General screen in the Search Name. For example, if ARCH is added to the field for Archive Limited, the entity's name will be displayed as Archive Limited - ARCH. When you implement this for the first time, you need to rebuild the Master file Labels in Configuration > Database Archiving > Troubleshooting.

 

Enable 'Local Address Cards'

Tick this to enable the local address card feature.

 

Show 'My Reminders' in Tools

Tick this to enable My Reminders tab in Tools. If this is not ticked, the tab will not be visible.

 

Use Accounting Exchange Rates

Tick this to use exchange rates in Exchange Rates (CA) throughout Viewpoint. Otherwise, different exchange rates will be used - the Exchange Rates (AD/PM) in Entity Administration and Practice Management, Exchange Rates (CA) in Accounting.

 

Allow Master File Coding in Special Characters

Tick this if you would like the Master File code to include special characters (ASCII characters 33 to 126) except asterisk (*), question mark (?), quotation mark (') and pipe (|).

 

Auto-Save Workflow Changes

Tick this if you want to enable the auto-save function in workflow. This will turn auto-save on by default.

 

Command buttons:

Click on the relevant command button to perform the necessary.

  • Update - Click this to save changes.

  • Cancel - Click this to cancel the changes.
