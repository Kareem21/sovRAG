Data Validation Review - Tags

For Data Validation reviews, tags are available to specify checking on standard Viewpoint fields and User Fields. These tags are to be entered in the Tag Values field of the relevant Data Validation review rule which must be active. For example, if the field is in Master File - Setup screen, then the tag to check on this field is to be defined in Master File > Setup - Data Validation [R_FMMSETUP] review rule.

 

Checking on Standard Viewpoint Fields

Standard Viewpoint fields refer to fields available in Viewpoint Connect program. The tag allows you to check on whether the field is completed and if the field value is a specific value.

Purpose

Syntax

Example and Explanation

Specify whether the field is required to be completed.

[FV.<DBField>.Required=Y]

 

where <DBField> is the Database Field name for the field.

 

[FV.KYC_Officer.Required=Y]

This tag will check on CDD Officer field. If the field is empty, it will show up in the File Review results.

 

Note: The tag is to be defined in the Master File > Compliance > CDD - Data Validation [R_FMMKYCVV] review rule as the field is in the CDD screen.

Compare the field value against specified value(s).

[FV.<DBField>.ValueList=<Value>]

 

where:

  • <DBField> is the Database Field name for the field.

  • <Value> is the acceptable value or values. For multiple values, separate each value with a comma.

[FV.KYC_Officer.ValueList=DD]

This tag will compare the CDD Officer field with the specified value of 'DD'. If the value is not 'DD', it will show up in the File Review results.

 

[FV.KYC_Officer.ValueList=DD,ET]

This tag will compare the CDD Officer field with the specified values of 'DD' and 'ET'. If the value is not 'DD' or 'ET', it will show up in the File Review results.

 

Note: The tag is to be defined in the Master File > Compliance > CDD - Data Validation [R_FMMKYCVV] review rule as the field is in the CDD screen.

Compare date fields with specified date.

[FV.<DBField>.DateValue=<Value>]

 

where:

  • <DBField> is the Database Field name for the field.

  • <Value> can be:

  • Actual date in the format of dd/MM/yyyy.

  • #NOW for today's date.

  • referring to a date field such as @EN.IncorpDate for the incorporation date

 The '=' can also be replaced by:

  • > to check that the date is after the specified date.

  • < to check that the date is before the specified date.

  • >= to check that the date is on or after the specified date.

  • <= to check that the date is on or before the specified date.

[FV.KYC_Obtained.DateValue>=#NOW]

This tag will check the Date CDD Status and compare with today's date. If the value is before today's date, it will show up in the File Review results.

 

[FV.KYC_Obtained.DateValue>=@EN.IncorpDate]

This tag will check the Date CDD Status and compare with the date of incorporation. If the value is before the incorporation date, it will show up in the File Review results.

 

Note: The tag is to be defined in the Master File > Compliance > CDD - Data Validation [R_FMMKYCVV] review rule as the field is in the CDD screen.

 

Note: To check on the field 'Surname' or 'Entity Name', the tag must be entered in the Master File > Setup - Data Validation [R_FMMSETUP] review rule.

 

Checking on User Fields

User Fields in Viewpoint are found in the User Field screens as well as in separate pop-up window accessed through screens like Shares > Capital. Tags are also available to check if the user field is completed and if the value is a specific value.

For user fields in User Field screens, the tags are as follows:

Purpose

Syntax

Example and Explanation

Specify if the user field is required to be completed.

[FV.<FieldKey>.<DBField>.Required=Y]

 

where

  • <FieldKey> refers to the key of the user field.

  • <DBField> can be

  • FldValue refers to the value of the user field

  • FldNote refers to the Note part of the user field

  • FldVCode refers to the Lookup Table or User Table value when the field type is 'Master File/Dates/Lookup [82]' or  'Master File/Dates/User TV [83]'

  • DateFrom refers to the Date - From part of the user field

  • DateTo refers to the Date - To part of the user field

[FV.REA_ID.FldValue.Required=Y]

This tag will check on the user field with the key 'REA_ID'.  If the field is empty, it will show up in the File Review results.

 

Note: The tag is to be defined in the Master File > General > User Fields - Master File - Data Validation [R_FMMUSRFM] review rule as the user field is in the User Fields - Master File screen.

Compare the value of the user field against specified value(s).

[FV.<FieldKey>.<DBField>.ValueList=<Value>]

 

where

  • <FieldKey> refers to the key of the user field.

  • <DBField> can be:

  • FldValue refers to the value of the user field.

  • FldNote refers to the Note part of the user field

  • FldVCode refers to the Lookup Table or User Table value when the field type is 'Master File/Dates/Lookup [82]' or 'Master File/Dates/User TV [83]'.

  • DateFrom refers to the Date - From part of the user field.

  • DateTo refers to the Date - To part of the user field.

  • <Value> is the acceptable value or values. For multiple values, separate each value with a comma.

[FV.REA_ID.FldValue.ValueList=10004]

This tag will compare the value of the user field with the key 'REA_ID' against the specified value of '10004'. If the value is not '10004', it will show up in the File Review results.

 

Note: The tag is to be defined in the Master File > General > User Fields - Master File - Data Validation [R_FMMUSRFM] review rule as the user field is in the User Fields - Master File screen.

Compare  date value for user field against the specified date.

[FV.<FieldKey>.<DBField>.DateValue=<Value>]

where

  • <FieldKey> refers to the key of the user field.

  • <DBField> is:

  • FldValue when the field type is Date [12].

  • DateFrom or DateTo when the field type is Lookup Value/Dates [70], Master File/Dates [81], Master File/Dates/Lookup [82], Master File/Dates, User TV [83]or User Table Value/Dates [71].

  • <Value> can be:

  • Actual date in the format of dd/MM/yyyy.

  • #NOW for today's date.

  • #INCORP for the incorporation date

 The '=' can also be replaced by:

  • > to check that the date is after the specified date.

  • < to check that the date is before the specified date.

  • >= to check that the date is on or after the specified date.

  • <= to check that the date is on or before the specified date.

[FV.KYC202.FldValue.DateValue<=#NOW]

This tag will check the user field with the key 'KYC202' and compare the date with today's date. The field type for KYC202 is date [12]. If the value is after today's date, it will show up in the File Review results.

 

[FV.KYC303.DateFrom.DateValue<=#NOW]

[FV.KYC303.DateTo.DateValue>#NOW]

These tags will check the user field with the key 'KYC303' and compare the Date - From and Date - To with today's date. The field type for KYC303 is Master File/Dates [81]. If DateFrom is after today's date and/or DateTo is on or before today's date, it will show up in the File Review results.

 

Note:  The tag is to be defined in the Master File > General > User Fields - Master File - Data Validation [R_FMMUSRFM] review rule as the user field is  in the User Fields - Master File screen.

 

For user fields that are in separate pop-up window, you need to also add the <Category> so that the syntax are as follows:

  • [FV.<FieldKey>.<Category>.<DBField>.Required=<Value>]

  • [FV.<FieldKey>.<Category>.<DBField>.ValueList=<Value>]

  • [FV.<FieldKey>.<Category>.<DBField>.DateValue=<Value>]

The <Category> depends on the location of the user field.

Category

Location of User Fields

ShareClass

  • File Maintenance > Entity Administration > Shares > Capital.

  • File Maintenance > Entity Administration > Partners > Classes.

Reference

  • File Maintenance > Entity Administration > Assets > Fixed Assets.

MatterCode

  • File Maintenance > Sales Ledger > Billing File > Matters.

PortId

  • File Maintenance > Accounting > Account File > Portfolios.

AcctKey

  • File Maintenance > Accounting > Journals > Debtor.

  • File Maintenance > Accounting > Journals > Creditor.

  • File Maintenance > Accounting > Journals > Bank Account.

  • File Maintenance > Accounting > Journals > Fixed Assets.

  • File Maintenance > Accounting > Journals > Securities.

  • File Maintenance > Accounting > Journals > Investments.

  • File Maintenance > Accounting > Journals > Share/Unit Holder.

  • File Maintenance > Accounting > Journals > Products/Services.

  • File Maintenance > Accounting > Journals > Tax Account.

 

Examples:

[FV.UF01.TS.FldValue.Required=Y]

<Category> is the value for the field Reference (can be text or code) i.e.'TS'. This tag is to be entered in the Entity Administration > Assets > Fixed Assets – Data Validation [R_FSAFIXED] review rule to check for user field with the key 'UF01'.

 

[FV.SELISTCO.OR01.FldValue.Required=Y]

<Category> is the value for the field Share Class Code i.e. 'OR01'. This tag is to be entered in the Entity Administration > Shares > Capital > User Fields - Data Validation [R_FSESHCAP] review rule to check for user field with the key 'SELISTCO'.
