User Fields (Configuration)

The facility to create user fields is designed for situations where you need extra fields that are not already in the program. You can create user fields for the following:

User fields can be created so that they can be retrieved and used for ALL Master Files, jurisdictions or sub-ledger types, or they can be created so that they are file-type, jurisdiction and sub-ledger specific.

The information in user fields can also be retrieved in documents.

 

User Field list

This lists all user fields for the selected category.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • New - Click this to create new user field.

    1. Enter the details for the User Field:

      • General Field - Tick this if the field is to be used for all files. This field should be ticked if you are setting up a user field as a reference for a journal script in Accounting.

      • Category/Jurisdiction/Share Type/Client Type/Matter Type/Sub-ledger Type/Classification/Asset Type - If you have not selected the General field, then specify when the user field can be used. The user field will only be available for selection if the category/jurisdiction/type or classification, as the case may be, matches. If the General Field is ticked, this field will be deactivated.

      • Key - Assign a unique key to the user field.

      • Allow Duplicates - Tick to allow multiple instances of this user field to be used. Also tick this box if you are setting up a user field as a reference for a journal script in Accounting.

      • Automatically Add - Tick to automatically add this user field to the relevant user fields folder when the related user field screen is first accessed for a specific Master File. The automatic adding of user fields only happens when the user field screen does not have any existing user fields in it. To automatically add a user field to a screen that already has existing fields, use the Add Globally function.

      • Mandatory - Tick this to make the user field mandatory.

      • Field Type - Select the field type.

        • Administrator Group - to show list of Administrator Groups

        • Amount - to accept numerical value. Select this if you are setting up an incremental number for automatic referencing in Accounting.

        • Date - to accept date value

        • Lookup Value - to allow selection of values from specified Lookup Table. The following can be added to the Key Values:

          • [TABLECODE=<Code>] to specify the Lookup Table where <Code> refers to the code of the Lookup Table.

          • [SHOWCOMBO] to display the values as a drop-down for selection. By default, the program will use the search control.

        • Lookup Value/Dates - same as Lookup Value but with dates

        • Master File - to accept a Master File value. To filter the list, you can add [MFTYPE=<Value>] to Key Values where <Value> can be:

          • CS (Company)

          • FG (Fund)

          • TU (Trust)

          • RM (Billing File)

          • CA (Account File)

          • CL (Client)

          • OC (Officer)

          • SH (Shareholder)

          • BA (Bank)

          • SI (Bank Signatory)

        • Master File/Dates - same as Master File but with dates

        • Master File/Dates/Lookup - same as Master File but with dates and Lookup Value. The following can be added to the Key Values:

          • [CODECAPTION=<Caption>] to set a caption for the  Lookup Value field.

        • Master File/Dates/User TV - same as Master File/Dates/Lookup but with User Table Value. The following can be added to the Key Values:

          • [CODECAPTION=<Caption>] to set a caption for the  User Table Value field.

        • Number - to accept numerical value

        • System User - to show list of user accounts

        • Text - to accept text value

        • User Table Value - to allow selection of values from specified User Table. The following can be added to the Key Values:

          • [TABLECODE=<Code>] to specify the User Table where <Code> refers to the code of the User Table.

          • [SHOWCOMBO] to display the values as a drop-down for selection. By default, the program will use the search control.

          • [ON_NEW...] - This tag is used to instruct Viewpoint to execute certain action(s) when a user field is selected for the first time.

            You can perform the following actions with ON_NEW:

            • EXEC-UTBL - Update User Table (Only modify existing user tables, not add new user tables)

            • EXEC-BKAC - Edit/Update Bank Record

            • CREATE-BA - Create New Bank Record (Hard coded to BankAccounts table). The following are the fields you should define with this code:

              • AccountName

              • CurCode

              • BAType

              • BankAddrCode

          • [ON_REMOVE...] - This tag is used to instruct Viewpoint to execute certain action(s) when a user field is changed, deleted or removed.

            You can perform the following actions with ON_REMOVE:

            • EXEC-UTBL - Update User Table (Only modify existing user tables, not add new user tables)

            • EXEC-BKAC - Edit/Update Bank Record

            • CREATE-BA - Create New Bank Record (Hard coded to BankAccounts table). The following are the fields you should define with this code:

              • AccountName

              • CurCode

              • BAType

              • BankAddrCode

          • [ON_EDIT...] - This tag is used to instruct Viewpoint to execute certain action(s) when a value is edited.

            You can perform the following action with ON_EDIT:

            • SETNOTE - Automatically sets the Note section of a user field according to the field value.

             

        • User Tables Value/Dates - same as User Table Value but with dates

        • Yes/No - to allow selection of Yes or No value

      • Caption - Enter a name that will be displayed for the User Field when it is retrieved.

      • Default Value - Enter a default value for the user field, if applicable. For example, if the user field type is Master File, enter the code of the Master File you would like to default to this field. If you are setting up an incremental number for automatic referencing in Accounting, then set this to whatever starting number you prefer. The field will auto-increment each time a new journal is created. You can change the default value for each Account File by editing the value in Accounting > Account File > User Fields - Account File.

      • Default Note - Enter a detailed description to explain the usage of the user fields.

      • Tag Values - Define additional information required for the user field. Apart from [TABLECODE=], [SHOWCOMBO], [MFTYPE=], [ON_NEW...], [ON_REMOVE...], and [ON_EDIT...] there is also this:

        • [ADD_MULTI=<Number>] - this is used to add multiple instance of the user field with <Number> referring to the number of instanced to add. This can be used when the options Allow Duplicates and Automatically Add are ticked.

    2. Click Update.

     

  • Edit - Click to edit the text fields of a user field.

    1. Change the relevant details.

    2. Click Update.

  • Delete - Click this to delete the selected user field. If the user field is used, deletion is not possible.

  • Add Globally - Click this to add the user field to the relevant user field screen for the appropriate Master Files.
