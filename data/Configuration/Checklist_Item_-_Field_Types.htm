Checklist Item - Field Types

A checklist item can be used to 'hold' information in a specific format or perform an action depending on the field type set. When used to contain information, it can be information that is:

  • Entered by user or selected from a list.

  • Retrieved from fields in the database.

  • Derived from another checklist item.

When used to perform an action, it may be to act as a link to a screen in Viewpoint or a document file. Other times, the checklist item may just be used as a 'label' - to display information.

The field type that you set for the checklist item will play a part in determining what the checklist item can do.

Date

Use this for date input or to retrieve date information from the database.

Connect, Engage

Disbursement Entry

(Practice Management component is required)

This is to create a disbursement entry without exiting the workflow. Details of the disbursement entry can be specified by clicking the Browse button in Values/Default column where you can:

  • Specify default values.

    For each detail, specify the information.

  • Retrieve 'Description' details from an item in the same workflow step or previous workflow step.

    This is applicable only for the 'Description' field.

    If the 'Description' is to be retrieved from the same workflow step, add the tag (+<Key>+) where <Key> refers to the assigned key of the item.

    If the 'Description' is to be retrieved from previous workflow step, add the tag (+WF.<Key>+) where <Key> refers to the assigned key of the item.

Connect

Document Holder

(Document Management component is required)

This allows for storing of document to Viewpoint without exiting the workflow. It can be to a new holder or an existing holder. For new holders, details of the holder can be specified by clicking the Browse button in Values/Default column where you can:

  • Specify default values.

    For each detail, specify the information.

  • Retrieve details from items in the same workflow step or previous workflow step.

    This requires the details to be specified in items with assigned key.

    If the detail is to be retrieved from the same workflow step, add the tag (+<Key>+) where <Key> refers to the assigned key of the item.

    If the detail is to be retrieved from previous workflow step, add the tag (+WF.<Key>+) where <Key> refers to the assigned key of the item.

  • Specify the limit for the file size (MB) and file type.

    Use the special character, semi colon ';', to separate the file types. An error will be prompted when the file size or the file type uploaded in the workflow is inconsistent with the requirements updated in the workflow template.

In the workflow, when the item is selected, you will see:

  • Drag document here - This is to create a new holder. You can drag and drop the document to this option.

  • Browse - This is to create a new holder. Click to browse to the location of the document in order to select it.

  • Link to existing - This is to add the document to an existing holder. Click View to see contents of the specified holder (stored or placed in In-tray).

  • Delete - This is to delete the document stored in the Quarantine Tray. The option is available only for documents stored in the Quarantine Tray.

  • Store - This is to store the documents from the Quarantine Tray into a defined holder. The option is available only for documents stored in the Quarantine Tray. The option will be disabled when the ESP Status of the workflow is 'Draft'.

  • Move to In-tray - This is to move the documents stored in Quarantine Tray to in-tray. The option is available only for documents stored in the Quarantine Tray. The option will be disabled when the ESP Status of the workflow is 'Draft'.

If required, the pop-up options can be moved around by dragging it across the workflow screen.

Connect, Engage

Document Link

 

This will open a document file or a website. The file or URL is to be defined in Values/Default column.

Refer to Field Type: Document Link for details.

Connect

Drop Down Value

This allows the selection of values from a specified drop-down table.

Connect, Engage

Entity Type

This allows users to select from the list of Entity Types. This requires a country code to be specified within the workflow step and the list will be based on that country code.

Connect, Engage

Excel Statement

This allows users to generate Excel statement within the workflow. The Excel statement will be linked to the workflow once it has been stored. Additional tags are available for the configuration of this field type, refer to Checklist Item - Tags.

Connect, Engage

File Review

This allows File Review to be performed from within the workflow. The review scope must be specified using tag.

Connect

Formula

This is to indicate that the checklist item will perform calculation based on the specified formula.

Refer to Field Type: Formula for details.

Connect

Go to Screen

This takes you to the pre-defined Viewpoint screen so you can update or verify the information on the screen. Additional tags are available for the configuration of this field type, refer to Checklist Item - Tags.

Connect

Input Field

This is to show different items in the workflow step based on the configuration for the selected type.

Refer to Field Type: Input Field for details.

Connect, Engage

List with Edit

Use this to provide a list of values to choose from. If the option is not there, the user can enter a new value.

Connect, Engage

List without Edit

 

Use this to provide a list of un-editable values to choose from.

Connect, Engage

Lookup Value

This allows the selection of values from a specified lookup table.

Connect, Engage

Macros

 

This is to have the item perform a predefined action when user clicks on the macro caption. There is also an icon that will allow access to the related screen, similar to Go to Screen.

Refer to Field Type: Macros for details.

Connect

Master File

This allows the checklist item’s value to be a Master File. For additional configuration for Master File type, see Field Type: Master File.

Connect, Engage (only if Data Source is defined)

Master File (ESP)

 

This allows users to select from list of Master Files available on ESP.

Note: ‘Master File (ESP)’ is available only in workflow templates that contain this field type as an existing checklist item. It is no longer available for new configuration.

Connect, Engage

Number

Use this for numerical values of up to ten decimal places.

Connect, Engage

Record Link

(Accounting component is required)

This allows the checklist item’s value to  be a record related to Accounting such as:

  • Sub-ledger types

  • Payee

  • Payment

Connect

Required

This is to list all or specific items configured as ‘Required’.  For additional configuration for Required field type, see Field Type: Required.

Note: The ‘Required’ feature do not support the following field types: Approval Card, Change Link, Disbursement, Conditional Group, Excel Statement, File Review, Formula, Go to Screen, Input Field, Macros, Not Applicable, Required, Task, Time Entry and Word Report. This field type is not applicable to checklist.

Connect, Engage

Section Break

This is to create a section break within the workflow to separate the checklist items into different sections. Checklist items after a section break will be in another section.

Connect

Service

This is to allow the workflow to create an invoice and details such as the service code and quantity are to be specified in the Values/Default column.

Connect, Engage

Standard Report

This allows users to generate Standard reports that are SSRS or Analysis report types within the workflow. Additional tags are available for the configuration of this field type, refer to Checklist Item - Tags.

Connect

System User

This allows for selection of a Viewpoint user.

Connect

Task

(Document Management component is required)

This sets the item as a task that can be assigned to a different user than the one assigned to the step. Details of the task can be specified by clicking the Browse button in Values/Default column where you can specify details of the task.

Note: You need to ensure that the task type has been set up first.

Connect

Text

Use this for text input or to retrieve text information from the database. There is no maximum length.

Connect, Engage

Time Entry

(Practice Management component is required)

This is to create a time entry without exiting the workflow. Details of the time entry can be specified by clicking the Browse button in Values/Default column where you can:

  • Specify default values.

    For each detail, specify the information.

  • Retrieve 'Desription' details from an item in the same workflow step or previous workflow step.

    This is applicable only for the 'Description' field.

    If the 'Description' is to be retrieved from the same workflow step, add the tag (+<Key>+) where <Key> refers to the assigned key of the item.

    If the 'Description' is to be retrieved from previous workflow step, add the tag (+WF.<Key>+) where <Key> refers to the assigned key of the item.

Connect

User Defined

This allows the selection of values from a specified data source.

Refer to Checklist/Workflow Step - Tags and Field Type: User Defined for details.

Connect, Engage

User Table Value

This allows the selection of values from a specified user table. The setup is similar to Lookup Value.

Connect, Engage

Word/PDF/XML Report

This allows users to generate XML, PDF, and Word reports within the workflow. The report will be linked to the workflow once it has been stored. Additional tags are available for the configuration of this field type, refer to Checklist Item - Tags.

Connect, Engage

Workflow Link

This is used to link one workflow to another workflow. It also allows users to start a new workflow from within a workflow.

In the workflow, when the item is selected, you will see:

  • Initiate new workflow - This is to start a new workflow from the current workflow.

  • Link to existing - This is to link the current workflow to a workflow that has been started.

Refer to Checklist Item - Field Type: Workflow Link for details.

Connect, Engage

Yes/No

This allows for user to specify a Yes or No value.

Refer to Checklist Item - Field Type: Yes/No for details.

Connect, Engage
