System Defaults - Engage Servicing Portal

The defaults for Engage Servicing Portal (ESP) are set up here.

 

Web Title

Enter the title that will appear on the Engage Servicing Portal.

 

Web URL

Enter the URL where the Engage Servicing Portal is hosted.

 

Token Expires in

Enter the time period (in Hours) before a token expired. The token is issued when user activation and password reset is requested.

 

Timeout of Data Synchronisation is

This refers to the timeout period for data synchronisation between the front-end (ESP web portal) and back-end server. Enter the time period (in Minutes). If value is '0' or negative, this indicates that timeout is disabled.

 

Login Failure Threshold

Enter the number of login attempts permitted. Enter '0' to disable the feature.

 

Terms and Conditions

Enter text for the terms and conditions that show during Engage Servicing Portal user login. HTML tags are supported. The terms and conditions will only show when 'Enable Terms and Conditions' option is ticked. User must agree to the terms and conditions in order to access the Engage Servicing Portal.

 

Maximum Search Result

Set the maximum number of records to display in the Engage Servicing Portal search result screen. The range allowed is from 1 to 1000.

 

Maximum Grid Records

Set the maximum number of grid records to display in the Engage Servicing Portal grid. Enter '0' for unlimited records.

 

Document Expiry

This setting restricts access of documents in the ESP by setting an expiry period. It includes documents in the Request (Workflow) document holder and linked binders for Meetings and Filings that are shared from the Back-office.

When expires, the document will be inaccessible to the ESP users. Attempt to download the document will prompt the message ‘This document has expired and can no longer be downloaded’ (or based on the message as customised in the ‘Expired Document Message’ field). The document, however, can still be accessed from the Back-office.

To define the document expiry, enter the value for the document expiration in Days/Months/Years from:

  • the date the Request (Workflow) is marked as ‘Completed'

  • the Meeting’s actual date

  • the Filing’s filed date

Set the value by entering the value followed by:

  • D - for days

  • M - for months

  • Y - for years

For example, a value of 30D means that the expiration period is 30 days after the request's completion date or the meeting's actual date or the filing's filed date.

Note: The value is not case-sensitive; hence, it can also be entered as d/m/y, for example: 30d.

 

Expired Document Message

This message will show when the ESP user opens an expired document in Request (Workflow), Meetings binder or Filings binder. You can customise the display message by entering the plain text. This is only applicable when Document Expiry is set (enabled).

 

Enable Terms and Conditions

Tick this to display the terms and conditions to user upon log in. The user must agree to the terms and conditions in order to access the Engage Servicing Portal.

 

Convert Document to PDF

Tick this to automatically convert registers to PDF when the registers are downloaded.

 

Enable Auto Load Profile

Tick this to enable auto load Back-office user profile to ESP. Refer to ESP Auto Load Profile.

 

Enable Direct Document Generation

Tick this to enable direct document generation. This allows the documents to be generated immediately without having a waiting time as compared to the existing 'Generate' function. When enabled, only the 'Download' and 'View' functions are seen in the Documents > Generate page for both Home and File.

 

Enable Real-Time Virus Scanning in File Upload

Tick this to enable real-time virus scanning in file upload. When this option is disabled, there will be no virus scan for file uploaded from the ESP.

 

Enable Saved Tabs

Tick this to allow for opened tabs to be saved. Saved tabs will be loaded when user next logs in.

 

No Approval Required for Configuration Change

Tick to bypass the need for approval in ESP configuration change.

 

Enable Export Calendar

Tick this to allow ESP user to export the calendar item from Engage Servicing Portal Calendar to Outlook or other calendar applications via .ics file. When this feature is enabled, ESP user will be able to see the ‘Add to Calendar’ option in each Calendar item.

 

Enable Multilingual

Tick this to enable the multilingual function in ESP. When this is enabled, user will be able to select their preferred language in the Engage Servicing Portal (by clicking on the globe icon in the portal). The available languages are English (default), Dutch, German, Chinese Simplified and French.

 

ESP Security Options

Password Expires in

Set the number of days before the password expires since the last password change. Enter '0' to disable the feature.

 

Minimum Password Length

Specify the number of characters for the ESP login password. The minimum password length should not be more than 20 characters. However, when 'Require Complex Passwords' option is enabled, the 'Minimum Password Length' should not be less than 6 or more than 20. Enter '0' to disable the feature.

 

Enable Forget Password

Tick this box to allow Engage users to reset their login password from the Engage portal.

 

Require Complex Passwords

Tick this box to enforce complex password. When this option is enabled, password should consist of a mix of alphanumeric, upper case, lower case, special characters and must NOT match or contain any part of the ESP user code. It will validate the password length as set in the 'Minimum Password Length' field and prompt an 'Invalid Minimum Password Length' message if the password length is less than 6 or more than 20.

 

Enable Two-Step Verification

Tick this box to enforce two-step verification. Upon enabled, the Engage portal login will prompt users to enter a six-digit verification passcode generated by the authenticator app installed in their mobile device.

 

Enable Authentication Using E-mail

Tick this to enable authentication using E-mail when the login option is OpenId Connect Single Sign-On login.

 

Command buttons:

Click on the relevant command button to perform the necessary.

  • Update - Click this to save changes.

  • Cancel - Click this to cancel the changes.
