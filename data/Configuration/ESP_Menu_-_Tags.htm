ESP Menu - Tags

Tag is used to define certain values in further configuring the ESP menus.

Menu Type

Tag Value

Description

Header [100]

[/link=<child menu ID>/]

Example:

[/link=CP2NOTBC/]

This will direct to the page when user clicks on the side menu. You can only link to a child menu ID (Level 2 menu in Configuration > Engage Servicing Portal > Global > Menus).

 

Grid Display [102]

[/link=<grid code>/]

Example:

[/link=CPANNOCE/]

The grid code can be retrieved from the table GridHeader in the Back-office database. This will direct to the selected grid display.

[/GridOptions=<bitwise value>/]

Example:

[/GridOptions=3/]

The option is based on the value:

  • 1 - system to filter the grid.

  • 2 - system to display the grid filter bar.

This is a bitwise value; hence, you may add up the value (1+2) if you wish to have both options.

[/Params=<key:value>/]

Example:

[Params=StatusCode:NW,PD/]

 

Note: This is handled by ESP framework; therefore, it is not recommended for you to change it without consulting Viewpoint.

This will further filter the grid display (Sub Menu View).

Normal Display [103]

 

Normal View Without Content Header [105]

[/link=<state name>/]

where the state name is defined in config.router.js

Example:

[/link=app.documentHolder/]

 

Note: This is handled by ESP framework; therefore, it is not recommended for you to change it without consulting Viewpoint.

This will direct to the page when user clicks on the menu.

[/Params=<key:value>/]

Example:

[Params=StatusCode:NW,PD/]

 

Note: This is handled by ESP framework; therefore, it is not recommended for you to change it without consulting Viewpoint.

This will further filter the grid display (Sub Menu View).

 

e-View Display [104]

Customised e-View Display [114]

[/link=<Form Code>/]

Example:

[/link=CPSTAT/]

 

The Master File type can be used to further indicate which eView template to be used for a specific Master File type:

[/link=[MF.<Master File type>=<Form Code>]/]

where the Master File type is represented by the following:

  • Company - C

  • Foundation - N

  • Individual - I

  • Other (non-individual) - O

  • Partnership - P

  • Trust - T

To specify the default, the code 'DF' is used.

Example:

[/link=[MF.C=CPSTAC] [MF.T=CPSTAU] [MF.P=CPSTAP] [MF.N=CPSTAN] [DF=CPSTAT] /]

This will direct the page to the e-View template page when user clicks on the menu. The e-View template is maintained in the Back-office Document Libraries.

Custom Display [106]

Customised Static Display [116]

[/link=<html file name>/]

Example:

[/link=content.html/]

This will link to a static html. The static html file is stored in <ESP Folder>/tpl/custom folder. By default, this folder does not exist. User is required to manually create the folder and copy the static html file into this folder.

Home Menu [11]

[/AlwaysShow=Y/]

This tag will enable the Home menus to be displayed in the File tab. Clicking on the Home menu from the File tab will redirect users to the Home tab and opens the Home menu which display the records for all the entities within the Organisation. Having the Home menu to be displayed in the File tab provides ease of navigation as users need not have to go to the Home tab to access the Home menu.

CP2CPINV, CPFCPINV

Note: This is used for the payment confirmation (in the Invoices page) to open a Request (Workflow).

A new button to open the Workflow will be available in the Invoices payment confirmation page when the following tags are defined.

[/WFButton=<Button ID>/]

This defines the button ID.

[/WFButton.<Button ID>.WKey=<Workflow Key>/]

This defines the Workflow that will be retrieved from the Back-office.

Note: The Home and File ‘Invoices payment confirmation page’ must use different Workflow Key due to the global Workflow template requires attributes setup.

Additional Setup to Enable Request Initiated from Home

If the Invoices page is opened from the Home tab, additional setup is required to allow the users to initiate Request (Workflow) from the ESP Home > Invoices payment confirmation page. Follow the steps below to configure:

  1. In the Front-office, go to Home > Configuration > Engage Servicing Portal > Configuration > Organisations, select the organisation and click Edit.

  2. In the Edit screen, select the 'Database' and 'Master File’. This will set the ESP to open the selected Database and Master File when initiating the Request (Workflow) from the ESP Home > Invoices payment confirmation page.

  3. Click Update.

  4. In the Back-office, go to Home > Configuration > Business Parameters > Workflow > Workflow Library, select the Workflow template that will be opened from the configured button.

    Note: The configured button will not appear if Workflow Key is invalid or Workflow is not published to the ESP.

  5. Click Attributes.

  6. In 'Attributes Configuration' screen, at Master File, select the Type 'Master File' and set the Value to the Master File that you have configured in step (2).

  7. Click Update to save the setting.

[/WFButton.<Button ID>.FldKey=<Workflow Field Key>/]

This defines the Workflow field that will display the invoice details for payment confirmation.

You can define more than one field by separating each with a comma (,).

Note: Additionally, the Back-office Workflow field requires to also include [+HD.<Workflow Field Key>+] at the 'Values/Default' to define the default value.

[/WFButton.<Button ID>.Text=<Button Name>/]

This defines the button name that will show in the Invoices payment confirmation page. This button will trigger to open the Request (Workflow) upon clicked.

[/WFButton.<Button ID>.<Workflow Field Key> =(+Field Name+)/]

where the (+Field Name+) can be:

  • (+Currency+) – This refers to the invoice currency.

  • (+DateDue+) – This refers to the invoice date due.

  • (+DateInvoice+) – This refers to the invoice date.

  • (+InvoiceType+) – This refers to the Invoice type. This should have DbId prefixed when the invoice is from multiple Back-offices.

  • (+FileName+) – This refers to the Master File name.

  • (+InvoiceNr+) – This refers to the invoice number.

  • (+InvoiceTotal+) – This refers to the invoice total.

  • (+PaymentAmount+) – This refers to the amount payable for the invoice.

  • (+TotalAmount+) – This will total up the (+PaymentAmount+) for all invoices.

  • (+ProformaNr+) – This refers to the invoice proforma number. It should have DbId prefixed when the invoice is from multiple Back-offices.

  • (+FileCode+) – This refers to the Master File code.

This defines the invoice details that will show in the Request field for payment confirmation.

[/WFButton.<Button ID>.<Workflow Field Key>.Format=LC or LL/]

where:

  • LC = List with comma

  • LL = List with lines

This defines the display format of the payment confirmation text.

List with comma (LC) will display each invoice details separated by a comma.

List with lines (LL) will display each invoice detail in a separate line.

See full example of the tag values.

Example tag values as shown:

[/WFButton.wire.INV1.Format=LL/]

[/WFButton.wire.INV2=(+PaymentAmount+)/]

[/WFButton.wire.INV2.Format=LC/]

[/WFButton.wire.INV3=(+Currency+)/]

[/WFButton.wire.INV4=(+TotalAmount+)/]
