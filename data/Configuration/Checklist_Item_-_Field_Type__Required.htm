Checklist Item - Field Type: Required

This field type is used to retrieve checklist items from previous step(s) that remain empty so that they can be completed in the current workflow step. These items are flagged as Required to allow users to complete the workflow step without entering a value for the items; and revisit these items again further along in the workflow. In the workflow, the caption for items will be in indigo.

To allow this, in the workflow step where you would like required items to be retrieved for completion, set up a checklist item with the field type 'Required' and in Values/Default, specify the following:

  • The key of the workflow step containing the required items

  • The key of the required items

 

As a result, all ‘Required’ items from previous workflow step(s) that remain empty will be displayed in the workflow step. If the checklist item is flagged as Mandatory, then it will be shown as mandatory items.

If this checklist item is not included, you can still access required items in the Required Fields tab by clicking the Required icon from the Ribbon.

 

If this checklist item is not included, you can still access required items in the Required Fields tab by clicking the Required icon from the Ribbon. When there are multiple steps with ‘Required’ checklist items, these will be displayed sequentially according to the order of the workflow steps. To go to the workflow step item, right click on the item and click ‘Navigate to Step Field’.

 

Note: The ‘Undo Step’ and ‘Remove Optional’ ribbon buttons, and the ‘New Link’, ‘New Holder’ and ‘Open’ button for Document Links will be disabled in the ’Required Fields’ tab.
