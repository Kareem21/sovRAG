Archiving

All forms and correspondence generated from Viewpoint are stored in the Archive folder. The benefit of this archiving process is that if you manually edit a document via your word processing software after it has been generated, the changes are saved in the documents that are stored in the Archive folder. This means that if you need to reprint the document you will not have to redo the changes. However, although the Archive is a useful time-saving tool for users, it will obviously grow as the system is used. This means that it will gradually require more and more storage space. For this reason we advise that you delete old documents from the Archive after a certain period of time.

The path to the location of the General Archive folder, is set up in Configuration > Settings > System Defaults > Folders. When you archive an item for the first time, the folder is automatically created by Viewpoint. You may use the Browse button to browse to the location of the Export folder.

 

General

The General section of the Archiving screen is where you archive the following:

  • Change Log entries, find out more here.

    Archiving Change Log entries will move the records in the ChangeLog database table to VpArchive_ChangeLog.mdb and delete the records displayed in the Change Log of the various Viewpoint modules.

    1. Tick Change Log.

    2. Select the period from the Change Date older than drop down list. Viewpoint will delete records older than this.

    3. Click Archive.

    4. Click OK to exit when the process is complete.

     

  • Correspondence, find out more here.

    Archiving correspondence will move information VpArchive_Administrator.mdb and the soft-copy of the documents to the Correspondence_Documents sub-folder under the Export folder.

    1. Tick Correspondence.

    2. Select the period from the Document Date older than drop down list.

    3. Click Archive.

    4. Click OK to exit when the process is complete.

     

  • Activity & Exception Log, find out more here.

    This will delete all activity and exception log entries from Viewpoint. The deleted activities and exception log will be archived in \VpArchive_ActivityLog.xml

    1. Tick Activity & Exception Log.

    2. Select the period for items you want to archive.

    3. Click Archive.

    4. Click OK to exit when the process is complete.

  • Agent Job Log, find out more here.

    This will delete the Job Scheduler log from Viewpoint. The deleted activities and exception log will be archived in \VpArchive_AgentJobLog.xml

    1. Tick Agent Job Log.

    2. Select the period for items you want to archive.

    3. Click Archive.

    4. Click OK to exit when the process is complete.

Note: The 'Web Document Generation Folder' option is obsolete starting from Viewpoint version 7.4.

 

Entity Administration

The Administrator section of the Archiving screen is where you archive the following items.

  • Form Queue entries, find out more here.

    The archiving function will delete entries in the Form Queue and any corresponding document, if they have already been generated. Form Queue entries are archived in VpArchive_Administrator.xml while generated documents are placed in the FormQueue_Documents sub-folder under the Export folder.

    Note: If you are Archiving by Master File, the physical documents will not be transferred to the FormQueue_Documents folder.

    1. Tick Form Queue.

    2. Select the period for items you want to delete.

    3. 'Documents generated older than' will delete items in the Pending and Completed lists.

    4. 'Documents not generated older than' will delete items in the New list. This includes items that encountered an error during generation.

    5. Click Archive.

    6. Click OK to exit when the process is complete.

     

 

Practice Management

The Practice Management section of the Archiving screen is where you archive the following items. Archived items are stored in VpArchive_Billing.mdb.

  • Time Posting, find out more here; and

    Use this feature to archive posted, un-posted, chargeable and un-chargeable time entries.

    Note: This will not affect your invoice history.

    1. Tick Time Posting.

    2. Select the period for items you want to delete:

      • Non-chargeable older than - This will delete non-chargeable time entries that have been posted.

      • Chargeable older than - This will delete chargeable time, including un-posted time.

      • Un-posted/non-chargeable older than - This will delete all non-chargeable time entries that have not yet been posted to an invoice. This includes all entries that have not yet been approved.

    3. Click Archive.

    4. Click OK to exit when the process is complete.

     

  • Disbursements, find out more here.

    Select this option to delete posted disbursement entries.

    Note: This will not affect your invoice history.

    1. Tick Disbursements.

    2. Select the period for the items to be deleted.

    3. Click Archive.

    4. Click OK to exit when the process is complete.

 

Document Management

This section is used to archive items in Document Management. Archiving of document management items involve deleting and archiving the following:

  • Document Holders & Tasks, find out more here; and

    Select this option to delete document holders and tasks according to the retention policy set for each Filing Cabinet. Items are archived in VpArchive_Documents.mdb and the physical documents are moved to the DocMgr_Holders sub-folder under the Export folder.

    Note: This is only applicable where the DocStore is located in the file structure (e.g., \\<server>\Viewpoint\DocStore) and has not been converted into a SQL database.

    1. Tick Document Holders & Tasks.

    2. Select the items you want to delete:

      • Delete documents according to the retention policy setting will delete items older than what was set for the Filing Cabinet. This is based on the month value of the date set in the Document Date index field.

      • 'Master File tasks older than' will delete Master File tasks older than the selected period.

    3. Click Archive.

    4. Click OK to exit when the process is complete.

     

  • Audit Log, find out more here.

    Deleting the audit log will also delete any images that could be retrieved if they were inadvertently deleted from a document holder. Images that were deleted in the following processes are moved to the DocMgr_Audit subfolder:

    • Holder - Remove File

    • In-tray - Remove File

    • Batch - Delete Scan Page

    Deleted entries are archived in VpArchive_DocAudit.mdb.

    1. Tick Audit Log.

    2. Select the items you want to delete:

      • In-tray audit trail older than - This will delete items the In-tray audit trail that are older than the selected period.

      • Batch audit trail older than - This will delete items in the Batch audit trail that are older than the selected period.

      • Task audit trail older than - This will delete items in the Task audit trail that are older than the selected period.

      • Holder audit trail older than - This will delete items in the Holder audit trail that are older than the selected period.

    3. Click Archive.

    4. Click OK to exit when the process is complete.

    See also Audit Trail Setup.
