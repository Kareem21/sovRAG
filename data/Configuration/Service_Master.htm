Service Master

The Service Master allows you to set up and maintain company-specific services and related information for quick and easy data input. With the Service Master, you have access to standard service information such as service details and prices every time you bill a client. This information should be set up before you start using Practice Management in order to minimize the amount of time spent working on each invoice.

 

Filter

The filter applies to the service list.

 

Select button

This allows for additional criteria to apply to the service list.

 

Service list

This shows the services that you have set up.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • New - Click this to add a new service.

    1. Enter details for the service.

      • For Billing Event - this is set to Standard [STD] unless the service is a disbursement payable to third parties. Then, it can be set as:

        • 3rd Party AR - Fees Payable [PAR] - if it is to be used for Fees Payable invoicing.

        • 3rd Party AR - Others [POT] - if it is not to be included for Fees Payable invoicing.

      • For Descr. Shortcuts - Time/Disb and  Descr. Shortcuts - Invoice Entry, these can be used to:

        • provide additional options that you can select for the Description field when creating time and disbursement entries (Descr. Shortcuts - Time/Disb); and in the Recurring Fees and Create/Edit Invoice screen (Descr. Shortcuts - Invoice Entry).

          For Descr. Shortcuts, you can create multiple descriptions for a service. Each new line denotes one description option. For example:

           

          In Service Master Configuration:

          Meeting with client

          Telecon with client

          Preparation of Correspondence

           

          On Description’s Screen Selection:

          Meeting with client

          Telecon with client

          Preparation of Correspondence

           

                Group multiple text as one description option by adding a pipe character '|' to the end of the group to get multiple lines of selection options. For example:

           

          In Service Master Configuration:

          Meeting with client|Telecon with client|Preparation of Correspondence

           

          On Screen Selection:

          Meeting with client

          Telecon with client

          Preparation of Correspondence

           

          The above two methods will provide three options and user can select one of them.

           

          If the description options are to be multiple lines, group the lines by adding a pipe character '|' to the end of the group. For example:

           

          In Service Master Configuration:

          Virtual meeting with client

          Send confirmation email|

          Meeting with client

          Obtain signature|

           

          The above will provide two options with each containing two lines.

           

          On Screen Selection:

          Virtual meeting with client

          Send confirmation email

          Meeting with client

          Obtain signature

           

          Note: The pipe character '|' will be treated as a separator only. It will not be treated as a part of descriptions. If the pipe character '|' is not being defined, the system will use a new line as a separator. If pipe character '|' is found together with a new line, the pipe character '|' will be used as a separator.

           

        • provide multi-language descriptions which will be filtered based on the Master File language defined in the General screen for the Master File. If no language is specified, the default language set up in System Defaults > General/Master File will be selected.

          Start with the language code in square brackets, followed by the description in that language. The language code is from the Language Codes [LANG] lookup table. Use one line per description. 

           

          For example:

          In Service Master Configuration:

          [ENG]Meeting with client

          [DUT]Ontmoeting met de klant

          [FRE]Réunion avec le client

           

          For the above, if the Master File's language is Dutch, then only the second option is seen.

           

          On Screen Selection:

          [DUT]Ontmoeting met de klant

           

          If no language code is specified, then the line is included, regardless of the language of the Master File. For example:

           

          In Service Master Configuration:

          [DUT]Ontmoeting met de klant

          [FRE]Réunion avec le client

          Prepare meeting minutes

           

          For the above, if the Master File's language is Dutch, then the first and third options are seen.

           

          On Screen Selection:

          [DUT]Ontmoeting met de klant

          Prepare meeting minutes

      • For Code, tick to select the matter types that can used this service. This is required especially when you enable the option 'Link Services to Matter Types' in System Defaults > Practice Management.

      • For Use For, this restricts the service to only specific areas. Tick to select the areas the service can be used. Note that this section adds on to the control by the service class and options in System Defaults.

        • Allow all Services for Time Entry

        • Use Recurring Services for Time Entries

        • Use Recurring Services for Disbursement Entries

        • D Class Services are Exclusively for Disbursements

        • Service Use For - if this is ticked, then the service can be used in all areas.

        • Sales Ledger - if this is ticked, the service can be set up as a recurring fee and selected for Sales Ledger invoicing.

        • Purchase Ledger - if this is ticked, the service can be selected for Purchase Ledger invoicing.

        • Time Entry - if this is ticked, the service can be selected in Time Entries screen.

        • Disbursement - if this is ticked, the service can be selected in Disbursements screen.

      • For Tag Values, please refer here for more details.

    2. Click Update.

    3. If you are using the Accounting component, you may be required to do the following:

      • complete the account mapping in the Account Mapping screen that will be displayed. This is seen when the value for PM Accounting Journals in System Defaults > Accounting is greater than or equals to 2.

      • set the tax code for the service in the Service Tax Codes screen if the option 'Use Accounting Tax Sub-ledger as Tax Codes' is enabled in System Defaults > Practice Management.

  • Edit - Click this to edit an existing service

  • Delete - Click this to delete the selected service. Services which are in use cannot be deleted but you can prevent future use by deactivating them. For a newly created service, you can delete the service provided that has no Account Mapping record created/mapped before.

  • Global Change - Click this to update the Recurring Fees and descriptions, Time/Disbursements (unposted entries only) or only unapproved entries, and unit price.
