Front-Office Import Sheet

Excel Import Sheets are available for use in CP-AD (Front-office) for updating records in:

  • Configuration > Engage Servicing Portal > Configuration > Organisations screen:

    • Organisations grid

    • File Configuration grid

  • Configuration > Engage Servicing Portal > Configuration > Users screen:

    • Users grid

    • User - Organisation grid

Prior to importing the records, the import sheet must be completed with the relevant details before they are imported through Viewpoint Connect Add-ins in Microsoft Excel.

It is mandatory to define the ‘record type’, ‘import action’ and ‘Configuration refresh’ to ensure that the data import works as desired.

There are two types of import sheet provided where you can define the ‘record type’ globally or individually.

When you define the ‘record type’ globally, the record type will apply to all the records in the import sheet.

When you define the ‘record type’ individually or per row basis, the record type will apply only for that row and each row would require to define its own ‘record type’.

The ‘record type’, ‘import action’ and ‘Configuration refresh’ are as explained below:

 

Record Type

The ‘record type’ defines which setting is being updated. It can be User, Organisation, User Authority, or combination of two or more settings.

There are seven record types available and each of these would require mandatory fields and optional fields to be entered into the sheet prior to the import:

Record Type

What Setting is Being Updated?

Mandatory (M) and Optional (O) Fields to Fill up in the Import Sheet

ORG

Organisation

  • OrgCode (M)

  • OrgName (M)

  • OrgType (O)

  • OrgNote (O)

  • GroupCode (O)

  • OrgDbId (O)

All the above fields will update the records into the ClpOrganisations database table.

COF

Configuration

  • DbId (M)

  • FileCode (M)

  • SearchType (O)

  • Note (O)

All the above fields will update the records into the ClpFileConfig database table.

Note: Please ensure that the ‘DbId’ is set correctly in the import sheet and is registered as a valid code in Configuration > Engage Servicing Portal > Global > Databases.

USR

User

  • UserCode (M)

  • UserName (M)

  • Email (M)

  • Telephone (O)

  • Company (O)

  • JobTitle (O)

  • TimeZone (O)

All the above fields will update the records into the ClpUsers database table.

OUA

User Authority

  • UserRole (M)

This field will update the records into the ClpOrganisationsUser database table.

OCF

Organisation and Configuration

Refer to the above for ORG and COF.

Note: This is not applicable for OrgType = 2

OCU

Organisation, Configuration, User and User Authority

Refer to the above for ORG, COF, USR and OUA.

Note: This is not applicable for OrgType = 2

UOA

User and User Authority

Refer to the above for USR and OUA.

 

Import Action

The ‘import action’ defines how the values are updated for the existing record:

Abbreviation

Action

AO

Add or overwrite value if code exists.

AI

Add or ignore value if code exists.

IN

Inactivate the record.

DL

Delete the record if it has no other relations, else inactivate the record.

Note: This is not applicable for RecType = OCF/OCU/UOA)

 

Configuration Refresh

The ‘configuration refresh’ defines how the File Configurations are refreshed for the existing record:

Abbreviation

Action

REF

Refresh the entire File Configuration. This will not remove the File Configuration.

RXL

Refresh the entire File Configuration but retain the excluded Master File(s). This will not remove the File Configuration.

RDL

Remove the entire File Configuration and import new File Configuration according to the import sheet.

 

Performing Import in Microsoft Excel

Upon completing the relevant details, you may proceed to do the import:

  1. Open the Import Sheet and enter the required information.

  2. In Microsoft Excel, select the Viewpoint add-in menu.

  3. Click Import Sheet. The Import Sheet screen will appear.

  4. Click Import.

  5. Click Close to exit the Import Sheet screen when the import is complete.
