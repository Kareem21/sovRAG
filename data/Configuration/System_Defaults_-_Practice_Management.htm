System Defaults - Practice Management

The defaults for Practice Management are set up here.

 

Reporting Currency

The reporting currency is the principal currency in which the system reports and summarises financial numbers. You must ensure that you make the correct choice when selecting your currency because it cannot be changed once the system has been set up.

 

Default Revenue Entity

Set the revenue entity to the most frequently used one, so that the system defaults to this when creating a new Matter.

 

Recharge - Default Service

The service specified in this field is the default for the Recharge button in the Settlements screen. If the option 'Use Recurring Services for Disbursement Entries' is enable but the specified service has not been set up as a Recurring Fee for the Billing File, the charge will still be created as a disbursement entry. However, you would not be able to edit the service code of this disbursement entry.

Recharge button is available when user clicks Receipt/Payment or Settle to Matter in the Settlements screen for Accounts Receivable.

 

File Review - Fee Date

Select the annual fee review date from the standards.

  • 1st Jan - current year [01] - Select this if the annual fee is due on the 1st of January every year.

  • 1st Month - current year [03] - Select this if you want the anniversary of the fee to be the 1st of the current month. For example, if you are running the fee review on 17 July, then the anniversary of the fee is 1 July of this year.

  • 1st Quarter - current year [02] - Select this if you want the anniversary of the fee to be the 1st of the current quarter. For example, if you are running the fee review on 17 September, then the anniversary of the fee is 1 July of this year.

  • Anniversary incorp. - current year [05] - Select this if you want the anniversary of the fee to be the anniversary of the incorporation.

  • Today's date [04] - Select this if you want the anniversary of the fee to be one year from today.

 

Recurring Fee - Pro-rata Method

Select the method to pro-rate recurring fees.

  • Day [D] - Total number of days for pro-rate period is calculated based on 365 days in a year.

  • Week [W] - Total number of days for pro-rate period is calculated based on 52 weeks in a year.

  • Month [M] - Total number of days for pro-rate period is calculated based on 12 months in a year.

 

Sales Ledger

Require Approval - Sales Order

Draft invoices can only be converted into an invoice if they are approved. If your approval system requires that drafts be approved before they are posted as invoices, activate this function by ticking the check box. If this option is not ticked, draft invoices are approved by default.

 

Split Authority for Approving and Invoicing Sales Orders

Tick this to prevent the same user from approving and posting a Sales Order.

 

Allow Overpaid - Sales Ledger

Tick this to allow for overpayment for sales orders and sales invoices.

 

Disable Use of Journal Date

If the invoice date is the same as journal date, you may tick to disable the use of the Journal Date field for sales invoices.

 

Round Invoice Line to #.00

Tick to round invoice lines to two decimal places. If this box is ticked the system will apply a small discount to invoice lines so that there are only two decimal places in the invoice line total. This applies to sales invoices only and when invoice is created using Create/Edit Invoice.

 

Add Administrative Markup

Tick this to automatically add a percentage/amount of invoice total to the invoice on generation of a draft. You will need to set up the service code to be used for the additional line as well as the percentage/amount of the increase. When this is enabled, a new checkbox named ‘Add Administrative Mark-up’ is added to the grid of the invoice line and is enabled by default.

See also:

  • Lookup Tables - Administrative Mark-up [ADMU]

Use Matters

Tick this if you want to use matter entries. Matters can be viewed as a project defined for a particular billing file. If this box is not checked all transactions will be related to a General (default) Matter.

See also:

  • Matters

 

Disable Use of 'Default' Matter

Tick if you do not want the system to default to a specified matter.

 

Link Services to Matter Types

Tick this if you want to limit the services that can be charged to a particular Matter. Further settings are done in the Service Master in Business Parameters. If you do not tick this, services can be used for all Matters. This option CANNOT be used in conjunction with the option Allow all Services for Time Entry (see below).

See also:

  • Service Master

 

Multi Rate/CCY List

Tick this if you are using multi-currency rates for services and Administrator rates. This allows users to charge a client based on the Matter currency. You can create new service rates lists in the Service Rates Lists [SRLT] lookup value table. Currencies for the list are to be specified in the Long Name field.

See also:

  • Service Rates

 

Multi Rate/CCY List Cost of Services

Tick this if you want the cost of service granularly entered, with different values for each Rate List and Currency (rather than having a single value for time/material in the PM reporting currency). This option will only work when the option 'Multi Rate/CCY List' is ticked.

See also:

  • Service Rates

 

Use Recurring Fee - Unit Price/Effective Date

Tick this if you want to add a new recurring fee unit price, based on a specific effective date. The new rate is applied immediately if the recurring fee's effective date is the same as or earlier than the effective date for the new rate and the service has not been invoiced before.

Otherwise, the fee will be reviewed during the posting of the recurring fee and made effective at the next due date. You may use the Global Change feature in the Service Master to update the unit price globally, or you may enter the new fee per individual Billing File.

See also:

  • Service Master

 

Disbursement Markup - Post to Separate Account

Tick this to post the Cost and Markup into separate account. The Cost will be posted into the Expense Account – R.PA and the Markup will be posted into the Income Account – R.SA of the disbursement service.

 

Allow Multi-CCY in Payment Batches

Tick if you want to allow multi currency invoice lines in Government fee payment batches.

 

Purchase Ledger

Require Approval - Purchase Order

Tick this option if your approval system requires the Purchase Orders to be approved. When this option is ticked, the draft Invoice will be assigned with an invoice number upon approval and cannot be deleted. It will prompt the user to set the invoice line as '0' value if user clicks the Delete button. When this option is not ticked, all the Purchase Orders entries are approved by default, however the draft invoice will not be assigned with an invoice number and can be deleted.

 

Split Authority for Approving and Invoicing Purchase Orders

Tick this to prevent the same user from approving and posting a Purchase Order.

 

Allow Overpaid - Purchase Ledger

Tick this to allow for overpayment for purchase orders and purchase invoices.

 

Disable Use of Journal Date

If the invoice date is the same as journal date, you may tick this to disable the use of Journal Date field for purchase invoices.

 

Disable Validation of Tax Amount on Purchase Line

Tick this to disable the validation of the Tax Amount field for a purchase order line in Create/Edit Purchase Orders. When enabled, users can amend the amount in the field.

 

Batch/Approval of Accounts Payable Payments

Tick this if you require purchase orders to be approved before payment can be done.

 

Create Accounting Payment Record for Payments

This option is applicable if you have the Accounting licence and would like to create Accounting payment record upon creation of supplier payments. System will only create payment record for 'Credit Note' supplier payments. See Bank Payment Export File for PO Settlements for details.

 

Other Options

No Exchange Translation for RC Invoice Line

Tick this if you do not want to translate the invoice lines entered in the Base Currency (‘Reporting Currency’) and invoiced in a different currency. When this option is unticked, the invoice lines entered in the Base Currency (‘Reporting Currency’) and invoiced in a different currency will be translated to the Base Currency (‘Reporting Currency’) using this formula:

Line Base Amount = [Header Exchange Rate * (Line Exchange Rate * Total Line)]

This is applicable for invoice calculations, not settlements.

 

Round VAT Down to 5 Cents Multiples

Tick to round Value Added Tax amounts down to 5-cent multiples. This will apply also to calculation of VAT amount in purchase orders.

 

Enable Intercompany Invoice Line Allocations

Tick this to allow a revenue entity that is different from the revenue entity for the invoice header to be set for the invoice lines. This requires the Revenue Entity field to be enabled for the screen Sales Invoice - Detail Lines and Purchase Invoice - Detail Lines.

Note: Journal posted will be based on the Invoice Line Revenue Entity instead of Invoice Header Revenue Entity.(Only applicable when the option 'PM Accounting Journals' in Configuration > Settings > System Defaults > Accounting is set to either CA Journals RC[2] or CA Journals RC + CE[6].

See also:

  • Intercompany Invoicing and Settlements

 

Enable Intercompany Settlements

Tick to allow invoice from one revenue entity to be settled to another revenue entity's bank account. Ensure that the account 'Intercompany Account - R. GI' is mapped in Account Mapping. This option applies to both Sales and Purchase Ledger.

See also:

  • Intercompany Invoicing and Settlements

 

Allow Negative DR/CR Notes to be Posted

Tick to allow Debit notes with a ‘credit’ total, and Credit notes with a ‘debit’ total to be posted.

 

Allow Receipt/Payment Date before Invoice Date

Tick this option to allow Receipt or Payment Date before Invoice Date. When this is disabled, system will prompt a validation message and user is not allowed to create transaction.

 

Enable Validation of Invoice Date in Posting

Tick this option to enable invoice date validation in Posting. The date will be validated against the Practice Management 'Journal - Date Open' and 'Date Close' in Posting Control Settings.

 

Use Accounting Tax Sub-ledger as Tax Code

Tick this if you want the tax sub-ledger codes to be available for selection for Tax Code field when creating invoice. This allows for different tax codes for different revenue entity. However, you must ensure that you map the tax codes in the Service Tax Codes screen. Once enabled, the Tax field will be enabled and mandatory. The option 'Always charge VAT' in Service Master screen will not be taken into account.

If you do not want to use this option, make sure it is unticked. When not used, system will refer to the following:

  • 'Always Charge VAT' option in Service Master where if this option is ticked, it will apply the rate set regardless of the setting for Tax Charge Type field in Matters screen

  • the rate in Service Master & Administrator Rates screen

  • the setting for Tax Charge Type in Matters screen - if the value for this field is 5 or greater, it will apply the rate set. Note that standard values are 0 - Not Applicable, 1 - Exempt and 9 - Applicable Rate. However, you can add more options in Sales Tax Group [STAX] lookup table.

Note: It is recommended that you consider properly before deciding to implement this due to its impact. Once you enable this option, the Tax Code field in Practice Management will be mandatory. You would not be able to perform further transactions on invoices that have been posted prior to enabling this function (e.g. cancelling the invoice).

If after enabling this and you wish to disable it, note that this will lead to inconsistency in tax report.

 

Enable Organisational Unit

Tick if you want to pass information on Organisational Units from Practice Management to Accounting. The selections for Organisational Units are set up in the Organisational Units [ORGUNITS] user table.

How do I enable/disable the Organisational Unit feature?

Click Settings > Tools & Utilities > Localise and then select Enable/Disable Organisation Unit under the General group. This ensures that the fields are enabled/disabled throughout Viewpoint.

Refer to Localise.

 

Use Change Codes for Document Generation

Tick this to use change codes to control the generation of Billing documents like draft invoices. The setup required would be to add the relevant change link to the templates. With this option, there is no need to specify the templates in Business Parameters > Practice Management > Setup > Document Templates. Change codes that are applicable:

  • PCDI - Print Draft Sales Invoice

  • PCOI - Post Sales Invoice

  • PCSI - Print Sales Invoice

  • PSDI - Print Draft Supplier Invoice

  • PSOI - Post Sales Invoice

  • PSSI - Print Sales Invoice

  • PMPR - Post Receipt

  • PMPP - Print Receipt

  • PMPS - Post Payment

  • PMPQ - Print Payment

  • PPCS - Print Creditor Statements

  • PPDS - Print Debtors Statements

  • PPMS - Print Matter Statements

See also:

  • Using Change Codes for (Invoice) Document Generation

 

Allow Receipt/Payment on Draft Invoices

Tick this option to allow receipt or payment date on draft invoices. When this is disabled, system will prompt a validation message and user is not allowed to create transaction.

 

Use Exchange Rates based on Journal Date

Tick this option to get the exchange rates based on the Journal date for draft invoice in both Sales and Purchase and Payment/Receipts. Once the exchange rate is defined in the draft invoice screen and if the user changes the journal date during invoice posting window, the exchange rate will not be recalculated based on the 'Journal Date' defined in the Sales Invoice Posting/Supplier Invoice Posting window. It will only follow the exchange rate in the draft invoice instead.  

Note: The total amount due for recurring fee/time entries/disbursement entries might differ during propose draft and prepare draft invoice stage in the Posting of WIP screen. This is because of the Journal date does not exist during the propose draft stage, whereas the Journal date is already defined in the prepare draft invoice stage.

This option only works when the option 'Use Accounting Exchange Rates' is enabled in Configuration > Settings > System Defaults > General/Master File.

 

Command buttons:

Click on the relevant command button to perform the necessary.

  • Update - Click this to save changes.

  • Cancel - Click this to cancel the changes.
