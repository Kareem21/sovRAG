Checklist Item - Values/Default

The Values/Default column are for additional configuration details of the checklist item. Depending on the field type, when clicking on the Values/Default column, you may be able to:

  • Enter information - This applies to the field types 'Formula' and 'Conditional Group' where you can click and enter the formula or condition.

  • See the Browse button - This applies to the other field types and you need to click on the button in order to add the configuration details. Different field types will require different  types of configuration details which can be entered in the following:

    • Values/Default - This is available for most field types and is used to set a default value which can be:

      • Hard-coded value by entering the value e.g. 'Individual'. For the field type 'Address Card', enter the Address Card number.

      • Retrieved from another checklist item using codes which you can enter or obtain from Workflow Field Wizard.

      • Fields with default value will be in read-only mode unless the ‘[ROW.RW]’ tag is present.

    • Values - This is available only for the field types 'List with Edit' and 'List without Edit' for you to specify the options for the list.

    • Default - This is available only for the field types 'List with Edit' and 'List without Edit' to allow you to set a default value.

    • Row Height - This is available only for the field types 'Text' and 'Go to Screen' and is used to:

      • Increase the row height for memo-style input when the field type is 'Text'.

      • Increase the row height to display multiple line Help/Description for checklist items with the field type 'Go to Screen'.

    • Max Characters - This is available only for the field type 'Text' to limit the number of characters that the users can input.

    • Validation Option - This is available only for the field type 'Text' and is used to ensure the value entered by users is of a valid format. Users cannot complete the step if the wrong format is entered. There are three options:

      • EMAIL - Select this for e-mail where the value entered must contain '@'.

      • WEBSITE - Select this for website where the value entered can start with 'http://', 'https://', or 'www.'.

      • REGEX - Select this for Regular Expression where the value entered must match with the specific character pattern. You can further configure this in 'Regular Expression Pattern'.

    • Regular Expression Pattern - This is available only for the field type 'Text'. For 'Validation Option' that is set to 'REGEX', you can confifure the character pattern according to a certain format. For example, if you want users to enter a Malaysian Identity Card number as the value, configure '\d{6}-\d{4}'. This means that the value must be numbers in the format of 'xxxxxx-xx-xxxx', where 'x' is a digit from 0-9.

    • Show List - This is available only for the field type 'Yes/No' to allow the checklist item to be a Yes or No value.

    • Clipboard Field - This is available for most field type to allow mapping of the item to the Database Field for copying and pasting using Viewpoint Clipboard.

    • Tag Values - This is available for most field types for checklist item tags to be entered. Refer to Checklist Item - Tags.

    • Approval Card - This is available only for the field type 'Approval Card' and it is to specify the approval card.

    • ESP Option - This is available only for the field type 'Master File (ESP)' to display the list of Master Files that can be selected. There are two options:

      • From Master File Relations - This retrieves all Master Files that are related to the Master file the workflow is for.

      • From Organisation - This retrieves all Master Files that are in the same Organisation of the Master File the workflow is for.

    • Output - This is available for most field types and is used to pass the value of the checklist item to the decision step.

    • Duplicate - This is available for most field types except 'Approval Card', 'Conditional Group', 'Change Link', 'Disbursement Entry', 'Document Holder', 'Go to Screen', /Macro', 'Task', 'Time Entry' and 'Workflow Link' field types to duplicate items and grouped items within the workflow step.

    • Refresh - This option is available for most field types. It is used to allow values for items to be refreshed automatically when the workflow is opened. The refresh will apply to all items AFTER the item where Refresh option is ticked. It is applicable only for checklist used as workflow step.

    • Display Name - This is available only for the field type 'Document Link' to describe the linked document.

    • Jurisdiction Key - This is available only for the field type 'Entity Type' to specify the key of a checklist item with the country value.

    • Field/Mask - This is available only for the field types 'Lookup Value', 'User Table Value', 'Record Link' and 'Drop Down Value' to specify the table and record type.

    • Multi Selection - This is available only for the field types 'Lookup Value' and 'User Table Value'. If ticked, users can select more than one values.

    • Show Search Control - This is available only for the field type 'Lookup Value' and 'User Table Value'. When ticked, the list of lookup values or user table values will be displayed in the 'Select' screen. Only ‘Multi Selection’ or ‘Show Search Control’ can be selected at a time, not both.

    • Formula - This is available only for the field types 'Lookup Value' and 'User Table Value' and is used with Multi Selection where you can apply a formula to the selections made. The available formulas are:

    • SUM - This will add the scores of the selected values.

    • AVG - This will calculate the average of the scores for the selected values.

    • MAX - This will return the biggest score for the selected values.

    • MIN - This will return the smaller score for the selected value.

    • Sort - This is available only for the field types ’Lookup Value’ and ’User Table Value’ to sort the available values alphabetically. Select ‘N’ to sort by the Name, and ‘C’ to sort by the Code. If the checkbox ’Multi Selection’ is ticked, the drop-down list will be sorted according to Name in alphabetical order.

    • Include - This is available only for the field types ’Lookup Value’ and ’User Table Value’ to filter the drop-down list of a lookup table or user table by including only certain values. The values to be included in the field can be obtained from the corresponding lookup table or user table. To include multiple values, separate each value with a ‘|’ (pipe) symbol.

    • Exclude - This is available only for the field types ’Lookup Value’ and ’User Table Value’ to filter the drop-down list of a lookup table or user table by excluding certain values. The values to be included in the field can be obtained from the corresponding lookup table or user table. To include multiple values, separate each value with a ‘|’ (pipe) symbol.

    • Screen/Type - This is available only for the field type 'Go to Screen' to specify the screen by selecting it from the list available.

    • Screen Entry - This is available only for the field type 'Go to Screen'. For screens with grids, this will give the option of command button for adding new record such as Appoint for Statutory Officers screen or New for Registrations screen, or updating information on the existing record as well as removing. When specified, information copied using Viewpoint Clipboard can be paste without the need for users to manually click Appoint or New - users can click on the icon and a new record will be created with the copied information. If this is not specified, users will still need to manually click New or Appoint command button in order to have the copied information automatically pasted.

    • Criteria - This is available only for the field type 'Go to Screen' to specify the record for screen shortcut.
      Note: This does not apply to Addresses and Statutory – Company screens.

    • Overwrite Caption - This is available only for the field type ‘Input Field’. This is to allow the captions for the checklist items within the Input Field group in workflow to be overwritten with the field captions specified in the corresponding field settings for the selected Screen/Type and/or Screen Entry.  

    • New Master File - This is available only for the field type 'Master File' and when ticked, will allow for creation of new Master File.

    • Data Source - This is available only for the field type 'Master File', and 'User Defined'. The field ‘Data Source Attribute’ can be specified for field type ‘Master File’ to further filter the list of Master Files as Individual or Non Individual. The option Show Combo List can be ticked/unticked when Data Source is defined.

    • Required Step Key - This is available only for field type ‘Required’ to specify the key for the workflow step(s). For multiple keys, separate each key with a comma symbol.

    • Required Field Key - This is available only for field type ‘Required’ to specify the key for the checklist item(s). For multiple keys, separate each key with a comma symbol.

    • Service Code - This is available only for the field type ‘Service’ to specify the list of service codes for selection for the invoice.

    • Quantity Key - This is available only for the field type ‘Service’ to specify the key of the checklist item that will contain the unit quantity for the service. This checklist item can be ‘Amount’, ‘List with Edit’, ‘List without Edit’, ‘Number’ and ‘Yes/No’.

    • Item information - This is available only for the field types 'Disbursement Entry', 'Task' and  'Time Entry' where you will need to enter the relevant information for the item.

      • Master File - This is to select the Master File.

      • Service - This is to select the Service.

      • Number of Units - This is to specify the number of units.

      • Description - This is to describe the item.

      • Task Type - This is to select the type available in the drop-down selection.

      • Task Description - This is to describe the task.

      • Assigned To - This is to specify the assignee for the task. This can be Entity File, Account File, Billing File, Supplier File’s Administrator, Manager and Controller, or the User. If this is not defined, the logged in user will be displayed by default in the workflow.

      • Date Due - This is to allow you to set the due date.

      • Note - This is to enter additional information for the item.

      • Task Type - This is to select the type available in the drop-down selection.

      • Task Description - This is to describe the task.

      • Assigned To - This is to specify the assignee for the task. This can be Entity File, Account File, Billing File, Supplier File’s Administrator, Manager and Controller, or the User. If this is not defined, the logged in user will be displayed by default in the workflow.

      • Date Due - This is to allow you to set the due date.

      • Note - This is to enter additional information for the item.

      • Task Type - This is to select the type available in the drop-down selection.

      • Task Description - This is to describe the task.

      • Assigned To - This is to specify the assignee for the task. This can be Entity File, Account File, Billing File, Supplier File’s Administrator, Manager and Controller, or the User. If this is not defined, the logged in user will be displayed by default in the workflow.

      • Date Due - This is to allow you to set the due date.

      • Note - This is to enter additional information for the item.

      • Master File - This is to select the Master File.

      • Service - This is to select the Service.

      • Number of Units - This is to specify the number of units.

      • Description - This is to describe the item.

      • Task Type - This is to select the type available in the drop-down selection.

      • Task Description - This is to describe the task.

      • Assigned To - This is to specify the assignee for the task. This can be Entity File, Account File, Billing File, Supplier File’s Administrator, Manager and Controller, or the User. If this is not defined, the logged in user will be displayed by default in the workflow.

      • Date Due - This is to allow you to set the due date.

      • Note - This is to enter additional information for the item.

    • Holder Settings - This is available only for the field type 'Document Holder' to specify the filing cabinet and index fields.

    • Store Action - This is available only for the field type 'Document Holder' to specify the actions when documents are uploaded. Each tag must be separated with a ‘|’ (pipe) symbol.

      The available tags are:

      • EXTRACTXML – This will extract the content of form into the clipboard and store it in the database.

      • REFRESH – This will refresh the workflow content.

      • AUTOSTORE  – This will store the documents automatically when the Document Holder is newly created.

    • Maximum Files - This is available only for the field type 'Document Holder' to specify the maximum number of files that is allowed to be stored into the holder.

    • File Size (MB) - This is available only for the field type 'Document Holder' to specify the maximum file size of every document that can be uploaded.

    • File Type - This is available only for the field type 'Document Holder' to specify the type of files that can be uploaded.

    • Note Editor - This is available for field types ‘Change Link’, ‘Workflow Link’, 'Macro', ‘Word Report’, ‘Excel Statement’, ‘File Review’ and 'Standard Report' to enter tags for these field types.

    • Hide on Screen - This is available for most field types except field types with Note Editor. When ticked, the item will not be displayed in the Workflow screen. For the field types with Note Editor, users may enter the [HideOnScreen=1] tag in the Note Editor screen to hide the item from the Workflow screen.

    • Hide on Report - This is available for most field types except field types with Note Editor. When ticked, the item will not be displayed in the Workflow report. For the field types with Note Editor, users may enter the [HideOnReport=1] tag in the Note Editor screen to hide the item from the Workflow report.

Setting Default Value

The default value can be a specific value or it can be codes to retrieve values from another checklist item or from the database. For the codes to retrieve values, you can use the Workflow Field Wizard. However, if SQL is used, then the field code will follow this syntax: (+<QueryName>.<Field>+).

  1. Click the Browse button.

  2. Enter the value or code in Values/Default.

  3. Click Update.

 

Defining List Options

When user is to select options from a list, this can be from a lookup table, user table, drop-down table or list created using the field type ‘List without Edit’ or ‘List with Edit’.

 

Lookup Table/User Table/Drop-down Table

  1. Set the field type accordingly.

  2. Click the Browse button.

  3. Select the table.

  4. Click Update.

Note: For Lookup table value and User table value, you can tick Multi Selection to allow more than one selection.

 

List without Edit/List with Edit

  1. Set the field type accordingly.

  2. Click the Browse button.

  3. Enter the options in Values. Separate each option with a pipe symbol '|'.

  4. If you would like to set a default value, enter this in Default.

  5. Click Update.

Note: The options can be fixed text options that you enter or it can be references to checklist items in a previous workflow step. For the latter, use (+WF.<Key>+) code to refer to the checklist item where <Key> is the key of the checklist item.

 

System User/Administrator Group/Master File

This does not require any additional setup apart from setting the field type appropriately.

 

Defining Yes/No as Options

When the field type is 'Yes/No', this will present a checkbox for the user to tick for 'Yes' or leave blank for 'No'. If you prefer for the user to select the option Yes or No, then do the following:

  1. Set the field type to 'Yes/No'.

  2. Click the Browse button.

  3. In Show List, enter Yes|No.

  4. Click Update.

 

Defining Screen

  1. Set the field type to 'Go to Screen'.

  2. Click the Browse button.

  3. Select the screen in Screen/Type.

  4. Click Update.

Note: Tags can be added to embed the screen or not. Refer to Checklist Item - Tags.

 

Defining Data Source for Master File

This allows for list of Master Files to be filtered based on pre-defined condition of Components Activated or Roles Allowed for the Master File.

  1. Set the field type to 'Master File'.

  2. Click the Browse button in the Values/Default column.

  3. Select the option for Data Source.

  4. Click Update.

If Show Combo List option is not ticked, user can:

  • Click on the magnifying glass icon to select the Master File.

  • Enter the full name of the Master File.

  • Enter the code of the Master File prefixed with a plus sign e.g. +ARCHER.

If Show Combo List option is ticked, then the list of Master Files will appear as follows:

 

Defining Data Source

This is similar to the above except that the list can be filtered based on condition(s) specified for the data source.

  1. For the workflow step, click Edit Properties to define the data source in the Tag Values field.

  2. Click Update.

  3. Set the field type to 'User Defined' or 'Master File'.

  4. Click on the Browse button in the Values/Default column.

  5. In the Data Source field, enter the name of the Data Source as defined in step 1.

  6. Click Update.

 

Accessing New Master File Wizard

  1. Set the field type to 'Master File'.

  2. Click on the Browse button in the Values/Default column.

  3. Tick the option New Master File.

  4. Click Update.

 

Defining Path for Document

If you would like the workflow to open a document file, ensure that the file is saved to the Viewpoint\Forms folder.

  1. Set the field type to 'Document Link'.

  2. Click the Browse button in the Values/Default column.

  3. In Values/Default, enter the path for the document. If the document is stored in Document Manager, use tags to specify the document.

  4. In Display Name, enter a meaningful name for the file. Otherwise, the path is displayed for the user.

  5. Click Update.

 

Defining URL for Website

  1. Set the field type to 'Document Link'.

  2. Click the Browse button.

  3. Enter the URL for the website in Values/Default.

  4. In Display Name, you can enter the name of the website. Otherwise, the URL is displayed for the user.

  5. Click Update.

Note: The URL must start with 'http' or 'www'.

 

Defining Decision

If the step is just before a decision step, you can have the input of an item in the step make the decision automatically.

  1. Click the Browse button.

  2. Tick Output option.

  3. Click Update.

  4. Select the decision step.

  5. Enter the input for a Yes decision.

    • Input Type - Select if the input is text or number.

    • If Yes - Set the value for a Yes decision. If the Input Type is Text, then the operator is In. If the Input Type is Number, then you can select these operators:

      • = means the value must be a number equal to the specified value to go to Yes branch.

      • >=  means the value must be a number that is equal to or greater than the specified value to go to Yes branch.

      • <=  means the value must be a number that is equal to or smaller than the specified value to go to Yes branch.

      • <> means the value must be a number that is not equal to the specified value to go to Yes branch.

      • > means the value must be a number that is greater than the specified value to go to Yes branch.

      • < means the value must be a number that is smaller than the specified value to go to Yes branch.

  6. Click Update.

If the item is of the 'Formula' field type, then the Output option is not seen but you can still use this item to define the decision. In this case, add the [OUTPUT=] tag. Refer to Checklist/Workflow Step - Tags.

 

Creating Conditional Group

  1. Enter a key for the checklist item that will be used as the condition.

  2. Create sub-items for the checklist item by indenting these checklist items.

  3. Set the field type for the main item to 'Conditional Group'.

  4. Click in Values/Default column.

  5. Enter the condition.

Note: The condition can be based on the value or score of the checklist item. If it is to be based on the score, specify it as <Key>.Score.

 

Defining Formula

  1. Enter a key for all checklist items that will be used in the formula.

  2. Set the field type for the checklist item to Formula.

  3. Click in Values/Default column.

  4. Enter the formula.

 

Assigning Approval Card

  1. Set the field type for the checklist item to Approval Card.

  2. Click on the Browse button in the Values/Default column.

  3. Select the Approval Cad in Approval Card.

  4. Click Update.

 

Defining Linked Workflow

  1. Set the field type for the checklist item to Workflow Link.

  2. Click on the Browse button in the Values/Default column.

  3. In the Note Editor, enter details for the other workflow.

  4. Click OK.

Note: Refer to Workflow Link - Tags.

 

Defining Macro

  1. Set the field type for the checklist item to Macros.

  2. Click on the Browse button in the Values/Default column.

  3. In the Note Editor, enter the tags for the macro(s).

  4. Click OK.
