Databases

In this screen, you can define the Back-office database that you want to be connected to the Engage Servicing Portal database (CP-DB). Note that you can define more than one Viewpoint databases.

 

Filters

The filters apply to the database list.

  • Database status - choose to view only active, inactive or all databases.

  • Database Purpose - choose to view a specific database purpose or all.

 

Database list

The list shows the databases set up for Engage Servicing Portal.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • New - Click this to define the Viewpoint database to be connected. All the newly added database record will have the record status 'New' until the record is approved via the Approve button.

    1. Enter the details.

    2. Click Update.

    Note: You can click on Test Connection button to test the connection with the Back-office.

  • Edit - Click this to edit the selected database record. This will set the record status to 'Edited' until it is approved. If you are editing a database record that is to be deleted but deletion remains unapproved, you will be prompted to confirm if you want to continue with the editing.

  • Delete - Click this to remove the selected database record. If the database is not linked to any Organisation, the Approve button will be enabled. Note that if the deletion is not approved, the 'Record Status' will be set to 'Deleted Record'.

  • Password - Click this to set or change the password for the Database Service Address.

  • Approve - Click this to approve addition of new database record, edit or deletion of a database record. User must have at least '8 - Power User' authority to this screen in order to approve. Self-approval is possible, i.e. you can approve database record that you have entered yourself.
