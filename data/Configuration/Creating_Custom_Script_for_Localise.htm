Creating Custom Script for Localise Function

You can create customised scripts to make the database changes that you want to apply provided you have sufficient authority. Create your custom script in any text file and then copy and paste it to the Edit script: <Settings> screen.

You will need to have the necessary access rights in order to use and/or customise the feature. To verify that you have the required access, do the following:

  1. In Configuration, go to Users & Security > Security Setup > User Group.

  2. Click View Authority.

  3. Select the Configuration tab, select the Windows Menus radio button and then scroll down to Settings > Tools & Utilities > Localise.

 

You will need at least authority level 6 – Regular User to run the Localise function, and 8 – Power User to customise the scripts.

The following will explain the tags used in custom scripts:

Tag

Usage

CAPTION

This must be defined in the FIRST line of the script.

Syntax:

CAPTION:<Caption Text>

Example:

CAPTION:Set the Workflow ratings on Country codes

Tags

There are specific tags to be entered depending on what you want the script to do. Find out more in the appropriate section:

  • System Defaults

  • Menus

  • Fields

  • Lookup Values

  • User Table Values

  • User Fields

  • Relationship Types

  • Identity/AEOI Register Types

  • Owners and Controllers Types

  • Index Fields and Filing Cabinets

  • Task Types

  • Risk Ratings

  • Link Type for Notifications

Remarks

Remarks can be added if you want to explain what a particular section of the script is meant to do.
