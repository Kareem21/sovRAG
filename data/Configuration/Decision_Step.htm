Decision Step

For each workflow template, a decision step is automatically included in the Step list. This decision step allows for two-way branching in the workflow where the flow can go in either one way. The decision step can be used multiple times in the workflow by adding it to the relevant order in the workflow diagram either by drag and drop or using the Add to Link.

When a decision step is added, it is also possible to automate the decision:

  • If the decision is not automated, the user will arrive at the decision step and be able to select the branch by clicking on command buttons that represent the Yes and No branch.

  • If the decision is automated, then the user will not encounter the decision. Instead, the decision will be based on a checklist item within the workflow step just before the decision step.

Changing Caption for Decision Step

When added to the workflow diagram, the default caption will be:

  • Decision for the decision step

  • Yes and No for the branches

These captions can be changed by doing the following:

  1. In the workflow diagram, click to select the decision step.

  2. In Step details, enter the caption in the Completed Field column for:

    • Decision - caption for the decision step shape in the workflow diagram

    • Yes - caption for Yes branch

    • No - caption for No branch

  3. Click Update to save the changes.

Note: The caption change for Yes and No branches will also apply to the command buttons seen if the decision is not automated.

 

Automating Decision Step

By automating a decision step, there is no need for the user to manually select the Yes or No branch. Instead, the required value that will decide on the branch will be a checklist item that the user has completed in the workflow step before the decision step. This requires the following:

  • define the item as a source for the decision step

  • define the value for the Yes branch

Define Item as Source for Decision Step

If the workflow step has one checklist item that will be the source for the decision step, then this can be indicated by ticking the Output checkbox for the checklist item.

  1. In the workflow step containing the item, select the item.

  2. In Values/Default, tick the Output checkbox.

  3. Click Update.

If the workflow step contains conditional groups and within each group is a checklist item that will be the source for the decision step, then do the following:

  1. In the workflow step, assign the same key to these checklist items and click Update.

  2. Click Edit Properties for this workflow step.

  3. Add the [OUTPUT=<Key>] tag in Tag Values with the <Key> referring to the key assigned in step 1.

  4. Click Update.

Note: If it is the score of the checklist item that is to be the source for the decision step, the tag is [OUTPUT=<Key>.SCORE].

 

Define the Value for the Yes Branch

  1. In the workflow diagram, click to select the decision step.

  2. In the Step details, specify the following:

    • Input Type - in Complete Field column, select if the Input is Text or Number

    • If Yes - in Complete Field column, select the operator, and in the Value field, enter the value for the Yes branch.

  3. Click Update.
