Checklist Item – Field Type: Document Link

Document allows for linking to a file or a website. For file, this can be a file that is stored in Document Manager or a file in a network path.

If the file is stored in Document Manager, then the following tags are available:

Code

Explanation

@VPFILE:<DocumentId>

where <DocumentId> is the number assigned to the document in the holder. This will be in the format of HolderNr.SequenceNr.

This will link the item to the specified document within the holder. Clicking on the icon will open the document file.

e.g. @VPFILE:358.1 means the first document stored in holder number 358.

 

@VPHOLDER:<HolderNr>

where <HolderNr> is the number of the holder.

This will link the item to the specified document holder. Clicking on the icon will open the document holder.

e.g. @VPHOLDER:357 means holder number 357.

 

If the file is not stored in Document Manager, then specify the location of the file in Values/Default.

Method

Explanation

UNC Path

This is the path to where the file is located.

e.g. \\VPMYN171\Checklists\Compliance Info.pdf

$FORMS$

This is used if the file is in Viewpoint’s Forms folder.

e.g. $FORMS$\Compliance\Compliance Questionnaire.pdf

For linking to a website, you need to specify the website’s URL in the Values/Default column where it must start with ‘http’ or ‘www’.
