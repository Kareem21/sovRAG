Document Store Validation and Update

In this screen, you can run two functions:

  • OCR Update - Which will recognise text in stored documents and images so the text can be used to locate specific documents.

  • Compression - Which will compress stored document files.

  • Revision - Which will create or undo revision groups in holders.

You can perform the both functions together or separately. However, note the following:

  • OCR update can only be performed on files that have not been compressed.

  • Compression is only for environments where Document Store is in file structure. It is not applicable if SQL database is used for Document Store.

  • Compression is not possible for the following file types: .jpg, .jpeg, .png, .gif, .tif, .tiff, .eps, .pdf, .zip, .rar and .7z

 

Select Holder Range

  • All holders (Require longer processing time).

  • Document holder range, Holder number - from and - to - Enter the range of document holders you want.

 

OCR Update

Tick to select OCR Update function and tick the document types you want to run the function on. Note that when using OCR Transym, the sequence will be the pdf text first, followed by the text in the image.

 

Note: For multi-paged TIF files, pages with JPEG compression are not supported and will not be recognised when performing OCR updates. Only pages with other compression types will be recognised.

 

 

Compression

Tick to select the Compression function and specify the minimum file size for compression. If this option is not active, it may be that the Document Store is an SQL database. Note also that if you selected All holders, this option will also be inactive. This is to ensure that you specify correctly the range of document holders for compression as compression is not reversible.

 

Revision

This allows you to apply revision management on existing holders. Tick to enable the revision grouping options where you can choose:

  • Build revision groups - To create revision groups for holders without any revision groups.

  • Includes holder with existing revision groups - Tick this to create new revision groups and rebuild existing ones according to the algorithm.

  • Undo revision groups - To remove all revision groups and all files and revision will be shown in the holder details.

Find out more on Revision Management here.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • Start - Click this to start the selected function(s). Ensure that you are the one person logged in to the database.

    Performing OCR Update

    This may take some time if you have a large database of stored documents. It is therefore recommended that you run this at the end of the day to avoid interruptions.

    1. Select the document holders for the OCR Update.

    2. Tick OCR Update.

    3. Tick document types that you want to include.

    4. Click Start.

    5. Click Yes to confirm.

    6. Click OK.

     

    Performing Compression

    1. Select the document holders you want to compress.

    2. Tick Compression.

    3. Enter the minimum file size for compression.

    4. Click Start.

    5. Click Yes to confirm.

    6. Type 'yes do compress' and click OK.

    7. Once completed, a log file - DMValidationAndUpdate_<DatePerformed>.csv - will be created and saved to Viewpoint\Program folder.

    8. Click OK.

     

    Performing OCR Update and Compression

    Once a document file has been compressed, it is no longer possible to perform OCR update on it. As such, if there is a need to perform OCR update and compress the document file, you can do that at the same time.

    1. Select the document holders.

    2. Tick OCR Update and select the document type.

    3. Tick Compression and enter the minimum file size for compression.

    4. Click Start.

    5. Click Yes to confirm.

    6. Type 'yes do compress' and click OK.

    7. Once completed, a log file will be created and saved to Viewpoint\Program folder.

    8. Click OK.

     

    Applying Revision Management to Holders

    1. Select the document holders.

    2. Tick Revision.

    3. Select one of the following options.

      • Build revision groups. The 'Includes holder with existing revision groups' option is optional.

      • Undo revision groups.

    4. Click Start.

    5. Click Yes to confirm.

    6. Once completed, a log file will be created and saved to Viewpoint\Program folder.

    7. Click OK.

     

     

  • Cancel - Click this to stop the process.
