User Roles

'User Roles’ is used to enable a ‘role-based’ access to various ESP menus. This allows different user types to have different authority level to each menu. The User Role is maintained in the Front-office Lookup Table 'User Role (ESP) [CPRL]' where administrator can create new User Role and activate it by enabling the 'Active' option. Only the active User Role will show in this screen.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • Authorities - This sets the authority for the User Role and allows you to customise the menu order for different User Role.

    Order (sort) the menus by using the Up, Down, Left and Right buttons.

    • Up - This will move the menu up in the list.

    • Down - This will move the menu down in the list.

    • Left - This will promote the menu as header (parent) menu.

    • Right - This will demote the menu to child menu. The menu can be demoted up to a maximum 3 levels (Parent – child – child).

    To set the menu authority for the User Role,

    1. Select the User Role and click on the Authorities button.

    2. For each of the menu tab, set the Authority for the menus in the Access column, by tick/untick the checkbox to grant user’s access to the menu.

    3. Click Save when done.

  • Reset - This will reset the menu to the standard default position/hierarchy structure based on the database table value.

  • Copy - This will create a new User Role by copying the menu settings from an existing User Role.

  • Validate - This will validate the menu against invalid hierarchy structure and authorisation setting.
