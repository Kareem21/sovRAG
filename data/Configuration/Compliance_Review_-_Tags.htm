Compliance Review - Tags

For Compliance reviews, tags are available to check on fields in screens such as Addresses, Identity Details, Identity Register and CDD; as well as update the status fields in the CDD screen.

The tags are to be entered to the Compliance review rule with the code ‘KYREVIEW- such as Compliance Review Rules KYCREVIEWA, KYCREVIEWB, KYREVIEWC, KYREVIEWD, KYREVIEWE and KYCREVIEWF -’ and enable the following:

  • Define which Master Files to include for Compliance review based on the Identity Details and CDD Required options.

  • Define which Master Files to include for Compliance review based on status fields in CDD screen.

  • Specify values that need to be flagged.

  • Update status fields in the CDD screen.

 

Identity Details and CDD Required Options

The Identity Details and CDD Required options are in a Master File’s Setup screen. By default, Compliance review will be performed on Master Files where the CDD Required option is ticked. However, you can use the tags below to specify which Master Files to include for the Compliance review.

Purpose

Syntax

Example and Explanation

Include Master Files based on Identity Details option

[RVW.MF.CP:<Value>]

[RVW.MF.CP:0]

Include Master Files even when the Identity Details option is not ticked.

 

[RVW.MF.CP:1]

Include Master Files when the Identity Details option is ticked.

Include Master Files based on CDD Required option

[RVW.MF.KY:<Value>]

 

[RVW.MF.KY:0]

Include Master Files even when the CDD Required option is not ticked.

 

Compliance Review Status Field in CDD Screen

Six status fields are available in the CDD screen – one each for the Compliance review rule type. The following tags can be used to include and/or update these compliance review status fields.

Purpose

Syntax

Example and Explanation

Specify status to be included for review

[RVW.Include:<StatusCode>]

 

[RVW.Include:P,V,O,-]

If the compliance review status field is set to P,V,O or left empty, the compliance review will be performed on the Master File.

To propose new status if match is found

[RVW.Find.Code:<StatusCode>]

[RVW.Find.Code:V]

Update the compliance review status field to 'In scope for review [V]' when there is a match when you choose to post update.

[RVW.Find.MSG:<Message to display>]

 

[RVW.Find.MSG:Found possible association - update to In Scope for review]

Display this message when there is a match.

To propose new status if the status is empty

[RVW.Empty.Code:<StatusCode>]

 

[RVW.Empty.Code:P]

Update the compliance review status field to 'Pending [P]' if it is empty during the review.

[RVW.Empty.MSG:<Message to display>]

[RVW.Empty.MSG:Review status is empty - update to Pending]

Display this message when there is no value in the compliance review status field.

To propose new status if there is no match

[RVW.NoFind.Code:<StatusCode>]

[RVW.NoFind.Code:N]

If there is no match, the compliance review status field will be updated to ‘Not Required[N]’ if you choose to post update.

[RVW.NoFind.MSG:<Message to display>]

[RVW.NoFind.MSG:There is no match – update to Not Required]

This message will be displayed for files without any match.

 

Note: The [RVW.Find.Code:<StatusCode>] and [RVW.Empty.Code:<StatusCode>] tags require the corresponding Compliance review rules with the review code ‘KYREVUPD’ to be active in order to work e.g. if the tags are defined for Compliance review rule ‘KYREVIEWA’, then the Compliance review rule ‘KYREVUPDA’ must be active. Otherwise, the review will not produce any update items.

 

Fields in Addresses, Identity Details, Identity Register and CDD Screens

These tags allow you to define values for addresses, contact numbers and various other fields that need to be flagged.

 

Country

The country tag applies to:

  • Addresses.

  • The ‘Deemed Residence’ and ‘Domicile’ fields in CDD screen.

  • The Country/Jurisdiction field for an Identity Register type in the Identity Register screen.

Purpose

Syntax

Example and Explanation

Specify country code that should be flagged during the review

[RVW.CNTR:<CountryCode>]

[RVW.CNTR:USA,US,US-DE,US-NV,US-NY]

This will flag files with the specified country codes used in addresses, the fields Deemed Residence and Domicile and/or the Contry/Jurisdiction field for an Identity Register type.

Specify address types to be excluded from the review

[RVW.Address.Exclude:<AddressType>]

[RVW.Address.Exclude:ST,SM]

This code is to exclude the address type ‘ST’ and ‘SM’.

Specify the Identity Register types to include in the review

[RVW.IDREG.Review:<Code>]

where <Code> refers to the Identity Register Type code

[RVW.IDREG.Review:PSP,BCT]

This code will check on the Country/Jurisdiction field for the Identity Register Type ‘Passport [PSP]’ and ‘Birth Certificate [BCT]’.

 

[RVW.IDREG.Review:*]

This code will check on the Country/Jurisdiction field for all Identity Register Types.

Display message when specified country code is found

[RVW.Address.MSG:<Message to display>]

[RVW.Address.MSG:US Address found (-CODE-)]

[RVW.KYC_Country1.MSG:<Message to display>]

[RVW.KYC_Country1.MSG:Deemed Residence is in (-CODE-)]

 

[RVW.KYC_Country2.MSG:<Message to display>]

[RVW.KYC_Country2.MSG:Domiciled in (-CODE-)]

 

[RVW.IDREG.MSG:<Message to display>]

[RVW.IDREG.MSG: (-Description-) issued in the United States]

 

The defined message will be displayed if the specified country code is found in the file.

Note: KYC_Country1 refers to Deemed Residence field, KYC_Country2 refers to Domicile field.

 

Nationality

The tags below are for nationality-related fields in Identity Details screen.

Purpose

Syntax

Example and Explanation

Specify value for the Nationality field that should be flagged during the review

[RVW.Nationality:<Text>]

[RVW.Nationality:Amer,USA,States]

This code will check if the value for Nationality field contains the text ‘Amer, USA, States’.

[RVW.Nationality.MSG:<Message to display>]

[RVW.Nationality.MSG:Nationality requires review  (-CODE-)]

This message will be displayed if the Nationality field contains the text ‘Amer, USA, States’.

Specify value for the Nationality Code field that should be flagged during the review

[RVW.NationalityCode:<Code>]

[RVW.NationalityCode:503,721]

This code will check if the code assigned to Nationality Code field is 503 or 721. The codes are from the lookup tables Nationality Individual Codes [NATI] and Nationality Corporate Codes [NATC].

[RVW.NationalityCode.MSG:<Message to display>]

[RVW.NationalityCode.MSG:Nationality code requires review  (-CODE-)]

This message will be displayed if the Nationality Code is set to 503 or 271.

Specify value for the Nationality of Origin field that should be flagged during the review

[RVW.NationalityOrigin:<Text>]

[RVW.NationalityOrigin:Amer,USA,States]

This code will check if the value for Nationality of Origin field contains the text ‘America, USA, States’.

[RVW.NationalityOrigin.MSG:<Message to display>]

[RVW.NationalityOrigin.MSG:Nationality requires review  (-CODE-)]

This message will be displayed if the Nationality of Origin field contains the text ‘Amer, USA, States’.

Contact Numbers

The following tags apply to contact numbers in Addresses screen.

Purpose

Syntax

Example and Explanation

Specify type of contact number to include in the review.

[TEL.Type:<Type>]

[TEL.Type:T,F]

This will include telephone numbers (indicated by T) and fax numbers (indicated by F).

Specify the prefix (for international numbers) to include in the review. To include numbers without any prefix, use ‘-‘.

[TEL.INT:<Prefix>]

[TEL.INT:00,0,+,-]

This code will review numbers starting with 00, 0 or +; and also numbers without any prefix (indicated by -).

Specify the number following the prefix to include in the review.

[TEL.Review:<Number>]

[TEL.Review:1]

This code will flag numbers starting with ‘1’ as a USA number.

Specify the numbers that will be excluded

[TEL.EXCL:<Number>]

[TEL.EXCL:1264,1268,1242,1246,1441,1284,1345,1767,1809,1829,1849,1473,1876,1664,1869,1758,

1784,1721,1868,1649,1403,1587,1780,1825,1236,1250,1604,1672,1778,1204,1431,1506,1709,1902,

1226,1249,1289,1343,1365,1416,1437,1519,1613,1647,1705,1807,1905,1418,1438,1450,1514,1579,

1581,1819,1873,1306,1639,1867]

This code will exclude the specified numbers from being flagged as a US number.

 

Wildcard

For text values, note that this is not case-sensitive and you can use the following wildcard options:

Purpose

Syntax

Example and Explanation

<text>

This will search for exact match to the specified text.

[RVW.PasPlaceIssue:<US>]

This will flag Master Files where Place of Issue is ‘US’ only.

<text

This will search for values starting with the specified text.

[RVW.PasPlaceIssue:<US]

This will flag Master Files where Place of Issue starts with ‘us’ such as USA.

text>

This will search for values ending with the specified text

[RVW.PasPlaceIssue:US>]

This will flag Master Files where Place of Issue ends with ‘US’ such as Cyprus.

text

This will search for values containing the specified text.

[RVW.PasPlaceIssue:US]

This will flag Master Files where Place of Issue contains ‘US’ such as USA, Brussels, Lausanne and Cyprus.
