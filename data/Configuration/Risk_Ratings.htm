Risk Ratings

Risk Ratings is how you can set scores required for checklists and workflows. The scores are for values in the following items:

  • Lookup Tables

  • User Tables

  • Identity/AEOI Register Types

  • Relationship Types

  • Service Master

To do this, the following steps are required:

  • The Risk Rating option must be ticked in the respective item.

    For Lookup Tables and User Tables, click Edit on the top grid.

    Tick the Risk Rating option.

     

    For Identity Register Types, Relationship Types and Service Master, select the relevant record and click Edit.

    Tick the Risk Rating option.

     

  • The score is set in the Risk Ratings screen.

    1. In Viewpoint - Home, go to Configuration > Business Parameters > General > Risk Ratings.

    2. Once you have specified the group, table and type, enter the score in the Value field.

    3. Click Update.

For Lookup Table and User Table values, if you had scores assigned using the Score Tags earlier, then you can use the Convert function in Risk Ratings screen.

 

Risk Rating Option

This option is available in the following screens:

  • Configuration > Business Parameters > General > Lookup Values.

  • Configuration > Business Parameters > General > User Tables/Values.

  • Configuration > Business Parameters > Entity Administration > Identity/AEOI Register Types.

  • Configuration > Business Parameters > Entity Administration > Relationship Types.

  • Configuration > Business Parameters > Practice Management > Service Master.

For Lookup Tables and User Tables, ticking the option is at the table level and will apply to all values of the table. For the other three, the option is available for the record and only applies to the record itself.

 

Enabling Risk Rating Option

If you are unable to see the Risk Rating option in the screen, it means that the option is turned off. In this case, go to Configuration > Settings > Customise > Field Settings to enable it.

For instance, to enable the option for Service Master, do the following:

  1. In Viewpoint - Home, go to Configuration > Settings > Customise > Field Settings for Service Master.

  2. Click to select Risk Rating.

  3. Click Edit.

  4. Set the Auth. Req. for Update to Regular User [6].

  5. Click Update.

 

Risk Ratings Screen

The Risk Ratings screen is located in Configuration > Business Parameters > General > Risk Ratings. If you are not able to see the screen, then check the following:

  • In Configuration > Settings > Customise > Windows, check to ensure the screen is Active.

  • In Configuration > Users & Security > User Roles, check to ensure that authority is given to the screen for the required User Role(s).

 

Rating Group

This refers to the scoring system where the default Standard [STDR] is the same as assigning scores using [WSCORE=] tag. If you require additional scoring system, then new group can be set up in the Risk Rating Groups [RRGR] Lookup Table.

 

Table

This will list out items where the Risk Rating option is ticked.

 

Type

By default, this shows Current [C] which means the current score. The other options available are:

  • Prepare Revision [R]

  • Scenario Evaluation [S]

The different types allow you to have another set of scores within the same scoring system for comparison.

 

Compare With

This is to select the type for comparison.

 

Value List

This shows value(s) that require scores to be set. You need to click on Refresh to view the value(s). In the list, you have the following:

  • Code - This is the code of the table value or selected record.

  • Name - This is the name/description of the table value or selected record.

  • Effective - This reflects the effective date of the score set and is populated when an effective date is set and updated.

  • Value - This reflects the score being set for the value/record and is populated once a score has been set and updated.

  • Change to Effective - This allows you to enter the effective date of the score.

  • Value - Score for the value/record is entered here.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • Convert - Click this to convert scores assigned to Lookup Table values and User Tables values using the [<ScoreSystem>=] tag in earlier version. This is available when Rating Group is other than Standard [STDR].

  • Refresh - Click this to load the selected record or values of the specified Lookup Table or User Table.

  • Copy - Click this to copy the scores set for a Rating Group/Type  to another Rating Group/Type. This can be used as a basis for new Rating Group/Type where you copy existing scores and make changes where relevant.

    1. Select the Rating Group/Type.

    2. Click Copy.

    3. In the Copy Risk Ratings Score From screen, specify the Rating Group and/or Type to copy from.

    4. Click OK.

  • Update - Click this to save the changes made.

  • Cancel - Click this to cancel the changes made.

 

Conversion from Score Tags

The use of [WSCORE=] and [<ScoreSystem>=] tags to assign scores to Lookup Table value and User Table values is no longer possible as this now needs to be done through Risk Ratings. However, functions are available to make the conversion for you and these are:

  • Upgrade Risk Ratings function in Configuration > Database Archiving > Troubleshooting.

  • Convert command button in Configuration > Business Parameters > General > Risk Ratings.

 

Using Upgrade Risk Ratings

This will work for scores assigned using:

  • [WSCORE] tag where the scores will be retrieved and updated to the Standard [STDR] Rating Group.

  • [<ScoreSystem>=] if the length of <ScoreSystem> is less than 5 characters where a Rating Group will be created automatically and the scores will be retrieved and updated to the new Rating Group. If the length of <ScoreSystem> is five characters or longer, then you need to use the Convert command button.

This is to be done after you have ensured that you are able to access Risk Ratings screen.

  1. In Viewpoint - Home, go to Configuration > Database Archiving > Troubleshooting.

  2. Click Upgrade Risk Ratings.

  3. Enter an effective date.

  4. Click Update.

  5. When complete, click OK.

Once completed, the conversion will automatically tick the Risk Rating option for Lookup Tables and User Tables if scores were retrieved from the table and updated to Risk Ratings screen.

 

Using Convert Command Button

For scoring system where the name/description is longer than five characters, you need to use this to get the scores into the Risk Ratings screen. This requires you to set up the <ScoreSystem> as its own Rating Group.

  1. In Viewpoint - Home, go to Configuration > Business Parameters > General > Lookup Values.

  2. Select Risk Rating Groups [RRGR[ Lookup Table.

  3. Click Edit in the bottom grid to add the <ScoreSystem> as a new Rating Group. Note that the code can only be a maximum of four characters.

  4. Click Update.

  5. Exit Viewpoint and log in again.

  6. Go to Configuration > Business Parameters > General > Risk Ratings.

  7. Select the Rating Group created earlier.

  8. Click Convert.

  9. Enter the <ScoreSystem> in the Score Tag field.

  10. Click Update.

  11. When complete, click OK.

  12. Click Refresh to see the scores.

Doing this will also update the existing [SCORETAG=] in the Tag Values field for the checklist/workflow step to show the new code used for the scoring system.

 

Using Localise Script

Entry to the Risk Ratings screen is also possible through the use of Localise Script. This will be a custom Localise Script with tags for Risk Rating. Refer to Localise Script - Risk Ratings.
