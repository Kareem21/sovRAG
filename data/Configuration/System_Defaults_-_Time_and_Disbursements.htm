System Defaults - Time and Disbursements

The defaults for Time and Disbursements are set up here.

 

Time Unit of Measure

This is the standard value in which all time entries are entered. For example, if you choose Hour(s) [1], one unit of time will represent one hour and if you wish to enter ten minutes you must enter it as one sixth of a unit or 0.16. The options available as per below:

  • Hour(s) [1]
  • Minute(s) [2]
  • 10 Minute(s) [3]
  • 6 Minute(s) [4]
  • 15 Minute(s) [5]
  • 5 Minute(s) [6]

Note: The time unit of measure cannot be changed once the system has been set up so it is vital that you consider your choice carefully and select the most appropriate unit from the system standards.

 

Blocked Matters - Time Service

This is to set a Time - Administrator Rate [TA] service code for time entries that will be logged for Matters with status 2 and 3.

 

Timesheet - Default Service for:

Public Holiday

This is to set the default service for the time entry that will be created when you click Public Holiday in the Timesheet screen.

Note: If there is no service specified, the Public Holiday command button will not be available.

 

Leave

This is to set the default service for the time entry that will be created when you click Leave in the Timesheet screen.

Note: If there is no service specified, the Leave command button will not be available.

 

Internal Meeting

This is to set the default service for the time entry that will be created when you click Internal Meetings in the Timesheet screen.

Note: If there is no service specified, the Internal Meetings command button will not be available.

 

Time

Allow All Services for Time Entry

Tick to allow all services created to be used for Time Entry. The option CANNOT be used in conjunction with the option Link Services to Matter Types (see above). If you tick this, then services in the Disbursement service class are charged at Administrator rate when entered in the Time Entry screen.

See also:

  • Service Rates

 

Use Recurring Services for Time Entries

Tick to enable the selection of recurring fees for Time entries. This will limit the services that the user can select in Time entries to the recurring fees set up for the Billing File. Services other than those in the 'Time' service class will be charged at the prevailing administrator rate, while the Time - Non Chargeable [TN] services will not be charged, as normal. You will need to set the Number of Units and Unit Price to 0 when setting up the services in the Recurring Fees screen in Sales Ledger. This is done automatically if the Sales Ledger in Service Master is unticked.

See also:

  • Recurring Fees - Time and Disbursement Services

 

Use Recurring Services for Time Postings

This option can be used together with the option 'Use Recurring Services for Time Entries'.

When 'Use Recurring Services for Time Entries' and 'Use Recurring Services for Time Posting' options are ticked:

  • users can choose to enter Time services in either Recurring Fees or Transaction Charging screen.

  • users would use Recurring Fees and Transaction Charging when they want to separate the authority i.e. Finances to set up Recurring Fees and Administrators to set up Transaction Charging.

If this option is not ticked, the draft invoice will include time entries that are within the date range specified for the Posting of WIP. The Date from and Date to for the invoice line will also follow the date range specified for the Posting of WIP.

If ticked, draft invoices prepared using Posting of WIP will:

  • include approved time entries dated BEFORE the Next Invoicing Due set for the service in the Recurring Fees screen.

  • NOT include time entries that are AFTER the Next Invoicing Due set for the service in the Recurring Fees screen.

In this case, if the Entry Date is earlier than Recurring Fees From Date, the system will roll one period before (based on Charge Frequency) of Recurring Fees From Date and assign the date as From Date of the Invoice Line.

[Date - from] = [Recurring Fees From Date] - [Charge per]

Example: When Recurring Fees From Date = 01 Jan 2019, Charge per = Annually, and earliest time entry is dated 01 Jan 2018, the [Date - from] will be 01 Jan 2018.

 

If the Entry Date is equal to or later than Recurring Fees From Date, the system will assign Recurring Fees From Date as the From Date of the Invoice Line.

Example: When Recurring Fees From Date = 01 Jan 2019, Charge per = Annually, and earliest time entry is dated 03 Jan 2019, the [Date - from] will be 01 Jan 2019.

 

If Next Invoicing Due of Recurring Fees is equal to or earlier then Recurring Fees To Date, the system will assign the Earliest Entry Date as the From Date of the Invoice Line.

Example: When Recurring Fees To Date = 31 Dec 2019, Recurring Fees Next Invoicing Due = 01 Jan 2019, earliest time entry is dated 31 Dec 2018, the [Date - from] will be 31 Dec 2018.

 

 

Allow Special Function on Approved Time

This option will apply the Special function on time entries that have been approved. If this option is not ticked, the Special function can only be applied on time entries that have not been approved.

 

Enable Analysis Code

Tick this to enable the Analysis Code field in Time Entries screen. This option is useful if you need to further analyse the allocation of time spent by users.

 

Default Description with Analysis Code

Tick this to have the description for time entry default to the Analysis Code's description.

 

Set Administrator Rates by Rate Level

Tick this if you are using Rate Level instead of Administrator Rate. You can create new Rate Level list in the Service Rate Admin Levels [SRAL] lookup value table.

 

Use Client Fee Agreement for Administrator Rates

Tick to apply rates in Client Fee Agreement for services of the service class 'TA'.

 

Use Budget & Rates for Administrators

Tick this to allow different time rates to be set up for Billing Files. The rates are to be set up in Budget & Rates screen.

 

Hide Unit/Total Cost

Tick this to hide the fields 'Unit Cost' and 'Total Cost' in the Time Entries screen.

 

Disallow Negative Entry

Tick this option to disallow negative value for the total amount of time entries. This option is not ticked by default.

 

Disallow Entry when 'Time Charge Limit' Exceeded

Tick this to disallow time entries if it exceeds the limit set for the Matter or Recurring Fees (whichever limit that exceeds first). If unticked, the time entry can still be added.

 

Show 'Time Clocks' in Tools

Tick this to enable Time Clocks tab in Tools. This will be seen only in Viewpoint - Home.

 

Time Management

Enable Time Management

Tick this to enable the Time Management feature. This will display the Summary folder in Timesheet screen and the date selection will default to the current week. If this option is not ticked, date validation for the time entries will use the dates for WIP Time in Posting Control Settings.

 

Create Two Timesheets when Month Ends on a Weekday

Tick this to create two timesheets for the week instead of one. This applies only when the end of the month falls on a weekday and the start of the next month is also a weekday. Both timesheets will have the same week number e.g.

  • 49-2015A (period from 30/11/2015 to 30/11/2015)

  • 49-2015B (period from 1/12/2015 to 6/12/2015)

Approve Time Entries on Timesheet Approval

Tick this to have related time entries approved when the timesheet is approved. This removes the need to approve the time entries separately.

 

Require Approval for Weekly Timesheet

Tick this if administrators are required to submit their timesheet for approval. This will set the timesheet status to 'Waiting for Approval' after submission. If this is not ticked, the timesheet will be automatically approved and closed.

 

Disbursement

D Class Services are Exclusively for Disbursements

Tick this to allow services of Service Classes starting with 'D' to be selected ONLY when creating Disbursement entries.

 

PO Recovery Code must exist as Recurring Fee

This option needs to be enabled with ‘Use Recurring Services for Disbursement Entries’ to be effective. After enabled, when performing Add Recovery in Purchase Ledger > Invoicing > Create/Edit Purchase Orders screen, the PO Recovery Code by the Supplier Entity will be validated against the Service Code of the Billing Entity Recurring Fee. The PO Recovery Code by the Supplier Entity must exist as a Service Code of the Billing Entity Recurring Fee in order for the Add Recovery to proceed.

 

Other Options

 

All Time/Disbursement Entries Link to an Activity

Tick this if you want to link Time/Disbursement entries to an activity.

 

Allow T&D Code must Exist as Budgeted Item

Tick this if Time/Disbursement must exist as budgeted item.

 

Allow Input with Decimal Places

Tick this if you want to enter in parts of a unit.

 

Default with Service Description

Tick this to use the service description for the time or disbursement entry.

 

Require Approval for Time & Disbursement Entries

Time and disbursements can only be posted to a draft invoice if they are approved. If your approval system requires the approval of time spent and disbursements entered before they are posted, tick the check box. If this option is not ticked, all time and disbursement entries are approved by default.

 

Reverse Time & Disbursement on Cancel Invoice

Tick this if you want time and disbursement entries to be reversed when you cancel an invoice. Posted time and disbursement entries will be offset with negative entries with the same activity date as the original entries, and new 'replacement' entries will be created in Time and Disbursements with the same activity date as the original. If you cancel the posting of the credit note during the invoice cancellation, the entries will not be made.

 

Command buttons:

Click on the relevant command button to perform the necessary.

  • Update - Click this to save changes.

  • Cancel - Click this to cancel the changes.
