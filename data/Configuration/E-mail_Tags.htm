E-mail Tags

E-mail tags can be added to word reports templates before the wording of the actual report. These tags are recognised (and removed) when you select the Convert e-mail formats and <option> during the generation of the report.

 

Code

Explanation

[$Mail]

Place this code at the beginning of the document. Any text before this code is ignored when the document is converted to e-mail.

[$Subject:]

This is the subject line of the e-mail after it is prepared. You can use any variable in the subject line, such as (+HD.EntClient\#AD-1-0+), provided this is returned by the header statement i.e., the [/HD-SQL=SELECT.../] line, found in the Key Information field of the template.

For example: [$Subject: Account Statement (+HD.EntClient\#AD-1-0+)]

[$Sender:]

This is to set the e-mail address of the sender and it only accepts the e-mail address which must also be an account already set up in Outlook.Otherwise, it will still default to the current selected e-mail account in Outlook.

[$Recipient_to:]

This is the recipient e-mail address that will appear in 'To' field. You can use code to retrieve e-mail address as it is recorded in Master File - Addresses - Contacts.

For example: [$Recipient_to:(+HD.EntClient\#CT-EB-1+)]

Note: It is possible to have multiple recipients.

[$Recipient_cc:]

If copies are to be sent, you can enter the recipients here. This will appear in 'cc' field.

For example: [$Recipient_cc:(+DF.USER.MailID+)]

Note: It is possible to have multiple recipients.

[$Recipient_bcc:]

If there are blind copies to be sent, you can enter the recipients here. This will appear in the 'bcc' field.

For example: [$Recipient_bcc:admin@manford.com]

Note: It is possible to have multiple recipients.

[$Attach_DOC:]

Any text, together with its formatting, below this code, will be generated as a separate document file and attached to the e-mail.

For example: [$Attach_DOC:(+HD.EntClient+) Client Statement (+HD.RevEntity+) (+HD.CurCode+)]

[$Attach_PDF:]

Any text, together with its formatting, below this code will be generated as a separate PDF file and attached to the e-mail.

For example: [$Attach_PDF:(+HD.EntClient+) Client Statement (+HD.RevEntity+) (+HD.CurCode+)]
