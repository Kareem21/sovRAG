Electronic PDF Forms

Electronic PDF forms are PDF files which accept input i.e. fillable forms. These forms generally contain fixed text which cannot be edited for the form title, heading or field labels; and control elements that allow information to be entered.

Such forms can be generated from Viewpoint with all or some of the information completed by having Viewpoint retrieve the information if they are already in the database. Other information can then be completed by the user on the form itself.

In order for this, you need to have the template for the form in Viewpoint so that the following can be take place:

  • SQL queries and Viewpoint field codes can be added to retrieve the information.

  • Users can generate the PDF forms from Viewpoint.

At the moment, only the XFA (XML Forms Architecture) PDF is supported.

 

PDF Form Templates

To have the template in Viewpoint, you need to have the .pdf file for the PDF form itself and the form's .xml file. If you do not have the .xml file, you can export the PDF form data using Adobe Acrobat or Adobe Reader. This will give you an .xml file - <FileName>_data.xml and you have to change this to <FileName>_data.txt.

  1. Save both the .pdf and .txt files to an accessible location.

  2. In Viewpoint - Home, go to Configuration > Document Libraries.

  3. Select Form Library or Word Reports.

  4. Click New.

  5. Enter the template properties and ensure the following:

    • Form Type is set to Electronic PDF Report [ER] if the template is in Word Reports, PDF Form [EP] if the template is in Form Library.

    • Path & File Name points to the .txt file in the location.

  6. Click Update.

  7. Select the template and click:

    • Edit - To add the SQL queries needed to retrieve information in the Key Information section.

    • Edit Document - To open the .txt file in your default text editor program (such as Notepad or Notepad++) to add Viewpoint field codes in the relevant areas. The field codes are the same as those used in Word templates.

Note: Viewpoint is used to code the template for retrieving information only and not for amending the PDF form itself. For changes to the PDF form, this is to be done using Adobe Acrobat program.

 

Once you have the template in Viewpoint, you can share the template to other Viewpoint environments using the Export and Import command buttons. When exporting the template, you will get:

  • .def file

  • .txt file

  • .pdf file

To import, you must ensure that you have all the three files placed in the same location. When you select the location, only the .def file is listed but performing the import will bring in all three files to Viewpoint.

 

Generate PDF Form

When the template is in Viewpoint, you can generate it like you would for Word Report or Forms. If you are generating a PDF form for the first time, Viewpoint will prompt you to enter a PDF Form/Report Registration Code. You can obtain this from your Viewpoint representative.

Once you have entered the code, you do not need to re-enter it for subsequent generations. Every generation will give you the PDF form with the retrieved information. The retrieved information is also contained in an .xml data file.

If you do not enter the code, then you will see an error message informing you of the location of the .xml data file. You can then import this data file into the PDF form using Adobe Reader.
