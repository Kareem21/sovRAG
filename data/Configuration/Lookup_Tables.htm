Address Types [ADRT]

Address Card Master

The following rules apply to address card codes:

  • Address types starting with ‘S’ are the Statutory locations of Companies, with SR being the Registered Office Address. These addresses are used in statutory forms.

  • RA is the registered address for individuals and RO for entities. These addresses are used in statutory forms where the Master File is an officer or shareholder.

  • Address types starting with ‘I’ are for individuals only.

  • Address types starting with ‘E’ are for non-individuals only.

  • If the address code contains an *, you can have more than one active record of the address type e.g. Postal Address [P*] to have multiple Postal Address records.

The address type listing on the Address Card screen can be sorted according to the Element/Sort Value of the lookup value.

 

For statutory address types (those where the code starts with 'S'), you can add the tag [EXCOMREC=1] to exclude from the Hong Kong Annual Return. With this tag, even if the address type is different from the SR address, it will not be included in the Hong Kong Annual Return.

 

Referencing Tag is possible in this lookup table.

Administrative Mark-up [ADMU]

Billing File - General

Enter the configuration options in the Tag Values field of the record. Find out more here.

[Markup_Service=<service code>]

Enter the service code that you want to use for the markup. For example:

[Markup_Service=ADSERV]

 

[Markup_Percent=<percentage>]

Enter the percentage as a number, without spaces and the percentage (%) symbol. For example, if the markup percentage is 3.5%:

[Markup_Percent=3.5]

 

[Markup_Exclude=<service type>]

Enter the service types that are excluded from the percentage increase, separated by a comma. For example:

[Markup_Exclude=PS,OT]

 

[Markup_Min=<amount>] OR [Markup_Min_EUR=<Value>]

Use this to specify the minimum markup amount in the matter/invoice currency. For example:

[Markup_Min=35]

You can also define the minimum value in the selected currency. For example:

[Markup_Min_EUR=50]

 

[Markup_Max=<amount>] OR [Markup_Max_EUR=<Value>]

Use this to specify the maximum markup amount in the matter/invoice currency. For example:

[Markup_Max=300]

You can also define the maximum value in the selected currency. For example:

[Markup_Max_EUR=100]

 

[Markup_Descr=<Text>]

Use this tag to have an alternative description for the charge. Without this tag, the description will follow the description in the Service Master. Enter the description in text. For example:

[Markup_Descr=Account Preparation]

 

[Markup_Cur=<CurrencyCode>]

Enter the markup currency code. For example:

[Markup_Cur=USD]

 

[Markup_AllowChange=<Value>]

Specify whether the markup is allow to change or delete. <Value> can be Y or N. For example:

[Markup_AllowChange=Y]

 

[Markup_Exempt=<TaxChargeType>]

Specify which tax charge type to exempt from being markup. For example:

[Markup_Exempt=1].

Note: Tax Charge Types is defined in Matter screen. New Tax Charge Types can be defined in Lookup Values ‘[TCHT]’.

 

[Markup_Excl_Class=<ServiceClass>] OR [Markup_Excl_ServCode=<ServiceCode>]

Specify which service class or service code to exclude from being markup. For example:

[Markup_Excl_Class=DI]

[Markup_Excl_ServCode=ARJER]

 

[Markup_Incl_Class=<ServiceClass>] OR [Markup_Incl_ServCode=<ServiceCode>]

Specify which service class or service code to include from being markup. For example:

[Markup_Incl_Class=DI]

[Markup_Incl_ServCode=ARJER]

 

Note: You can only use either [Markup_Excl_Class=<ServiceClass>] or [Markup_Incl_Class=<ServiceClass>] at one time. Similarly, you can only use either [Markup_Excl_ServCode=<ServiceCode>] or [Markup_Incl_ServCode=<ServiceCode>] at one time.

 

[Markup_InclTax=<Value>]

<Value> can be Y or N. This tag will include the VAT tax and the service's total for calculation of the additional charge if value is Y. By default, the value is Y. For example:

[Markup_InclTax = Y]

If this tag is not present, the system will exclude VAT in the calculation.

 

[Markup_TaxOnly=<Value>]

<Value> can be Y or N. This tag will takes only the VAT tax amount for the calculation of the additional charge if value is Y. By default, the value is N. For example:

[Markup_TaxOnly = N].

 

[Markup_Rounding=<Method>]

Specify if the calculated charge should be round up or down. Values for <Method> can be:

  • N - Normal rounding according to the currency and whether or not 'Round VAT Down to 5 cents Multiples' is enabled.

  • U2 - Round up to nearest cent

  • D2 - Round down to nearest cent

  • N0 - Normal rounding to whole number

  • U0 - Round up to whole number

  • D0 - Round down to whole number

For example:

[Markup_Rounding=N]

 

Can I set a fixed markup amount instead of percentage?

Yes, this is possible. To do so, set the minimum and maximum markup to the same amount. For example:

[Markup_Min=50]

[Markup_Max=50]

 

Referencing Tag is possible in this lookup table but only General Referencing Tags.

 

Note: It is not recommended to use the same Service Code for Administrative mark-up and Recurring Fees, otherwise the amount for Recurring Fees will become 'Nil' when perform invoicing.

Contact Types [ADRC]

Master File - Addresses/Contact

The contact type listing on the Addresses screen can be sorted according to the Element/Sort Value of the lookup value.

To allow more than one record of a contact type, use an * in the code e.g.

  • Element Code - T*

  • Name - Telephone number

Tags can be added to the Tag Values field of a contact detail type to:

  • Require Country information be entered for the contact detail type.

    [REQCNTR=Y]

    This is to make the Country column mandatory for the contact detail type. If the information is not provided, user will see a prompt when trying to update.

  • Have a format mask and validate data entry.

    For phone number and fax numbers, the tag is:

    [PHONE=<FormatMask>]

    where <FormatMask> can include:

    • special characters like (, ), -  and space

    • 0 to indicate mandatory input

    • 9 to indicate optional input

     

    For e-mail address and Web site address, to check that the data entry is of a valid format, use the tag:

    [EMAIL=Y]   
    [WEBSITE=Y]

    where

    [EMAIL=Y] will check that what is entered contains '@'

    [WEBSITE=Y] will check that what is entered starts with 'http://' or 'https://'

     

    Additionally, you can also use the following tag to set a default format:

    [DEFAULT=<Format>]

    e.g. [DEFAULT=http://www.] means that when entering a new Web site address, you do not need to type the  'http://www.' part of the address.

Country Codes [CNTR]

Throughout Viewpoint

Tags can be added to:

  • Set format for address cards in a jurisdiction

    Address Lines

    The following code is used to format the address lines in the Address Card Master into specific address fields. When this is done, each of the lines will have a caption, enabling the user to enter the appropriate portion of an address in that field.

    [ACF=CY|DC=Cyprus Address format|A1=Street|A2=No|A3=Building|A4=Floor|A5=Flat|FM=A3 A2+A1, A5, A4]

    Where:

    • ACF = Address Card Format identifier. We recommend using the country code as the identifier so that you can easily match the country selected when you set up a new address. Therefore, the above example would be placed in the Cyprus [CY] look value of the Country Codes [CNTR] lookup value table.

    • DC = Format Description. This is what the user will see when selecting a format option for an address card.

    • A1 - A5 = Captions for address lines 1 to 5. Leave these blank if you do not want a caption, and ‘^’ to disable the line.

    • FM = Address Card Format. This determines how the lines will appear. '+' is used for a line feed.

     

    See also:

    • Edit/New Address

  • Set mandatory address card and local address details in a jurisdiction

    Mandatory Lines

    The following tag is used to make the named fields of address card and local address (if local address feature is enabled) mandatory for completion:

    [AFMAN=RS,CI,A1,A2,A3,A4,A5,PC,DE]

    Where:

    • RS = Region/State

    • CI = City

    • A1–A5 = Address line 1 to 5

    • PC = Postcode

    • DE = Description

     

    See also:

    • Edit/New Address

  • Set the country calling code.

    [TELPREFIX=<Code>]

    This is to set the country’s calling code.

    Example

    [TELPREFIX=+60]

     

    See also:

    • Contacts Cleanup

     

  • Set starting prefixes that is not allowed for a country.

    [TELEXCL=<Code>]

    This is an optional tag to alert the user if the entered number starts with the specified code. For countries with the same country calling code, this adds a layer of validation.

    Example

    [TELEXCL=+1 242,+1 246,+1 264,+1 268,+1 284]

  • Define presenter in a jurisdiction

    [PRESENT.RELTYPE=<RelCode>]

    This is to define the presenter as a relationship type with <RelCode> being the code for the relationship type. Use this tag if the presenter is different for different managed entities within the jurisdiction.

    Note: The default code is 'PRS'. If you have an existing relationship type for Presenter with the code 'PRS', you can omit this tag. Otherwise, add this tag with your own code.

     

    [PRESENT.MFCODE=<MFCode>]

    The <MFCode> refers to the code of the Master File for he presenter. This tag is to define the presenter for all managed entities in the jurisdiction.

     

    See also:

    • Presenter Information

Engage Portal Calendar Type [CPIT]

Engage Servicing Portal - in Calendar and Workflow Action widget

The values here apply to the ESP Calendar and the Workflow Action widget.

For ESP Calendar, the values here define what to include in the ESP Calendar when the calendar is updated by the execution of the 'Engage Servicing Portal - Calendar Update [CPUC]' job. This is done through:

  • Long Name

    This is to retrieve the item include in the calender based on a date field and must be specified in the following syntax:

    <Table>.<DBField>|<Condition>

     

    where

    • <Table> - refers to the name of the database table containing the date to retrieve

    • <DBField> - refers to the database field name for the date that you want to retrieve

    • <Condition> - refers to condition that you want to apply and must include (-MFs:EntCode-)

    Example:

    EntMeetings.DateSchedule|(-MFs:EntCode-)

    The above retrieves the scheduled date for the meeting.

     

    Entity.AcctYearEnd|(-MFs:EntCode-)

    The above retrieves the year end date.

     

    Note: The condition (-MFs:EntCode-) must be included but if you would like to have multiple conditions, it is possible. For example, DocFinancial.DateDue|Category='AVG'|(-MFs:EntCode-)

  • Element/Sort Value

    Specify a number value here to define the period for Due Soon and Further Away. For instance, if 30 is entered then:

    • Due Soon - is when the retrieved date is within 30 days

    • Further Away - is when the retrieved date is more than 30 days

  • Tag Values

    Additional detail for the item can be defined using the tag below:

    [Detail=<Details>]

     

    where <Details> can be text or field code(s) enclosed within (-...-). The field must also be in the same database table defined in the Long Name.

     

    Example:

    [Detail=Incorporation Number:(-IncorpNr-)]

    The above will include the incorporation number.

     

    [Detail=Incorporation Number:(-IncorpNr-), incorporated on (-IncorpDate-)]

    The above will include the incorporation number and date.

Filing Actions [FLAC]

Master File - Folders - Meetings

Entity Administration - Activities - Filings

Action codes can be defined in the Tag Values field to do the following:

  • Update fields for Filing record upon completion of Filing Action.

    [DONE.UPDATE.RC:<RCField>=_<ActionField>] or [DONE.UPDATE.RC:<RCField>=<Value>]

    To update multiple fields, separate with '|' symbol.

    For example:

    [DONE.UPDATE.RC:DateSigned=_DateDone|DateFiled=_DateDone|FlActive=0]

  • Trigger change upon completion of Filing Action.

    [DONE.RUN:<Code>]

    <Code> can be replaced by:

    • RVW - perform File Review for Master File.

    • RVT - perform File Review for the Meeting/Filing/Review type only.

    • AFF - trigger Annual Accounts for foreign registered company. This will update date fields in Foreign Registrations screen with dates from the Filing Record:

      • Accounts - Last field will be updated with the date in Date Covered - From field minus one day.

      • Date Next field will be updated with the date in Date Due field

    • ANR - trigger Annual Return change code. This will update date fields in M&A/Dates screen with dates from the Filing Record:

      • Annual Return - Date Last field will be updated with the date in Date Covered - From field minus one day.

      • Date Next field will be updated with the date in Date Covered - To field

      • Date Due field will be updated with the date in Date Due field

Tax (VAT/GST etc) Charge Types [TCHT]

Practice Management - Sales Ledger and Purchase Ledger

Tags can be used to:

  • Allow for reverse tax charge to be created for purchase order lines.

    [REV_PUR=Y]

    This indicates that reverse charge is required for purchase invoices.

     

    [REV_DESCR=<Description>]

    This is to define the description for the invoice line.

     

  • Specify which tax code can be reversed and its corresponding reverse code.

    [REV_CODE=|<Code1>=<RCode1>|<Code2>=<RCode2>|]

    This is to set the tax codes that can be reversed <Code1>, <Code2>... and the reverse tax codes <RCode1>,<RCode2>.. For example: [REV_CODE=|STD=RSTD|]

     

    With the tag in place, for a new Purchase Order, when a line is added for a tax code with a corresponding reverse code, a new line for the reverse tax will be created.

     

    When editing a Purchase Order, if the line is for a tax code without a corresponding reverse code, and this is changed to one with a reverse code, the line for the reverse tax will not be created.

     

    Note:

    The tax codes must be Tax Accounts of the type 'Tax Account - Both Invoiced & Received [WBSP]' or 'Tax Account - Received [WPUR]'
