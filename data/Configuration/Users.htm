Users

Users refer to user accounts which grant access to the Viewpoint Connect program. The maintenance of user accounts is performed in this screen.

 

Filters

The filters apply to the user account list.

  • User account status - You can choose to view only active user accounts, inactive user accounts or all user accounts.

  • Administrator Group - You can choose to view user accounts that are assigned to a specific Administrator Group.

 

Select button

This allows for additional criteria to apply to the user account list.

 

User Listing

The list shows all existing user accounts.

 

Command buttons

These apply to the User account list. Click on the relevant command button to perform the necessary.

  • New - Click this to create new user accounts.

    1. Enter the details for the new user account. If required, tags can be added for the user account.

    2. Click Update.

    3. The Assign/Reset Password screen will be displayed so the password can be set for the new user account. This is not seen if the User Login Mode is set to 'Windows account login'.

    When assigning the User Group, if this results in the user account exceeding the number of licence, the User Access Matrix screen will appear. You need to acknowledge this before you can click OK to close this screen.

     

    Note: If Engage Servicing Portal is used, the Profile Picture field can be enabled to allow the users' pictures to be uploaded. The default file type, dimension and file size is set in the Field Settings as TYPE=JPG,PNG,BMP|DIMENSION=300,300|SIZE=100.

  • Edit - Click this to view/change details of an existing user account.

  • Delete - Click this to remove an existing user account.

  • Reassign - Click this to reassign items for the selected user account to another, including access to encrypted Master Files and Address Cards. The reassignment can be performed in the event of a user resignation where you reassign the items for the resigning user before you deactivate his/hers user account.

    1. In the User Reassign screen, select the types of item that you want to reassign.

    2. Enter the user to whom the items are being reassigned to.

    3. Click Reassign.

    4. Click Yes to confirm.

    Note: If you are transferring only one item type, then you can choose to transfer all of the item type or only selected ones. If more than one item type, then all of each item type will be included.

     

  • Password - Click this to reset the password and/or change the user account status.

    To change password:

    1. In the Assign/Reset Password screen, tick the Change password option.

    2. Enter the new password in New Password and Confirm Password field.

    3. Click Update.

    Other Options

    • Auto generate password and e-mail user login link - This will send a password reset e-mail to the user. This option is only available in Engage Servicing Portal > Configuration> Users screen for ESP Front-Office.

    • Require password change on next log-on - Tick this to prompt the user to change the password on the next log in.

    • User login active - Tick this to set the user account as active. If this is not ticked, the user account becomes inactive.

    • Allow Select - When ticked, the user's name will be displayed for selection in Viewpoint.

    • Administrator Rate - When ticked, the Administrator Rate for the user is set to active.

    • Reset secret key - Tick this to reset the secret key of the two-step verification feature. This option is only available if two-step verification is enabled in Configuration > Settings > System Defaults > Security.

    How can I activate or reset a password via e-mail?

    For more information on this, please refer to the Password Activation and Reset via E-mail topic.

     

     

  • Administrator Groups - Click this to view all assigned Administrator Groups or assign additional Administrator Group additional Administrator Group.

    1. In the Assign Administrator Group To <User> screen, select the Administrator Group to assign from the Available Administrator Groups list..

    2. Click >> to add the selected Administrator Group to the list of Assigned Administrator Groups.

    3. Repeat steps 1 and 2 until all required Administrator Groups have been added.

    4. Click Update.

    To remove, select the Administrator Group from the list of Assigned Administrator Groups and click <<. Click Assign

     

    Note: The primary Administrator Group is assigned using the Administrator Group field in the user account record.

     

  • Access Groups - Click this to view the assigned Access Groups or assign Access Group to the user account.

    1. In the Assign Access Group To <User> screen, click to select the Access Group to assign.

    2. Click >> to add to the Assigned Access Groups list.

    3. Repeat steps 1 and 2 until you have assigned the necessary Access Groups.

    4. Click Update.

    To remove, click to select the Access Group from the Assigned Access Groups list and click <<. Then click Update.

     

     

  • Relation Enquiry - Click this to view all Master Files where the selected user account has been set up as a relation. This will be shown in a separate User Relations enquiry screen.

  • List User Access - Click to view user access to confidential database. This will be shown in an Excel report. This is only available if Confidential Database is set up.

    You will be prompted to select Yes or No depending on whether you want to view access to confidential database for selected user account or all user accounts.

    The report will be created once you have made your choice.

     

Administrators and Rates

The list is of rates set for a selected user who will be recording time. The rate set here is used when the user selects services from the Time - Administrator Rate [TA] class.

 

Command buttons

These apply to the rate list. Click on the relevant command button to perform the necessary.

  • New - Click this to create a new administrator record for a new user to enter the rate.

    1. Enter the details for the record.

    2. Click Update.

     

  • Renew - Click this to renew an existing administrator with new information such as rates and/or job grade. This creates a new administrator record for the selected administrator and sets the previous administrator record to inactive.

    1. Enter the starting date for the new record in the Date From field.

    2. Update the relevant details.

    3. Click Update.

     

  • Edit - Click this to edit information for an existing administrator record.

  • Delete - Click this to delete the selected administrator record.

  • Validate - Click this to validate the active period. This Validate function validates all active periods for the selected administrator.

    When you click this, you will see the following options:

    • Create Records/Update Target - Tick this to create records and update targets for the selected administrator for all Open periods.

    • Delete Not Used - Tick this to remove timesheet for administrators if not required for the period e.g. administrator has resigned. This will apply to the relevant periods if the period status is still set to Open.

    • Recalculate Total - Tick this to recalculate the total units for the selected administrator.

Additional Fields for Information Recording

The following are fields that can be used to record additional information:

  • In User details:
    • Employee Code - It is a text field with a maximum length of 50 characters. The value for each user must be unique. A message will be prompted if the Employee Code value is already in use.
  • In Administrator record:
    • Line Manager - This is a search field allowing the selection of a Viewpoint User. The Reassign function is applicable with this field.
    • User Number <number> (There are seven fields, each numbered from 1 to 7) - This is a field to enter decimal value of a maximum of 16 digits before the decimal point and 12 digits after the decimal point.
      Note: The User Number fields are limited to a maximum of 27 characters per field. Please ensure the values keyed in do not exceed this limit.

These non-mandatory fields are not requirements for any feature’s logic in Viewpoint Connect. They are not enabled by default and the correct authority must be assigned in Field Settings to use these fields.

Once valid values are updated in these fields, the information can be seen in the View Details function. These fields are available in Custom Reports – Design under the Users/Workgroups report name.
