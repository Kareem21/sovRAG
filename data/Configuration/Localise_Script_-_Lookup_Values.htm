Localise Script Tags - Lookup Values

You can use tags to perform three types of actions in relation to lookup values:

  • Deactivate an existing value.

  • Delete an existing value.

  • Add and define a new value.

 

Deactivating a Value

The tag below will deactivate the specified lookup table value.

 

Syntax

LVINACT:<TCode>+<VCode>

 

where:

  • <TCode> is the Lookup Table code, obtained from Configuration > Business Parameters > General > Lookup Values.

  • <VCode> is the code of the lookup table value to be deactivated.

 

Example

To deactivate the Low [01] code from the CDD Classification [KYCC] lookup table: LVINACT:KYCC+01

 

Note: To deactivate all values in a lookup table, use asterisk e.g. LVINACT:KYCC+*

 

Deleting a Value

The tag below will remove the specified lookup table value.

 

Syntax

LVDEL:<TCode>+<VCode>

 

where:

  • <TCode> is the Lookup Table code, obtained from Configuration > Business Parameters > General > Lookup Values.

  • <VCode> is the code of the lookup table value to be deleted.

 

Example

To delete the value 'Low [01]' from the CDD Classification [KYCC] lookup table: LVDEL:KYCC+01

 

Note: To delete all values in a lookup table, use asterisk e.g. LVDEL:ROSU+*

 

Adding a Value

When you add a value to a lookup table, you need to first create the value in the table and then specify the settings for the value. The new value will have the Customised and Active checkboxes ticked by default.

 

Syntax

To add a new value:

LVADD: <TCode>+<VCode>=<VName>

 

To specify the value settings:

LVVAL:<TCode>-<VCode>+<DBField>=<Value>

 

where:

  • <TCode> is the lookup table code, obtained from Configuration > Business Parameters > General > Lookup Values.

  • <VCode> is the code of the Lookup Table value to be added. There should not be duplicates of any (existing) code.

  • <VName> is the name of the value.

  • <DBField> is the database field for the lookup table value

    • Tag - Long Name field

    • eFile - Electronic Filing Code field

    • ElValue - Element/Sort Value field

    • ConvertValues - Tag Values field

  • <Value> is the value for the field.

Example 1

To add Resident [2] to the User Defined – 1 [USR1] lookup table: LVADD:USR1+2=Resident

 

Example 2

To add a Document Management index tag to the Minute [MI] code in the Form Type [FRMT] lookup table:

LVVAL:FRMT-MI+ConvertValues=Document index code:<BR>[/DMW.DocIndex|Descr=(-DCM.Description-)|Cabinet=CSFORM|VpDocId=(-FRM.FQnumber-)|Note=(-FRM.FieldDetails-)|/]<BR><BR>Main Master File index<BR>[/DMW.Relation|KeyTag=INDEX|KeyValue=(-FRM.EntCode-)|/]<BR><BR>Index field<BR>[/DMW.Relation|KeyTag=DOCDATE|KeyValue=(+EVT.Effective+)|/]<BR><BR>Index field - Document type<BR>[/DMW.Relation|KeyTag=DOCTYPE|KeyValue=AI|/]<BR><BR>Add the task that the document must be signed<BR>[/DMW.Task|TaskType=SIGNDOC|TaskDescr=(-DCM.Description-) |AssignTo=(-APP.UCode-)|TaskDue=(-APP.NOW-)+15|/]

 

Note: <BR> is used for line feeds.

 

Adding Tag Values

The tag below allows you to add tags to existing lookup table values in the Tag Values field. This applies only to certain lookup tables.

 

Syntax

LVTAG:<TCode>+<Code>=<Tag>

 

where:

  • <TCode> is the lookup table code, obtained from Configuration > Business Parameters > General > Lookup Values.

  • <Code> is the code you want to set up the WSCORE tag for.

  • <Tag> refers to the tag for the lookup table values.

 

Example

To add the required country tag for 'Telephone - Office [TB] in Contact Types [ADRC] lookup table:LVTAG:ADRC+TB=REQCNTR=Y

 

Note: You can use either the LVVAL tag or the LVTAG tag to add score information. However, LVVAL will over-write existing information in the ConvertValues field, whereas LVTAG will append information.
