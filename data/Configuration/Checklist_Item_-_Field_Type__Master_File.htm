Checklist Item - Field Type: Master File

The Master File field type is used to hold a Master File value which can be:

  • a new Master File created by the user

  • existing Master File selected by user or retrieved for the workflow

For new Master File, this will bring the user to the New Master File Wizard for user to create a new Master File. For existing Master File, this can be all active Master Files or the list can be filtered based on predefined option or user-defined data source.

 

Master File List based on Predefined Option

These are options that you select as the Data Source and can be based on

  • Components activated for a Master File

    • #CS - Entity Administration component is enabled.

    • #RM.B - Practice Management component (Sales Ledger) is enabled.

    • #TR - Practice Management component (Purchase Ledger) is enabled.

    • #CA - Accounting component is enabled.

  • Roles allowed for a Master File

    • #CL - Client/Group role is enabled.

    • #SH.B - Shareholder/Partner/Interest Holder role is enabled.

    • #OC - Statutory/Trust/Foundation Officer role is enabled.

    • #PR - Relation role is enabled.

    • #MF - Master Files with any role enabled.

    • #BA - Bank/Broker role is enabled.

    • #SI.B - Bank/Broker Account Signatory role is enabled.

     

  • How a Master File is related

    • #RAP - Master File appointed as any relation.

    • #RT - Master File appointed as Trust officer.

    • #RE - Master File appointed as Entity relation.

    • #RO - Master File appointed as Statutory officer.

    • #RM - Master File appointed as Master File relation.

    • #BE - Master File added as beneficiary.

    • #SI - Master File added as bank/broker account signatory.

    • #SH - Master File that holds shares.

    • #IN - Master File added as Interest holder.

    • #OW - Master File added as Owners & Controllers.

    • #MRT - Master File is linked to the workflow's Master File.

    • #MRT.CHL - Master File is added as a child relation of the workflow's Master File.

    • #GRT - Master File is in the same ESP Organisation as the workflow's Master File or linked to any of the Master File in the same ESP Organisation as a relation (e.g. Officers, Relations and Beneficiaries).

    • #GRT.CHL - Master File is in the same ESP Organisation as the workflow's Master File or linked to any of the Master File in the same ESP Organisation as a child relation.

    • #GRT.ORG - Master File is in the same ESP Organisation as the workflow's Master File.

    • #CA.ASL - Master File is linked to the sub-ledger account.

As you select an option, the code is seen. You can also customised the code to further filter by relationship type by specifying the data source like below:

<OptionCode>:<Type>

where

  • <OptionCode> refers to the code of the selected option.

  • <Type> refers to the code of the type.

Option Code

Explanation and Example

All Relation options:

  • RT

  • RE

  • RO

  • RM

This applies to all Relation options such as Relations [Entity Relations] and Relation [Statutory Officers]. You can add code of the relationship type.

 

Example:

RO:DIR – the list will show only Directors.

RO:D* - the list will show only Statutory Officers where type starts with ‘D’ such as DIR (Director) and DAL (Alternate Director).

RO:D*,S* - the list will show Statutory Officers where type starts with ‘D’ or ‘S’.

BE

This is only for Beneficiaries option.

 

Example:

BE:I – the list will show only beneficiaries where Beneficiary Type is Income [I].

Note: The types are from Beneficiary Type [BEFT] lookup table.

SI

This is only for Bank Signatories option.

 

Example:

SI:A – the list will show only signatories belonging to Group A.

Note: The types are from Signing Auth. Group [SIGR] lookup table.

SH

This is only for Shareholders option.

 

Example:

SH:O*,P* – the list will show only Shareholders holding share class type starts with 'O' or 'P'.

Note: The types are from Share Types [SHRT] lookup table.

IN

This is only for Interests option.

 

Example:

IN:BEO – the list will show only Beneficial Owners.

Note: The types are from Interest Types [INTP] lookup table.

CA.ASL

This is only for sub-ledger account type related to Master File.

 

Example:
CA.ASL:B,F – the list will show only sub-ledger where the account type is 'B' or 'F'.

Note: The types are from sub-ledger account types.

 

For the #MRT and #GRT options, you can filter by customising the code as follows:

<OptionCode>:<RelationID>

where

  • <OptionCode> refers to the code of the selected option.

  • <RelationID> refers to the ID of the relationship type which can be obtained using Workflow Field Wizard under the category 'Master File Data Source Type'.

Example:

#MRT:560 - Will retrieve Master Files that are bankers/brokers for the workflow's Master File.

#MRT:560,566 - Will retrieve Master Files that are bankers/brokers and bank/broker signatories for the workflow's Master File.

#MRT:56* - Will retrieve Master Files that are relations with RelationID starting with '56' for the workflow's Master File.

#GRT:560 - Will retrieve Master Files that are bankers/brokers for all Master Files in the same ESP Organisation as the workflow's Master File.

 

Show Additional Information in Master File drop-down list

Additional information such as the caption from database field can be configured to show in a column in the Master File drop-down list. Ensure that the 'Show Combo List' option is ticked to have the column displayed.

To configure, add the following tag:

<OptionCode>:<DBField>

where:

  • <OptionCode> refers to the code of the selected option and is currently supported for all Relation options except #GRT.

  • <DBField> is the database field required to be shown as the additional information. Only one database field can be configured for each Master File field type.

Note: For #MRT, the <DBField> supported are RelCaption, RelFrom, RelTo and RelNote only.

 

The field ‘Data Source Attribute’ is only applicable when the Data Source value is #OC, #PR or #MF. This is to allow Master Files to be further filtered according to Individual ‘I’ or Non Individual ‘N’.

Note: In ESP, when the 'Master File' field type is configured without a Data Source, the default value will be set to #GRT.ORG (ESP Global Relations [From Organisation]).

 

Master File List based on User-Defined Data Source

This allows you to filter the list using other conditions. This requires the following:

  • Define the data source in the workflow step properties using [/<DataSource>=/]. Refer to Checklist/Workflow Step - Tags.

  • Add the name of the data source to the Data Source field instead of selecting an option.
