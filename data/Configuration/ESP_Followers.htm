Followers

The ESP Followers settings screen consists of a list of Followers types that the administrator can activate in the CP-WP. Once activated, users can ‘follow’ the components in Engage Servicing Portal (ESP). The Followers feature is available for these types:

  • Entity Information

  • Meetings

  • Filings

  • Documents

  • Workflows (Requests)

When the component is followed, users will receive notification in ESP if there is any update (change) in their followed items.

 

Configuration

A few configurations are required in the Front-office prior to enabling the Followers feature in ESP.

Activate the Follower Components

In the Front-office, the Administrator is required to activate the Followers component.

This is done in the Configuration > Engage Servicing Portal > Global > Followers screen.

Note: The Followers screen is hidden by default due to authority settings. To access this screen, you may grant authority in Configuration > Users & Security > Security Setup > Users Roles.

  1. In the Configuration > Engage Servicing Portal > Global > Followers screen, select the Followers component and click Edit.

  2. Update the settings by tick/untick the options.

  3. Click Update once done.

  4. Repeat step (1) to (3) for other Followers component.

 

Enabling Task Scheduler Job ‘Engage Servicing Portal – Follower Notification [CPFN]’

In the Front-office, the Task Scheduler job ‘Engage Servicing Portal – Follower Notification [CPFN]’ processes any Notification triggered by the Change from the Back-office.

  1. Go to Configuration > Business Parameters > Task Scheduler > Agent Job Scheduler.

  2. Click New. In the ‘Setup’, select Job Type ‘Engage Servicing Portal – Follower Notification [CPFN]’.

  3. Click Next >.

  4. In the ‘Criteria’, enter the Database where the notification is retrieved.

  5. Click Next >.

  6. If an e-mail notification for the job is required, tick the option ‘Enable E-mail Notification’ and specify details for the e-mail. If you prefer different types of e-mail notification, tick ‘Advanced’ and you will get the following:

    • Success - this is the notification when the job is completed successfully.

    • Fail - this is the notification when the job is not completed.

  1. Click Next >.

  2. In the ‘Schedule’, enter the job frequency.

  3. Click Done.

Note: Whenever there is a configuration update in the Front-office, it is recommended to recycle the Web IIS Application Pool to refresh the settings in the Web Portal. This is followed by restarting the Web IIS Application Pool.

 

Filters

The filters apply to the Followers type list. You can select either Active, Inactive or All.

 

Followers item list

The list shows the Followers type available in the ESP.

 

Command button

Click on the relevant command button to perform the necessary.

  • Edit - Click this to edit the selected Follower component.

  • Configure - Click this to configure the Follower component. This includes assign and edit a Change Code or deleting a Change Code.

    1. To assign a Change Code, click on the magnifier glass icon to select a Change Code.

    2. Select the Change Code and click Select.

    3. Click Assign>>>.

    4. Click Close once done.

    For a newly assigned Change Code, it is important to include the tag value, otherwise no notification will be generated for the Change.

    The Edit button in the Select Change Code window is available to allow the administrator to enter/customise the tag value for the Change Code. To include the tag value:

    1. In the Select Change Code window, select a Change Code and click Edit.

    2. This will launch the Configure Tag Values window.

    3. Enter the Tag Values accordingly. The available syntax are as follows:

      Syntax

      Description

      Example

      NOTIFY_MSG

      This is the message template for the notification that is shown to the portal users. It uses standard Viewpoint template variables. The defined variables will be replaced by values in the Back-office.

      [/NOTIFY_MSG=(+CC.AddrCode\#AD-40-0+) has been appointed as (+CC.OfficerText+)./]

      FILTER

      This defines the conditions to filter the EventLog records retrieved from the Back-office.

      [/FILTER=CSPUserCode <> '' AND CSPUserCode IS NOT NULL/]

       

       

    4. Click OK.

    5. Click Close.
