Assign To Codes

These apply to checklists/workflow steps and workflows where you can specify the assignee for:

  • The checklist/workflow step

  • The workflow

 

Checklist/Workflow Step

For checklists and workflow steps, the Assign To refers to the user who will be completing the checklist/workflow step. The code is to be entered in the Assign To field of the checklist/workflow step record in Step Library and can be:

  • The user code - This means that the checklist/workflow step will always be assigned to the specified user.

  • Field code - This allows for the checklist/workflow step to be assigned to different users based on the Master File the checklist/workflow step is for. This can be:

    • The user specified as the file's Administrator, Manager, Controller or CDD Officer. Select from the list of options.

      Code

      Explanation

      (-EN.AD-)

      Entity File Administrator

      (-EN.MG-)

      Entity File Manager

      (-EN.CT-)

      Entity File Controller

      (-AF.AD-)

      Account File Administrator

      (-AF.MG-)

      Account File Manager

      (-AF.CT-)

      Account File Controller

      (-BF.AD-)

      Billing File Administrator

      (-BF.MG-)

      Billing File Manager

      (-BF.CT-)

      Billing File Controller

      (-SF.AD-)

      Supplier File Administrator

      (-SF.MG-)

      Supplier File Manager

      (-SF.CT-)

      Supplier File Controller

      (-MF.AD-)

      Master File Administrator

      (-MF.MG-)

      Master File Manager

      (-MF.KO-)

      Master File CDD Officer

       

    • The user related to the checklist/workflow step - refers to the user who started the workflow or is assigned to work on a workflow step. Select from the list of options.

      Code

      Explanation

      (-WF.CM-)

      The current logged in user's manager. This is defined in the user's record in Configuration > Users & Security > Users screen in the 'Manager' field.

      If there is no manager specified, the current logged in user will be the  assignee.

      (-WF.CR-)

      The user who started the checklist or workflow.

      (-WF.OW-)

      The owner of the workflow. This is only applicable for workflow steps.

      (-WF.SP-)

      The user assigned to the previous workflow step. This is only applicable for workflow steps.

      (-WF.SF-)

      The user assigned to the first workflow step. This is only applicable for workflow steps.

      (-KEY.<ItemKey>-)

      where <Item Key> refers to the key of the checklist item with ‘System User’ field type.

      This will assign the next workflow step within the same workflow to the user specified in the ’System User’ field type.

      If the key does not match, the workflow step will automatically be assigned to the workflow owner.

       

    • User named in the Service Office - This refers to the user specified in the Service Office screen. Enter the relevant code.

      Code

      Explanation

      (-SOF.AM-)

      This is the user in the Account Manager field for the first retrieved Service Office.

      (-SOF.TO-)

      This is the user in the Trust Officer field for the first retrieved Service Office.

      (-SOF.AM.<ServCode>-)

      This is the user in the Account Manager field for the first retrieved Service Office for the specified Service Code.

      (-SOF.TO.<ServCode>-)

      This is the user in the Trust Officer field for the first retrieved Service Office for the specified Service Code.

       

    • User specified as a relationship type - This refers to the user appointed as a user relation for a relationship type. Enter the relevant code.

      Code

      Explanation

      (-REL.<RelCode>-)

       

      where <RelCode> refers to the code of the relationship type.

      This will assign it to the user appointed as the user relation of the specified relationship type.

      If the relationship type does not have any user relation, it defaults to the logged in user.

      (-REL.<RelCode>.<Field>-)

       

      where  <Field> refers to Master File fields for the relationship type and can be:

      • USER - This is the User Relation for the relationship type.

      • MF_USER - This refers to the Principal Relation field for the relationship type.

      • MF1USER - This refers to the Additional Relation 1 field of the relationship type.

      • MF2USER -  This refers to the Additional Relation 2 field of the relationship type.

      • MF3USER -  This refers to the Additional Relation 3 field of the relationship type.

      When the <Field> is USER, this will be the same as (-REL.<RelCode>-).

      For other values, it will look for the user where the appointed Master File is specified in the user's 'User -> Master File' field.

       

       

 

Workflow

For workflows, the Assign To refers to the user who is the owner of the workflow. The code is to be entered in the Assign To field of the workflow template record in Workflow Library and can be:

  • The user code - This means that the workflow will always be assigned to the specified user.

  • Field code - This allows for the workflow to be assigned to different users based on the Master File the workflow is for. This can be:

    • The user specified as the file's Administrator, Manager, Controller or CDD Officer. Select from the list of options.
