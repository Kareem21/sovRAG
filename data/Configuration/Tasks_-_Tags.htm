Tasks - Tags

When setting up a task, you can add these tags in the Tag Values field to specify if the task is a Delivery Manager or a follow-up task.

 

Delivery Manager Task

Tag

Explanation

[DD.ConvertMail=Y]

This will check for mail tags in the template. If the tag [$Recipient_to:] exists with a valid e-mail address, the document will be converted to an e-mail. If there is no e-mail address then the instruction specified by the [DD.NoMail] tag will be used.

[DD.NoMail=<Option>]

This tag will tell Viewpoint how to handle the document if there is no valid e-mail address retrieved for the document you have generated, where <Option> can be:

  • P - This will print out the document together with the e-mail tags and text for the e-mail body.

  • R - This will print out the document without the e-mail tags and separate the text for the e-mail body from the attachment. The original document in the holder will be replaced by the new version without the e-mails tags and separate page for the e-mail body text.

  • S - This will print out the document without the e-mail tags and the text for the e-mail body. The original document in the holder will be replaced by the new version without the e-mail tags and e-mail body text.

  • Blank - If you prefer to handle documents that cannot be converted to e-mail manually, use [DD.NoMail=]. The document holder will not be stored regardless of the selection in the 'When the e-mail is sent' section of the Delivery Manager screen.  

 

Follow-up Task - Complete

The following will create a follow-up task when you click Complete for the earlier task.

Tag

Explanation

[On_Complete.Codes=<Key Tag>]

This creates a follow-up task automatically created with partial details when you click the Complete button.

Enter as many task key tags as required, separated by a comma (,). User will be able to select the correct task when creating the ‘follow-up’ task.

Example:

[On_Complete.Codes =DOCPREP,DOCSIGN]

[On_Complete.Default=<Key Tag>]

If you have multiple key tags specified in [On_Complete.Codes=<Key Tag>], above, then this allows you to specify a default.

Example:

[On_Complete.Default=DOCPREP]

[On_Complete.Descr=<Text>]

This sets the default description for the follow-up task. The default is blank

[On_Complete.Assign=<user>]

This will assign the follow-up task to the user defined. The default selection is blank, but you may use any of the codes below to pick up the user that you want:

  • @ADM1.<FileType> - This picks up the Administrator of the <FileType> where A - Account File, B - Billing File, E - Entity File and S - Supplier File.

  • @ADM2.<FileType> - This picks up the Manager of the <FileType> where A - Account File, B - Billing File, E - Entity File and S - Supplier File.

  • @ADM3.<FileType> - This picks up the Controller of the <FileType> where A - Account File, B - Billing File, E - Entity File and S - Supplier File.

  • @ORIG - This retrieves the originator of the task.

  • @UMGR - This will select the Manager of the user completing the task (set up in Configuration > Users & Security > Users).

  • <UserCode> - This assigns the follow-up task to the specified user.

[On_Complete.Due=<DueDate>]

 

This sets the due date of the follow-up task and can be:

  • Dependent on the originating task - Enter a positive number. This will set the due date as n number of days from the date you complete the originating task.

    For example:

    [On_Complete.Due=3] means the due date for the follow-up task is three days from the day you clicked Complete for the originating task.

  • A specific date - enter the date in this format #YYYY-MM-DD# where:

  • YYYY refers to the year in 4-digit format.

  • MM refers to the month in 2-digit format e.g. 01 for January, 02 for February.

  • DD refers to the day in 2-digit format.

[On_Complete.Action.ConfirmDraft=<Option>]

This will automatically approve a draft invoice on the completion of an invoice-related task. To identify the draft number that is to be approved, place the information in either the index values for the related document holder, or place the information in a specific index field. This is done in the Document Management tag values that are found in the Key Information of the invoice template.

  • VPDocID - If you intend to use the index values of the document holder to record the draft number information, place (-HDF.ProformaNr-) in the VpDocID section of the tag:

    [/DMW.DocIndex|Descr=Draft No (-HDF.ProformaNr-)|Cabinet=ACCINV|VpDocId=(-HDF.ProformaNr-)|DocConf=0|/]

    The task tag will then be:
    [On_Complete.Action.ConfirmDraft=VpDocId]
  • DRAFTNO - This is used if the draft number is placed in its own index field which you will need to create the with key tag DRAFTNO. This index field must be linked to the filing cabinet. The draft number is placed in the DRAFTNO index field in the Key Information of the invoice template:

    [/DMW.Relation|KeyTag=DRAFTNO|KeyValue=(-HDF.ProformaNr-)|/]

    The task tag will then be
    [On_Complete.Action.ConfirmDraft=DRAFTNO]

 

 

Follow-up Task - Reject

The following will create a follow-up task when you click Reject for the earlier task.

Tag

Explanation

[On_Reject.Codes=<Key Tag>]

This will show the Reject button on the originating task. This will allow the user to select the follow-up task from the selections provided. Separate multiple selections with a comma (,).

Example:

[On_Reject.Codes=DOCSUB,DOCREV]

[On_Reject.Caption=<Text>]

This determines the button caption. The default is ‘Reject’, but may be changed, for example, to ‘Decline’.

[On_Reject.Default=<Key Tag>]

This sets the default task when there are multiple choices set for [On_Reject.Codes=<Key Tag>] The default is none.

[On_Reject.Assign=<user>]

This will assign the follow-up task to the user defined. The default selection is blank, but you may use any of the codes below to pick up the user that you want. Refer to the explanation provided in [On_Complete.Assign=<user>], above.

On_Reject.Due=<DueDate>]

This sets the due date of the follow-up task and can be:

  • Dependent on the originating task - Enter a positive number. This will set the due date as n number of days from the date you rejected  the originating task.

    For example:

    [On_Reject.Due=3] means the due date for the follow-up task is three days from the day you clicked Reject for the originating task.

  • A specific date - enter the date in this format #YYYY-MM-DD# where:

  • YYYY refers to the year in 4-digit format.

  • MM refers to the month in 2-digit format e.g. 01 for January, 02 for February.

  • DD refers to the day in 2-digit format.

[On_Reject.Complete=Y]

Valid values are Y or N (Yes or No). If Yes, it will ‘Complete’ the related (originating) task. If you use N, then nothing will be updated to the originating task, and it will remain in the Assigned Tasks until you complete it.

[On_Reject.Descr=<Text>]

This customises the text for the default description of the follow-up task. You may use (-Copy-) to include the description of the originating task. For example: [On_Reject.Descr=Declined (-Copy-)]
