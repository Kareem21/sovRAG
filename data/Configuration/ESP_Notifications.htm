Notifications

The ESP Notifications settings screen consists of a list of standard notifications that the Front-office administrator can activate for ESP.

The Notifications feature keeps the ESP users informed of the changes and updates to the following:

  • Document generation status

  • Status of requests

  • Messages relating to requests

  • Online payment status

  • Updates to the followed items

There are two types of notifications:

  • Instant Notification - The notifications will be seen in the Notifications panel as well as at the bottom right corner of the ESP screen. This is a standard feature and cannot be turned off unless all the notifications are deactivated in the Front-office (CP-AD).

  • E-mail Notification - This is applicable when the 'Send Report on Notifications Pending Overnight' and/or 'Send Weekly Summary of Calendar Reminders and Notifications' are activated in the Front-office Notifications settings screen. When the notification is activated in the Front-office Notifications settings screen, the ESP users can see the option to turn on/off the subscription of the e-mail notification in ESP Home > Profile > Notification tab. Users will be receiving e-mail notification as per their subscribed option.

 

Filters

The filters apply to the Notification list. You can select either Active, Inactive or All.

 

Notification list

The list shows the Notifications available in the ESP:

Notification Name

Description

Document Generation - Success

When activated, user will receive instant notification in the ESP when a document (registry) is generated successfully.

This requires a running task scheduler job 'Engage Servicing Portal - Register Printing [CPPR]' to retrieve the document generation status and prompts a notification.

Document Generation - Fail

When activated, user will receive instant notification in the ESP when a document (registry) has failed to generate.

This requires a running task scheduler job 'Engage Servicing Portal - Register Printing [CPPR]' to retrieve the document generation status and prompts a notification.

My Request - Message Received

When activated, user will receive instant notification in the ESP when there is any message received from the Back-office on the user's Request (workflow).

This requires a running task scheduler job 'Engage Servicing Portal - Get Notification [CPGN]' to retrieve the My Request message and prompts a notification.

My Request - Status Update

When activated, user will receive instant notification in the ESP when there is any status update on the user's Request (workflow).

This requires a running task scheduler job 'Engage Servicing Portal - Get Notification [CPGN]' to retrieve the My Request status update and prompts a notification.

Online Payment Completed

When activated, user will receive instant notification in the ESP and e-mail notification to notify them that the payment is completed.

This requires a running task scheduler job 'Engage Servicing Portal - Get Notification [CPGN]' to check the payment status and prompts a notification.

Send Report on Notifications Pending Overnight

When activated, it will show the option in the ESP Home > Profile > Notification tab for user to subscribe to the e-mail notification. When the users subscribe to this, they will receive an e-mail that consists of a report on pending (unread) notifications. It will only send out the e-mail if the user did not log in to the ESP overnight.

It sends the report on the following pending (unread) notifications:

  • Documents that the user generated (both successful and failed).

  • User ‘Requests’ status updates.

  • User ‘Requests’ messages received.

  • All items that the user followed, which include:

    • Entity Information

    • Documents

    • Filings

    • Meetings

    • Requests

This requires a running task scheduler job ‘Engage Servicing Portal – Daily Report [CPDR]’ to retrieve the pending notifications and send out the e-mail.

Send Weekly Summary of Calendar Reminders and Notifications

When activated, it will show the option in the ESP Home > Profile > Notification tab for user to subscribe to the e-mail notification. When the users subscribe to this, they will receive an e-mail that consists of a report that summarises all notifications on a weekly basis.

It sends the report on the following:

  • Calendar items that are ‘Overdue’ and ‘Due Soon’.

  • Documents that the user generated (both successful and failed).

  • User ‘Requests’ status updates.

  • User ‘Requests’ messages received.

  • All items that the user followed, which include:

    • Entity Information

    • Documents

    • Filings

    • Meetings

    • Requests

This requires a running task scheduler job ‘Engage Servicing Portal – Weekly Report [CPWR]’ to retrieve all the notifications during the week and send out the e-mail.

 

Command button

As the Front-office administrator, you can only edit the Notifications. Click on the relevant command button to perform the necessary.

  • Edit - Click this to edit the selected Notification. You can edit and customise the Instant Content and E-mail Body/ E-mail Template.

    Applicable only to 'Online Payment Completed' notification. Upon completion of an online payment, an e-mail will be sent to the users to notify them of the successful transaction.

    To use an e-mail template, tick the option 'Use E-mail Template' and select the template from the drop down list. The e-mail templates are maintained in Configuration > Document Libraries > Other Document Templates, under Document Type 'Email Notification Templates [D03]'.

    Alternatively, you may opt not to use e-mail template but instead construct the e-mail content in the E-mail Body. To do so, untick the option 'Use E-mail Template' to activate the 'E-mail Body' field box. Clicking on the browse button will open the parameter wizard to select the tag for the parameter that you would like to show in the e-mail.

    The Instant Content refers to the content that will be displayed in the ESP notifications panel as well as at the bottom right corner of the ESP screen. You can configure the Instant Content for the following notifications:

    • Document Generation - Success

    • Document Generation - Fail

    • My Request - Message Received

    • My Request - Status Update

    Clicking on the browse button will open the parameter wizard to select the tag for the parameter that you would like to show in the notification.
