Verify & Correct

Verify & Correct is used to verify the setup of Document Management. You can also use this function to repair incorrectly set-up areas:

 

Fixing

  • Document Index File Source - This routine is used to fix the DocOrigin field in the DocIndex table. The DocOrigin field contains information about the type of document stored in the database. It is not generally used but Viewpoint support will advise you if you need to run this.

  • Document Storage - This allows for the Document Storage to be verified. It checks the existence of documents between the physical storage and the Document Management Document Holder. The result of the process will be in the log file VerifyRepair_date.log. It flags missing files, records and files with extensions that are more than 10 characters long.

 

Delete

  • Delete tasks done - This deletes all Master File tasks that have been flagged as 'Done'.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • Start - Click this to start the selected process.

  • Close - Click this to exit the screen.
