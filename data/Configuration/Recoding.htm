Recoding

This function is used to re-code Master Files. Always backup your database and ensure you are the only user logged in before starting the process.

Important Note: Re-coding Master Files can lead to forms being lost from the form queue. If this happens, re-trigger them from the Change Confirmation/Approval screen (in Viewpoint - Home screen, click To Do > Jobs > Change Confirmation/Approval, in File Maintenance screen, click General > To Do > Jobs > Change Confirmation/Approval).

 

Recode Selected

Click to change the Master File code for a single Master File.

  1. Ensure you have made a backup of your database, and that no other user is logged in, before clicking OK to proceed.

  2. Select the Existing File Code.

  3. Enter the New File Code.

  4. Click Start.

  5. Confirm the re-coding by clicking Yes.

  6. Click OK when the process is complete.

If the new code already exists, then information for the Master File specified as the Existing File Code will be lost, and the Master File status will be updated as Marked for Removal [9]. A RecodeMasterFiles_<yyyyMMddhhmm> log file will be generated and placed in the program's Export folder to show fields that will be updated due to the recoding, as well as records that are deleted.

For example, if you have the following Master Files in your database:

  • Viewpoint Ltd [VP0001]

  • Viewpoint Limited [VP0002]

when you specify VP0001 in the Existing File Code and VP0002 in the New File Code, the current Viewpoint Ltd [VP0001] will have its Master File status updated to Marked for Removal [9] and instances of Viewpoint Ltd [VP0001] recorded in other Master Files will be replaced with Viewpoint Limited [VP0002].

 

Note: After recoding, if you are using Document Management, navigate to Configuration > Database Archiving > Troubleshooting and select 'Document Management Master File Field' to rebuild the Master File details in the Document Holders.

 

Recode Using 'RecodeBatch'

Click to re-code all the Master Files listed in the RecodeBatch table. This feature is used when you want to re-code multiple Master Files at one time and requires some knowledge of SQL database  and the use of a database table. Make sure your database has been backed up and that no one else is logged into the system before you start the recoding. This function CANNOT be used to merge Master Files.

  1. Access the database using Enterprise Manager or SQL Server Management Studio Express where appropriate.

  2. Locate the 'RecodeBatch' table. This table will be used to identify the Master Files codes to be changed.

  3. RecodeBatch consists of four columns. Specify the current Master File codes in the 'OldCode' column and their related new codes in the 'NewCode' column. The 'Is Done' column will be 0 (zero) indicating that no change has been made yet. This value will change to -1 when a successful re-code has been done.

  4. In Viewpoint – Home screen, go to Configuration > Database Archiving, and then select 'Recoding'.

  5. Click the 'Recode using 'RecodeBatch’' button.

  6. Type in 'YES DO RECODE' in the text box of the window that pops up, and then click 'OK'.

  7. The system will automatically change the codes as specified in the 'RecodeBatch' table.

  8. Click 'OK' when the process is complete.
