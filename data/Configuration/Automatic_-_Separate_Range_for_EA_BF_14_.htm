Automatic - Separate Range for EA/BF[14]

The Automatic - Separate Range for EA/BF [14] coding method will assign a Master File code that is made up of prefix and a running number where different prefixes can be set for the following:

  • Master Files that are managed entities - the Entity Administration option is ticked

  • Master Files that are non-managed Billing Files - the Time & Billing (Sales Ledger) option is ticked

  • Master Files that are non-managed entities - both the Entity Administration and Time & Billing (Sales Ledger) options are NOT ticked

The prefix for each type will be set in Configuration > Settings > Customise > System Running Numbers for the following System Running Number Types:

  • Entity File Codes [ADE] for managed entities

  • Billing File Codes [ADB] for non-managed Billing Files

  • Master File Codes [ADR] for non-managed entitiees

Note: For Master Files with both Entity Administration and Time & Billing (Sales Ledger) options ticked, these are considered as managed entities and will follow the prefix set for Entity File Codes [ADE].

 

Format of the Master File Code

The first few characters will be the prefix that you have specified and the remaining will be the number for the the Entity File Codes [ADE], the Billing File Codes [ADB] or the Master File Codes [ADR]

Both the numbers are running numbers that will start at 1 and increases so the length of the Master File code will depend on the prefix that you defined as well as the number.  

Note: The Master File code cannot be longer than 10 characters.

 

Using 'Automatic - Separate Range for EA/BF [14]' coding method

To start using this coding method, do the following:

  • Select the coding method in System Defaults > General/Master File

  • Set up the required System Running Number types

Select Coding Method

  1. In Viewpoint – Home, go to Configuration > Settings > System Defaults > General/Master File.

  2. Under Coding Method for Master File, select Automatic - Separate Rage for EA/BF [14].

  3. Click Update.

 

Set up System Running Number Types

The three System Running Number types may not yet be available in your system. If you are not able to locate it, then you need to add the type by doing the following:

  1. In Viewpoint – Home, go to Configuration > Settings > Customise > System Running Numbers.

  2. Click New.

  3. In Document Type, select the required type e.g. Billing File Codes [ADB].

  4. Specify the Current Doc. Number as 0.

  5. Specify the required prefix in the Prefix & Format field.

  6. Click Update.

  7. Repeat for the other types - Entity File Codes [ADE], Master File Codes [ADR].

If the type is already available, you can select it and click Edit to specify the required prefix.

 

Prefix and Format

The Prefix & Format field can also include the following:

  • Zero '0' to set a uniform length

  • User Office code

 

 

 

How do I assign a user to a User Office?

This is done in Configuration > Users & Security > Users but you must ensure that:

  • User Office field is enabled

  • Values have been added to the User Office [USOF] Lookup Table
