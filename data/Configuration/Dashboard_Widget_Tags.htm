Dashboard Widget Tags

Each widget type comes with tags that control the design, functions, and the data to display for the widget. While most tags come with predefined values, users can change the tag values for existing widgets or create new widgets with new values.

The following table lists the available tags. Note that some tags are only applicable to certain types of widgets or at Home or File level only. Tags that require SQL queries must use SELECT queries.

 

Tag

Explanation

Example Tag Values

[/SQL=<Query>/]

(Mandatory) For chart construction to retrieve values based on the Data Source columns. This tag must be used with an SQL query.

Applicable for all widget types except Navigation Listing and Summary Widget.

[/SQL=SELECT ER.RvwDescription, DATEDIFF(DD,GETDATE(),ER.DateDue) AS CountNr, RT.[Name]

FROM EntReviews ER LEFT JOIN EntReviewType ERT ON ER.EntCode=ERT.EntCode AND ER.ReviewCode=ERT.ReviewCode

LEFT JOIN qLvRVTP RT ON ERT.ReviewType=RT.ElCode

WHERE ER.EntCode='(-MF-)' AND ER.RvwActive=1 AND ER.RvwStatus<>'CP' AND ER.DateApproved IS NULL

ORDER BY CountNr ASC/]

* Sample Tag taken from Widget ID: VPFMFRVW004

[/Filter=<Query>/]

(Optional) This tag defines which Data Source column from the query result in [/SQL=/] tag to populate the filter’s drop-down option.

Applicable for all widget types except World Map, Navigation Listing, Summary Widget, and Score Cards.

[/Filter=Name/]

* Sample Tag taken from Widget ID: VPFMFRVW004

[/Filter.Default=<Query or plain text>/]

[/List=<ID>/]

(Mandatory) Refer to Configuration > Settings > Customise > Windows Menu for Menu Id. Three methods to define the tag value:

1. User Defined listing

·       [/List=<Header=VPMENUID><MN= VPMENUID><DIV>/]

·   <Header=> is the first menu of the grouping.

·   <DIV> is the divider.

2. Menu range

·       [/List=< VPMENUID >:< VPMENUID >/]

·       This will construct from the beginning of the Menu ID until the end of the Menu ID.

·       The grouping/indent level of the menu will auto follow Viewpoint’s level.

3. Single Menu

·       [/List=< VPMENUID >/]

·       This will auto construct all menus under this Menu ID.

·       The ID define in the tag will be grouping head.

·       The indent level is only 1 and 2 for Header and Child menu.

Note: Replace the < VPMENUID > with Viewpoint Menu ID.

If there is any Workflow bound to the Menu ID, the group of Workflow option under the Menu ID will be auto created.

The following are variables that can be used to specify conditions in SQL Query tags:

SQL Query Variable

Explanation

Example Tag Values

(-MF-)

(Optional) Only applicable to Widgets under File level. This will retrieve data based on the opened Master File.

This tag supports ALL SQL Query

SELECT * FROM TABLENAME WHERE RefCode=’(-MF-)’

[/SQL=SELECT TOP 10 [Name] AS AcctName, CurCode, SUM(CurBalance) AS SumAmount  FROM AcctItems   WHERE AcctType='B' AND Active=1 AND EntCode='(-MF-)' AND CurBalance > 0  GROUP BY CurCode, [Name]  ORDER BY SumAmount DESC/]    Mandatory field. Put column name from the SQL query result to generate as Axis.  [/Axis=AcctName/]

* Sample Tag taken from Widget ID: VPFCABANK002

(-UC-)

(Optional) Only applicable to Widgets under File level. This will retrieve data based on the current logged in user.

This tag supports ALL SQL Query

SELECT * FROM TABLENAME WHERE UCode=’(-UC-)’

[/SQL=WITH TimeBucket AS (   SELECT '<1' AS IncorpYear, 0 AS Num UNION ALL  SELECT '1-5' AS IncorpYear, 1 AS Num UNION ALL  SELECT '6-10' AS IncorpYear, 2 AS Num UNION ALL  SELECT '11-15' AS IncorpYear, 3 AS Num UNION ALL  SELECT '16-20' AS IncorpYear, 4 AS Num UNION ALL  SELECT '>20' AS IncorpYear, 5 AS Num),   CompAge AS (  SELECT E.IncorpPlace, LV.[Name] AS Country, CASE WHEN DATEDIFF(YY,E.IncorpDate, GETDATE()) <1 THEN 0  WHEN DATEDIFF(YY,E.IncorpDate, GETDATE()) BETWEEN 1 AND 5 THEN 1  WHEN DATEDIFF(YY,E.IncorpDate, GETDATE()) BETWEEN 6 AND 10 THEN 2  WHEN DATEDIFF(YY,E.IncorpDate, GETDATE()) BETWEEN 11 AND 15 THEN 3  WHEN DATEDIFF(YY,E.IncorpDate, GETDATE()) BETWEEN 16 AND 20 THEN 4  WHEN DATEDIFF(YY,E.IncorpDate, GETDATE()) >20 THEN 5 END AS IncorpYr   FROM Entity E  LEFT JOIN qLvCNTR LV ON E.IncorpPlace=LV.ElCode  INNER JOIN (SELECT Ucode As 'UniqueUCode', RefCode As 'UniqueRefCode', MfNr, IsReadOnly FROM VwSecurityUserFile) AS VpSecurityFile1 ON E.EntCode = VpSecurityFile1.UniqueRefCode  WHERE E.[Status]='0' AND (-SCOPE-) AND VpSecurityFile1.UniqueUCode = '(-UC-)')   SELECT A.Country, T.IncorpYear, COUNT(IncorpYr) AS CountNr  FROM TimeBucket T  LEFT JOIN CompAge A ON T.Num=A.IncorpYr  GROUP BY T.IncorpYear, T.Num, A.Country   ORDER BY T.Num/]

* Sample Tag taken from Widget ID: VPFCABANK002

(-USG-)

(Optional) Only applicable to Widgets under File level. This will retrieve data based on the current logged in user’s User Group.

This tag supports ALL SQL Query

SELECT * FROM TABLENAME WHERE UserGroup=’(-USG -)’

* Currently no demo widget is using this tag but would be similar to (-UC-)

(-AGP-)

(Optional) Only applicable to Widgets under File level. This will retrieve data based on the logged in user’s default Administrator Group.

This tag supports ALL SQL Query

SELECT * FROM TABLENAME WHERE AdminGroup=’(-AGP-)’

* Currently no demo widget is using this tag but would be similar to (-UC-)

(-SCOPE-)

(Optional) Only applicable to Home level Widgets. This will append the scope selection record filter query condition based on the [/SQL=/] tag. 

This tag supports ALL SQL Query

This tag MUST be together with the tags below:

·       [/SCOPE.Fcode=/]

·       [/SCOPE.Ucode=/]

[/SQL=SELECT CASE WHEN RL.[Name] IS NULL THEN 'Unassigned' ELSE RL.[Name] END AS BankName, AI.CurCode, SUM(CurBalance) AS SumAmount   FROM AcctItems AI   INNER JOIN EntityAccount EA ON AI.EntCode=EA.EntCode   LEFT JOIN RefLabels RL ON AI.RefCode=RL.Code   WHERE AI.AcctType='B' AND CurBalance<>0 AND (-SCOPE-) GROUP BY RL.[Name], AI.CurCode/]

* Sample Tag taken from Widget ID: VPCABANK003
