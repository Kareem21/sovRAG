Engage Servicing Portal (ESP)

Engage Servicing Portal (ESP) is an interactive web portal for clients (end-users) to:

  • access information and documents related to their entities.

  • view calendar of upcoming events.

  • act as a medium of communication with the relevant parties of the entity (directors, secretary, lawyers, shareholders, etc.).

  • act as a medium to request and track services and changes.

The portal is designed to connect through the web with one or multiple Viewpoint core environments (effectively the 'Back-Office' databases or VP-DB, for short) making it possible for portal users (clients) to instantly and transparently access their authorised information. The ESP system consists of the following:

Back-Office:

Application Block

Description

Viewpoint Connect (VP-BO)

This refers to the Viewpoint Connect (Back-Office application).

Viewpoint Back-Office database (VP-DB)

This refers to the Viewpoint database in Viewpoint Connect.

Viewpoint Connect REST Services

This refers to the REST API host in IIS and is connected to the Back-Office database (VP-DB). It acts as an endpoint for various requests from CP-WP.

Note: Viewpoint Connect REST Services needs to be same version as ESP.

 

Front-Office:

Application Block

Description

Client Portal Administrator (CP-AD)

This is the Viewpoint Connect program for various Engage Servicing Portal configuration.

Client Portal Database (CP-DB)

This is the Engage Servicing Portal database. Information and data to be published to the Engage Servicing Portal are retrieved from VP-DB and cached in this database.

Task Scheduler & Notification Agent (TSNA)

This refers to the Windows service that manages various Viewpoint scheduled activities such as update ESP calendar, review To-Do list etc.

Viewpoint Connect REST Services

This refers to the REST API host in IIS and is connected to the Front-Office database (CP-DB). It acts as an endpoint to receive requests from TSNA.

Client Portal Web (CP-WP)

This is the Engage Portal (web portal) that is available to the end client.

Note: Microsoft Internet Information Service (IIS) is required for this block.

Configuration of Engage Servicing Portal

Setting up of ESP is done via Engage Servicing Portal Administrator (CP-AD) through the following screens in Viewpoint Connect - Home > Configuration > Engage Servicing Portal:

  • Global > Databases -  to define the database(s) for the ESP

  • Global > Menus - to define properties of the screens in ESP

  • Global > Relations - to define the caption for the relation types seen in ESP

  • Global > Dashboard - to customise caption and/or activate/deactivate available widgets for the Dashboard screen in ESP

  • Global > Notifications - to define the notifications that the user will receive

  • Global > Followers - to define the items in ESP that a user can follow

  • Configuration > Organisations - to set up the organisations in ESP

  • Configuration > Users - to set up user accounts for accessing the ESP

  • Notice Board - to configure items for the Notice Board in ESP

  • User Role - to configure access to the screens in ESP

Optionally, you can also proceed with the following once configured:

  • approval of Organisations and Users, if required. Refer to Approval.

  • sharing of documents stored in Viewpoint Back-Office Database (VP-BO). Refer to Document Sharing on Engage Servicing Portal.
