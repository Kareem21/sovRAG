Identity Register Type - Tags

These tags are to be added to the Tag Values field of the Identity Register Type:

Tag

Explanation and Example

[On_Update|From_Field=<FromField>|To_Field=<Prefix>.<ToField>|DESCR=<Descr>]

where

  • <FromField> can be:

  • IdCode - Value entered in Identity Text and can be passed to a text field.

  • FromDate - Value entered in From Date and can be passed to a date field.

  • ToDate - Value entered in To Date and can be passed to a date field.

  • Country - Value entered in Country/Jurisdiction and can be passed to a text field.

  • Country_Code - Value entered in Country/Jurisdiction and can be passed only to a lookup value field.

  • LookupValue - Value entered in Lookup Value 1 and can be passed only to a lookup value field.

  • LookupValue2 - Value entered in Lookup Value 2 and can be passed only to a lookup value field.

  • <Prefix> can be C for Compliance table, R for RefMaster table.

  • <ToField> would be field codes for fields in CDD screen or Identity Details screen.

  • <Descr> this would be the text that appears for the option.

 

This is to allow the information entered for the Identity Register Type to be updated to fields in other screens such as:

  • Identity Details

  • CDD

  • Entity File - General (when Identity Register Type is not for Individual Master File)

  • Statutory - Company (when Identity Register Type is not for Individual Master File)

 

Example:

Update Passport Number field in Identity Details screen - for Individual Master File

[On_Update|From_Field=IdCode|To_Field=C.PassportNr|DESCR=Update Passport No.]

 

Update the Gender in Identity Details screen for Individual Master File

[On_Update|From_Field=LookupValue|To_Field=C.Gender|DESCR=Gender]

 

Update the Marital Status in Identity Details screen for Individual Master File

[On_Update|From_Field=LookupValue2|To_Field=C.MaritalStatus|DESCR=Marital Status]

 

Update Place of Incorporation in Identity Details screen - for Non-Individual Master File

[On_Update|From_Field=Country|To_Field=C.PasPlaceIssue|DESCR=Update Place of Incorporation]

 

Update Incorporation Number in Identity Details screen - for Non-Individual Master File

[On_Update|From_Field=IdCode|To_Field=C.PassportNr|DESCR=Update Incorporation Number]

 

Update Date of Incorporation in Identity Details screen - for Non-Individual Master File

[On_Update|From_Field=FromDate|To_Field=C.PasDateIssue|DESCR=Date of Incorporation]

 

Note: When you do use this tag, set the authority level for the fields in the other screens e.g. Identity Details or CDD screen to authority level 9 so information is only entered in the Identity Register screen.

[UKey=<Field>+<Field>+...]

where <Field> can be:

  • IdType - Identity register type

  • Country - Country/Jurisdiction

  • UdCode - User Defined Table

  • LookupValue - Lookup Table

  • MasterFileLink - Master File Link

If the tag is used, when the user creates a new record, the program will check on value entered for the defined field(s) with existing records. The record will not be accepted if the information is similar to an existing record.

 

Example:

To only allow one instance of the identity type 'Birth Certificate' - enter this tag in Tag Values field for the identity register type 'Birth Certificate':

[UKey=IdType]

 

To only allow one instance of passport per country - enter this tag in Tag Values field for the identity register type 'Passport':

[UKey=Country]

[<DBField>.Mandatory=<Value>]

where:

  • <DBField> is the Database Field name for the field. You can obtain this from Configuration > Settings > Field Settings for the Identity Register [FMMCOMPR] screen or the Classification and Indicia [FMMREVIC] screen.

  • <Value> can be 'Y' or 'N' which sets the field to be mandatory or not mandatory.

This is to specify if the field is mandatory or not.

 

Example:

To set the field 'Country/Jurisdiction' to mandatory:

[Country.Mandatory=Y]

[<DBField>.Caption=<Value>]

where

  • <DBField> is the Database Field name for the field. You can obtain this from Configuration > Settings > Field Settings for the Identity Register [FMMCOMPR] screen or the Classification and Indicia [FMMREVIC] screen.

  • <Value> is the caption for the field.

This is to specify a caption for the following fields:

  • Lookup Value 1 - default caption is the name of the specified lookup table

  • Lookup Value 2 - default caption is the name of the specified lookup table

  • User Table 1 - default caption is the name of the specified user table

  • User Table 2 - default caption is  the name of the specified user table

  • Country/Jurisdiction - default caption is Country/Jurisdiction

  • User/Account Name and Password - default caption is User/Account Name and Password

  • From Date - default caption is From Date

  • To Date - default caption is To Date

  • Reminder - default caption is Reminder Date

Example:

To set the caption for 'Country/Jurisdiction' to Country Issuing Passport:

[Country.Caption=Country Issuing Passport]

 

[<DBField>.Authority=<Value>]

where:

  • <DBField> is the Database Field name for the field. You can obtain this from Configuration > Settings > Field Settings for the Identity Register [FMMCOMPR] screen or the Classification and Indicia [FMMREVIC] screen.

  • <Value> can be '1','6','8','9' which sets the field authority. It does not include the following values '0' and '99' as control of whether the field is available or not is by ticking or unticking the field for the Identity Register type.

This is to specify the authority required for the field.

 

Example:

To set the authority for the 'Country/Jurisdiction' to [8]:

[Country.Authority=8]

[<DBField>.Default=<Value>]

where

  • <DBField> is the Database Field name for the field. You can obtain this from Configuration > Settings > Field Settings for the Identity Register [FMMCOMPR] screen.

  • <Value> varies depending on the field:

  • List fields - Code for the lookup value or user table value e.g. 'FCST'.

  • Checkbox field - 'Y' for checked or 'N' unchecked.

  • Text field - Default text, e.g. 'Identity Type'.

  • Date field - Date, e.g '12/12/2016', 'NOW'

This is to set the default value for the field.

 

Example:

To set the 'From Date' as today's date.

[FromDate.Default=NOW]

[SUBSCRIBER=<Subscriber>]

where <Subscriber> is the subscriber group code.

 

 

 

This is to specify the Subscriber group(s) that will require the Identity Register type. Use a comma to separate multiple values.

 

Example:

To replicate the Identity Register type to subscriber groups 'GEN' and 'ASA'

[Subscriber=GEN,ASA]

 

Note: This is applicable if you have Master Configuration Environment (MCE).

[SUBSCRIBER<><Subscriber>]

where <Subscriber> is the subscriber group code.

This is to specify the Subscriber group(s) that will be excluded when replicating the Identity Register type.

 

Example:

To not replicate the Identity Register type to subscriber group 'EUR'

[Subscriber<>EUR]

 

Note: This is applicable if you have Master Configuration Environment (MCE).
