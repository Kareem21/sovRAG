Approval Cards

Approval cards are used when an item requires an approval from a user. These can be for any of the following:

  • Payee and bank payments - the relevant approval card will be automatically applied to the payment record based on matching values in the Master File Code, Sub-ledger/Account, Currency and Up to Amount fields.

  • Changes - the approval card must be linked to the relevant Change Code through Change Codes screen.

  • Workflow steps - the approval card must be specified for the workflow step.

 

Create Approval Cards

The approval cards are created in the Approval Cards screen which can be accessed through Configuration > Users & Security > Approval Cards.

 

Approval Card list

The list shows all existing Approval Cards.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • New - Click this to create a new Approval Card.

    1. Enter details for the Approval Card.

    2. Click Update.

    Note: If the option 'Restrict Own Record Approval' is ticked, this means that if the item for approval was created by the approver, then approval is only possible by another user in the Approval List.

  • Edit - Click this to edit the details of an existing Approval Card.

  • Rebuild Approval List - Click this if you made changes to the Approval List of an existing Approval Card. This will also update the Approvals screen for the relevant approvers.

  • Rebuild All - Click this rebuild the Approval List of all existing Approval Cards.

 

Assign Approval Cards

If the approval card is for a Change or workflow, then the approval card must be assigned to the Change Code or workflow step.

 

Assigning Approval Card to Change

  1. In Viewpoint - Home, go to Configuration > Business Parameters > General > Change Codes.

  2. Select the Change Code.

  3. Click New.

  4. In the Change Link screen, set the Link Type as Approval Cards and select the approval card.

  5. Click Update.

Assigning Approval Card to Workflow Step

Approval card in workflow is used to have a workflow step approved before the step can be completed. To assign the approval card, the workflow step must contain a checklist item with the field type set as 'Approval Card' and the relevant approval card specified in the Values/Default column. Refer to Linking Approval Card to Workflow Step in Configuring Checklist Item.
