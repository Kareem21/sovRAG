Localise Script Tags - System Defaults

The tags below are to set values for settings and options in System Defaults.

 

Setting up Non-checkbox Fields

This refers to settings in System Defaults that require a value to be selected or entered.

 

Syntax

SETTS:<Application>.<Key>=<Value>

 

where:

  • <Application>.<Key> can be retrieved from the Table field in Configuration > Settings > Customise > Field Settings.

  • <Value> can be any value that is accepted by the field in the actual screen. This may be a Currency code EUR, a Country code IM, an e-mail address like support@goviewpoint.com or a value like 6, depending on the field that you are setting up. In cases like these the Database Field will display SetStr:

 

Example

For instance, to set the Coding Method for Master File to Manual – 6 Characters [6], the tag would be: VPCROSS.CDADDR=6

 

Setting up Checkbox Fields

This refers to options in System Defaults where the checkbox is ticked to enable the option and unticked to disable the option.

 

Syntax

SETNR:<Application>.<Key>=<Value>

 

where:

  • <Application>.<Key> can be retrieved from the Table field in Configuration > Settings > Customise > Field Settings.

  • <Value> is the value next to the text SetNos in the Database Field. Use a dash ‘-’ to untick a field.

 

Example

To tick the checkbox for Enable detailed ‘Relationship Roles’, the tag would be: VPCROSS.GENOPT=131072

 

To untick the checkbox, the code would be: VPCROSS.GENOPT=-131072
