SQL Selection

eView profiles can be customized to display information that is required by a particular User Group.

 

Customizing Reports

The number of reports a User can view or access in the eView Home Category can be customised so that reports that are not applicable to a particular User or User Group can be removed from the menu box. The eView profile type MUST be H (Home). Reports can only be listed in your Home page.

 

Configuring URL Query String

The URL query string tells the system which report it should be redirected to.

Minimum information - The minimum information required in the query string is the form code, which indicates the HTML template to be linked to the report:

fc=<form code>

Database table keys - If database table keys are required to select a particular record, supply the key information. Multiple keys are delimited by a pipe:

keys=<field name>:<field value>

keys=<field 1 name>:<field 1 value>|<field 2 name>:<field 2 value>...

 

Example:

A complete query string can contain just the form code, or the form code and the key information:

fc=SRH_NOTE

fc=SRH_NOTE&KEYS=Ucode:(-UCode-)

In the following query string,

'Master File Menu box

Master File\?BOX

Addresses and Contacts\?fc=MSADCT

Master File Acts as Officer for\?fc=MSOFFR

 

Code

Explanation

'Master File Menu box

This is a remark which starts with an apostrophe

Master File\?BOX

This is the menu header name and always ends with \?BOX.

Addresses and Contacts

and

Master File Acts as Officer for

This is the report caption or title and always ends with \?.

\?

This is the delimiter that separates the report caption and query string.

fc=MSADCT

and

fc=MSOFFR

This is the query string and comes after the delimiter (\?).

 

 

Customizing Other Profiles

Other profiles in eView may have specific search parameters in order to retrieve specific files. These parameters can be used to limit the files a user can access. There are currently four search parameters that can be used:

Search Criteria

Query format

Example of resulting search result

Administration Group

...IN (-AGROUP-)...

where Administrator has access to Admin Group1, Admin Group2 and Admin Group3 files:

Files from 'GROUP1', 'GROUP2' and 'GROUP3'.

User Code

...=(-UCODE-)...

where Administrator's User Code is DD (Donald Daly):

Files where Administrator is Donald Daly.

User Group

...=(-UGROUP-)...

where Administrator's User Group is SYSMGR:

Files that can be accessed by System Managers.

First Administration Group (i.e. the default Administrator Group for the Administrator}

...=(-FAGROUP-)...

 where Administrator's default Admin Group is Group1:

Files from 'GROUP1'.

 

Each profile can have more than one search parameter, separated by the pipe symbol '|', as shown below:

fc=RepMenu&KEYS=Ucode:(-UCODE-)|AGROUP:(-AGROUP-)
