Relationship Type - Tags

These tags are to be entered in the Tag Values field for the Relationship type.

 

Tag

Explanation and Example

[OFFICERTEXT=<Text>]

where <Text> can be just text string or field codes such as:

  • (-AddrCode-) - The appointed Master File.

  • (-RDescr-) - The description of the relationship type.

  • (-AltDescr-) - The alternative description of the relationship type.

  • (-RelatedAddress-) - This is Additional Relation 1.

  • (-RelatedAddress2-) - This is Additional Relation 2.

  • (-RelatedAddress3-) - This is Additional Relation 3.

This is to define the description for the column 'Relation type' in the grid display.

 

Example:

[OFFICERTEXT=(-RelatedAddress1\#AD40-0-) is the Auditor in charge]

 

Note: With the exception of (-RDescr-) and (-AltDescr-), you can also include format options.

[<Component>.<FieldName>]

where

  • <Component> - This refers to the component where the Master File is active:

    • EA - Entity File in Entity Administration.

    • BF - Billing File in Practice Management > Sales Ledger.

    • SF - Supplier File in Practice Management > Purchase Ledger.

    • CA - Account File in Accounting.

  • <FieldName> - This is the field that is to be updated with the relation:

    • AdminCode - This refers to the Administrator field.

    • Admin2Code - This refers to the Manager field.

    • Admin3Code - This refers to the Controller field.

When a relationship type has a User relation, this tag will update the specified user to the Administrator, Manager or Controller field of the Entity File, Billing File, Supplier File or Account File.

If you want the relation to update more than one field, add all the relevant tags.

 

Example:

[CA.AdminCode]

will update the Administrator field for the Account File.

 

Note: The relation type must be a user relation and the update is done when the relation is created or an existing relation is updated.

[SUBSCRIBER=<Subscriber>]

where <Subscriber> is the subscriber group code.

This is to specify the Subscriber group(s) that will require the relationship type. Use a comma to separate multiple values.

Example:

To replicate the relationship type to subscriber groups 'GEN' and 'ASA'

[Subscriber=GEN,ASA]

[SUBSCRIBER<><Subscriber>]

where <Subscriber> is the subscriber group code.

This is to specify the Subscriber group(s) that will be excluded when replicating the relationship type. Use a comma to separate multiple values.

Example:

To not replicate the relationship type to subscriber group 'EUR'

[Subscriber<>EUR]
