System Defaults - Entity/BPM

The defaults for the Entity and Forms are set up here. Before you proceed, ensure that you have set up the Master File for the Presenter, and codes for the various jurisdictions. Viewpoint comes pre-installed with two-letter ISO-standard codes for most countries. However, if the country code you need is not in the list, you can create it in the Country Codes [CNTR] lookup table.

 

Jurisdiction

Enter the default country for the Form Library. Each time you open the form library, the jurisdiction will automatically default to the country entered here.

See also:

  • Form Library

 

Presenter

Enter the Presenter's Master File. This information appears on some statutory forms and can be coded to appear in your own customised forms. There are special codes in the Field Wizard that will retrieve the Presenter's information on statutory forms.

See also:

  • Field Wizard

 

Master File Bearer Shares

Enter the Master File that is to be used as shareholder for bearer shares. This information is used to report on Bearer shares in some statutory forms.

 

Exclude General Forms from Selections

Tick this box if you want to EXCLUDE Forms in the 'General' category from being triggered in Viewpoint Administrator. When ticked, the program will only look through the Jurisdiction-related forms when processing an event.

 

Deactivate Validation of Share Certificate Number

Tick if you wish to inactivate the validation of share certificate numbers.

 

Restrict Distributions to Beneficiaries

Tick this box if you want to restrict distributions only to the beneficiaries of a Trust. If you do not tick this box then you can do a distribution to any Master File.

 

Allow Multiple Foreign Registration in one Jurisdiction

Tick this box if you want to allow more than one Foreign Registration to be created for one jurisdiction. This is useful if you want to record multiple details such as the VAT number and the Income Tax number for the same jurisdiction in the Foreign Registration screen.

 

Trigger Change Processing for Global File Review

Tick this box if you want to display the Change Processing screen when posting updates from Global File Review. This will list all items linked to the changes applied (updates posted).

 

Global Change of Particulars

The options below are to determine if the Global Change of Particulars for statutory filing will be triggered due to changes made for Master Files that are appointed different roles.

 

Trigger for Statutory/Trust/Foundation Officers

This option is ticked by default and will trigger Global Change of Particulars for statutory filing of changes made for active Statutory, Trust and Foundation Officers.  

 

Trigger for Entity Relations

This option is ticked by default and will trigger Global Change of Particulars for statutory filing of changes made for active Entity Relations.

 

Trigger for Shareholders

Tick this option to trigger the Global Change of Particulars for statutory filing of changes made for active shareholders.

 

Trigger for Beneficiaries

Tick this option to trigger the Global Change of Particulars for statutory filing of changes made for trust beneficiaries where Date Ceased is empty or later than the date of the change.

 

Trigger for Entity Interest Holders

Tick this option to trigger Global Change of Particulars for statutory filing of changes made for interest holders of managed entities where Date Ceased is empty or later than the date of the change.

Note: This does not apply to interest holders for non-managed entities.

 

Trigger for Owners & Controllers

Tick this option if requires changes to Owners/Controllers details to trigger the Global Change for statutory filing. This will only trigger for Owners/Controllers without ‘Date Ceased’ defined or ‘Date Ceased’ with a later date.

 

Mandatory to Select Change of Particulars

Tick this option to trigger the Global Change without prompting a message to confirm Yes/No whenever the details of the Master File has been changed.

 

Workflow

Create Activity for Workflow If None Exists

Tick this option to trigger the creation of activity if the workflow does not has a linked activity when saved. Activity will be created based on the activity template that is linked to the workflow template. Activity's details such as Category, Sub-Category, Description, Assign To, Due Date and Note will be the same as the workflow.

 

Command buttons:

Click on the relevant command button to perform the necessary.

  • Update - Click this to save changes.

  • Cancel - Click this to cancel the changes.
