System Defaults - Document Management

The settings for Document Management are defined here.

 

Document Store

Enter or browse to the folder path where documents are to be stored. Use relative paths to assign drives (for example: \\server\Viewpoint\DocStore). Special attention must be paid when Viewpoint programs are installed on a server and workstations are used to run the programs. To assign drives, relative paths must be used to ensure that the programs will properly run from the workstations. Therefore, Viewpoint recommends using the network name instead of the assigned drive name.

Folder path validation is disabled when Viewpoint Connect is set up in Cloud Solution ('Enabled Viewpoint Connect to connect through Application Server' is ticked in Viewpoint Management with Application Server running).

Note: If you have moved your Document Store to the SQL database, then this field will show ;[DMSTORAGE].

 

Audit Trail Folder

Enter or browse to the path where scanned batches and the audit trail are stored. Folder path validation is disabled when Viewpoint Connect is set up in Cloud Solution ('Enabled Viewpoint Connect to connect through Application Server' is ticked in Viewpoint Management with Application Server running).

 

Holder Security

This feature allows you to increase the security level on a document holder. Select the appropriate security type from the system standards. The following table shows the conditions to access document holders determined by the Holder Security and File Access selected.

Holder Security

File Access

 

No Files [0]

Administrator Groups [2]

Administrator Groups + View-only for Others [4]

No Additional [0]

No authority to view any holder.

 

Has authority to view and access all holders in their authorised Filing Cabinets based on the Document Management Authority Settings in User Groups.

 

Security Model - Master File [1]

 

Has authority to view and access holder when user:

  1. Has access to Master File.
  2. Has Filing Cabinet authority.

Has authority to view and access holder when user:

  1. Has access to Master File. Note: Without access, holders will be view-only. User cannot Edit or Delete a holder. Holder Details is viewable and documents can be opened.
  2. Has Filing Cabinet authority.

Administrator Group - Index Field [5]

 

Has authority to view and access holder when:

  1. Holder's Administrator Group matches the User's Administrator Group.
  2. User Has Filing Cabinet authority.
  3. User has access to Master File.

Has authority to view and access holder when:

  1. Holder's Administrator Group matches the User's Administrator Group.
  2. User has access to Master File. Note: If 1 or 2 is not fulfilled, holders will be view-only. User cannot Edit or Delete a holder. Holder Details is viewable and documents can be opened.
  3. User has Filing Cabinet authority. Note: If the Administrator Groups do not match, the holder will not appear in Documents Enquiry screen.

Note: If a user does not have the authority to access the holder, the Detail Window will show a 'Group Security - Not authorised to view this document holder' status when the grid record is selected.

 

If you are using Administrator Group - Index Field [5], you need to set up the Administrator Group index field, which must be made mandatory for completion:

  1. In Business Parameters > Document Management > Index Fields, click New.
  2. Enter the details of the new index field, but ensure that you do the following:
    • Mandatory - Tick this check box.
    • Field type - Select Admin. Group (dropdown) [73].
    • Format/Mask - Select Assigned Groups [A]. This option means that the user will be able to select his Administrator Group(s) or the common Administrator Group  when creating a new holder.

      The common Administrator Group allows for users to access the holder even if the user is not a member of the Administrator Group. To apply, define the common Administrator Group in System Settings and set this as the Administrator Group for the holder.

       

       

  3. Click Update.

Ensure that this index field is applied to all filing cabinets using Apply Global.

 

 

Processed E-mail/Attachments

Select the default method for storing attachments of e-mails. This applies to processed e-mails and e-mail sent through Delivery Manager function.

  • Separate Documents in Holder - Attachments are stored as separate documents from the e-mail.

  • Embedded in e-mail - Attachments are stored embedded in the e-mail. You would not know if the stored e-mail contains any attachments unless you open the e-mail.

  • Both Embedded and Separate Documents - Attachments are stored embedded with the e-mail as well as separately from the e-mail.

Electronic Signatories

Select ARX Cosign, DocuSign OAuth, Skribble, or Adobe if you use either one of these program and you would like to turn on Electronic Signatories option for stored holders. Otherwise, choose No Integration. Depending on the selected option, additional details may be required.

Note that Viewpoint only allows the use of these program in Document Management for PDF and scanned documents that have been stored.

Default Reject Task

Select a task that you want to be added to the workflow's step when user choose to reject the current step. The task will be assigned to the administrator of the rejected step and the due date will follow the rejected step's date due or set to today's date. This works only if the option 'Task for Workflow Undo/Reject' is ticked. If there is no task selected, user will manually select the task to be added.

 

Default Undo Task

Select a task that you want to be added to the workflow's previous step when user choose to undo the current step. The task will be assigned to the administrator of the previous step and the due date will follow the previous step's date due or set to today's date. This works only if the option 'Task for Workflow Undo/Reject' is ticked. If there is no task selected, user will manually select the task to be added.

The task list is maintained in Configuration > Business Parameters > Document Management > Tasks.

 

Display Completed Task up to (days ago)

You can view completed tasks up to the number of days specified here. For example, if you enter 10, it means you can see tasks that are completed up to 10 days ago in the Task Created By (Me) screen.

 

Maximum File Size (MB)

This field limits the maximum file size (MB) during document storing. Warning will prompt if maximum file size has exceeded and user will not be able to store the file. By default, this field is blank (no limit).

 

Include File Format

This field includes specific file formats (extension) during document storing. Warning will prompt for file formats that are not in this field and user will not be able to store the file. To enter more than one file format, separate the file format with ';'. By default this field is blank (no file extension to include). Both the Include File Format and Exclude File Format fields cannot be used simultaneously.

 

Exclude File Format

This field excludes the file format (extension) during document storing. Warning will prompt if the prohibited file format is found and user will not be able to store the file. To enter more than one file format, separate the file format with ';'. By default this field is blank (no file extension to exclude). Both the Include File Format and Exclude File Format fields cannot be used simultaneously.

 

Global Outlook Correspondence

This acts as an indicator to Outlook on which Filing Cabinet to retrieve files from/store to for each of the Master Files.

 

Retrieve Correspondence Files up to (days ago)

Enter the number of days from the 'Date Last Updated' of the document Holder where Outlook shall retrieve the files.

 

SharePoint Integration

Tick this to enable SharePoint Integration feature where you can publish a document stored in Document Management to SharePoint for collaboration and editing and check in documents on completion of collaboration.

 

Integrate DM for Generated Documents

Tick this if you are using the Document Management module and would like to be able to store generated documents using "Document Management Index Tags". You would not use this option together with the Archive by Master File option in System Defaults > General/Master File.

 

Allow to Delete Holder/Task in Same Day

Tick this to allow users to delete a stored document holder only on the same day it was stored. This means a user will be able to delete a holder that he stored, even if he was not given the authority to do so in his User Group.

 

Delete Forms when Stored in DM

Tick this if, after you have stored a form from Viewpoint, you want to delete it from Form Queue. The file will also be deleted from the Archive folder. Note that the form must have Document Manager index tags.

 

Disable Auto-check for Add-in Upgrades

Tick this if you want to disable the Verify/upgrade Office Add-ins pop up box, which will appear after every Document Management upgrade to remind you to upgrade the Document Management Add-ins.

 

Engage Servicing Portal Document Sharing

Tick this if you want to enable document sharing through Client Servicing Portal (ESP). However, you need to ensure that you have a licensed ESP environment set up.

See also:

  • Document Sharing on ESP

 

Add Master File Code to Master File Name Display

Tick this to also include the Master File code in Master File index fields. Otherwise, only the Master File name is seen. This applies only to document holders created henceforth. For existing holders, you need to rebuild the information using 'Document Management Master File Field' in Database Archiving > Troubleshooting.

If you would like to enable Notifications feature for Document Management change codes, ensure that this is ticked.

 

Enable Audit Trail

Tick this if you want to enable the Audit Trail function in Document Management. The Audit Trail is set up in Business Parameters > Document Management > Audit Trail Setup.

Note: The feature will only begin the trail when you enable it. There is no information on actions done prior to the time you enable it. If you would like to enable Notifications feature for Document Management change codes, ensure that this is ticked.

 

Enable Scan-Batch 'Carbon Copy'

Tick this if you want to assign copies of scanned batches to multiple users in Document Management. You may want to set up default assignments of the owner and copy recipients in Settings > System Settings.

 

Enable Full Text Searching

This option is automatically enabled when you have properly set up the Full Text search catalogs in the Viewpoint Database Management utility (ViewPointDbMgt.exe). Full Text searching provides an additional search field in Document Management's Advanced Search feature, but is not available to users of Access, MSDE or SQL Express databases.

 

Enable Folder Templates

This feature allows you to set up templates so that you can quickly complete the details in a Document Holder before storing.

 

Task for Workflow Undo/Reject

This option allows for a task to be added to:

  • the workflow's previous step when user chooses to undo a workflow step.

  • the workflow's current step when user chooses to reject approval of the current workflow step.

Automatically Link Viewpoint Generated Forms

Tick this to automatically add record link to forms generated from Viewpoint when the form is stored.

 

Enable Revision Management

When ticked, the Revision Management will group versions of a document in a holder into revision groups.

Untick to disable the feature. This will undo all existing revision groups, and all Ribbon icons and right-click context menu options relating to Revision Management will be hidden.

 

Open Safe Document Only

Tick this to enable the option to only open documents that have been scanned for virus and considered safe. A warning message will be prompted when trying to open unsafe or unscanned documents. Various document related functions will also be restricted for such documents.

 

Store Scanned Image as PDF

Tick this to enable the option to save scanned images in PDF format. This will enable the 'Store as PDF' icon in the Scan and Batch Management Preferences Ribbon group.

 

Primary Master File Accessible Security

Tick this to restrict access to a holder for users who can access any of the Master Files specified as the holder's Master File index field with the Key Tag 'INDEX'. Other Master File index fields with other Key Tags will not be considered. For example, Correspondent index field with Key Tag 'CORR'.

 

Command buttons:

Click on the relevant command button to perform the necessary.

  • Update - Click this to save changes.

  • Cancel - Click this to cancel the changes.
