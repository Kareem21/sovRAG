Account Mapping

You can post appropriate journal entries in Accounting for invoice transactions in Practice Management. In order to correctly post these journal entries, the mapping of the components of the invoice in Practice Management to the Nominal Accounts in Accounting must be done. Ensure that you have selected the appropriate setting for the posting function in Configuration > Settings > System Defaults > Accounting - PM Accounting Journals, then set up the mapping of the accounts. Depending on your selection for this, you may need to map one or both of the following:

  • Revenue Entities - 'inhouse' - Set up the revenue entities and their bank accounts before proceeding.

  • Sales Ledger - Client Accounts.

You should have already set up the Chart of Accounts in Accounting before you proceed.

 

Mapping Table

The mapping table is split into two columns. The first column is divided into two sections: the first for Revenue Entities and their bank accounts, and the second section for the Client Accounts. In this second section, all the Charts of Accounts that you have created in Accounting will be displayed. Selecting one of these will display the related components in the second column of the table.

  • Components - This is a list of all the sales and purchase invoice components. The Income and Expense Accounts entries are specified by the Service Code in the Service Master. These are grouped by Service Type. Note that for Expense Accounts - Supplier Invoices - R.PA, the mapping can be empty as long as the nominal account is specified in Nominal Account field (in Create/Edit Purchase Orders screen) when creating the Purchase Order.

  • Type - Select the account type and code from the list. Your selection here will load the available accounts in the Account field (see below).

  • Account - Press F12 on your keyboard to display the list of accounts. When you have selected the account you want to map to, the Account Name field will be completed.

  • Line Description and Narrative - The default line description and narrative that will be used for the generated journal line in Accounting will displayed here. Click the field to display the browse button and amend the description as necessary.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • Add Matter map - Click this to map account code based on Matter type.

    This button is only active for the following components:

    • Control Accounts - Sales Ledger

    • Control Accounts - Receipt/Payments

    • Control Accounts - Intercompany

    • Income Accounts - Sales Invoices - R.SA

    These would include also the Accounting Posting rule for Debtors Provision Accrual, Recurring Fees in Arrears Accrual and WIP Accrual Posting rule.

    If there is a matter type specified, the account mapping will take preference on the matter type specified.

    Example:

    For Components 'Accounts Preparation - TACCS', the journal will be generally posted to Account code '6132'. However, for the specified matter 'Company Secretarial (CO)', it will be posted to Account code '6133'.

    In this example, the Journal '880' for Matter Code 'CO' has been posted to account code '6133' in the Revenue Entity DREAMW.

  • Remove Matter map - Click this to remove the Matter map.

  • Save - Click this to save changes made to the mapping table.

  • Cancel - Click this to cancel any changes made.

  • Reset Mapping - Click this if you want to clear the 'Account Mapping', 'Line Description', and 'Narrative' for the selected template.

  • Reset Description/Narrative - Click this to reset the 'Line Description' and 'Narrative' of the selected template to the system defaults. The defaults are set up in the Account Types Practice Management [PMAT] Lookup Table.

  • Copy - Click this to copy the mapping data. You can paste the results elsewhere, such as in an Excel spreadsheet.

  • Close - Click this to exit the screen.
