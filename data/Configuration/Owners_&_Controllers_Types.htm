Owners & Controllers Types

Types of owners and controllers are set up in this screen so that the records can be added in the Owners & Controllers screen. For each type, specify the following:

  • Jurisdiction

  • Description

  • User Table fields which include:

    • Register Type

    • Exemption Reason

    • Capacity

    • Nature of Interest

    • Status

    • Extra 1

    • Extra 2

    • Extra 3

    • Action Type

  • Non-User Table fields such as:

    • RefCode, RefDirect - Master File fields

    • DateFrom, DateTo, DateIn, DateOut - Date fields

    • PercInt, PercVote - Percentage fields

    • EntOwnNote - Note field

  • Checkbox fields such as Customised and Active

Note: The jurisdiction will allow the type to be available to entities that are incorporated in the same jurisdiction or has a foreign registration for the jurisdiction.

 

Filters

You can filter the types by the status.

 

Select/Apply button

This allows for additional criteria to apply to the identity register type list.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • New  - Click this to create a new type.

    1. Enter details for the new type:

      • For Jurisdiction, select from the list. This will allow the type to be available only for entities in the same jurisdiction and has the same first 2 character of the country code.

      • For Description, enter a description for the type. This is only seen here.

      • For User Table fields, specify the User Table code. Ensure that the User Table has been set up. If the User Table field is not required for the type, leave the field blank.

      • For Tag Values, enter the tags to define field settings for non-User Table fields.

        The syntax is:

        [<DBField>.<Property>=<Value>]

         

        where

        • <DBField> refers to the database field name of the field

        • <Property> refers to the field setting such as

          • Caption – caption for the field with <Value> being the caption

          • Authority – authority required to edit the field with <Value> being the authority level of 0, 1, 6 or 8

          • Visible – if the field is visible or not with <Value> being Y for Yes and N for No

          • Mandatory – if the field is mandatory or not with <Value> being Y for Yes and N for No

    2. Click Update.

  • Edit - Click this to edit an existing type.

  • Delete - Click this to remove an existing type

 

Note: New types can also be created by running jurisdiction-specific Localise Scripts related to Owners and Controllers from Viewpoint.
