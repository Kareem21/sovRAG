Remove Master Files

To ensure data security and integrity, users cannot delete a Master File within any of the Viewpoint components. They must first change the Master File status to Marked for Removal [9], then ask their System Manager to remove the file. This procedure ensures that no Master File is accidentally removed.

When you remove a Master File, the system automatically checks if there are any dependencies attached to the file, such as a shareholding, and if there is, will warn you not to remove the file. You should transfer or remove any dependencies before removing a Master File.

 

Master File list

This shows Master Files where the Master File status is set to 'Marked for Removal'.

 

Remove Master File

Click to remove any Master File that has been marked for removal.

 

Anonymise Master File

Click to anonymise any Master File that has been marked for removal. This should be performed before you remove the Master File.

The Anonymise process will perform the following:

  1. Recode Master File Code to format as configured in System Running Numbers for Document Type 'Master File Anonymised [MFA]'

  2. All fields' value in the database will be masked accordingly.

Note: An Address Cards can only be masked if it is attached only to the Master File that is going to be anonymised.
