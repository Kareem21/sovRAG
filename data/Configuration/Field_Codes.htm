Field codes

Field codes are used to retrieve information for a form, report, invoice or HTML template. The codes will be replaced by data in the corresponding field in the generated document, ESP or Summary Screen. Some field codes can be found in the Field Wizard while others can be retrieved using SQL queries placed in the Key Information.

Defining the field code in the template follows this syntax:

(+<Prefix>.<Field>+)

 

where:

  • <Prefix> can be:

    • HD when the field is retrieved by the header statement for the template.

    • the name of the SQL query used to retrieve the field.

    • EVT when the field is linked to a change.

    • DF when the field is a special field hard-coded to the program as listed in the Field Wizard.

    • DC when the field is a default Entity Information field as listed in the Field Wizard.

    • DU when the field is a User Field.

    • SUB when the field is from a sub-routine.

  • <Field> is the database field name.

If you want to define how the retrieved field is displayed, then you can also add the format option at the end: (+<Prefix>.<Field>\<Format>+). You can refer to the Field Wizard for format options as well as Format Options.

(+CODE\#IP-FilePart.Ext+)

This code is used to insert client-specific logos or pictures into letters or other word templates.

The program will evaluate the CODE and use the value returned by the code (CODERETURN) to construct the following file name:

FormsFolder\DocumentPictures\CodereturnFilePart.Ext

where 'DocumentPictures' is the designated location of the picture files.

If the file name returned doesn't exist the code will return an error message. See example below.

 

(+CODE\#IP-FilePart.Ext#NoValue+)

As above.

If the file name returned doesn't exist and there is a second format specified (starting with #) it will format the CODERETURN with this format and return this value. If the second format only has the # character, the code will return an empty string.

For example:

A word document generated for the Master File ('RefCode') ABCCOM:

  • The code (+HD.RefCode\#IP-.logo.jpg#AD-5-0+) will look for the file  ...Viewpoint\Forms\DocumentPictures\ABCCOM.logo.jpg

If the file exists, the picture will be inserted on the location of the code. If the file doesn't exist, the program will replace the code with the name and address (format code #AD-5-0).

  • The code (+HD.RefCode\#IP-.logo.jpg+) will, if the picture file doesn't exist, replace the code with the message "Picture file '...\Viewpoint\Forms\DocumentPictures\ABCCOM.logo.jpg' doesn't exist"

  • The code (+HD.RefCode\#IP-.logo.jpg#+) will, if the picture file doesn't exist, remove the code only.

(+DOCAPPEND+)

This code is used to combine multiples Word reports into a single Word report. It will generate a document based on the form code and selection criteria specified in the syntax. The generated document will be appended at the end of the document that has the specified syntax and will produce a single document.

Where:

 

FORMCODE: The Template Code to be generated and a ! symbol can be include or exclude at the end of the template code.

  • Without ! symbol - TemplateCode, it is to indicate that is for Draft Invoice.

  • With ! symbol - TemplateCode, it is to indicate that is for Posted Invoice.

HD-SEL: Selection Criteria of the template.

  • without Attach_[Template Type], the generated document will append at the end of the main document

  • Attach_PDF: Attach the generated document as PDF when converting to an e-mail

  • Attach_DOC: Attach the generated document as Word when converting to an e-mail

(+WF.TAB.<KEYTYPE>:<KEY>\FieldFormat+)

This code is used to display in table format and support format option.

(WF.TAB.<KEYTYPE>):<KEY>\FieldFormat

The prefix WF.TAB is used only when the information being retrieved is linked to the Workflow and it is to be shown in the tabular format.

Replace <KEYTYPE> with a valid FLDKEY, GROUPKEY or STEPKEY.

The FieldFormat is to define multiple format strings for fields of tabular display such as:

1:<Format>\2:<Format>\3:<Format>

  • (+WF.TAB.FLDKEY:ABCD\...+) – This tag is for one row to display field for that Key.

  • (+WF.TAB.GROUPKEY:G1\...+) – This tag is to list all the rows in the Duplicate Group of the Workflow Step.

  • (+WF.TAB.STEPKEY:S1\...+) – This tag is to list all the rows in the Workflow Step.

  • (+WF.TAB\...+) – This tag is to list all rows in the Workflow. It will be split by Workflow Step Description as Heading.

  • (+WF.TAB.STEPKEY:S1\CAP*IS+) – This tag is to show the caption of the Workflow Key in Indent level, add one space for additional one level.

  • (+WF.TAB.STEPKEY:S1\CAP*IT+) – This tag is to show the caption of the Workflow Key in Indent level, add one tab for additional one level.

Note: \... refers to the displayed columns - Caption, Field Value, Field Description, Score and Help.

(+BARCODE.<Name>&BarcodeType=<BarcodeTypeCode>&<Property>+)
 
Ending code with (+BARCODE.<Name>&END+)

This code is used to generate a QR Code for Word Reports generation. For Swiss QR Code, the code will be integrated into Invoice Template and will have all the invoice details.

Name - This is a user-assigned name for the barcode block. This needs to be unique.

BarcodeTypecode - Specify the barcode symbology to be inserted for this Barcode Block. For example:

  • SWISSQR - for Swiss QR Code

  • QR - for normal QR Code

Property - Specify the properties of barcode in accordance to the specified barcode symbology such as language and the output size.

 

Swiss QR Code would require Language and OutputSize:

  • Language:

    • EN - English (Default)

    • DE - German

    • FR - French

    • IT - Italian

    • RM - Romansh

  • OutputSize:

    • QrBillOnly - This is the default size of the QR code. The size is 105mm x 210mm. This size is suitable if the QR code has no horizontal line. If the horizontal line is needed and the A4 sheet size is not suitable, use QrBillExtraSpace instead.

    • A4PortraitSheet - A4 sheet in portrait orientation. The QR code is at the bottom.

    • QrCodeOnly - QR code only. The size is 46mm x 46mm.

    • QrBillExtraSpace - QR code only with additional space at the top for the horizontal line. The size is about 110mm x 210mm. The extra 5mm at the top create space for the horizontal line and optionally for the scissors.

Normal QR Code would requires ModuleSize.

    • ModuleSize -  This is to set the size of QR code. This is not mandatory to be set. When it is not set, the value will be defaulted to 4. Note: The size of generated QR code will vary depending on the size of the data.

Ending code with (+BARCODE.<Name>&END+)

Swiss QR Code:

(+BARCODE.CHQR&BarcodeType=SWISSQR&Language=EN&OutputSize=QrBillOnly+)

 

Successful QR Code generation will have the following:

  1. A valid address must contains the name, postal code, town and country ISO code.

  2. The creditor's name, address and account number are mandatory.

  3. The debtor's details is optional. If it is not used, all address fields must be empty or null, or they must be completely omitted.

  4. The account number needs to be a valid IBAN from Switzerland or Liechtenstein.

  5. If a reference number is given, it must be a valid QR/ISR reference number or an ISO 11649 creditor reference number (i.e. the applicable check digits must be valid)

  6. The currency must either be CHF or EUR.

Normal QR Code:

(+BARCODE.QR&BarcodeType=QR&ModuleSize=10+)

 

 

 

 

HTML Templates

These templates use the same codes as the ones used for Word Reports with the addition of '(+ftyp=evw|' before the code. For example, where you would use (+HD.EntCode\#AD-0-0+) to retrieve the entity name in a Word report, you would use (+ftype=evw|HD.EntCode\#AD-0-0+) for HTML templates.

Below are additional codes that apply only to HTML templates:

Code

Explanation

Example

(+ftyp=evw|HD.<Field>+)

(+ftyp=evw|HD.<Field>\<Format>+)

 

where:

  • <Field> is the database field name

  • <Format> is the format option code

This is to retrieve a single field from the header statement of the template.

(+ftyp.evw|HD.EntCode+)

The above retrieves the Master File code of the entity.

 

(+ftyp.evw|HD.EntCode\#AD-0-0+)

The above retrieves the Master File code of the and displays its name.

(+ftyp=evw|DT.<Query>+)

 

where <Query> is the name of the SQL query

This is to retrieve multiple fields using an SQL query and display the information in a table format.

It requires the following:

  • The SQL is defined in the Key Information of the template

  • HTML codes to define the table properties

(+ftyp.evw|DT.AddrList+)

The above executes the query ‘AddrList’ specified in the Key Information of the template and displays it in a table.

(+ftyp=evw|<Query>.<Field>+)

(+ftyp=evw|<Query>.<Field>\<Format>+)

 

where:

  • <Query> is the name of the SQL query

  • <Field> is the database field name

  • <Format> is the format option code

This is to retrieve a single field from an SQL query. It requires the SQL to be defined in the Key Information of the template.

(+ftyp=evw|MasterFile.Status+)

The above executes the query ‘MasterFile’ and displays the field ‘Status’.

(+ftyp=evw|CM.<Field>+)

 

where <Field> is the Master File field that will have the context menu

This allows for enabling context menu in the Summary Screen or ESP screen when users right-click on a Master File information. The default options are:

  • View Details - opens the Information window for the Master File.

  • Open File - opens the File Maintenance screen of the Master File. This is not available for ESP screens.

  • Enquiry - opens the File Enquiry screen of the Master File. This is not available for ESP screens.

  • Relations - opens the Relations Enquiry screen of the Master File. This is not available for ESP screens.

The code must be added to the field code that retrieves the Master File information with the ftyp=evw prefix.

When the information is retrieved using the HD prefix, an ending tag is also required.

(+ftyp=evw|CM.EntClient+) (+ftyp=evw|HD.EntClient\#AD-0-0+)(+ftyp=evw|CM.EntClient|END+)

 

Note: If the information is in a table retrieved using DT prefix, then the ‘CM’ code and the Master File field is to be added to the [/DT.<QNAME>-FORMAT=…] in Key Information e.g.

[/DT.EntClients-FORMAT=\1:CM=EntCode\2:#LT-CNTR\3:DMB\5:DMB\6:#LT-COST\7:*NS/]

(+ftyp=evw|CM.<Field>\#<Identifier>+)

 

where <Identifier> refers to the item that contains the screen(s)

Similar to the above but allows for more options to be added to the context menu. The options act as links to other screens.

The screens must be specified in the Key Information as [/CM.<Identifier>=<Screen>/] where <Screen> is the NavigationID of the screen.

This code applies only for Summary Screens.

In Key Information:

[/CM.Info=FSEGENER,FSEMNADT/]

 

In the template:

(+ftyp=evw|CM.EntClient\#INFO+)(+ftyp=evw|HD.EntClient\#AD-1-0+) (+ftyp=evw|CM.EntClient\#INFO|END+)

(+ftyp=evw|auth=<Screen>+)

 

where <Screen> is the NavigationID of the screen and can be obtained from Configuration > Settings > Customise > Windows Menu.

This is the beginning code to apply User Group authority to hide/show information. When this code is added, if the user does not have access to the screen from where the information is obtained, he/she will not be able to see the information.

The ending code is also required.

 

(+ftyp=evw|auth=<Screen>|END+)

This is the ending code to apply User Group authority to hide/show information. It is to be used with the beginning code (refer to above).

 

(+fgrd=<Query>+)

 

where <Query> is the name of the SQL query

This is to retrieve multiple fields using an SQL query and display the information in a grid:

  • with scroll bar and Refresh button

  • with page options like navigation buttons and number of records per page

  • that allows sorting by column

It requires the SQL to be defined in the Key Information of the template along with properties of the grid such as:

  • the caption for each grid column

  • the width of each column

  • the format of each column

(+fgrd=OEREL+)

The above executes the query ‘OEREL’ and displays the information in a grid.

 

Note: When using this code, there is no need to use HTML code to define a table.
