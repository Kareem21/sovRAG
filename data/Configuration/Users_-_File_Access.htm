Users - File Access

The File Access controls two areas:

  • Access to Master Files in an Administrator Group and applies if your security model is Administrator + User Group Model [2], Access Group Model [3], User Relation Links [4], Services Provided Model [5] or Service Office Model [8]. With access, you can also view details of the Master File through Grid Details tab and the View Details screen. Otherwise, you may see only the Master File code and name.

  • Tabs in the Selection Setup screen which is seen when you click Select button.

 

Authority to Master Files

The table below explains the different File Access options and shows the condition for a user to have authority to a Master File, based on the user's and Master File's Administrator Group:

Security Model

File Access Option

No Files [0]

Administrator Groups [2]

Administrator Groups + View-only for Others [4]

User Group authority only [1]

No authority to any Master Files.

All Master Files in the same Administrator Groups are accessible to the user, and those not in the same Administrator Groups can both open and edit them.

Authority within the Master Files is based on the User Group.

Admin + User Group [2]

User can access Master File in their Administrator Group(s).

Authority within the Master File that can be opened is based on User Group.

Master Files belonging to different Administrator Groups are unable to be accessed at all.

User can access to the Master Files that do not belong to their Administrator Group, but the authority is limited to View-only based on the User Groups authority.

Access + User Group [3]

The user has authority to the Master File if there is an Access Group Security that comprises the user's Access Group and the Master File's Administrator Group.

The authority is to be based on the Access Group Security.

User Relation Links [4]

The user has authority to the Master File and can open the Master File if:

  • the user is appointed is an active User Relation of the Master File; or

  • the Master File's Administrator Group is the same as the user's Administrator Group

Services Provided Model [5]

The user has authority and can only access Master Files that are in the same Administrator Group.

The user will NOT be able to see Master Files in another Administrator Group.

The authority to edit is to be based on the Service Provided.

Service Office Model [8]

The user has authority to the Master File if the Administrator Group of the Master File is the same as the user's Administrator Group.

Master File belonging to different Administrator Groups are unable to be accessed at all.

The authority to edit is to be based on the Service Office and Master File owner.

 

For the following security models. being authorised to a Master File does not mean the user will be able to open the Master File. The ability to open a Master File will depend on other configuration of the respective security models:

  • for Access + User Group [3], the ability to open a Master File depends on the Access Group Security.

  • for Service Office Model [8], the ability to open a Master File depends on the Service Office Authorisation method.

 

Screen Filter icon and Selection Setup

The File Access option set for the user will determine if they are able to apply filter to the screens.

File Access Option

User Ability

No Files [0]

User is not able to click on the Filter icon.

Administrator Groups [2]

User is able to click the Filter icon and see the following tabs in Selection Setup screen:

  • Favourites

    Within this tab, are the following options:

    • Me - will filter the screen to show only for the logged-in user

    • I am the Viewpoint Users' Manager - will filter the screen to show for the logged-in user and subordinates. This is available only if the logged-in user is specified as Manager for other users.

    • Me and my Workgroups (Unassigned Work) - will filter the screen to show for the logged-in user and workgroups where the logged-in user is the workgroup manager.

    • My primary Administrator Group - will filter the screen to show for the default Administrator Group of the logged-in user.

    • All my Administrator Groups - will filter the screen to show for all Administrator Groups of the logged in user.

  • Users - the list includes the logged-in user and subordinates.

  • Workgroups - the list contains workgroups where the logged-in user is the workgroup manager.

  • Administrator Groups - the list contains all Administrator Groups of the logged-in user.

  • Organisational Units - the list contains the Organisational Unit of the logged-in user.

  • User Relations - the list includes the logged-in user and subordinates.

Administrator Groups + View-only for Others [4]

User is able to click the Filter icon and see the following tabs in Selection Setup screen:

  • Favourites

    Within this tab, are the following options:

    • Me - will filter the screen to show only for the logged-in user

    • I am the Viewpoint Users' Manager - will filter the screen to show for the logged-in user and subordinates. This is available only if the logged-in user is specified as Manager for other users.

    • Me and my Workgroups (Unassigned Work) - will filter the screen to show for the logged-in user and workgroups where the logged-in user is the workgroup manager.

    • My primary Administrator Group - will filter the screen to show for the default Administrator Group of the logged-in user.

    • All my Administrator Groups - will filter the screen to show for all Administrator Groups of the logged-in user.

    • My User Group - will filter the screen to show for all users in the same User Group as the logged-in user.

    • All Users - will filter the screen to show for all active users.

  • Users - the list includes all active users.

  • Workgroups - the list includes all active workgroups.

  • Administrator Groups - the list includes all active Administrator Groups.

  • User Groups - the list includes all active User Groups.

  • Client/Group - the list includes Master Files that are specified as Client/Group (in To Do Summary and To Do Dashboard screens).

  • Organisational Units - the list includes all active Organisational Units.

  • User Relations - the list includes all active users.

 

Document Holder Security

File Access is used in the setup for document holder security. The selection of File Access and Holder Security option in System Defaults - Document Management determines the security level for a user's access to document holders. For more information, go to the Holder Security section in the topic.
