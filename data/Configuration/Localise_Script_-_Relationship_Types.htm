Localise Script Tags - Relationship Types

For Relationship Types, tags are available for Localise Scripts to be used to deactivate existing relationship types and to add new relationship types. For new relationship types, the relationship types will have the Customised and Active checkboxes ticked by default.

 

Deactivating Relationship Types

The tag below will deactivate all relationship types in the specified class.

 

Syntax

RELTYPEINAC:TYPE+'<ClassCode>'

 

where:

<ClassCode> is the relationship class code:

  • E – Entity Relations

  • I – Internal Relations

  • M – Master File Relations

  • O – Statutory Officers

  • T – Trust Officers

To deactivate relationship types from multiple relation classes, separate the <ClassCode> with a comma.

 

Example

To deactivate all Master File Relations and Entity Relations:RELTYPEINAC:TYPE+‘E’,‘M’

 

Note: Relationship Types are generally not deleted as Viewpoint standard types will be reinserted on the next upgrade if they have been deleted.

 

Editing and Adding New Relationship Types

When adding a new type, you need to specify the required fields and values for the type. Each field will be separated with the pipe '|'. You can use the same tag to edit and make changes to existing relationship types.

 

Syntax

RELTYPEADD:<RelCode>+<ClassCode>=<DBField>=<Value>|...

 

where:

  • <RelCode> is the relationship type code.

  • <ClassCode> is the relationship class for the relationship type.

  • <DBField> is the database field referring to the field for the relationship type

    • RDescr - Description field

    • DisplayText - Display Caption field

    • Reciprocal - Reciprocal Relation field

    • AltDescr - Alternative Description field

    • PrincipleRel - Principal Relation field

    • CaptionPrincipleRel - Caption for Principal Relation field

    • RelType1 - Additional Relation 1 field

    • RelType2 - Additional Relation 2 field

    • RelType3 - Additional Relation 3 field

    • RelCaption1 - Caption for Additional Relation 1 field

    • RelCaption2 - Caption for Additional Relation 2 field

    • RelCaption3 - Caption for Additional Relation 3 field

    • CaptionUserRel - Caption for User Relation field, when this is specified, User Relation checkbox will be ticked

    • CaptionFrom1 - Caption for 'From' Date 1 field; when this is specified, the date checkbox will be ticked

    • CaptionFrom2 - Caption for 'From' Date 2 field; when this is specified, the date checkbox will be ticked

    • CaptionFrom3 - Caption for 'From' Date 3 field; when this is specified, the date checkbox will be ticked

    • CaptionTo1 - Caption for 'To' Date 1 field; when this is specified, the date checkbox will be ticked

    • CaptionTo2 - Caption for 'To' Date 2 field; when this is specified, the date checkbox will be ticked

    • CaptionTo3 - Caption for 'To' Date 3 field; when this is specified, the date checkbox will be ticked

    • UdCode1Table - Code for User Table 1 field; when this is specified, User Table 1 checkbox will be ticked

    • UdCode2Table - Code for User Table 2 field; when this is specified, User Table 2 checkbox will be ticked

    • With_Reelect - Re-election field

    • With_Position - Position Text field

    • With_Powers - Describe Powers field

    • With_Reason - Reason Resignation field

    • TagValues - Tag Values field

    • RelNote - Note field

    • WithRating - Risk Rating field

  • <Value> is the value for <DBField>. Use 1 to tick a checkbox field and 0 to untick the checkbox. For Note and Tag Values fields, the <Value> can include

    • <br> for a line break

    • <pipe> for a | symbol

 

Example

To edit an existing relationship type: RELTYPEADD:AGT+E=RDescr=Agent|DisplayText=Agent|PrincipleRel=1|CaptionPrincipleRel=Relation|With_Reelect=0|With_Powers=0|With_Position=1|With_Reason=1|

To add a new relationship type: RELTYPEADD:AGT+E=RDescr=Agent|DisplayText=Agent|PrincipleRel=1|CaptionPrincipleRel=Relation|CaptionFrom1=Date Appointed|CaptionTo1=Date Resigned|

 

Note: Active and Customised options will be ticked by default so there is no need to specify these in the script.
