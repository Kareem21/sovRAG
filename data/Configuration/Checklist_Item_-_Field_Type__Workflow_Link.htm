Checklist Item - Field Type: Workflow Link

This field type is used to start another workflow from within a workflow step. When there is a workflow link, clicking on the item will show the following options:

  • Initiate new workflow - This is to start a new workflow from the current workflow.

  • Link to existing - This is to link the current workflow to another workflow that has been started. If the selected workflow is incorrect, click to select it and click Remove Link.

 

The options appear in the Complete Field column but you can click and drag around the options to place them in another location. However, the options will return to its original location after you close the workflow.

When configuring a Workflow Link, ensure that you do the following:

  • Item field type is 'Workflow Link'

  • Tags are added to specify additional information such as the linked workflow, information to pass to the linked workflow, etc.

 

Workflow Link - Tags

Tag Explanation Example

[WF.Code=<WorkflowKey>]

where <WorkflowKey> is the Key of the new workflow.

This sets the workflow that will be initiated when user clicks on the Workflow Link icon.

Ensure that the <WorkflowKey> entered is valid. When configuring the Workflow Link item, if the key is not specified or invalid, a message prompt will appear so that this can be corrected. Otherwise, the user will also be alerted when running the Verify function.

[WF.Code=VPWF1]

will initiate the workflow with the Key ‘VPWF1’.

[WF.FileCode=<MasterFile>]

where <MasterFile> can be:

  • A specific Master File code.

  • A checklist item key with a Master File value.

This is to set the Master File the new workflow is for.

 

Note: This tag is supported on workflows published to Viewpoint Engage.

[WF.FileCode=BRIGHT]

will initiate the workflow for the Master File ‘BRIGHT’.

 

[WF.FileCode=(+CurAppoint+)]

will initiate the workflow for the Master File that is entered in the checklist item with the key ‘CurAppoint’.

[WF.StopIfNotComplete=<Value>]

where <Value> can be:

  • Y for Yes

  • N for No (default)

This is to specify whether the current step of the main workflow can be completed if the linked workflow is not yet completed.

[WF.StopIfNotComplete=Y]

will not allow main workflow to be completed if the linked workflow is not yet completed.

 

[WF.StopIfNotComplete=N]

allows the main workflow to be completed even if the linked workflow is not yet completed.

[WF.StopIfNotSubmit=<Value>]

where <Value> can be:

  • Y for Yes

  • N for No (default)

Note: This is applicable only for Engage Servicing Portal workflow.

This is to specify whether the current step of the main workflow can be completed if the linked workflow is not yet submitted.

[WF.StopIfNotSubmit=Y]

will not allow main workflow to be completed if the linked workflow is not yet submitted.

 

[WF.StopIfNotSubmit=N]

allows the main workflow to be completed even if the linked workflow is not yet submitted.

[WF.AutoComplete.Keys=<Key>]

[WF.AutoComplete.<Key>=(+WF.Key+)]

where:

  • <Key> refers to the checklist item of the main workflow where the value will be pasted.

  • <WF.Key> refers to the checklist item of the linked workflow where its value will be retrieved.

This is to retrieve the value of the linked workflow's checklist item and paste it to the main workflow's checklist item. Multiple keys can be specified, separated by commas.

The supported field formats are:

  • VALUE

  • VALSHOW

  • SCORE

Note: This tag is supported on workflows published to Viewpoint Engage. In Viewpoint Engage, the supported format is only VALUE or VALSHOW.

 

[WF.AutoComplete.Keys=A,B]

will retrieve value for items with the key A and B in the main workflow.

 

[WF.AutoComplete.A=(+WF.MF\VALUE+)]

will retrieve the value in item with the key MF in linked workflow to item A in main workflow.

 

[WF.AutoComplete.B=(+WF.MF\SCORE+)]

will retrieve the score in item with the key MF in linked workflow to item B in main workflow.

 

[WF.LinkStatus=<Value>]

where <Value> can be:

  • I for Pending

  • C for Completed

This is to filter the list of available workflow by status when the 'Link to existing' option is selected.

 

Note: This tag is supported on workflows published to Viewpoint Engage.

[WF.LinkStatus=I]

will limit the list to show only pending workflow for selection to link to the workflow.

[WH.<Field>=<Value>]

where:

  • WH.<Field> refers to workflow header field such as:

  • WH.WflowDescr

  • WH.AdminCode

  • WH.DateDeadLine

  • WH.WflowNote

  • <Value> is the default value to assign to the field and this can be text or retrieved from the main workflow.

This is to specify the value for a workflow header field for the linked workflow.

The available fields are:

  • Description

  • Administrator

  • Date Due

  • Note

The above fields can be configured for the child workflow to have the same workflow header details by adding (+WH.<Field>+) to the tag. This is applicable when initiating a new child workflow which is not linked to any existing workflow.

 

Note: This tag is supported on workflows published to Viewpoint Engage.

 

[WH.WflowDescr=Resignation of (+OFR\#AD-1-0+)]

will set the description for the linked workflow’s header with the text ‘Resignation of’ followed by the name of the Master File specified in the checklist item with the key ‘OFR’  from the main workflow.

 

[WH.AdminCode=(+WH.AdminCode+)]

will copy the Administrator from the main workflow to the linked workflow.

[WS.<Field>=<Value>]

where:

  • WS.<Field> refers to the workflow step field such as:

  • WS.WfPageDescr

  • WS.AdminCode

  • WS.DateStart

  • WS.DateDeadLine

  • <Value> is the default value to assign to the field and this can be text or retrieved from the main workflow.

This is to specify the value for a workflow step field for the linked workflow.

The available fields are:

  • Step Description

  • Administrator

  • Due

  • Completed

The above fields can be configured for the child workflow to have the same workflow header details by adding (+WS.<Field>+) to the tag.

 

Note: This tag is supported on workflows published to Viewpoint Engage.

 

[WS.DateStart=(+WS.DateStart+)]

will copy the Date Start of the step from the main workflow to the linked workflow.

[WK.<Key>=<Value>]

where:

  • <Key> refers to the key of the checklist item for the workflow.

  • <Value> is the default value to be assigned and this can be a fixed value or retrieved from the main workflow.

This is to specify the value for the linked workflow's checklist item.

Note: This tag is supported on workflows published to Viewpoint Engage.

[WK.RMF2=(+OFR+)]

will set the value specified for the checklist item with the key ‘OFR’ (in the main workflow) to be the default value for the checklist item ‘RMF2’ in the linked workflow.

[DisplayName=<Value>]

where <Value> can be any custom text to recaption the Workflow Link.

This is to customise the displayed caption for the Workflow Link in the Complete Field column.

[DisplayName=Click to Initiate New Master File Creation]

will set the value to this new caption.
