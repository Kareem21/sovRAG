Filtering List Values

Fields that require users to choose from a list of options can be retrieved from lookup tables, user tables and other types such as Entity Types, Relationship Types, etc where the values seen are those that are active. However, if the list is long, then it is possible to filter the values by applying one of the following tags:

  • [FILTER:] - to specify values to include for the field

  • [EXCLFILTER:] - to specify values to exclude for the field

Syntax  for both tags are as follows:

[FILTER:<FIeldDef>=<ValudeCode1>,<ValueCode2>,...]

[EXCLFILTER:<FIeldDef>=<ValudeCode1>,<ValueCode2>,...]

 

where

  • <FieldDef> refers to the Field Definition code of the field that uses the lookup table. You can obtain this from Configuration > Settings > Field Settings - Field Definition column.

  • <ValueCode> refers to the code of the value that you want to include.

The filtering can be applied:

  • on a user basis - the tag is to be added to the Tag Values field for the User account

  • by entity type - the tag is to be added to the Tag Values field for the Entity Type

  • by Master File type - the tag is to be added to the Tag Values field on the type in the Master File Type [ADTP] lookup table

If the tag for the same field is applied in a user account, Entity Type and Master File Types [ADTP] lookup table with different values, the one in a user account will take precedence, followed by the entity type and then the Master File type. If the same field contains both [FILTER:] tag and [EXCLFILTER:] tag, then the [FILTER:] tag will take precedence.

In addition, it is also possible to filter the value for a lookup table (secondary) based on value of another lookup table (primary). This requires the tag to be added to the primary lookup table. For example, to filter the list of Compliance Register Action Types based on the selected Compliance Register Type, add the tag to the Compliance Register Types [CRTP] lookup table.

Note: If the list is already filtered due to the field being a child of another field, then it is not possible to filter this further. For example, values for Compliance Register Sub-Type field depends on the selection for the Compliance Register Type field. As such, it is not possible to apply the [FILTER:] tag for the Compliance Register Sub-Type field.
