System Defaults - Accounting

The defaults for Accounting are set up here.

 

PM Accounting Journals

Select the format for the accounting journals in Practice Management from system standards.  

  • None [0] - No postings are made, and no journals are kept.

  • PM Journals [1] - Journals are generated in Practice Manager and you can create an Excel report on these journals for manual import or entry into a third party accounting software.

  • CA Journals RC [2] - Journals are posted to the Revenue Entity's account(s) only. This option is used when you use Accounting for internal (in-house) accounts, but not for your Client's accounts.

  • CA Journals CE [4] - Journal are posted to the Client's Entity account(s) only. This option is used when you keep accounts for the Client's entities, and do not use Accounting for your internal (in-house) accounts.

  • CA Journals RC + CE [6] - Journals are posted to the Revenue Entity's account(s) as well as the Client's accounts.

 

Generate Journals Automatically

Tick this option if you want the journals created in Practice Management to be generated automatically in Accounting. If this option is not ticked, you can manually post journals to Accounting. You can find out more in the Practice Management and Accounting help files.

 

Post Client File Journal as Draft

Tick this option to post invoices and receipts to Accounting as draft journal for client file.

This requires the Draft Journals option to be ticked in File Maintenance > Accounting > Account File > General > Settings. Otherwise, it will still post as Final.

 

Global Reporting Currency

Select the reporting currency for global reports. Global reports can be accessed from Viewpoint - Home screen through the Reporting section.

 

Reporting Location

Enter the default location for global reports.

 

Disable use of Line Reference Date

Tick this option if you want to disable the use of the Line Reference Date. Once disabled, you will not be able to enter a Line Date during journal entry. In Journal Quick Entry – Entries Tab, the ribbon button ‘Line Date’ will not be shown and the grid column ‘Line Date’ will be hidden.

 

Auto Calculate Reverse Exchange Rate

This option allows Accounting to automatically calculate the reverse exchange rate to the one entered into the Exchange Rates Master. For example, if the exchange rate for USD to Pound Sterling is entered into the system, it will automatically calculate the rate for Pound Sterling to USD. Select either this option, or the Use 'Triangular/Cross' Exchange Rates option (see below) to calculate the exchange rate. You should not use both at the same time.

 

Use 'Triangular/Cross' Exchange Rates

When ticked, the system will automatically calculate the exchange rate between two currencies if there are already rates available for these two currencies against a third currency. For example, if the rate between Euro and USD, and Pound Sterling and Euro are available, Viewpoint will be able to calculate the rate between USD and Pound Sterling. Select either this option, or the Auto Calculate Reverse Exchange Rate option (see above) to calculate the exchange rate. You should not use both at the same time.

  1. Direct rate (A → B); or

  2. Inverse of the direct rate from B to A (1/B → A); or

  3. Indirect rate through another currency (C), where the priority is the most recent rate (A → C x C → B); or

  4. As per 3, but using the inverse rate (A → C x 1/(B →C))

Note:

  • The program will search for the most recent rate ON/BEFORE the journal date.

  • For 3 and 4, the rates (of A → C and C → B) must be on the same date.

If you opt for this method after having used the Auto Calculate Reverse Exchange Rate option (see above) prior to this, then it is advisable to clean up the Exchange Rates Master and delete unused exchange rate currency combinations, i.e. those which you no longer update, to avoid referring to them and returning old or outdated rates. If the old rates are left as is, Viewpoint could pick them up as it will first look for the direct rate - even though it might be years old. We suggest that you have only the exchange rates for your currency of choice (e.g. GBP -><Currencies>) to avoid confusion. Note that if you do a backdated valuation, Viewpoint will use the cross rates.

 

Disable Automatic Tax Entries

Tick this if you DO NOT want a tax journal line to be automatically created in the general journal and journal script screens when a tax account is selected or specified in a journal line.

 

Allow Editing of Automatic Tax Entries

Tick this option to allow editing of tax amount in Journal Quick Entries and Journal Script.

 

Split Authority for Draft & Posting

If the Draft Journals function is used in the Account File, when this option is ticked, the program will not allow the user who last edited a draft journal to post it. This option is only applicable on ‘Post as Final’.

 

Enforce Credit Limit for Sub-ledgers

Tick the check box if you want to enforce the credit limit for the Creditor and Bank Account sub-ledgers. You will not be able to post any more journals to the sub-ledger when you reach the credit limit.

 

Line Reference Date Must be in Posting Window

This option is ticked if you want to validate that the Line Reference date is within the posting period.

 

Show Assigned Journal No. on Posting

Tick this to show the assigned Journal Number on posting. This does not apply to Journal Quick Entry.

 

Roll Year-End: Warn for Un-posted Transactions

Tick this if you want to be informed of un-posted transactions such as invoices, receipts, payments or draft journals when you roll the year-end.

 

Use Unit Codes Default Values in Journal Entry

Tick this to apply default values for Unit Codes to Journal entry screen.

 

Include Unrealised Revaluation in Cost Calculations

When this option is ticked, transactions assigned to the Unrealised Revaluation movement code ending with 'U'(*U) will be included in the disposal cost calculation when using journal scripts, depending on the Security Valuation method used.

 

Command buttons:

Click on the relevant command button to perform the necessary.

  • Update - Click this to save changes.

  • Cancel - Click this to cancel the changes.
