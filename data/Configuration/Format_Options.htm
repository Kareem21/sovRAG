Format Options

Format codes are used to convert the information retrieved by a field code to useful information. For example, (+EVT.EntCode+) will only display the Master File code of the Entity File, such as ASIA-P. If you add the format option to the field code (+EVT.EntCode\#AD-0-0+) you will get Asia-Pac (HK) Ltd.

 

General Codes

Format Option

Explanation

#AD-X-Y

Formats a Master File code to produce a specific result.

In the format option #AD-X-Y:

  • X = the main format option, used to retrieve the name or address of the field code. See the Field Wizard for a listing of all options.

  • Y = indicates additional formatting and may be:

  • 0 = No additional formatting.

  • 1 = Format multiple (name and address) lines as one line, with each line separated by a comma.

  • 2 = Format multiple (name and address) lines, add a comma at the end of each line except the last, and a period at the end.

  • 3 = Format multiple (name and address) lines as one line without commas or period.

  • 8 = Add a line before the retrieved information. This is useful for signature lines.

For example: (+EVT.EntCode\#AD-0-0+) or (+HD.AddrCode\#AD-40-8+)

#MF&TP-X-Y

Formats a Master File code with the address card information specified by the address type code TP, format option X and additional formatting Y.

In the format option #MF&TP-X-Y:

  • TP = the address type required. Address types are defined in the Address Types [ADRT] lookup value table. Standard address types are:

  • BA = Business Address

  • MA = Mailing Address

  • RO = Registered Office Master File

  • RA = Residential Address

  • SR = Statutory Registered Office

  • SM = Register of Members

  • SO = Register of Dir/Sec

  • SD = Register of Debentures

  • SC = Register of Charges

  • SI = Location of Minute Book

  • SA = Instrument Creating Charges

  • SS = Location of Common Seal

  • ST = Location of Duplicate Seal

    Note: You can have multiple values for TP to have the system look for and retrieve a single address based on the Address Types specified. If unable to find the first Address Type, it will look for the next one. The different Address Types are to be separated with a plus '+' sign and the order of retrieval depends on the order specified. For example, MA+RA+BA will first look for Mailing Address.  If there is a Mailing Address, it will return the Mailing Address, else it will look for Residential Address followed by Business Address.

  • X = the main format option, used to retrieve a specific portion of the address specified by the address code:

  • 1 = Full address as displayed by the system with the address card

  • 2 = Address card number

  • 12 = Address without country

  • 20 = Address lines

  • 21 = City

  • 22 = State

  • 23 = Country code

  • 24 = Country description

  • 25 = Postal code

  • 26 = Postal code position (number)

  • 27 = Confidential (Yes/No)

  • 28 = Statutory (Yes/No)

  • 29 = Active (Yes/No)

  • 31 = Address line 1

  • 32 = Address line 2

  • 33 = Address line 3

  • 34 = Address line 4

  • 35 = Address line 5

  • Note: If you utilise the Local Address Card feature, add 100 to the code to retrieve the local address. For example, 21 retrieves City, so 121 will retrieve the local City. This applies to values 1 to 35. The exception is 102 which will retrieve both the main and local addresses.

  • 36 = Address lines 3 - 5 followed by City, State and Postal Code in one line with commas. Note that the position of the Postal Code depends on the Postal Code position. This format option is for Hong Kong forms. When using this format option, Y value of 0, 1 and 2 will have the same result.

  • 42 = Address lines with city.

  • 50 = Address card format code.

  • Y = indicates additional formatting. Refer to #AD-X-Y (above) for more information.

For example: (+EVT.EntCode\#MF&SR-20-1+) or (+HD.EntCode\#MF&BA-42-1+) or (+HD.RefCode\#MF&MA+RA+BA-42-1+)

#MF&TP~CC-X-Y

Formats a Master File code with the address card information specified by the address type code TP, with formatting specified by country code CC, format option X and additional formatting Y.

In the format option #MF&TP~CC-X-Y:

  • TP = the address type required. Address types are defined in the Address Types [ADRT] lookup table. Refer to #MF&TP-X-Y for address type codes.

  • CC = country code of the address format. Acceptable values are obtained from the Country Codes [CNTR] lookup value table.

  • X = indicates the format option. See #MF&TP-X-Y for valid codes.

  • Y = indicates additional formatting and may be:

  • 0 = No additional format. If the country code has been formatted, this is the default format.

  • 1 = In one line, separated by commas. If there are commas in the country code format, these are retained.

  • 2 = Line feed with commas.

  • 3 = In one line without commas.

  • 11 = Returns empty if the country code does not match the address format. This code is only used with ~CC, and X=31-35, 131-135 (see above).

  • 12 = Returns address lines 1 - 5 if the country code does not match the address format. This code is only used with ~CC, and X=31-35, 131-135 (see above). The lines will be separated with line feeds, similar to format code 0.

  • 13 = Returns address lines 1 - 5, if the country code does not match the address format. This code is only used with ~CC, and X=31-35, 131-135 (see above). The lines are separated by commas, similar to format code 1.

  • 30 = Returns two dashes '--' if the name, address or other requested value is empty. This code is for Hong Kong electronic forms.

For example: (+EVT.EntCode\#MF&SR~CY-31-12+)

#LD-X

Formats the value for Nationality Individual Codes [NATI] drop-down table to value from corresponding Country Codes [CNTR] lookup table where X can be:

W

Formats a number to full wording, with decimal places specified as xx/100.

For example: 6,551.34 is Six Thousand Five Hundred Fifty One 34/100.

W:XX

Formats a number to word with additional formatting for decimal places as indicated by XX:

  • W:DP - This will format decimal places as words. For example: 6,551.34 is Six Thousand Five Hundred Fifty One point Three Four.

  • W:DW - This will format the decimal places as words and specify the decimal as cent. For example: 6,551.34 is Six Thousand Five Hundred Fifty One and Thirty Four cent.

DXY

Formats a date value, whereby XY indicate the format type. See the Field Wizard for the various date formats.

C2/C4

Formats a number with the field format defined in Configuration > Settings > System Defaults > Field Formats where:

  • C2 - Will take on the format set in Amounts (2 d.p.).

  • C4 - Will take on the format set in Amounts (more than 2 d.p.).

N0/N4/N6

Formats a number with the field format defined in Configuration > Settings > System Defaults > Field Formats where:

  • N0 - Will take on the format set in Numbers without d.p.

  • N4 - Will take on the format set in Numbers - shares with d.p.

  • N6 - Will take on the format set in Numbers - others with d.p.

N

Formats the number as per the 'General Number' format set at the control panel.

C

Formats the number as a currency. You can also use a variation of #,##0.00### to format amounts with more than two decimal places.

B99

Inserts 99 spaces if the value of the field is empty, so that an empty space if created in the form for manual completion.

BHYP

Inserts a hyphen (-) if there is no result returned by the field code.

For example:  (+EVT.AddrNumber\#AD-15-0\BHYP+)

BHY2

Inserts a hyphen (-) if the field code returns no result. It is only used on the 2014 Hong Kong forms.

For example: (+EVT.AddrNumber\BHY2+) or (+EVT.AddrNumber\#AC-31-1\BHY2+)

*NS

This will hide the field from the report.

*H

If [/HYPHENFIELDS/] is placed in the Key Information, i.e to place hyphens in all empty fields, this option will suppress hyphenation for the formatted field, i.e. will NOT place hyphens in the fields if the field is empty.

*N

Updates value for a change field that was changed in the transaction that generated the change. This format code can be immediately followed with a second format code. See also *O below.

For example: (+EVT.Field\*N#AD-1-0+)

*O

Old value for a change field that was changed in the transaction that generated the change. See also *N above.

*L

Used in tables to place the next field in the same cell as this field, but in a new line. This option is placed BEHIND any other format option.

For example: (+DIRS.ALL-TAB\1:#AD-40-0*L\2:#AD-42-1+)

*A

Used in tables to place the next field in the same cell as this field. This option is placed BEHIND any other format option.

For example: (+DIRS.ALL-TAB\1:#AD-40-8*A+)

#LP

List in paragraph format. This format code can be used for forms with multiple changes linked, or SQL statements that (might) retrieve multiple records. The multiple values will be listed as a paragraph, example: Value A, Value B and Value C.

For multiple linked changes, this code will find the selected field in the related changes, and format accordingly. For SQL statements, this code will pick the selected field (that must be a return field of the statement) for all rows returned by the statement, and format accordingly.

The #LP code can be followed by a second format code separated by a dash. The system will first apply the second format code before the #LP List paragraph.

For example: (+EVT.EntCode\#LP-#AD-40-0+)

#LS

This is similar to #LP except that the multiple values are separated by a line feed so the values are listed on separate lines. In addition, the #LS code can be followed by a second format code separated by a dash only for fields linked to a change code. For example: (+EVT.AddrCode\#LP-#AD-40-0+)

For SQL statements, a second format code is not supported.

#TC

Same as #LP except that the multiple values are placed in separate table cells: i.e. the first value is placed in the table cell where the code is placed, and each subsequent value in the next cell (note that new cells are automatically added to the table 'as if' you press 'Tab' in a word document).

This format option is valid only if placed in a table cell.

#CV-11-0

Formats the Master File code to show the following:

  • For individual Master Files: Title, Initials/First Name and Surname.

  • For non-individual Master Files: Current and Former Name. The former name is displayed as (formerly known as <Former Name>), and centred on the page for one year from the effective date of the name change.

#CV-11-8

Similar to #CV-11-0, however there will be a signature line before the retrieved information.

BTXT:<Text>

This will display the specified text if the information is not available. It can be used on its own or with other format options.

For example:

(+DC.IncorpNr\BTXT:Information is missing+)

  • If there is no incorporation number entered for the Master File, the generated document will show 'Information is missing'.

  • If the incorporation number has been entered as WH1234, the generated document will show 'WH-1234'.

 

(+HD.AuditReq\#LT-AUDR\BTXT:Information is missing+)

  • If there is no option selected for Audit Requirement, the generated document will show 'Information is missing'.

  • If the value is Arrange Audit [F], the generated document will show 'Arrange Audit'.

 

Note: This format option does not apply to Change fields i.e. those with EVT prefix such as (+EVT.DateResolution+). For Excel templates, if the query returns an empty record set, this will remain empty instead of returning the text.

#FV-X-Y-Z

This will retrieve a specific record from the database table name and column.

  • X refers to the Database Table Name.

  • Y refers to the Column Name of the Database Table.

  • Z indicates the filtering.

For example: (+BF.RefCode\#FV-RefMaster-Name-RefCode+)

where in SQL Query, this is equivalent to:

SELECT Name FROM RefMaster WHERE RefCode = ‘ABC’

*SUBSTR=N:N

Defines a character limit for a retrieved value. 'N' will be a number that refers to the position of a character (including space) in a sequence of text with

  • the first N - limit begins at this character

  • the second N - limit ends at this character

For example: (+HD.AdminCode\#US+) retrieves the user name of the administrator - 'International Corporate Services (Hong Kong) Ltd'.

Applying the format option (+HD.AdminCode\#US*SUBSTR=1:25+) will retrieve the value starting from the first character 'I' and end at the twenty-fifth character 'S'. Hence, the value will be 'International Corporate S'.

Any other format string

Any valid Microsoft for Application format string can be used.

*MASK

This code is used to hide a specific amount of retrieved data's alphanumeric characters. It can be hidden from either the first or the last alphanumeric character.

*MASK=W-X-Y-Z

W = 1 or 0, to mask alphanumeric only or to mask entire characters including special characters.

  • 0 - Mask alphanumeric only. All characters that are not alphanumeric will be filtered out.

  • 1 - Mask entire characters including special characters.

X = 1 or 0, to define the masking position, starting of the character and ending from the character.

  • 0 - Mask from the last character of the string.

  • 1 - Mask from the first character of the string.

Y = 0...n,  to specify how many characters will be masked.

  • If even number, it will be divided by 2.

  • If odd number:

    •  If 00, it will be length +1 divided by 2. Total length = 7 + 1 = 8. 8/2 = 4 is the number of masked characters. For example, 1234567 will be written as 123****.

    •  If 0, it will be length -1 and then divided by 2. Total length = 7 - 1 = 6. 6/2 = 3. 3 is the number of masked characters. For example, 1234567 will be written as 1234***.

  • Any number other than 0, for example, 4, will mask four characters.

Z = If nothing is defined, it will be replaced with a string of blank characters. Otherwise will be replaced with the defined symbol.

 

Only W, X, and Y are required to be defined. If Z is not specified, the string is cut without masking.

 

In the case of invalid input, W, X, and Y will remain as original value. For Z, any character can be used.

BIDN-X

 This code is used to hide or show certain length of alphanumeric characters.
 
X is an integer.
 
Example: BIDN-3, it will only show the first 3 of the characters. e.g. ABCDEFGH will be shown as ABC.
Note: This only use on Hong Kong forms.

BIDC

 This code is used to get the check digit inside the bracket. Format is like A123456(9) where 9 is the check digit inside the brackets. Output = 9.
Note: This only use on Hong Kong forms.

BIDF

 This code is used to get the output from BID2 and BIDN except the check digit inside the bracket. Format is like A123456(9). Output = A123456
Note: This only use on Hong Kong forms.

BID2

 This code is used to get the first or two alphabet characters. Format is like A123456(9) where A is the first or two alphabet character. Output = A
Note: This only use on Hong Kong forms.

BIDN

 This code is used to get the middle characters between the first alphabet characters and the check digit. Should consists of numbers only. Format is like A123456(9) where 123456 are the middle characters. Output = 123456
Note: This only use on Hong Kong forms.

Word Reports, Correspondence and Billing Templates Only

Format Option

Explanation

@SubTotal

Calculates the sub total for the column. Add -N or -C to format the results as a number or as currency. If there is no additional format option, the result defaults to C (amount).

For example: \2:@SubTotal-N\3:...

@RUNTOTAL

Calculates the running total in the column. Add -N or -C to format the results as a number or as currency. If there is no additional format option, the result defaults to N (number).

For example: \6:@RUNTOTAL-C

Dx.ABC

Formats the date in the language specified by ABC, where x determines whether the long date or medium date is returned:

  • ABC = supported language:

  • ENG = English

  • FRE = French

  • GER = German

  • DUT = Dutch

  • ITA = Italian

  • x = date format:

  • F = Long format

  • H = Medium format

Note: The dates use Word 2007 defaults for the capitalization of the month and the dots/separators in the medium date format.

 

Long format:

(+DF.DateField\DF.ABC+)

For example:

  • (+DF.Now\DF.ENG+) = 18 August 2011

  • (+DF.Now\DF.FRE+) = 18 aout 2011

  • (+DF.Now\DF.GER+) = 18. August 2011

  • (+DF.Now\DF.DUT+) = 18 augustus 2011

  • (+DF.Now\DF.ITA+) = 18 agosto 2011

 

Medium format:

(+DF.DateField\DH.ABC+)

For example:

  • (+DF.Now\DH.ENG+) = 18-Aug-11

  • (+DF.Now\DH.FRE+) = 18-aout.-11

  • (+DF.Now\DH.GER+) = 18. Aug. 11

  • (+DF.Now\DH.DUT+) = 18 aug. 11

  • (+DF.Now\DH.ITA+) = 18 ago. 11
