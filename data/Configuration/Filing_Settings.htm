Filing Settings

This screen contains Viewpoint's standard settings for the different filing types.

 

Filing Settings list

The list shows settings of filing types such as:

  • Global - these are settings that are required by both FATCA and CRS Reporting Schemas for all Reporting Authorities

  • CRS - these are settings that are specific to CRS schema for all Reporting Authorities C_E*/E_*) such as C_JE, E_GI

  • FATCA - these are settings that are specific to FATCA schema for all Reporting Authorities (F_*) such as F_IM

  • Reporting Authority-specific (e.g. C_BM for Bermuda CRS) - these are settings that are as per guidelines set by the jurisdiction's Tax Authority. Changes to the settings are as per the Tax Authority's update on their portal.

For each filing type, are filing information and values retrieved.

 

Edit

Click this to make changes to the selected filing information such as:

  • Filing Value

  • 0 – Enable all validations

  • 2 – Enable GIIN format validation only

  • 4 – Replace space on XML character restriction

0

Fix Address Format

FADFMT

Specify address mapping for Fixed Address format.

For AddressFree, leave blank.

 

For AddressFix(structured address),

[BuildingIdentifier=]

[Street=]

[District=]

 

Customise how the structured/ fixed address is to be retrieved according to your data entry for Addresses. If not applicable, leave the field blank as default, and the address will be extracted as AddressFree.

For AddressFix, please noted that there is restriction on maximum length allowed for each field in the address e.g. 70 for Building Identifier (name and/or number).

Format:

[<XML field name>=<Viewpoint address field name>]

 

Viewpoint:

Address1- referring to the first line in Address

Address2- referring to the second line in Address

Up to and including Address5

 

Example:

[BuildingID=Address1]

[Street=Address2]

[District=Address3]
