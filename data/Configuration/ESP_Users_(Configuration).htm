Users (Configuration)

This is where you configure users for CP-WP but approvals are to be performed in Configuration > Engage Servicing Portal > Approval > Users.

  • a user can be assigned to an Organisation(s) and/or Group(s).

  • a user will be able to see all Master Files belonging to their Organisation in ESP Web portal under Home > Entities. If the user is assigned to a Group with more than one Organisation, the user will be able to view all Master Files within those Organisations.

 

Filters

The following filters apply to the user list:

  • User status - Select to view only active, inactive or all users.

  • Organisation Type - Select to view users that are assigned to a specific Organisation Type.

  • Organisation - Select to view users that are in a specific Organisation.

 

User list

The list shows all users that have been set up manually, added through the Enable Auto Load Profile option in System Defaults > Engage Servicing Portal, or through the  Excel Import Sheet for ESP.

 

Command buttons

These apply to the User list. Click on the relevant command button to perform the necessary.

  • New - Click this to add a user. The record status will be set to 'New' until the record is approved.

    1. Enter the details.

    2. Click Update.

    3. In the Assign/Reset Password screen, specify the password settings. The password must follow the password requirements as set in Configuration > Settings > System Defaults > Engage Servicing Portal in CP-AD.

    4. Click Update.

    Note: A new User - Organisation record will be automatically created for this user. See User - Organisation for more details.

     

  • Edit - Click this to edit the selected user record. The user record status will be set to 'Edited' until the change is approved.

    Note: If you choose to edit a user record set to be deleted, a message will appear asking for confirmation to proceed. Clicking 'Yes' allows you to edit the record and change the record status from 'Deleted' to 'Edited'. Otherwise, the record status remains as 'Deleted'.

  • Delete - Click this to delete the selected user. The record status will be set to 'Deleted'.

  • Password - Click this to reset the password. This button is disabled for Auto Load Profile users.

    1. In the Assign/Reset Password screen, tick the Change password option.

    2. Enter the new password in New Password and Confirm Password field.

    3. Click Update.

    Note: This will not send a notification e-mail to the user after the password reset/change.

     

    Other Options

    • Auto generate password and e-mail user login link - This will send a password reset e-mail to the ESP user.

    • Require password change on next login - Tick this to prompt the user to change the password on the next login session.

    • User login active - Tick this to set the user account as active. If this is not ticked, the user account becomes inactive.

    • Allow Select - This is not applicable for ESP.

    • Administrator Rate - This is not applicable for ESP.

    • Reset secret key - Tick this to reset the secret key for the ESP user. This option is applicable only when the 'Enable Two-Step Verification' option is enabled in the Configuration > Settings > System Defaults > Engage Servicing Portal. A reset on secret key is required when the user needs a new unique secret key to register him/her into the mobile authenticator app. This could happen when the user switches to another mobile device or has reinstalled the authenticator app.

    • Reset login attempt - Tick this to reset the user's password attempts to zero.

  • Import Users - Click this to import users from VP-BO. This will copy the User Code, User Name, E-mail address and Telephone number. This is not available when the Enable Auto Load Profile option is ticked.

    1. Select the VP-DB database where the user records are to be imported from.

    2. Select the user group. Note that only active user groups will be displayed.

    3. Select the user record(s) to import:

      • Tick the checkbox for the user record(s) to import; or

      • Click Select All to select all displayed user records. Note that if the user already exists in CP-AD, this will not be selected.

    4. In ‘Assign Users to Organisation’ field, select an Organisation to which the imported users would be assigned to. Note: As 'Import Users' function is used to import existing Viewpoint users from the Back-Office (internal users), only 'Internal' type Organisations will be listed in the 'Assign Users to Organisation' drop-down list.

    5. Click Import.

     

Note: The Edit, Delete and Password command buttons are deactivated if the selected user was added through Enable Auto Load Profile.

 

User - Organisation list

This lists the 'User - Organisation' records that define the User Roles on CP-WP. These may have been added manually, through the Enable Auto Load Profile option or the Excel Import Sheet for ESP.

For user that belongs to multiple Organisation (internal/external) and Groups, the User Role that is assigned for 'Group' supersedes all the other User Roles that are assigned to 'Internal' or 'External' Organisations where the user is assigned to. For example, User A is assigned to a 'Group' with User Role 'Normal User'. At the same time, the User is also assigned to an 'Organisation' that is part of the Group with the User Role 'Web Portal Manager'. The User Role for the 'Group' will supersede the 'Organisation'. Hence, User A will have User Role 'Normal User' authority for that Organisation.

 

Command buttons

These apply to the User - Organisation list. Click on the relevant command buttons to perform the necessary.

  • New - Click this to add a User to the Organisation. The 'Record Status' will be set to 'New' until the record is approved. The New button is disabled if the user belongs to an External organisation.
    Note: The approval is only applicable if 'No Approval Required for Configuration Change' option is disabled in the Engage Servicing Portal settings screen.

    1. Enter the required details.

    2. Click Update.

     

  • Edit - Click this to make changes to the User record. The 'Record Status' will also be set to 'Edited' until the change is approved. If you choose to edit a user record set to be deleted, a message will appear asking for confirmation to proceed. Clicking 'Yes' allows you to edit the record and change the 'Record Status' from 'Deleted' to 'Edited'. Otherwise, the 'Record Status' remains as 'Deleted'.
    Note: The approval is only applicable if 'No Approval Required for Configuration Change' option is disabled in the Engage Servicing Portal settings screen.

  • Delete - Click this to delete the selected User from the Organisation. The 'Record Status' will be set to 'Deleted'. This button is disabled if the selected Organisation is the User's default Organisation.

Note: All command buttons are deactivated if the selected user is added through Enable Auto Load Profile option.
