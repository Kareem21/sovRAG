Field Settings

The Field Settings screen lists out the fields that are available for customisation. This is where the usage of, and the access levels required to a particular field in Viewpoint could be specified. You may also re-name the fields caption and specify the default values that you would like to use for the fields.

 

Note: Items with captions containing '&' refers to command buttons in the screen but these can also be customised in terms of the caption and authority required.

 

Menu Group

Select the menu group to display the sections within the menu group.

 

Use In

Select to view menus from Windows Menu (Viewpoint Connect) or Web Menu (Viewpoint.WEB) or both. The default value is Windows Menu.

 

Section

Select the relevant section.

 

Find

Click after you have selected the Menu Group, Use In, and Section. This will list the screens for the selected section. Click on the desired screen and click Select.

 

Screen

This displays the screens available in the section that you have selected. If you want to select a different screen, click Find to access the screen list.

 

Field list

This shows the fields in the selected screen.

 

Command buttons

Click on the relevant command button to perform the necessary.

  • New - Click this to create a Custom Field. This is only available if the selected screen is a General screen. For other screens, this button is not active.

    1. Enter the details for the field.

      • Field Type - Select the field type.

      • Field Key - Define the identifier for the field.

      • Caption - Define the caption for the field.

      • Mandatory - Tick this if the field is to be a mandatory field. The caption for mandatory fields will appear in blue until you have entered a valid value for the field. If the field is left blank or the value is invalid, then you will not be able to save changes made in the screen.

      • Auth. Req. for Update - Specify the minimum authority required for the user to update the field. Users without the minimum authority required will not be able to update the field.

        Note: If the caption shows "Do not set to 'FIELD NOT USED'!", the system will disallow user to update the changes whenever the 'FIELD NOT USED' option is selected.

      • Auth. Req. for New - Specify the minimum authority required for the user to access the field for a new record.

        Note: If the caption shows "Do not set to 'FIELD NOT USED'!", the system will disallow user to update the changes whenever the 'FIELD NOT USED' option is selected.

      • Mask - This controls the format of the field. For example, if you want a date to appear in medium date format, you should type “Medium Date” into this field. The mask must be entered correctly as any spelling mistake will affect the data. All standard Microsoft Office input masks can be used.

      • Default - This allows you to set a default value for the field.

      • Row Order - This refers to the location of the field in the Y position (vertical). Field with a higher value will be placed lower down the screen. It is recommended that the value entered is different from the values for the standard Viewpoint fields in the same screen.

      • Column Order - This refers to the location of the field in the X position (horizontal). Field with a higher value will be placed closer to the right side of the screen.

      • Tab Order - This refers to the order of using the TAB key to reach the field.

      • Tag - This is for additional information for certain field types.

    2. Click Update.

    3. Re-launch Viewpoint to see the Custom Field.

  • Edit - Click this to change settings for the selected field.

    1. Make the necessary changes.

    2. Click Update.

    For Viewpoint Standard fields, you can do the following:

    • Set the field as mandatory field - Tick the Mandatory option.

    • Set the authority for the field - This can be used to restrict who can enter/update the field value as well as whether the field is seen or not. The available options are:

      • ALL ACCESS !! [9] - This will show the field as non-editable to all users. It is normally used for system-controlled fields.

      • FIELD NOT USED [99] - This will turn off the field so it is not seen.

      • No Access [0] - This makes the field accessible to everyone.

      • Power User [8] - This requires the user to have authority level 8 for the screen containing the field in order to edit the field.

      • Regular User [6] - This requires the user to have at least authority level 6 for the screen containing the field in order to edit the field.

      • View only [1] - This requires the user to have at least authority level 1 for the screen containing the field in order to edit the field.

    • Set a mask for the field - This determines:

      • the format of the field value.

      • whether or not a drop-down value can be updated by user input if the field gets its values from a drop-down table. Enter 1 to disable automatic update for the drop-down table.

    • Set default value for the field - This is done by entering the value in Default. You may also use the following defaults:

      • UCODE - to set the logged in user as the default value for fields that require user such as the Administrator or Manager field.

      • NOW - to set today's date for Date fields. Note: If it is blank, the Date field will show blank.

      • <Code> - for fields which accept values comprising of a code and name/description, enter the code to set as the default value. For example, enter 'I' to set the default Master File type to Individual [I].

    • Set up incremental numbering for the field - This is also done using the Default field and applies to certain fields like the Document No. field for Safe Custody Register entries and Number field for File Archives entries. Each of these can have numbers that are generated by the system with a predefined prefix using the code 'REF:<DocumentType>' ensuring that you have set up the document type in Configuration > Business Parameters > Lookup Values > Document Number Codes [DOCT].

      For example:

      If the document type is SC, enter REF:SC in Default.
      If the document type is BOX, enter REF:BOX in Default.

     

    For Custom Fields, clicking Edit allows you to change all settings except the Field Type and Field Key.

  • Edit Caption  - Click this to change the field caption.

    1. In the Caption Cascading screen, enter the new caption.

    2. Tick to specify where the new caption should be applied:

      • screens containing the field

      • grids containing the field

      • custom reports containing the field

      • Relation Enquiry screen containing the field

    3. Click Update.

  • Help Text  - Click this to add/update the Help Text for fields in the selected screen.

    1. Select the field and double-click the Help Text cell for the field.

    2. Enter new help text or amend the existing text.

    3. Click OK.

    4. Click Update.

     

    Can I print out the Help Text for all fields?

    You can copy and paste the help text for all fields in the screen to another program such as Microsoft Word or Excel to print. Perform the copying using the Help tab in the screen containing the fields. For example, to print the help text for Setup screen, do this in the Setup screen.

     

  • Delete - Click this to delete a Custom Field. This is only available if the selected field is a Custom Field.
