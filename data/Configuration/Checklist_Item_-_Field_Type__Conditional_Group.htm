Checklist Item - Field Type: Conditional Group

The condition or conditions must be defined in the Values/Default column and can refer to the value or score of a checklist item in the same workflow step, in a previous workflow step or information for the Master File. For each condition, you need to specify:

  • The Key of the item that will become the condition.

  • If the condition is based on the item's value or score.

  • The operator and value for the condition.

Examples:

  • If the condition is that the value for item 'A1' is 'Yes', then the condition will be defined as A1.Value=Yes or A1=Yes

  • If the condition is that the score for item 'B1' is more than 5, then the condition will be defined as B1.Score>5

 

The below lists the Conditional Group operators and values that are supported in both Viewpoint Connect and Engage.

Operator

Explanation

=

Equal to a single value.

>

Bigger than the specified value.

<

Smaller than the specified value.

>=

Bigger or equal to the specified value.

<=

Smaller or equal to the specified value.

<>

Not equal to the specified value.

IN

Is the same as any of the specified values.

NOTIN

Is not the same as any of the specified values.

 

When the condition is based on the value, the value can also be ‘EMPTY’ if the condition is for the checklist item to be empty. For example:

  • The condition – F1=EMPTY – Means the checklist item ‘F1’ must not contain any value.

  • The condition – FI<>EMPTY – Means as long as the checklist item ‘F1’ is not empty.

Note: The value 'EMPTY' must be in uppercase.

 

If the checklist item is in a previous step, then you need to add the prefix #WF before the Key. For example:

#WF.F1<>EMPTY

means the condition is that checklist item ‘FI’ in a previous workflow step must not contain any value.

 

If referring to the Master File information, these apply only to fields that can be retrieved from the following:

  • RefMaster table – Prefix #MF.

  • Compliance table – Prefix #MC.

  • Entity table – Prefix #EN.

  • EntityAccount table – Prefix #AC.

  • RmClients table – Prefix #BF.

  • RmSupplier table – Prefix #SF.

  • UserFieldSetup table – Prefix #UF.

where it must be defined as <Prefix>.<Field>. For example:

#MF.RefType=I

means the condition is that the Master File type must be I for Individual.

 

#EN.IncorpPlace=HK

means the condition is that the Jurisdiction of the Entity File is Hong Kong.

 

Note: Refer to the Workflow Field Wizard to see the list of available fields for each table.

 

For multiple conditions, it is possible to use:

  • AND - This requires all specified conditions to be met. The conditions are to be defined as follows:

    AND(<Condition1>;<Condition2>;<Condition3>:...)

    For example: AND(F1.Value=Yes,F2.Value=Yes;F3.Value=Yes).

  • OR - This requires just one of the conditions to be met. the conditions are to be defined as follows:

    OR(<Condition1>;<Condition2>;<Condition3>;...)

    For example: OR(F1.Value=Yes;F2.Value=Yes;F3.Value=Yes).

To configure the field type with ‘Conditional Group’ as duplicate, insert a pipe ‘|’ symbol, followed by the [Duplicate=1] tag.

For example:

F1<>EMPTY|[Duplicate=1]

means that the checklist item ‘FI’ in the current workflow step which must not contain any value can be duplicated, if required.
