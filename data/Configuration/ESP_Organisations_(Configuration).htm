Organisations (Configuration)

Organisations can be divided into:

  • Internal - When an Organisation is Internal, the 'Database' and 'Viewpoint User' fields are enabled in Configuration > Engage Servicing Portal > Configuration > Users, which allow the user to be linked to their Viewpoint User ID (in Back-office).

  • External - When an Organisation is External, the 'Database' and 'Viewpoint User' fields are disabled in Configuration > Engage Servicing Portal > Configuration > Users. User which is assigned to an External Organisation will not be able to link to a Viewpoint User ID (in Back-office).

  • Group - Organisations can be grouped. When a user is assigned to a Group, they can have access to all Organisations under that Group. A group can consists of multiple Sub-Groups with each having Organisations below it.

    In the Organisation grouping,

    • a Group can only have multiple Sub-Groups and/or Organisations, but cannot have Master Files with it.

    • only Organisations can have Master Files.

 

Filters

The filters apply to the Organisation list.

  • Organisation status - You can choose to view Active, Inactive or All.

  • Organisation Type  - You can choose to view External, Internal, Group or All.

 

Organisation list

The list shows all Organisations that have been set up manually, added through the Enable Auto-Load Profile option in System Defaults > Engage Servicing Portal, or through the Excel Import Sheet for ESP.

 

Command buttons

These apply to the Organisation list. Click on the relevant command buttons to perform the necessary.

  • New - Click this to add an Organisation. The record status will be set to 'New' until the record is approved.

    1. Enter the required details.

    2. Click Update.

    Note: If you would like the requests to be initiated from the Home tab in ESP for this organisation, make sure to specify the Database and Master File. The Master File refers to the Master File that will be used when initiating the request and must also be added as an attribute for the workflow template of the request in Configuration > Business Parameters > Workflow Library.

    For Auto Load Profile mode, the Database field is automatically filled with value and is not editable.

     

  • Edit - Click this to edit the selected Organisation record. The record status will be set to 'Edited' until the change is approved.

    You will be able to change the following:

    • Name

    • Type

    • Parent Group

    • Database

    • Master File

    • Note

    If the organisation record is from the auto-load profile, then you can only edit the following:

    • Master File

    • Note

    Note: If you would like the requests to be initiated from the Home tab in ESP for this organisation, make sure to specify the Database and Master File. The Master File refers to the Master File for the request and must also be added as an attribute for the workflow template of the request in Configuration > Business Parameters > Workflow Library.

     

  • Delete - Click this to delete the selected Organisation record. The record status will be set to 'Deleted' but you will be able to see the Organisation record as long as the deletion has not been approved. This is deactivated if the selected organisation is added through Enable Auto Load Profile option.

  • Users - Click this to navigate to the Users screen and filter it based on the selected Organisation.

Note: You can only delete an Organisation if there is no user or Master File linked to the Organisation.

 

File Configuration list

It is necessary to define the Master File(s) for each Organisation. If approval is required, it can be done in Configuration > Engage Servicing Portal > Approval > Organisations.

This lists the Master Files accessible for an Organisation which would have been set up manually, added through Enable Auto Load option in System Defaults > Engage Servicing Portal, or through the Excel Import Sheet for ESP.

 

Command buttons

These apply to the File Configuration list. Click on the relevant command buttons to perform the necessary.

  • New - Click this to add a file configuration for the selected Organisation.

    1. Enter the required details.

    2. Click Update.

    3. Click Configure. In the Master File screen, select the Master File and click Include. If you have selected the wrong Master File for inclusion, you may re-select again and click Exclude.

    4. Click Close when done.

  • Edit - Click this to edit the selected file configuration.

  • Delete - Click this to delete the selected file configuration record.

  • Configure - Click this to open the Master File screen to view Master Files retrieved by the selected file configuration. You can customise the grid via right-click.

    In this screen, you can filter the list by the following record status:

    • Approved [1] - Master File is approved to be accessed on CP-WP

    • New [2] - Master File is new to the list

    • Deleted [4] - Master File is to be deleted from the list

    • Approved Excluded [6] - Exclusion of the Master File from CP-WP has been approved

    • Included [7] - Master File is included in the list, pending approval

    • Excluded [8] - Master File is to be excluded, pending approval

    • All [*] - show all Master Files

  • Configure All - Click this to open the Master File screen to view Master Files retrieved by the selected Organisation.

    In this screen, you can filter the list by the following record status:

    • Approved [1] - Master File is approved to be accessed on CP-WP

    • New [2] - Master File is new to the list

    • Deleted [4] - Master File is to be deleted from the list

    • Approved Excluded [6] - Exclusion of the Master File from CP-WP has been approved

    • Included [7] - Master File is included in the list, pending approval

    • Excluded [8] - Master File is to be excluded, pending approval

    • All [*] - show all Master Files

  • Requery - Click this to synchronise CP-DB with VP-DB. This is to check the validity of the Master Files for the selected File Configuration. A Master File is invalid if the status is no longer Active.

  • Requery All - Click this to synchronise CP-DB with VP-DB. This is to check the validity of the Master Files for all the File Configuration.

Note: All command buttons are deactivated if the selected File Configuration is added through Enable Auto Load option.
