Checklist Item - Field Type: Macros

The Macros field type is used to automate certain actions. In workflow steps, a checklist item configured as a macro will have the following:

  • Macro caption - Which the user can click to run the macro(s). This will result in the macro message prompt where user needs to select Yes to continue. Clicking No will not run macro.

  • Icon - Which the user can click to go to the screen of the action, similar to Go to Screen field type.

The actions can be for:

  • Updating of existing records or adding new records.

  • Triggering items linked to User Defined Change Codes.

  • Storing documents that are created within the workflow.

When setting up a macro, the following are required:

  • Item field type is 'Macros'.

  • Macro tag is added in the Values/Default column the Note Editor.

 

Add or Update Records

When using the macros field type, instead of requiring the user to manually perform the actions, user can just click to run the macro and have the actions performed. These actions could be:

  • Updates that should be done when closing a file e.g. resign all statutory officer. If performing the resignation manually, users would select a record in the Statutory Officers screen and click Resign button to enter resignation details. Using a macro, the action of clicking Resign button and entering the resignation details will be performed automatically.

  • New information such as appointment of a statutory officer. If performing the appointment manually, users would need to click on Appoint and enter the details. Using a macro, the action of clicking Appoint button and entering the appointment details will be performed automatically. However, these require the appointment details to be available in the workflow step containing the macro.

  • To print the executed workflow and store the document in a document holder. If performing the printing manually, users would need to click Print Workflow in the workflow’s Ribbon and save the document in their machine. Users would need to manually store the document in a document holder. Using a macro, the action of printing and storing of document will be performed automatically.

To configure a macro for this, you need to specify the following:

  • The screen where the action will take place.

  • The command button for the action.

  • The details for fields related to the action.

This is done by entering the tags in the Values/Default column. For further information on the tags, you can refer to Macros - Tags.
