Approval

If the 'No Approval Required for Configuration Change' option is not ticked in System Default > Engage Servicing Portal, then configurations of organisations and users require approvals which can be performed via these screens:

  • Viewpoint Connect - Home > Configuration > Engage Servicing Portal > Approval > Organisations Approval.

  • Viewpoint Connect - Home > Configuration > Engage Servicing Portal > Approval > Users Approval.

In both screens, you will find list of items relating to organisations or users that require approval. Note that an approver can only approve items created by another user.

 

Organisations Approval

This screen allows for approval of organisations and file configurations.

 

Filters

The filters apply to the organisation list.

  • Organisation status - You can choose to view organisations that have been approved, pending approval or all.

  • Organisation Type  - You can choose to view a specific type or all.

 

Organisation list

The list shows all organisations that have been set up.

 

Command buttons

These apply to the Organisation list. Click on the relevant command buttons to perform the necessary.

  • Select All - Click this to select all listed organisations.

  • De-select All - Click this to de-select all selection.

  • Approve Organisation - Click to approve selected organisation item.

    Click Yes to confirm the approval.

    The Master File screen will appear with a list of Master File(s) pending approval associated with the organisation. If there is no Master File to be approved, the list will be empty.

    Once approved, it will not be seen in the list any more.

     

    Note: The deletion of an organisation will not be approved if there are:

    • users belonging to the organisation

    • users linked to the organisation via the User - Organisation authority setting

    • file configurations linked to the organisation

    • Master Files relating to the organisation

 

File Configuration list

This shows the files for the selected organisation.

 

Command buttons

These apply to the File Configuration list. Click on the relevant command buttons to perform the necessary.

  • Select All - Click this to select all listed file configuration.

  • De-select All - Click this to de-select all selection.

  • Approve File Configuration - Click to approve selected file configuration item.

    The Master File screen will appear with a list of related Master File(s) that are included, excluded or deleted.

    Click Approve.

    • If the Master File's record status was 'Included', it will be updated to 'Approved'.

    • If the Master File's record status was 'Excluded', it will be updated to 'Approved Excluded'.

    • If the Master File's record status was 'Deleted', the Master File will be removed from the list.

    A Files Approval Summary message will be displayed to show the following:

    • The number of Master File now Approved.

    • The number of Master File now Approved Excluded.

    • The number of Master File now deleted from the list.

    • The number of Master File that failed to be approved.

    • The number of Master File still pending approval.

    • The list of organisations pending approval with no related Master Files.

    Note: The deletion of a Master File will not be approved if there are:

    • relations associated to the Master File.

    • document shares related to the Master File.

    • summary pages linked to the Master File.

 

Users Approval

This screen allows for approval of users and User - Organisations.

 

Filters

The filters apply to the user list.

  • User status - You can choose to view users that have been approved, pending approval or all.

 

Select button

This allows for additional criteria to be applied to the user list.

 

User list

The list shows all users that have been set up.

 

Command buttons

These apply to the User list. Click on the relevant command button to perform the necessary.

  • Select All - Click this to select all listed users.

  • De-select All - Click this to de-select all selection.

  • Approve User - Click to approve selected user item.

    Click Yes to confirm the approval.

    Once approved, it will not be seen in the list any more. All User - Organisations relating to the user will be automatically approved as long as the user listed in the Change by column is not the approver.

     

    Note: If approving a deletion, the record will be removed permanently.

 

User - Organisation list

This shows the organisations that the selected user is in.

 

Command buttons

These apply to the User  - Organisation list. Click on the relevant command buttons to perform the necessary.

  • Select All - Click this to select all listed user - organisations.

  • De-select All - Click this to de-select all selection.

  • Approve User - Organisation - Click to approve selected User - Organisation item. If all the User - Organisation for a particular user have been approved, the associated user will also be approved.

    Click Yes to confirm the approval. This will set the record status to Approved.
