Change Codes: Link Item - Tags

When an item is linked to a Change Code, tags can be added to the Tag Values field to specify additional condition for the linked item and/or additional details for the linked item. The tags will depend on the type of linked item.

 

Multiple Linked Item Types

Tag

Explanation

Example

[/SQL.<QueryName>=<Query>/]

 

where

  • <QueryName> is the name of the SQL

  • <Query> is the SELECT query that will be retrieving the required information

 

This can be used for:

  • specifying the condition for the change parameter.

  • retrieving information that will be used as values for other tags. This applies to the following link item types:

  • E-mail Notifications

  • Notifications

  • Tasks

[/SQL.SQLOFC=SELECT EntCode FROM Officers WHERE AddrCode='(-AddrCode-)' AND Left(OfficerType,1) IN ('D', 'S') AND SeqNr=(-SeqNr-)/]

The above will refer to the specific change relating to officer types starting with 'D' or 'S'. This means that the linked item will be triggered once for each instance of the change.

 

[/SQL.EntityAdmin=SELECT AdminCode AS FileAdm FROM Entity WHERE EntCode='(-EntCode-)'/]

The above looks for the AdminCode field of the Entity File of the change. This information can be used as value for other tag such as tag to define assignee for a task.

 

Checklists

Available tags for Checklists linked item type are used to specify details of the checklist and action for the checklist.

Tag

Explanation

Example

[WS.Category=<CategoryCode>]

 

where <CategoryCode> refers to the code of the checklist category. You can obtain this from the File Note Types [MEMT] lookup table.

 

This is to specify the category of the checklist using the code.

[WS.Category=20]

The tag above will set the Category as 'Work & Outstanding'.

[WS.Index=<Value>]

 

where <Value> can be plain text or field codes related to the change.

This is to specify information for the Index field.

[WS.Index=Check information entered]

The above will result in the value for the Index field as 'Check information entered'.

[WS.DateEntered=<Value>]

 

where <Value> can be an actual date or a number.

This is to specify information for the Date Entered field.

[WS.DateEntered=0]

The tag above will set today's date as Date Entered.

 

[WS.DateEntered=15/06]

The tag above will set the Date Entered as 15th of June for the current year.

[WS.RequireDone=<Value>]

 

where <Value> can be:

  • 1 for Yes

  • 0 for No

This is to specify whether or not the option 'Confirmation Required' is ticked.

[WS.RequireDone=1]

The above will tick the Confirmation Required checkbox.

 

[WS.DateDue=<Value>]

 

where <Value> can be an actual date or a number.

This is to specify information for the Date Due field.

[WS.DateEntered=3]

The tag above will set the Date Due as three days from today.

[WS.Recurring=<Code>]

 

where <Code> refers to the code of the frequency which are:

  • 0 for 10 years

  • 2 for 2 years

  • 3 for 3 years

  • 4 for 4 years

  • 5 for 5 years

  • Y for Annually

  • F for Fortnightly

  • M for Monthly

  • - for Not Applicable

  • = for Others - see due date/note

  • Q for Quarterly

  • X for Semi-Annually

  • B for Two Months

  • W for Weekly

This is to specify the recurring frequency.

[WS.Recurring=Y]

The tag above sets the recurring frequency as Annually.

[WS.AdminAssign=<Value>]

 

where <Value> is the user code.

This is to specify information for the Administrator field.

[WS.AdminAssign=sm]

The tag above sets the Administrator to the user 'sm'.

[WS.Note=<Value>]

 

where <Value> can be plain text or field codes related to the change.

This is to specify information for the Note field.

[WS.Note=Appointment of (+AddrCode+)]

The tag above will retrieve the 'AddrCode' to be included in the Note field.

[WK.<Key>=<Value>]

 

where

  • <Key> is the key of the checklist item

  • <Value> is the default value for the checklist item

This is to specify default values for items in the checklist. Ensure that the checklist items are assigned unique keys.

Note: This can be used if the field type is Text, Date, Number, Amount, Yes/No, List with Edit, List without Edit, Lookup Value, User Table Value, System User or Administrator Group

[WK.CI1=Y]

The tag above sets the default value 'Yes' for checklist item with the Key 'CI1'.

 

E-mail Notifications

Tags are to be used to define details of the e-mail.

Tag

Explanation

Example

[/MAIL_SUBJECT=<Subject>/]

where <Subject> can contain plain text and field codes.

This is to specify the e-mail subject.

[/MAIL_SUBJECT=New Master File created/]

[/MAIL_TO=<Recipient>/]

where <Recipient> can be:

  • a specific e-mail address

  • field codes to retrieve the e-mail address

This is to define the recipient in the To field. If the information is to be retrieved using an SQL, ensure that the [/SQL.<QueryName>/] tag is also defined.

[/MAIL_TO=(+SQL.SQLTO.ADMEM+)/]

The above gets the recipient from the 'SQLTO' query.

[/MAIL_CC=<Recipient>/]

Similar to the above except that this is for the CC field.

[/MAIL_CC=(+SQL.SQLCC.MGREM+)/]

The above gets the recipient from the 'SQLCC' query.

You can have multiple email address by using semicolon (;)

[/MAIL_CC=sm@manford.com;anne@manford.com/]

[/MAIL_BCC=<Recipient>/]

Similar to the above except that this is for the BCC field.

[/MAIL_BCC=sm@manford.com/]

The above sets the recipient for BCC as 'sm@manford.com'.

You can have multiple email address by using semicolon (;)

[/MAIL_BCC=sm@manford.com;anne@manford.com/]

[/MAIL_REPLYTO=<E-mail>/]

where <E-mail> can be:

  • a specific e-mail address

  • field codes to retrieve the e-mail address

This is to specify the e-mail address for replies.

[/MAIL_REPLYTO=(+MailID+)/]

The above retrieved the e-mail address of the user who performed the change.

You can have multiple email address by using semicolon (;)

[/MAIL_REPLYTO=sm@manford.com;anne@manford.com/]

[/MAIL_BODY=<E-mailBody>/]

where <E-mailBody> can contain plain text, field codes and HTML formatting tags.

This is to define the e-mail content.

[/MAIL_BODY=Dear Sir,<br/> A new Master File has been created./]

(+CC.<FIELDNAME>+)

To retrieve relevant values from relevant field in the event string.

Without stating the new or old value, the new value will be retrieved by default.

 

(+CC.AddrLine+)

(+CC.Adline\*O+)

(+CC.Adline\*N+)

 

\*O = Old value before change.

\*N = New value after change.

 

(+CC.USERCODE+)

This is to retrieve user code for logged in use.

Trimmed Username: (+CC.USERCODE\#US*SUBSTR=1:5+)

 

E-mail Notifications supports Format Options.

(+CC.USERNAME+)

 

This is to retrieve username for logged in use.

Username: (+CC.USERNAME+)

(+CC.USER+)

To retrieve user and administrator details.

Email: (+CC.USER.MailD+)

 

API Integration

Tag

Explanation

Example

[/MESSAGE=<Message>/]

This defines the content to send to an external REST API in JSON format.

[/MESSAGE=

{

  "X_TRANSACTION_ID": "(+CC.EventNr+)",

  "Payload": {

    "eventChangeType": "UPDATE",

    "eventTypeName": "MATTER_CHANGED",

    "identifiers": [

      {

        "identifierName": "Matters.RefCode",

        "identifierValue": "(+CC.RefCode+)"

      },

      {

        "identifierName": "Matters.MatterCode",

        "identifierValue": "(+CC.MatterCode+)"

      }

    ]

  }

}

/]

[/SERVICE=<Service name>/]

This defines the Service Name that indicates which Task Scheduler and Notification job to pick up the Change Link.

[/SERVICE=PrimaryApiService/]

 

Excel Statements

For Excel Statements, tags are available to define:

  • Selection criteria - This is required.

    [<Field>=<Value>]

    This allows you to define the selection criteria to trigger the Excel Statement. Note that the fields available depend on the Excel template and the value can be a defined value or a field code.

    Example of defined value:

    [EDT=25-Feb-14]

    [SLNO=0-9978635]

     

    Example of field code:

    [EC=(+CC.RefCode+)]

    - this means the value is to be retrieved by the RefCode field

     

    [PB=(+SQL.Filings.PeriodFrom+)

    - this means the value is the PeriodFrom field that is to be retrieved by an SQL query 'Filings'.

  • Generation option - If no option is defined, it will revert to default option.

    [$GENERATE=<Option>]

    The option can be Q - to send to Document Queue (default), P - to print, E - to generate and open the document for editing.

 

Notes

For Notes, the available tag allows you to define the type of note for the change

[Type=<Value>]

where <Value> can be:

  • I - This have the Info icon to indicate that the Note is informational in nature.

  • W - This will have the exclamation mark icon to indicate that it is a warning message.

  • E - This will have a red square icon and red text to indicate that it is an error message.

Note: If the Tag Values field is left blank, by default, the type will be informational.

 

Notifications

For Notifications, tags are required to define properties of the notification item such as the category, priority, etc.

Tag

Explanation and Example

[/NOTIFY_CAT=<Value>/]

This defines the category for the Notification item and values can be:

  • MF – Master File

  • WF – Workflow

  • DC – Document Management

This tag is mandatory.

 

Example:

[/NOTIFY_CAT=MF/]

[/NOTIFY_SYSTEMFLAGS=1/]

This indicates that the notification is for Viewpoint Connect and is mandatory.

[/PRIORITY=<Value>/]

This is to set the notification priority and is mandatory. The values can be 0 or 1. Priority 1 will include an exclamation mark icon for the notification.

 

Example:

[/PRIORITY=1/]

[/NOTIFY_HEADER=<Header>/]

This is to specify the header or subject for the notification item and is mandatory. It can be just text or include Field Codes related to the change.

 

Example:

[/NOTIFY_HEADER=Master File Detail Changed/]

[/NOTIFY_MSG=<Msg>/]

This is to specify the content or message for the notification item and can be just text or include Field Codes related to the change.

 

Example:

[/NOTIFY_MSG=Contact Change for (+CC.RefCode\#AD-1-0+)/]

[/NOTIFY_ADMINGP=<Value>/]

This is to specify the Administrator Group for whom the notification will appear. Only the default Administrator Group can be selected. You can specify this by:

  • Entering the code of the Administrator Group.

  • Using an SQL query to retrieve the Administrator Group by defining the SQL and referring to the SQL.

 

Example:

[/NOTIFY_ADMINGP=GROUP1/]

means the notification item will appear for all users belonging to the Administrator Group ‘GROUP1’.

 

[/NOTIFY_ADMINGP=(+SQL.RefMaster.AdminGroup+)/] [/SQL.RefMaster=SELECT AdminGroup FROM RefMaster WHERE RefCode=’(-EntCode-)’/]

means the notification item will appear for users whose default Administrator Group is the same as the Administrator Group retrieved by the ‘RefMaster’ SQL.

 

Note: This tag cannot be used with the [/NOTIFY_USER=<Value>/], [/NOTIFY_USERGP=<Value>/], [/NOTIFY_ACCESSGP=<Value>/], [/NOTIFY_ADMINGPALL=<Value>/], tags.

[/NOTIFY_ADMINGPALL=<Value>/]

This is to specify the Administrator Group for whom the notification will appear. Unlike [/NOTIFY_ADMINGP=<Value>/], this tag allows the selection of non-default Administrator Groups. You can specify this by:

  • Entering the code of the Administrator Group(s).

  • Using an SQL query to retrieve the Administrator Group by defining the SQL and referring to the SQL.

 

Example:

[/NOTIFY_ADMINGPALL=GROUP2,GROUP1/]

means the notification item will appear for all users belonging to the Administrator Groups 'GROUP2' and 'GROUP 1'.

 

[/NOTIFY_ADMINGPALL=(+SQL.VpAGroupUsers.AdminGroup+)/] [/SQL.VpAGroupUsers=SELECT AdminGroup FROM VpAGroupUsers WHERE GroupType = 1 AND UCode = '(-CC.AssignTo-)'/]

means the notification item will appear for users whose Access Group is the same as the Access Group retrieved by the ‘VpAGroupUsers’ SQL.

 

Note: This tag cannot be used with the [/NOTIFY_USER=<Value>/], [/NOTIFY_USERGP=<Value>/], [/NOTIFY_ACCESSGP=<Value>/], [/NOTIFY_ADMINGP=<Value>/] tags.

[/NOTIFY_USERGP=<Value>/]

This is to specify the User Group for whom the notification will appear. You can specify this by:

  • Entering the code of the User Group(s).

  • Using an SQL query to retrieve the User Group by defining the SQL and referring to the SQL.

 

Example:

[/NOTIFY_USERGP=SYSMGR,admin/]

means the notification item will appear for all users belonging to the User Groups 'SYSMGR' and 'admin'.

 

[/NOTIFY_USERGP=(+SQL.VpUser.Ugroup+)/] [/SQL.VpUser=SELECT Ugroup FROM VpUser WHERE UCode ='(-CC.AssignTo-)'/]

means the notification item will appear for users whose User Group is the same as the User Group retrieved by the ‘VpUser’ SQL.

 

Note: This tag cannot be used with the [/NOTIFY_USER=<Value>/], [/NOTIFY_ACCESSGP=<Value>/], [/NOTIFY_ADMINGPALL=<Value>/], [/NOTIFY_ADMINGP=<Value>/] tags.

[/NOTIFY_ACCESSGP=<Value>/]

This is to specify the Access Group for whom the notification will appear. You can specify this by:

  • Entering the code of the Access Group(s).

  • Using an SQL query to retrieve the Access Group by defining the SQL and referring to the SQL.

 

Example:

[/NOTIFY_ACCESSGP=COSEC,FINANCE/]

means the notification item will appear for all users belonging to the Access Groups 'COSEC' and 'FINANCE'.

 

[/NOTIFY_ACCESSGP=(+SQL.VpAGroupUsers.AdminGroup+)/] [/SQL.VpAGroupUsers=SELECT AdminGroup FROM VpAGroupUsers WHERE GroupType = 2 AND UCode = '(-CC.AssignTo-)'/]

means the notification item will appear for users whose Access Group is the same as the Access Group retrieved by the ‘VpAGroupUsers’ SQL.

 

Note: This tag cannot be used with the [/NOTIFY_USER=<Value>/], [/NOTIFY_USERGP=<Value>/], [/NOTIFY_ADMINGPALL=<Value>/], [/NOTIFY_ADMINGP=<Value>/] tags.

[/NOTIFY_USER=<Value>/]

This is to specify the user(s) for whom the notification will appear, and it can be for:

  • Specific user(s) by – Specify the user code or user codes

  • User related to the Master File – Specify the relevant Assign To Code:

    • (-EN.AD-) for Administrator of the Entity File

    • (-EN.MG-) for Manager of the Entity File

    • (-EN.CT-) for Controller of the Entity File

    • (-BF.AD-) for Administrator of the Billing File

    • (-BF.MG-) for Manager of the Billing File

    • (-BF.CT-) for Controller of the Billing File

    • (-AF.AD-) for Administrator of the Account File

    • (-AF.MG-) for Manager of the Account File

    • (-AF.CT-) for Controller of the Account File

    • (-SF.AD-) for Administrator of the Supplier File

    • (-SF.MG-) for Manager of the Supplier File

    • (-SF.CT-) for Controller of the Supplier File

    • (-MF.AD-) for the Administrator of the Master File

    • (-MF.MG-) for the Manager of the Master File

    • (-MF.KO-) for the CDD Officer of the Master File  

For certain change codes, it is also possible to notify users who are related to the change. This requires the change field code that will retrieve the user. For instance, when a new workflow is started and assigned to another user, to notify the assigned user, the tag is [/NOTIFY_USER=(+CC.AdminCode+)/].

 

Example:

[/NOTIFY_USER=SM,ANNE/]

means the notification item is for the users ‘SM’ and ‘ANNE’.

 

[/NOTIFY_USER=(-EN.MG-),(-BF.MG-)/]

means the notification item is for the manager of the Entity File and the Billing File.

 

Note: This tag cannot be used with the [/NOTIFY_ADMINGP=<Value>/] tag.

[/NAVIGATION_KEY=<NavigationID/]

This is to set the screen for the Go To button for the notification item. This is so that when the notification item is selected, clicking Go To button in the Ribbon will display the specified screen. This tag is optional and if it is not included or no Navigation ID is defined, then clicking Go to will open the File Maintenance screen.

 

Example:
[/NAVIGATION_KEY=FSESTATC/]

 

Note: The <NavigationID> can be retrieved from Configuration > Settings > Customise > Windows Menu.

 

 

Tasks

For Tasks, tags are available to define:

  • Task details.

    Tag

    Explanation

    Example

    [TaskType=<TypeCode>]

     

    where <TypeCode> refers to the Key Tag of the Task.

     

    This is to specify the task to be triggered.

    [TaskType=APPROVE]

    [AssignedTo=<Value>]

     

    where <Value> can be:

    • specific user account

    • field code to refer to information retrieved using [/SQL.<QueryName>=/] tag

    This is to specify the assignee for the task.

    [AssignedTo=SM]

    The above sets the assignee to the user 'SM'.

     

    [AssignedTo=(+SQL.EntityAdmin.FileAdm+)]

    The above sets the assignee to the 'FileAdm' field value retrieved by the 'EntityAdmin' query.

    [TskDescr=<Value>]

     

    where <Value> can be plain text or field codes related to the change.

    This is to specify the task description. You can leave this blank to have it default to the description of the change.

    [TskDescr=Review and approve if in order]

     

    [TskNote=<Value>]

     

    where <Value> can be plain text or field codes related to the change.

    This is to specify the Note field for the task. You can leave this blank if no note is required.

    [TskNote=Inform administrator if incorrect information]

    [DateDue=<Value>]

     

    where <Value> can be an actual date or a number.

    This is to specify the Date Due for the task.

    [DateDue=2]

    The above will set the Date Due as two days from today.

     

  • Action for the task - If no action is defined, it will revert to default action.

    [$ACTION=<Value>]

    This is to specify if the task is to be seen in the Change Processing screen or not. [Value] can be:

    • P - this is the default action and will show the task in the Change Processing screen. To create the task, click the Browse button in the Effective Date/Action column and if necessary, change the task details.

    • S - this will automatically create the task as a Master File task as long as the task details are defined correctly. If invalid details have been specified, the task will be seen in the Change Processing screen. You need to click on the Browse button to specify the correct information to create the task.

    • T - this is similar to S. However, if there is invalid details, this task will be aborted so it does not get created or seen in the Change Processing screen.

Note: You need to have Document Management component to use task. Task can only be created if the Change details contain a Master File code.

 

Standard Report

For Standard Reports, tag can be used to define the selection criteria.

[<Field>=<Value>]

This allows you to define the selection criteria to trigger the SSRS report.

 

Word Reports

When the linked item type is Word Reports, tags are available to define:

  • Selection criteria - This is required.

    [HD-SEL=<Criterion>]

    The <Criterion> can be any field returned by the header statement for the report with a specific value.

    For example:

    [HD-SEL=RefCode='(+CC.RefCode+)']

    [HD-SEL=RefCode='(+EntCode+)']

     

    For  multiple selection criteria, the below tag is used:

    [HD-SEL=<Criterion> AND <Criterion>]

    For example:

    [HD-SEL=WflowNr='(+CC.WflowNr+)' AND WfPageNr='(+CC.WfPageNr+)']

  • Generation option - If no option is defined, it will revert to default option.

    [$GENERATE=<Option>]

    The option can be Q - to send to Document Queue (default), P - to print, E - to generate and open the document for editing.

 

Workflows

For Workflows, tags are available to define:

  • Details for the workflow header.

    Tag

    Explanation

    Example

    [WH.WflowDescr=<Description>]

    This is to specify the Description for the workflow header.

    [WH.WflowDescr=]

    The tag above will use the Workflow's description.

    [WH.AdminCode=<Administrator>]

    This is to specify the Administrator assigned to the workflow.

    [WH.AdminCode=sm]

    [WH.DateDeadLine=<Days>]

    This is to specify the Date Due for the workflow.

    [WH.DateDeadLine=7]

    The tag above will set the Date Due as seven days from today.

    [WH.WflowNote=<Note>]

    This is to specify the Note for the workflow .If not required, you can exclude this tag.

    [WH.WflowNote=Include supporting documents]

     

  • Details for the first workflow step.

    Tag

    Explanation

    Example

    [WS.WfPageDescr=<Description>]

    This is to specify the Description for the first step of the workflow.

    [WS.WfPageDescr=]

    The tag above will set the description to the step's description.

    [WS.AdminCode=<Administrator>]

    This is to specify the Administrator for the first step of the workflow.

    [WS.AdminCode=anne]

    [WS.DateStart=<Days>]

    This is to specify the Date Start for the first step of the workflow.

    [WS.DateStart=0]

    The tag above will set today as the Date Start.

    [WS.DateDeadLine=<Days>]

    This is to specify the Date Due for the first step of the workflow.

    [WS.DateDeadLine=4]

     

     

  • Default value for items in the first workflow step.

    This is done using the tag [WK.<Key>=<Value>] explained in tag values for checklist.
