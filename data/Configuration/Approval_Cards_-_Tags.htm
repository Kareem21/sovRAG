Approval Cards - Tags

For approval cards, tags are available to define the approval list for the approval card.

 

Specify approvers based on responsibilities

Tag

Explanation

Example

@ADM1.<FileType>

Approval is required from the user appointed as the file administrator. Replace <FileType> with one of the following:

  • A - Account File

  • B - Billing File

  • E - Entity File

@ADM1.A

means the user appointed as the Administrator of the Account File.

@ADM2.<FileType>

Approval is required from the user appointed as the file manager. Replace <FileType> with one of the following:

  • A - Account File

  • B - Billing File

  • E - Entity File

@ADM2.A

means the user appointed as the Manager of the Account File.

@ADM3.<FileType>

Approval is required from the user appointed as the file controller. Replace <FileType> with one of the following:

  • A - Account File

  • B - Billing File

  • E - Entity File

@ADM3.A

means the user appointed as the Controller of the Account File.

@RELC.<RelationTypeCode>

Approval from user appointed as the specified User Relation. Replace <RelationTypeCode> with actual code for the User Relation.

@RELC.LAW

means the user appointed as the 'LAW' relation.

@USGR.<UserGroupCode>

Approval from any user belonging to the specified User Group. Replace <UserGroupCode> with actual code for the User Group.

@USGR.MNGR

means any user belonging to the User Group 'MNGR'.

@ADGR.<AdminGroupCode>

Approval from any user belonging to the specified Administrator Group. Replace <AdminGroupCode> with actual code for the Administrator Group.

@ADGR.GROUP1

means any user belonging to the Administrator Group 'GROUP1'.

@ORGU.<OrgUnitCode>

Approval from any user belonging to the specified Organisational Unit. Replace <OrgUnitCode> with actual code for the Organisational Unit.

@ORGU.ORGUNIT1

means any user belonging to the Organisational Unit 'ORGUNIT1'.

@AGUG.<UserAdminGroup>+<UserGroup>

Approval from any user belonging to the specified User Group has the specified Administrator Group as default Administrator Group. Replace <UserAdminGroup> with actual code for the Administrator Group and <UserGroup> with actual code for the User Group.

@AGUG.GROUP1+MNGR

means any user belonging to the User Group 'MNGR' and has the Administrator Group 'GROUP1' as default Administrator Group.

@AAUG.<AssignedAdminGroup>+<UserGroup>

Approval from any user belonging to the specified User Group and has the specified Administrator Group as assigned Administrator Group. Replace <AssignedAdminGroup> with actual code for the Administrator Group and <UserGroup> with actual code for the User Group.

@AAUG.GROUP1+MNGR

means any user belonging to the User Group 'MNGR' and has the Administrator Group 'GROUP1' as assigned Administrator Group.

@USER.MGR

Approval from the user appointed as the Manager in Configuration > Users & Security > Users.

  

@MASF.KYC_Officer

Approval from the user appointed as the CDD Officer in General > Master File > [Master File] > Compliance > CDD screen.

  

@WH.ADMIN.MGR

Approval from the user appointed as the Manager of the Workflow Header Administrator as specified in Configuration > Users & Security > Users.

 

@WS.ADMIN.MGR

Approval from the user appointed as the Manager of the Workflow Step Administrator as specified in Configuration > Users & Security > Users.

 

@WK.<Key>.MGR

Approval from the user appointed as the Manager of the selected system user as specified in Configuration > Users & Security > Users.

<Key> refers to the checklist item's key.

  

 

Define approver group

It is also possible to group the approvers and specify the number of approvers required per group or the order of approval.

Tag

Explanation

Example

[GROUP.<GroupName>:<GroupMembers>]

This is to define the group of approvers and members within the group. You can have more than one group specified.

When specifying the <GroupMembers>, this can be:

  • specific user codes

  • tags to define users such as @USGR.<UserGroupCode

 

Note:When using tags to define to <GroupMembers>, ensure that you leave two spaces after the colon e.g. [GROUP.Managers:  @USGR.MGR]

[GROUP.Managers:  @USGR.MGR]

[GROUP.Directors:sm

anne]

means there are two groups - Managers and Directors. For Managers, the members are users in the User Group 'MGR'. For Directors, the members are the users 'sm' and 'anne'.

[GROUP.Require:

<GroupName1>:<Number of approval>

<GroupName2>:<Number of approval>

]

This is to specify the number of approvals required from a group.

 

Note: You cannot use this tag with the tag below.

[GROUP.Require:

Managers:1

Directors:0

]

means one approval is required from the Managers group. Where the quantity is 0, this gives flexibility to the approval card in which members of the Directors can be counted to fulfil the card's minimum required approvers i.e. if 3 approvals are required, one needs to be from Managers group while the other two can be from Directors group.

[GROUP.Order:

<GroupName1>:<Number of approval>

<GroupName2>:<Number of approval>

]

This is to specify the number of approvals required from a group and which group should approve first.

 

Note: You cannot use this tag with the tag above.

[GROUP.Order:

Managers:2

Directors:1

]

means that two managers must approve before a director will be able to complete the approval.

 

Note: If the number required if not specified, the default value is one.
