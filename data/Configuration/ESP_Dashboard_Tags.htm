ESP Dashboard - Tags

Multiple tags are available for ESP Dashboard and are used to define properties of the widget.

Tag Explanation Apply to... Example
[/FormCode=<FormCode>/]
 
where <FormCode> refers to the code of the template.
 
 
If you would like different template to be used based on the Master File type, then the Master File type can also be added to the tag as follows:
[/FormCode=[MF.<Type1>=<FormCode1>] [MF.<Type2>=<FormCode2] .../]
 
where <Type> is

  • C for Master File Type 'Company'

  • P for Master File Type 'Partnership'

  • T for Master File Type 'Trust'

  • DF for Default Template for when the Master File type is not any of the specified ones.

 This is used to specify the template used for the widget. The template is maintained in the Engage Servicing Portal Templates in Document Libraries.
 		 Widget type = e-View [104] [/FormCode=CPBFDA/]
 
The above will use the template 'CPBFDA' for the widget.
 
 
[/FormCode=[MF.C=CPCSSS] [MF.P=CPCSSS] [DF=CPMFSS]/]
 
The above will use the template 'CPCSSS' for the widget when the Master File type is Company and Partnership. Otherwise, the template 'CPMFSS' is used.
[/Page=rss/] This indicates that the widget gets feeds from an online site. This tag requires the [/URL=/] tag below. Widget type = RSS Feed [105] [/Page=rss/][/Url=http://feeds.reuters.com/reuters/businessNews/]
 
The above gets RSS feeds from Reuters.
 
 
[/Page=rss/][/Url=http://feeds.reuters.com/reuters/businessNews/]
[/Size=5/]
 
The above gets RSS feeds from Reuters and shows only five records.
[/URL=<URL>/]
 
where <URL> refers to the URL for the online site.		 This tag is mandatory and is used with [/Page=rss/] tag.
[/Size=<Number>/]
 
where <Number> refers to the number of records to display		 When used with [/Page=rss/] tag, it will restrict the number of items seen for the RSS feed widget. It is optional and if not defined, then all retrieved feeds will be shown.
[/Page=calendar/] This indicates that the widget gets information from the ESP Calendar. Widget type = General [103] [/Page=timesheet/][/Size=5/]
 
The above gets pending timesheets for the user and shows the earliest five.
 
 
[/Page=workflowlist/][/Size=5/]
 
The above gets pending requests and displays the first five.
[/Page=calendar_month/] This is for the Upcoming Events widget and it gets items from the ESP Calendar as well but only for the current month.
[/Page=workflow/] This indicates that the items are requests (workflows) and the display will be shown in Tile view.
[/Page=workflowlist/] This indicates that the items are requests (workflows) and the display will be shown in List view.
[/Page=timesheet/] This indicates that the items are timesheets that have not been submitted.
[/Page=timeheet_manager/] This indicates that the items are timesheets that include those that have been submitted for approval.
[/Page=debtorage/] This indicates that the items are information from Debtor Ageing.
[/Page=shareddocument/] This is for the Shared Documents widget and the items are documents that are shared on ESP.
[/Page=entityMap/] This is for the Group Overview widget where highlights will appear on the world map to indicate location of entities.
[/Page=feed/] This is for the Notice Board widget which gets items from Configuration > Engage Servicing Portal > Notice Board. The display is based on a grid and this tag requires the [/GridCode=/] tag below.
[/GridCode=<Code>/]
 
where <Code> refers to the to code assigned to the grid field.		 This is to specify the grid field and is used with the [/Page=feed/] tag.
[/Page=download/] This is for the Generated Documents widget and the items are documents that have been selected for generation.
[/Size=<Number>/]
 
where <Number> refers to the number of records to display		 This tag is used to restrict the number of items seen in the widget and is optional. It can be used with most of the [/Page=] tag where list of items are retrieved.
[/Page=workflowAction/] This tag is used for the Workflow Action widget where items are retrieved based on values in the Engage Portal Calendar Type [CPIT] lookup table.
 
Note: Refer to the tags for Engage Portal Calendar Type [CPIT] in Lookup Tables.		 Widget type = Workflow Link [106]

[/Page=workflowAction/]

[/ItemTypes=EN_FLDT,EN_MEDT,EN_REDT/]

[/Line1=(-ItemDetail.EntityName-)/]

[/Line2=(-ItemDetail.Name-), (-ItemDetail.Item1-) (-ItemDetail.Item2-), (-ItemDetail.Item3-)/]  

[/Button=Linked Request/]

[/WfCode=(-ItemDetail.LinkWfCode-)/]  

[/WfLinkCode=(-ItemDetail.LinkCode-)/]  

[/WfLinkRecId=(-ItemDetail.LinkRecId-)/]

[/GroupText=(-ItemDetail.GroupText-)/]

 

The above includes the three lookup table values for Workflow Action widget with information to be displayed in two lines. The caption for the button is defined as 'Linked Request'.
[/ItemTypes=<LookupValueCode>/]
 
where <LookupValueCode> refers to the code of the value in the Engage Portal Calendar Type [CPIT] lookup table. Use comma to separate multiple values.		 This tag is used to specify the relevant values for the widget.
[/Line1=<Info>/]
 
where <Info> can include text and also reference to information defined for the lookup table value using the [/Detail=/] tag. In this case, it must be specified as (-ItemDetail.<Code>-) 		This defines the information that will appear in the first line of the record. Use comma to separate multiple values.
 
Note: Refer to the tags for Engage Portal Calendar Type [CPIT] in Lookup Tables.
[/Line2=<Info>/]
 
where <Info> can include text and also reference to information defined for the lookup table value using the [/Detail=/] tag. In this case, it must be specified as (-ItemDetail.<Code>-)		 This defines the information that will appear in the second line of the record. Use comma to separate multiple values.
This is optional.
 
Note: Refer to the tags for Engage Portal Calendar Type [CPIT] in Lookup Tables.
[/Button=<Caption>/]
 
where <Caption> is the text label for the button.		 This is to set the caption for the button that will start a linked workflow.
[/WfCode=(-ItemDetail.LinkWfCode-)/] This is to refer to the workflow template that is defined for the lookup table value using the LinkWfCode code.
 
Note: Refer to the tags for Engage Portal Calendar Type [CPIT] in Lookup Tables.
[/WfLinkCode=(-ItemDetail.LinkCode-)/] This is to refer to the screen for linking the workflow that is defined for the lookup table value using the LinkCode code.
 
Note: Refer to the tags for Engage Portal Calendar Type [CPIT] in Lookup Tables.
[/WfLinkRecId=(-ItemDetail.LinkRecID-)/] This is to refer to the condition for the workflow linking that is defined for the lookup table value using the LinkRecID code.
 
Note: Refer to the tags for Engage Portal Calendar Type [CPIT] in Lookup Tables.
[/GroupText=(-ItemDetail.GroupText-)/] This is to refer to the caption for the tag that is defined for the lookup table value using the GroupText code.
 
Note: Refer to the tags for Engage Portal Calendar Type [CPIT] in Lookup Tables.

Note: The Engage Servicing Portal - Calendar Update job must be set up to ensure that the Calendar and Workflow Action widgets can retrieve the latest records.
