User Group - Document Management Authorities

The authorities here apply to the Document Management component and controls what the users can do in relation to document storing and sharing as well as task management.

 

Global Document Authority

This is to grant Document Management authorities that are not tied to Filing Cabinets. Tick the authorities you want to give to the User Group:

  • Export of document selections - This allows the group to export the results of a search performed in Document Management.

  • Edit Master File tasks - This allows the group to edit Master File tasks.

  • Delete Master File tasks - This allows the group to delete Master File tasks.

  • Global e-mail rules setup - This allows the group to edit the global e-mail rules.

  • Engage Servicing Portal - Share - This allows the group to set a document for sharing on Engage Servicing Portal (ESP).

  • Engage Servicing Portal - Approve - This allows the group to perform approval of document sharing on ESP. Self-approval is not allowed, therefore the login user is not able to approve document that is listed under 'Shared by Me' in the Document Sharing Approval screen.

  • Access to documents up to - This determines which level of documents the user can access. Users will not see grid records of holders they do not have access to.

Assigned Level

Documents user can access

Unrestricted Access

Document holders set to Unrestricted Access

Restricted Access

Document holders set to Unrestricted Access and Restricted Access.

Confidential

Document holders set to Unrestricted Access, Restricted Access and Confidential.

Highly Confidential

Document holders set to Unrestricted Access, Restricted Access, Confidential and Highly Confidential.

 

Filing Cabinets Access and Authorities

This is to grant access to the filing cabinets and the authorities for the cabinets. Tick the relevant checkbox(es) in the grid to grant access and authorities. The grid contains the following:

  • Filing cabinets - This follows the filing cabinet setup and includes groups that have been setup. The checkbox next to the cabinet functions as select/deselect so you can tick to all to grant all authorities or remove all authorities.

  • Search - This is to give access to users to search in the cabinet. If unticked, the cabinet will not appear in the Filing Cabinet structure in document search. If a holder grid record of the cabinet is selected, the holder details will not be viewable in the Detail Window ('Group Security - Not authorised to view this document holder' status is shown instead).

  • Holders - This is to grant holder-related authorities so users can:

    • Add new holders to the cabinet (A).

    • Edit existing holders in the cabinet including deleting of links for the holders (E).

    • Delete existing holders or documents within the holders in the cabinet (D).

    • Add new documents to existing holders in the cabinet (A.E).

  • Tasks - This is to grant document task-related authorities so users can:

    • Add document tasks to holders in the cabinet (A).

    • Edit existing document tasks for holders in the cabinet (E).

    • Delete existing document tasks on holders in the cabinet (D).

 

Copy

Click to copy the Filing Cabinets Access and Authorities grid. When you paste the information, a ticked checkbox has a 'True' value and unticked checkbox has a 'False' value.
