Using Notifications

User can have messages to inform the changes made or work that have been assigned. These messages are notification items that are linked to Change Codes so that as the related change occurs, the notification items will be created. However, only Change Codes that include File Code detail will trigger notifications. These notification items are configurable so that they appear only for certain user/users.

To use the Notifications function, the following is required:

  • Enable Notifications option is ticked in System Defaults > General/Master File

  • Notification items are linked to the relevant Change Codes. Refer to Link Type: Notifications

When a notification item is created, users will be made aware of it by the following:

  • Windows Action Center

  • Viewpoint Notification pop-up

  • Notifications button in Viewpoint Ribbon

 

Windows Action Center

Viewpoint notification items can be included in the Windows Action Center. In the Taskbar for Windows 10, the icon for Windows Action Center will be active when there is a notification item. It will remain as active for as long as the notification item is not dismissed.

You can dismiss Viewpoint's notification item the same way as dismissing Windows Notification.

Note: Display of Windows Action Center depends on the version of operating system used, e.g. display seen on a computer running on Windows 10 will be different from the display seen on a computer running on Windows 7.

 

Viewpoint Notification Pop-up

This is seen even if the Viewpoint Connect program has been minimised but the pop-up is seen only for a moment. the pop-up will appear on the lower right side of your computer screen.

 

Notifications Button in Viewpoint Ribbon

This button is seen in various Viewpoint Ribbon. Where there is no notification item, the Ribbon will be blue. It will turn red when there is a new or unread notification item and remain so, as long as there is an unread notification item. The number of new and unread notifications will be displayed.

Note: The change of colour to red does not apply when the selected theme in User Options is Black.

 

Clicking on this button will open the Notifications screen where you will be able to:

  • View and manage all notification items.

  • View the message for each notification item.

Note: Checking for notification items are done according to the interval set for Refresh/Notification Interval (Minutes) in User Options.

 

Notifications Screen

This screen is accessed by clicking on the Notifications button in the Ribbon in Viewpoint - Home screen, File Maintenance screen or File Enquiry screen. In the Notifications screen, you have the following:

  • Ribbon

  • Notification Items List

  • Preview Pane

 

Ribbon

The following ribbon buttons are available in the Notifications screen.

 

Home tab:

  • Windows Manager - Click to open Windows Manager.

  • Go to Home - Click to return to Viewpoint - Home screen.

  • Close - Click to close the screen.

  • Select All - Click to select all notification items.

  • Refresh - Click to refresh the display. This will retrieve the latest notifications.

  • Go To - Click to go to the related screen as defined for the notification item. If no value is configured in the event link tag values, clicking this will open the Master File.

  • Read/Unread - Click to mark the selected notification item as read or unread. Unread item will appear in bold text.

  • Follow Up - Click to flag the selected notification item for follow up.

  • Remove - Click to remove the selected notification item. If you would like notification items to be removed from the list automatically, you can also schedule a 'Notification - Delete Read/Unflagged Notifications [DNOT]' job type to be run by Task Scheduler and Notification Agent. This will remove the notification item based on the age of the notification item as long as it has been read and not flagged for any follow-up.

  • Unread - Click to show unread items only.

  • Flagged - Click to show items that have been flagged for follow up.

  • Priority - Click to show items with high priority.

  • Advanced - Click to use the additional criteria. Click OK to save the criteria or Reset to clear all criteria that have been entered. This will also untoggle the button.

    • Master File - This is to show notification items for a Master File.

    • Category - This is to show notification items based on the category i.e. Document, Master File or Workflow.

    • Subject - This is to show notification items containing the specified subject.

    • Date - This is to show notification items for the selected date.

  • Preview Pane - Click to show or hide the preview pane.

  • Show in Windows Action Center - Click to show or hide notifications in the Windows Action Center.

Notification Items List

The grid lists the details of the available notification items. All read notification items will disappear after closing the Notification screen. However, items that are flagged, high priority, and unread will remain.

  • Checkbox - Tick the checkboxes to select or deselect notification items.

  • Priority - This shows the notification items which have been marked as priority.

  • Flag - This shows the notification items which have been flagged for follow up.

  • Category - This shows the category for the notification items.

    Categories available are:

    • Approval
    • Document
    • Master File
    • Workflow

  • Master File - This shows the Master File.

  • Date Time - This shows the date and time of the change.

  • Subject - This shows the subject for notification items.

To copy or export the information, right-click within the grid and select the relevant option.

 

 Note: You can also double-click on a record in the grid to open the screen for the notification item. This depends on the configuration of the notification item.

 

Preview Pane

This will show the header and message for the selected notification item. The icon seen will depend on the category of the notification item.
