Localise Script Tags - Document Management

Document Management tags can be used to set up your filing cabinet and index fields.

 

Setting up Filing Cabinets

When using Localise Script to add a new filing cabinet, tags for creation of the filing cabinet and defining the cabinet settings are required.

 

Syntax

To create a new cabinet:

DFCADD:<FolderNr>+<FolderKey>=<CabinetName>

 

To define the cabinet settings:

DFCVAL:<FolderNr>+<DBField>=<Value>

 

where:

  • <FolderNr> is the number for the filing cabinet. This should start at 1.

  • <FolderKey> is the Key Tag for the filing cabinet. This should be a unique key.

  • <CabinetName> is the name for the filing cabinet. If you want to group cabinets together, enter the group name and a colon ‘:’ before the cabinet name. In this example, “Correspondence” is the name of the group, and Confidential is the name of the cabinet: Correspondence:Confidential

  • <DBField> is the database field for the cabinet and you require a DFCVAL tag for each <DBField>.

    • HelpNote - Help Text field

    • RetentionPolicy - Retention Policy field

    • ClosedPost - Closed for new documents option

    • ClosedSearch - Closed for searching option

    • fNote - Note field

Additional <DBField> fields:

    • ParentNr – this is used when you are creating a sub-cabinet under another cabinet. If the main cabinet FolderNr=1 then a sub-cabinet under this would have ParentNr=1

    • KeyWords – this field is not currently used.

    • SizeLimit – this field is not currently used.

    • FolderSize – this field is used to indicate the limit on the size of the folder in the file structure. Normally the field is left as 0.

  • <Value> is the value for <DBField>. Use 1 to tick a checkbox field and 0 to untick the checkbox.

 

Example

To set up a new cabinet as shown above:

DFCADD:1+CORCON=Correspondence:Confidential

DFCVAL:1+HelpNote=

DFCVAL:1+RetentionPolicy=000 (you can obtain the appropriate code from the Document Retention Policy [DRTP] lookup table)

DFCVAL:1+ClosedPost=0

DFCVAL:1+ClosedSearch=0

DFCVAL:1+fNote=

DFCVAL:1+ParentNr=0

DFCVAL:1+SizeLimit=0

DFCVAL:1+FolderSize=0

 

Setting up Index Fields

When creating index fields, you also need both the tags to create the index field and to specify the settings for the index field.

 

Syntax

To create a new index field:

DIXADD:<IndexNr>+<IndexKey>=<IndexName>

 

To specify the index field settings:

DIXVAL:<IndexNr>+<DBField>=<Value>

 

where:

  • <IndexNr> is the number for the index field. Viewpoint comes with two index fields that should not be over-written, INDEX and DOCDATE. Therefore, when you create a new index field, the <IndexNr> should start at 3.

  • <IndexKey> is key tag for the index field.

  • <IndexName> is the name of the index field.

  • <DBField> is the database field for the index field. You need to have one DIXVAL tag for each <DBField>.

    • IxMandatory - Mandatory option

    • SortNr - position of the index field in the index field list

    • IxHelpNote - Help/Note field

    • IxFldType - Field Type field

    • FldMask - Format/Mask field

    • IxAllowDups - Allow duplicate index relations for this field option


  • <Value> is the value for <DBField>. Use 1 to tick a checkbox field and 0 to untick the checkbox.

 

Example

To set up an index field as shown above:

DIXADD:3+CORR=Correspondent

DIXVAL:3+SortNr=4

DIXVAL:3+IxHelpNote=

DIXVAL:3+IxMandatory=0

DIXVAL:3+IxAllowDups=1

DIXVAL:3+IxFldType=40

 

Adding Index Fields to Filing Cabinets

When you have created an index field, you need to specify which cabinet will require the index field. Additionally you will need to specify the settings for the index field in each cabinet.

 

 

Syntax

To add the index field to a cabinet:

DCIXADD:<FolderNr>+<IndexNr>=<IndexName>

 

To specify the settings for the field in the cabinet:

DCIXVAL:<FolderIndexNr>+<DBField>=<Value>

 

where:

  • <FolderNr> is the folder number of the filing cabinet (see To create a new cabinet, above).

  • <IndexNr> is the number of the index field you want to add (see To create a new index field, above).

  • <IndexName> is the name you want the field to use in the selected cabinet. For instance, the Correspondent index field can be renamed ‘Sender’ in the Correspondence:Confidential cabinet.

  • <FolderIndexNr> is derived from the folder number for the cabinet and the index number for the index field in this manner using the formula (<FolderNr>*100)+<IndexNr>. For example:

    • If <FolderNr> is 1 and <IndexNr> is 1, <FolderIndexNr> would be 101

    • If <FolderNr> is 11 and <IndexNr> is 2, <FolderIndexNr> would be 1102

  • <DBField> is the database field for the index field settings within the cabinet. You need to specify one DCIXVAL tag for the following <DBField>:

    • Mandatory – define index field as mandatory.

    • AllowDups – allow duplicate index relations for this field option.

    • SortNr – this should match the SortNr for the index field (see Setting up Index Fields, above).

    • FType – this should match the FldType for the index field (see Setting up Index Fields, above).

    • FMask – this should match the FldMask for the index field (see Setting up Index Fields, above).

  • <Value> is the value for <DBField>. Use 1 to tick a checkbox field and 0 to untick the checkbox.

 

Example

To set up the Correspondent index field in the Correspondence:Confidential Filing Cabinet:

DCIXADD:21+3=Sender

DCIXVAL:2103+Mandatory=0

DCIXVAL:2103+AllowDups=1

DCIXVAL:2103+SortNr=4

DCIXVAL:2103+FType=40

DCIXVAL:2103+FMask=

 

Adding Task Types

When you create a new task type, you also need to specify the settings for the task.

 

Syntax

To add a new task:

DTSKADD:<TaskNr>+<TaskKey>=<TaskName>

 

To specify task settings:

DTSKVAL:<TaskNr>+<DBField>=<Value>

 

where:

  • <TaskNr> is the task number, and starts from 1.

  • <TaskKey> is key tag for the task. This should be a unique code.

  • <TaskName> is the description of the task.

  • <DBField> is the database field for the task settings.

    • BtNr - position of the task in the task type list

    • BtNote - Help/Note field

    • BtTagValues - Tag Values field

  • <Value> is the value for <DBField>.

 

Example

To create a task as shown above:

DTSKADD:21+SINV=Send invoice

DTSKVAL:21+BtTagValues=[DD.ConvertMail=Y]<BR>[DD.NoMail=R]

 

To create a task with follow up tasks:

DTSKADD:22+APPROVAL=Approval Required for Processing

DTSKVAL:22+BtTagValues=[On_Complete.Codes=DEVTASK]<BR>[On_Complete.Default=DEVTASK]<BR>[On_Complete.Descr=Send Invoice]<BR>[On_Complete.Assign=ANNE]<BR>[On_Complete.Due=3]<BR>[On_Complete.Action.ConfirmDraft=VpDocId]<BR>[On_Reject.Codes=DOCSUB,DOCREV]<BR>[On_Reject.Default=DOCREV]<BR>[On_Reject.Descr=Rejected (-Copy-)]<BR>[On_Reject.Caption=Decline]<BR>[On_Reject.Complete=Y]<BR>[On_Reject.Assign=ANNE]<BR>[On_Reject.Due=3]
