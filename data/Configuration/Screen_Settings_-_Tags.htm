Screen Settings - Tags

The display on a summary screen is based on the template assigned to the summary screen. The assignment is done using this tag:

[<CmdType>=<FormCode>]

where

  • <CmdType> can be:

    • MF.<TypeCode> to set condition based Master File type e.g. MF.I, MF.C.

    • ET.<JurisdictionCode>-<EntityTypeCode> to set condition based on Entity's jurisdiction and type e.g. ET.HK-CG.

    • OS.<OwnershipStructureCode> to set condition based on Entity's ownership structure e.g. OS.SA.

    • JU.<CountryCode> to set condition based on jurisdiction e.g. JU.HK.

    • DF to set as default template for the summary screen.

  • <FormCode> refers to the form code of the template.

CmdType

Description

Example

MF

Template is based on Master File Type.

  • [MF.I=WNMFSSI] - WNFMSSI template is used for Individual [I] Master Files.

  • [MF.C=WNMFSSC] - WNMFSSC template is used for Company [C] Master Files.

  • [DF=WNMFSS] - Default template. See DF, below.

ET

Template is based on the Entity File's Jurisdiction and Entity Type.

  • [ET.HK-CG=WNCSSSCG] - WNCSSSCG template is used if the Entity is incorporated in Hong Kong [HK] and the Entity Type is Company Limited by Guarantee [CG].

  • [ET.HK-CL=WNCSSSCL] - WNCSSSCL template is used if the Entity is incorporated in Hong Kong [HK] and the Entity Type is Private Company Limited by Shares [CL].

  • [DF=WNCSSS] - Default template. See DF, below.

OS

Template is based on the Entity's ownership structure.

  • [OS.SA=WNCSSSSA] - WNCSSSSA template is used for entities using the Shares – authorised capital [SA] structure.

  • [OS.TR=WNCSSSTR] - WNCSSSTR template is used for entities set up as Trust [TR].

JU

Template is based on jurisdiction.

[JU.HK=WNCSSSCL]

DF

Default template, used when none of the tags match.

[DF=WNMFSSS]
